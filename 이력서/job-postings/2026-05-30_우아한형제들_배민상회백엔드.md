## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Java 17/21 + Spring Boot 3.x + JPA + QueryDSL**: 자격 요건 4개 중 3개 정확 매칭. 트럼본 결재·티켓 도메인을 JPA/QueryDSL로 구현한 경험 직접 어필
- **객체지향 설계 + 도메인 모델링**: 헥사고날 아키텍처로 결재/티켓 도메인 분리 경험 → "객체지향 설계 능력" 정합
- **테이블 설계 역량**: 변경관리 도메인의 결재선·승인 상태 테이블 직접 설계 → "DB 테이블 설계" 충족
- **3년차 경력**: 경력 3년 이상 하한 충족
- **커머스 백오피스 패턴 친화**: 트럼본 백오피스 단독 구축 경험이 "판매사용 관리 시스템"과 구조적으로 유사

**공백**
- B2B 커머스(판매사·상품·쿠폰) 도메인 직접 경험 없음 — 결재/실행엔진 도메인과 거리
- 대용량 검색·트래픽 정량 메트릭 부재 (알려진 공백 2순위)

**매칭 신뢰도**: 상 — 필수 스택 4개 중 3개가 지원자 핵심 강점과 1:1 매칭, 경력 하한도 충족

## 도메인 한 줄 + 예상 기술부채
**도메인**: 배민상회 — 배달의민족 입점 사장님 대상 식자재·소모품 B2B 커머스 (상품·검색·멤버십·쿠폰·정산)
**예상 기술부채 2~3개**
1. **쿠폰·멤버십 발급/소진의 동시성 정합성** — 한정 쿠폰 중복 발급·초과 사용 위험. 지원자의 비관적 락 직렬화 경험으로 발급 게이트 직렬화 가능
2. **판매사 상품 등록 ↔ 검색 인덱스 간 데이터 정합성** — 등록 시스템과 검색 시스템 분리 시 eventual consistency 깨짐. 도메인 이벤트 + Outbox 패턴 적용 가능
3. **레거시 모놀리식 ↔ 신규 서비스 트랜잭션 경계** — 커머스 도메인 확장 중일 가능성. SAGA/분산 트랜잭션 경험으로 경계 보강 가능
