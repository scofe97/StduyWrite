## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Java/Spring + RESTful API**: Java 17/21, Spring Boot 3.x, REST API → 자격 요건 정확 충족
- **3년 이상 경력 정확 충족**: 3년차 → 자격 하단 정확 매칭, 연차 갭 없음
- **동시성 직렬화·비관적락**: 거래 체결·잔고 갱신은 동시성 정합성의 극단 — 비관적락·FOR UPDATE 직렬화·디스패치 게이트 경험이 거래소 백엔드의 핵심 난제에 직결
- **이벤트드리븐 + Kafka/Avro**: 주문·체결·정산 이벤트 파이프라인 경험으로 전이 가능
- **RDBMS 설계**: JPA/MyBatis + RDB 모델링 → Oracle RDBMS 자격 정합
- **관측성 스택**: Grafana/Loki/Tempo/Prometheus → 거래 시스템의 실시간 모니터링 요구와 정합

**공백**
- **공고 업무 설명 최소("신규 개발 및 유지보수")** — 세부 기술 요구가 불투명. 면접 전 실제 도메인(거래/지갑/정산) 확인 필요
- **대용량 실시간 거래 트래픽 정량 경험 부재** (§14 누적 2순위) — 거래소 피크 트래픽 규모 경험 부족
- **Oracle 직접 운영 경험 적음** — MariaDB 중심, Oracle 신규
- **가상자산·블록체인 도메인 무경험** — 지갑·온체인 정산 신규
- **금융 규제·보안 요구 강도** — 거래소는 보안·감사 요구가 높음(OWASP 숙지로 일부 대응)

## 도메인 한 줄 + 예상 기술부채
**도메인**: No.1 가상자산 거래 플랫폼. 주문·체결·지갑·정산을 실시간으로 처리하는 거래소 백엔드가 본업.

**예상 기술부채 2~3개**
1. **주문 체결·잔고 갱신의 극단적 동시성 경합** — 동일 종목에 매수/매도가 폭주하며 잔고 차감·체결의 정합성이 깨질 위험. 비관적락·FOR UPDATE 직렬화·멱등 처리 경험으로 직접 보강 가능
2. **정산·입출금 이벤트의 정확성·중복 방지** — 이벤트 누락/중복이 자산 불일치(자금 사고)로 직결. Outbox + 멱등 컨슈머 + Kafka로 보강 가능
3. **피크 트래픽 백프레셔·격리** — 급등락 시 주문 폭주로 시스템 과부하. Kafka 백프레셔·서비스 격리·관측성으로 병목 가시화·튜닝 가능
