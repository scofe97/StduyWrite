## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Java/Spring + RESTful API**: Java 17/21, Spring Boot 3.x, REST API → 자격 요건 정확 충족, 명확한 핵심 매칭
- **Kafka 경험**: 트럼본 Kafka/Redpanda + Avro 이벤트드리븐 → 기술 스택의 Kafka에 직접 매칭. 멤버십·커머스 이벤트 처리에 전이
- **커머스·결제성 도메인 친화**: 결재(변경관리)·실행엔진의 트랜잭션·상태 관리 경험 → "유료 멤버십·디지털 상품·커머스" 도메인에 전이 가능
- **JPA/RDBMS**: JPA/MyBatis + MariaDB → RDBMS 활용 자격 정합
- **연차 무명시·경력 무관 가능성**: 자격에 N년 명시가 없어 3년차 진입 장벽 낮음

**공백**
- **Redis 직접 운영 경험 부족** (§14 누적 5순위) — 멤버십·세션·캐시에 Redis 핵심 사용 가능성. 학습 검토 필요
- **Kotlin 직접 경험 없음** — JVM 기반, Java 깊이로 대체 어필
- **글로벌 팬덤 트래픽(대용량) 정량 경험 부재** (§14 누적 2순위) — 아티스트 콘텐츠 발매 시 스파이크성 트래픽 경험 부족
- **팬덤/엔터 플랫폼 도메인 무경험** — 신규 도메인

**매칭 신뢰도**: 중 — 기술 스택·자격은 무난한 매칭이나 자격 요건이 일반적이라 차별화 어필(Kafka·동시성 깊이)이 합격 변수

## 도메인 한 줄 + 예상 기술부채
**도메인**: 글로벌 팬덤 플랫폼 위버스. 아티스트-팬을 잇는 커뮤니티·멤버십·커머스를 한 백엔드로 받치는 슈퍼앱.

**예상 기술부채 2~3개**
1. **발매·컴백 시 스파이크 트래픽의 동시성 경합** — 한정판 멤버십·디지털 상품이 동시에 몰려 매진 처리·중복 결제 경합 발생. 비관적락·FOR UPDATE 직렬화·멱등 처리 경험으로 보강 가능
2. **커뮤니티·멤버십·커머스 도메인 SSOT 미정착** — 한 백엔드가 세 도메인을 떠안으며 데이터 경계가 흐려짐. 도메인 경계(Bounded Context) 재정의 + 도메인 이벤트로 보강 가능
3. **멤버십 권한·결제 이벤트 정합성** — 결제 성공/실패 이벤트 누락이 멤버십 권한 부여 오류로 직결. Outbox + 멱등 컨슈머 + Kafka로 보강 가능
