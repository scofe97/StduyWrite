## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Kotlin/Java + Spring + MSA**: 필수 스택 3개 정확 매칭. JVM 기반이라 Kotlin 러닝 커브 작음
- **고가용성·확장 시스템 설계**: 트럼본 실행엔진을 헥사고날 + 이벤트 드리븐으로 설계 → "확장 가능한 시스템" 직접 어필
- **장애 트러블슈팅 + 관측성**: Grafana/Loki/Tempo/Prometheus 스택 직접 운영 → "장애 트러블슈팅" 정합. 분산 추적 경험 강점
- **분산 트랜잭션·동시성 직렬화**: 금융 도메인 핵심 요구인 정합성 보장 경험 보유
- **무중단 배포 친화**: K8s/ArgoCD/GitOps 자가호스팅 → 토스의 잦은 무중단 배포 문화와 정합

**공백**
- **대규모 실시간 트래픽 정량 메트릭 부재** (알려진 공백 2순위) — 토스 핵심 요건. 수십억 건 트래픽 운영 경험 없음
- **Kotlin 직접 프로덕션 경험 없음** — Java 깊이로 보완 + "필요 시 학습" 준비
- **Redis 미경험** (알려진 공백 5순위) — 캐시/세션 레이어에서 자주 요구

**매칭 신뢰도**: 중 — 스택·설계·관측성은 강하게 매칭되나, 토스가 1순위로 보는 "대규모 실시간 트래픽" 정량 실적이 약함

## 도메인 한 줄 + 예상 기술부채
**도메인**: 토스 — 송금·결제·자산관리·생활금융을 묶은 금융 슈퍼앱. 수십억 건 트래픽을 금원 사고 없이 처리
**예상 기술부채 2~3개**
1. **50+ 마이크로서비스 간 분산 트랜잭션 정합성** — Product 서버 50개 이상 = 서비스 간 정합성 보장이 핵심 난제. SAGA/Outbox + 비관적 락 경험 적용 가능
2. **실시간 트래픽 폭증 시 동시성 직렬화 비용** — 이벤트성 트래픽(행운퀴즈 등)에서 락 경합. 직렬화 전략 튜닝 경험 기여 가능
3. **무중단 배포 환경의 호환성·롤백 안전성** — 하루 수백 회 배포 = 스키마/계약 호환성 부채. GitOps·점진 배포 경험으로 보강 가능

## 90일 가설
입사 후 첫 30일은 담당 Product 서버의 호출 그래프와 트래픽 패턴, 트랜잭션 경계를 파악합니다. 다음 30일은 실시간 트래픽이 몰리는 흐름 1건에서 동시성·정합성 결함을 재현 테스트로 검증하고, 관측성 스택으로 병목을 계량화합니다. 마지막 30일은 분산 트랜잭션 정합성 또는 락 직렬화 개선 PoC를 1개 서비스에 적용하고 무중단 배포 안전성 가설을 검증합니다.
