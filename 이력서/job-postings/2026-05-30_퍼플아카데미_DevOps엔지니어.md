## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **ArgoCD GitOps CD**: 트럼본 ArgoCD/GitOps 운영 + 개인 GCP 클러스터 ArgoCD 자가호스팅 → 필수 직접 매칭
- **Kubernetes(EKS) + CKA**: K8s 클러스터 운영·트러블슈팅 + CKA → 필수 정확 매칭
- **GitHub Actions / CI 워크플로우**: Jenkins 파이프라인 자동화 경험 + GitOps CI 설계 사고 → 필수 매칭(도구 전이)
- **Helm**: Jenkins 컨트롤러 Helm 차트 배포 경험 → 필수 매칭
- **모니터링·장애 대응**: 개인 GCP에 Grafana·Loki·Tempo·Prometheus 관찰성 스택 자가 구성 → "모니터링 환경 구축" 직접 매칭
- **경력 밴드 정합**: 3년차 = 요구 3~10년 하단에 안착

**공백**
- **Terraform 직접 작성 경험 부족** (IaC 최우선 공백) — 개인은 수기 kubeadm. Terraform 실무 학습 필요
- **AWS 상용 운영 경험 부재** (자격증만) — EKS/RDS/S3/IAM은 AWS 고유. GCP 운영을 AWS로 매핑하는 답변 준비
- **GitHub Actions 직접 운영 이력 명시 부족** — 본인 CI는 Jenkins 중심. Actions 워크플로우 작성 PoC 권장
- **Redis 운영 경험 부족** (알려진 공백) — 우대사항 RDS/Redis 중 Redis 측 보강 필요

**매칭 신뢰도**: 중 — GitOps·ArgoCD·K8s·Helm·모니터링은 정확 매칭이나, 필수 IaC가 Terraform이고 CI가 GitHub Actions라 본인 Jenkins/수기 운영 대비 도구 갭이 존재

## 도메인 한 줄 + 예상 기술부채
**도메인**: 빅데이터·LMS 기반 온라인 영어 홈스쿨링 솔루션. 개발 환경 리뉴얼 직후로 기존 서비스 통합·관리 체계 수립이 진행 중인 단계.
**예상 기술부채 2~3개**
1. **리뉴얼 환경 ↔ 레거시 서비스 통합 불일치** — "기존 서비스 통합·관리 체계 수립"이 명시 = 신구 인프라 혼재로 배포·설정 표준 부재. ArgoCD App 표준화 + Helm 템플릿으로 일원화 가능
2. **수기/Terraform 미적용 인프라 잔존** — IaC 자동화가 진행형이면 콘솔 수기 리소스가 남아 드리프트 발생. terraform import + 모듈화로 코드화 가능
3. **교육 트래픽의 시간대 편중(방과 후·주말)** — 홈스쿨링은 특정 시간 트래픽 집중. EKS HPA·캐싱 정책 정착 필요 (관찰성 스택 기반 부하 측정으로 접근)
