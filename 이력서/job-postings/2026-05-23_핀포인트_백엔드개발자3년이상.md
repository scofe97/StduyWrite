## 매칭 분석 (vs RESUME.md)

**강점 (이 공고에서 어필 가능한 내 마일스톤)**
- **Spring Boot 3.x 상용 운영 3년+**: 트럼본 본체·실행 엔진 2024.01~ 운영 = 약 2.4년. 자격 요건 "3년 이상"에 미세하게 미달이나 "또는 그에 준하는 실력" 미명시 → 면접에서 풀스택 트랙 병행을 시간 환산으로 보강
- **GCP 클라우드 환경**: 개인 GCP `asia-northeast3-a` 3노드 K8s 자가호스팅 + GitOps 운영 → 필수 "GCP 클라우드" 정확 매칭. **6건 공고 중 GCP를 명시한 유일한 공고**로 본인 강점 직접 어필
- **REST API + Git 협업**: 트럼본 결재 47회·티켓 24회·실행 엔진 185회 커밋 → 필수 매칭
- **다도메인 모듈 분할 경험**: 트럼본 본체 = 결재·티켓·워크플로우·외부 PMS·보고서 다도메인 단독 신규 구축 → 자격 "회원·상품·계약·빌링·예약·…" 다도메인 분할에 직접 정합
- **상태 머신·감사 추적 사고**: 결재·티켓 상태 머신 동기화 + 감사 로그 자동 적재 → "출입·방문·워크오더" 등 빌딩 운영 도메인의 상태 추적 사고와 정합

## 도메인 한 줄 + 예상 기술부채

**도메인**: 빌딩 통합 운영 솔루션 "빌딩OS". 회원·계약·빌링·예약·출입·IoT·워크오더 등 다도메인을 단일 플랫폼에 통합. 삼성전자 C-Lab 2024 선정 + CES 2025 혁신상 수상 프롭테크 스타트업.

**예상 기술부채 3개**
1. **다도메인 13개 모듈의 도메인 SSOT 미정착** — 주요 업무 명시 = 회원·상품·계약·빌링·예약·캘린더·출입·방문·IoT·CS·워크오더·요금·이용안내 13개 모듈 병행 = 도메인 모델 응집력 약화 가능성. 헥사고날 + 도메인 분리 패턴 직접 적용 가능
2. **IoT 외부 디바이스 의존 누적** — 출입·방문·IoT 명시 = 외부 디바이스 + 게이트웨이 호출 표면이 다양함. 외부 호출 단일 추상화 + 멱등 + webhook 폴링 fallback 패턴 적용 가능 (트럼본 Jenkins 통합 사례)
3. **빌링·요금 트랜잭션 외부 시스템 의존** — 빌링·요금 명시 = 결제·세무·정산 외부 시스템 연동 부담. 결재 트랜잭션·외부 호출 분리 + 멱등 보장 사고 그대로 적용 가능
