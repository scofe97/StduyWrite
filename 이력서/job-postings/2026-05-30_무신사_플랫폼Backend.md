**우대**: 본문 정보 부족 — 별도 우대사항 섹션 미명시 (향후 Golang / Kotlin 언어 확장 예정 명시)

## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Java/Spring + Kafka 이벤트 플랫폼**: 자격 요건 4·5번("Kafka/Kinesis 이벤트 스트림" + "Java/Spring")이 지원자 핵심 스택과 거의 완전 매칭. Redpanda(Kafka 호환) + Avro 신규 구축 경험 직접 어필
- **Event Bus / API Gateway 도메인**: 트럼본 실행 엔진을 이벤트드리븐 + 헥사고날로 구축한 경험이 "Event Bus Platform·API Gateway 개발·운영" 업무와 정합. 플랫폼성 컴포넌트를 단독 설계한 사례가 강점
- **MSA + 관측성**: "MSA 환경 플랫폼 운영"에 K8s(CKA)/Docker + Grafana/Loki/Tempo/Prometheus 관측성 스택으로 보강. Log Collector 업무와 관측성 경험 정합
- **분산 트랜잭션·동시성**: 이벤트 기반 플랫폼의 정합성·멱등성 이슈에 비관적 락·직렬화 경험 어필

**공백**
- **Golang 경험 없음** — 향후 확장 언어로 Golang 명시 (§알려진 공백: JVM외 언어, 단 시급도 하향). Kotlin은 JVM 기반이라 러닝 커브 작음
- **대용량 트래픽 정량 메트릭 부재** — 무신사는 대규모 커머스 트래픽. Log Collector·Event Bus는 처리량이 핵심. 정량 사례 보강 필요

**매칭 신뢰도**: 상 — "Kafka 이벤트 스트림 + Java/Spring + MSA 플랫폼"이 지원자 강점과 정면 일치. 플랫폼 컴포넌트 단독 구축 경험이 차별점

## 도메인 한 줄 + 예상 기술부채
**도메인**: 패션 커머스 무신사의 사내 플랫폼 팀. 서비스 도메인이 아니라 Log Collector·Event Bus·API Gateway 같은 공통 인프라 플랫폼을 만드는 조직.

**예상 기술부채 2~3개**
1. **Event Bus 스키마·토픽 거버넌스** — 전사 이벤트 버스는 사용처가 늘수록 스키마 호환성·토픽 난립이 부채화. Avro + 스키마 레지스트리 + 버전 호환 정책 경험으로 풀 수 있음
2. **Log Collector 처리량·유실 보장** — 대규모 로그 수집은 백프레셔·재처리·유실 트레이드오프가 핵심. Kafka 컨슈머 멱등성·재처리 경험으로 보강 가능
3. **API Gateway의 인증·라우팅 일관성** — 다수 마이크로서비스 진입점의 정책 일관성 유지. 헥사고날 포트/어댑터로 게이트웨이 정책을 경계화 가능

입사 후 첫 30일은 Event Bus Platform의 토픽·스키마 카탈로그와 컨슈머 그룹별 처리 보장 수준을 매핑하고, 스키마 호환성이 깨질 수 있는 지점을 식별합니다. 다음 30일은 Log Collector 또는 Event Bus 경로 1개에서 유실·중복 재처리 시나리오를 재현하고 멱등성 보강 PoC를 제안합니다. 마지막 30일은 Avro 스키마 호환 정책을 1개 토픽군에 표준화하고, 관측성 대시보드로 처리량·지연을 정량화합니다.
