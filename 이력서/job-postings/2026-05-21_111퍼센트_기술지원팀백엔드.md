## 매칭 분석 (vs RESUME.md)

> ⚠️ **이 공고는 본인 스택과 정합이 약합니다.** TypeScript + Node.js가 필수인데 RESUME.md는 Java 17·21 + Spring Boot 3.x가 핵심. JVM 외 언어 경험 부재가 §14 점검에서 반복 지적되던 공백과 맞물려 1순위 약점으로 노출됩니다. 지원 결정 전 후술 "지원 신중 검토" 단락을 먼저 읽으세요.

**강점 (이 공고에서 어필 가능한 내 마일스톤)**
- **공통 서비스 아키텍처 설계 경험**: 트럼본 본체 도메인을 단독 신규 구축하면서 결재 정책·동시성 정책을 재사용 가능한 어댑터로 분리한 사례 → 주요 업무 "재사용·확장성 높은 공통 서비스 아키텍처 설계"와 패턴 동일
- **인하우스툴·운영툴 사고 회로**: 결재 회복·자동 만료·외부 장애 자동 처리 → "인하우스 운영툴 유지보수"의 운영 우선 사고와 정합
- **Public Cloud 설계·구축·운영**: 개인 GCP 3노드 K8s 자가호스팅(Jenkins·ArgoCD·Redpanda·Nexus·Grafana·Loki·Tempo) + CKA → 필수 "GCP·AWS Public Cloud 실제 서비스 설계·구축·운영" 직접 매칭
- **문서화·기술 가이드 작성**: 동시성 제어 정책 문서 신규 작성으로 팀 의사결정 기준 명문화 → 우대 "문서화로 지식 공유 기여" 직접 매칭
- **라이브 서비스 운영 + 장애 대응**: 외부 호출 실패·반려·시간 초과 자동 회복 + webhook 폴링 fallback → 필수 "라이브 서비스 운영 + 장애 신속 대응" 매칭 (단, 컨슈머 트래픽은 아님)
- **신기술 도입 적극성**: Side Project에서 Outbox·SAGA·Break-and-Resume·DAG 등 EDA 패턴 직접 실험 → 우대 "신기술 도입 적극성" 정합

**공백 (보강하거나 학습 계획을 세워야 할 부분)**
- **TypeScript + Node.js 직접 경험 0** — 필수 스택. JVM 경험을 일반화해도 면접에서 "Node.js로 짠 코드 보여주세요" 요청이 들어오면 답변 불가. 지원 결정 전 최소 2주 안에 Side Project 1개를 Node.js + TypeScript로 마이그레이션 또는 신규 작성 필요. 또는 지원 자체를 보류 권장
- **AWS 상용 운영 경험 부재** — GCP는 직접 구축, AWS는 자격증만 (이미 §14 누적 공백). 면접에서 "GCP 경험은 어떻게 AWS로 이전하나" 답변 카드 준비
- **컨슈머 대용량 트래픽 정량 메트릭 부재** — 5번째 공고 연속 (5/5로 굳어짐). 모바일 게임은 트래픽 스파이크 + 글로벌 분산이 핵심인데 RESUME 트래픽 수치 없음
- **AI 서비스 통합 경험 없음** — 우대사항 "AI 기술 서비스 접목". RESUME에 AI/LLM 관련 경험 부재. 우대라 필수 아니나 다른 후보 대비 약점
- **게임 산업 도메인 직접 경험 없음** — 라이브 게임 서비스의 실시간성·인게임 경제·매출 펀치 등 도메인 사고가 RESUME에 없음. §5 슬롯 작성으로 우회

**지원 신중 검토 (4건 평균 매칭과 비교)**
- 4건 평균(여기어때·채널톡·라포랩스·클래스101)에서 본인 강점이 3/4~4/4 매칭이었던 반면, 이 공고는 **필수 스택 0건 매칭**입니다.

3. **AI 서비스 신규 진입 단계 인프라 부재** — "AI 활용 신규 서비스 서버 개발"이 주요 업무 = AI 통합 패턴(임베딩 캐싱·LLM 호출 추상화·비용 관리) 아직 정착 안 됨. 외부 시스템 호출 추상화 + 멱등 보장 패턴 그대로 적용 가능
