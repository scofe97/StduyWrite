## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **분산시스템 + 대용량 트래픽 기술 스택**: Kafka·MSA·분산 직렬화·이벤트드리븐 경험 → 기술 스택(분산시스템·대용량)에 정면 매칭
- **Spring 백엔드 + API 설계**: Java 17/21, Spring Boot 3.x, REST API → 자격 요건 정확 충족, 1년+ 경력 요구를 3년차로 여유 충족
- **공용 플랫폼(계정·카탈로그) 설계**: 결재·티켓·실행엔진을 도메인 경계로 설계·구축 → "공용 플랫폼 설계/개발" 정합
- **빌링·정산 도메인 친화**: 결재(변경관리)·실행엔진의 트랜잭션·상태머신 경험 → "빌링/비즈플랫폼" 도메인 이해에 전이 가능
- **K8s(CKA)/GitOps 인프라**: 모빌리티 대규모 서비스 운영 인프라와 정합, DevOps 보조 목표와도 부합

**공백**
- **Ruby on Rails 미경험** — Spring 단독 경로로 지원 가능(택1), Java 깊이로 어필
- **대용량 트래픽 정량 메트릭 부재** (§14 누적 2순위) — 모빌리티 실시간 트래픽 규모 경험 부족
- **NoSQL 운영 경험 부족** — MariaDB 중심, MongoDB/Redis 노출 적음 (§14 누적 5순위 Redis)
- **모빌리티 도메인 무경험** — 위치·배차·실시간성 도메인 신규

**매칭 신뢰도**: 중 — 기술 스택·연차는 상급 매칭이나 자격 요건이 일반적(1년+)이라 분산·대용량 어필 강도가 합격 변수

## 도메인 한 줄 + 예상 기술부채
**도메인**: 택시·대리·내비·주차·전기차 충전을 아우르는 모빌리티 플랫폼. 실시간 위치·배차·정산을 받치는 서버가 본업.

**예상 기술부채 2~3개**
1. **실시간 배차의 동시 요청 경합** — 한 기사/차량에 다수 호출이 동시에 매칭되며 중복 배차·경합 발생. 비관적락·FOR UPDATE 직렬화·디스패치 게이트 경험으로 직접 보강 가능(트럼본 실행엔진 디스패치 게이트와 동형)
2. **공용 플랫폼(계정·카탈로그) 서비스 간 정합성** — 다수 서비스가 공용 데이터를 참조하며 캐시·이벤트 동기화 드리프트 발생. 도메인 이벤트 + CDC로 보강 가능
3. **빌링·정산 이벤트의 정확성** — 충전·요금 정산 이벤트 누락/중복이 금액 오차로 직결. Outbox + 멱등 처리 + Avro 스키마 호환으로 보강 가능
