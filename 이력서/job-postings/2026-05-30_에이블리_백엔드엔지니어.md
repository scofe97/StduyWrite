## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Kotlin/Spring + RDBMS**: Java 17/21, Spring Boot 3.x, JPA/MyBatis, MariaDB → 자격(Spring·RDBMS) 정확 충족. Kotlin은 JVM 기반이라 러닝커브 작음
- **3년 이상 경력 정확 충족**: 3년차 → 자격 하단 정확 매칭, 연차 갭 없음
- **Kafka + 이벤트드리븐**: 트럼본 Kafka/Redpanda + Avro 이벤트드리븐 구축 → 인프라 스택의 Kafka에 직접 매칭. 모놀리식→MSA 전환의 이벤트 파이프라인에 직결
- **ArgoCD/GitOps/Grafana 관측성**: ArgoCD·GitOps·Grafana/Loki/Tempo/Prometheus 경험 → 인프라 스택(ArgoCD/Grafana/Datadog) 정합. DevOps 보조 목표와도 부합
- **헥사고날·도메인 설계**: 결재·티켓·실행엔진을 도메인 경계로 단독 구축 → "셀러 플랫폼·Open API·커머스" 도메인 분리에 전이

**공백**
- **Python/Django 미경험** — 모놀리식이 Django라 Django/DRF 우대 항목에 공백. Spring 트랙으로 어필 + "필요시 학습" 준비
- **ScyllaDB/Elasticsearch 운영 경험 부족** (§14 누적 5순위 Redis 포함) — MariaDB 중심, NoSQL/검색 신규
- **대용량 커머스 트래픽 정량 메트릭 부재** (§14 누적 2순위) — 스타일 커머스 피크 트래픽 규모 경험 부족
- **패션·스타일 커머스 도메인 무경험** — B2C 커머스 신규

**매칭 신뢰도**: 상 — Spring·RDBMS·3년+ 연차·Kafka·ArgoCD/Grafana가 지원자 강점과 정밀 정합. Django 공백은 Spring 트랙으로 상쇄 가능

## 도메인 한 줄 + 예상 기술부채
**도메인**: 스타일 커머스 플랫폼 에이블리. 누구나 스타일을 팔고 사는 NEXT COMMERCE. Django 모놀리식과 Kotlin/Spring MSA가 공존하며 전환 중.

**예상 기술부채 2~3개**
1. **Django 모놀리식 ↔ Kotlin/Spring MSA 이중 스택의 경계·트랜잭션** — 두 스택 병기 = 호출 경계·트랜잭션 경계가 모호. 헥사고날·SAGA·분산 트랜잭션 경험으로 전환 구간의 정합성 보강 가능
2. **커머스 주문·재고의 동시성 경합** — 인기 상품 동시 구매 시 재고 차감·결제 경합. 비관적락·FOR UPDATE 직렬화·멱등 처리 경험으로 직접 보강 가능
3. **이종 저장소(MySQL·Redis·ScyllaDB·ES) 간 정합성** — 다수 저장소에 상품·검색·캐시 데이터가 흩어지며 드리프트. Kafka + CDC + 도메인 이벤트로 동기화 보강 가능
