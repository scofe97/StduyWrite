- **이미지·동영상 처리 도메인 경험 없음** — 채팅·피드·계정 트랙 지원이 매칭 더 강함

## 도메인 한 줄 + 예상 기술부채

**도메인**: 당근 슈퍼앱의 공통 플랫폼 레이어 — 채팅·피드·계정·이미지·동영상·푸시. 신사업 Product가 의존하는 가로지르는 기술 자산.

**예상 기술부채 3개**
1. **공통 플랫폼이 신사업마다 다르게 호출되는 추상화 부족** — "플랫폼" 명시 = 호출 표면 일관성·SDK 정착이 미흡할 가능성. 외부 호출 단일 추상화 패턴 적용 가능
2. **MSA 분리 진행 중 gRPC + REST 혼재** — 두 프로토콜 병기 = 통신 표준화·관찰성 부담. OpenTelemetry + 통합 로깅 도입 가능
3. **푸시·이미지 처리의 백프레셔 + 큐 운영 부담** — 대용량 비동기 처리 = 큐 정원 초과·재시도 폭주 가능성. 트럼본 큐 정원 초과 0 건 사례 직접 적용 가능
