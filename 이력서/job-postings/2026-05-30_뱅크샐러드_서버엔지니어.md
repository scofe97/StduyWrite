## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Kotlin/Spring/Java 공용 스택**: 회사 스택이 Kotlin·Spring·Java(채용 검색 기준)로 지원자 핵심 JVM 스택과 정합. Spring Boot 3.x 주력으로 API 신규 개발·유지보수 업무에 직접 어필
- **리눅스 서버 운영 + 인프라**: 자격 요건 2번("리눅스 개발·서버 운영")에 개인 GCP 3노드 K8s 자가호스팅 + Docker/Jenkins/ArgoCD 실경험으로 강하게 어필
- **RDBMS + 데이터 정합성**: "금융 데이터 활용 백엔드"는 정합성이 생명. 비관적 락 동시성 직렬화·분산 트랜잭션 경험이 금융 데이터 무결성 업무와 정합
- **변경관리·결재 도메인 경험**: 트럼본 결재(변경관리) 엔진 담당 경험이 금융 도메인의 승인·이력 관리 감각으로 전이 가능

**공백**
- **금융/마이데이터 도메인 경험 없음** — 뱅크샐러드는 마이데이터 사업자. 금융 규제·데이터 표준 학습 필요
- **NoSQL 운영 경험 부족** — 자격 요건 3번이 NoSQL 명시. Redis가 §알려진 공백(5순위). 보강 필요
- **공고 스택 불투명** — 본문에 Kafka·구체 스택 미명시라 사전 정보 부족. 면접 전 회사 기술 블로그·테크 스택 직접 확인 권장

**매칭 신뢰도**: 중 — JVM 스택·리눅스 운영은 잘 맞으나, 공고 본문이 기술 스택을 구체화하지 않아 핀트 확인이 필요. 금융 도메인 격차도 변수

## 도메인 한 줄 + 예상 기술부채
**도메인**: 마이데이터 기반 개인 자산관리 핀테크 뱅크샐러드. 흩어진 금융 데이터를 모아 분석·추천하는 데이터 집약 금융 서비스.

**예상 기술부채 2~3개**
1. **마이데이터 외부 기관 연동의 정합성·재시도** — 다수 금융기관 API 연동은 응답 지연·부분 실패가 잦음. 지원자의 멱등성·재시도·보상 트랜잭션 경험으로 데이터 수집 안정성을 높일 수 있음
2. **금융 데이터 무결성·이력 추적** — 자산 데이터는 변경 이력·감사 추적이 필수. 지원자의 변경관리(결재) 도메인 + Outbox 이벤트 경험으로 감사 로그를 이벤트화 가능
3. **모든 API 단일 백엔드의 결합도** — "모든 API 신규·유지보수"는 모놀리식 결합 부채 신호. 헥사고날 포트/어댑터로 도메인 경계를 점진 분리 가능
