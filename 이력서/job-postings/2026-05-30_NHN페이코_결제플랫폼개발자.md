## 매칭 분석 (vs 지원자)
**강점 (어필 가능한 마일스톤)**
- **Java 3년+ 정확 매칭**: 트럼본 본체·실행 엔진 운영 경력으로 "Java 3년 이상" 충족
- **Spring Framework + RDBMS**: 필수 스택 정확 매칭 (Spring Boot 3.x + PostgreSQL/MySQL)
- **Linux 환경 운영**: 개인 GCP 3노드 K8s 자가호스팅 + 트럼본 운영으로 Linux 서비스 운영 경험 정합
- **결재(변경관리) 도메인 ↔ 결제 플랫폼 인접성**: 결재·티켓 도메인의 상태 추적·트랜잭션 경험이 결제 서버 개발에 정합
- **PG 연동 ↔ 외부 시스템 연동 추상화**: "PG 연동 결제 서버"에 트럼본 외부 PMS·Jenkins 연동 추상화 경험 전이
**공백**
- **PG 연동 결제 서버 실무 경험** — 자격에 "결제 플랫폼 서버 개발 경험"이 명시됨. 본인은 결재 도메인이라 결제 정산·PG 직접 경험은 인접하나 동일하진 않음. 면접에서 인접성으로 풀이 필요
- **대용량 결제 트래픽 정량 메트릭 부재** — 결제 플랫폼은 트래픽 규모가 핵심. 정량 부하 지표 경험 약함
- **간편결제 도메인 무경험** — 멤버십·포인트·청구서 등 PAYCO 특유 도메인 학습 필요
**매칭 신뢰도**: 중상 — Java/Spring/Linux/RDBMS 필수 4종 정확 매칭 + 결재 도메인 인접성. "결제 플랫폼 서버 개발 경험" 자격을 결재 인접성으로 풀고, 트래픽 메트릭만 보강하면 강함

## 도메인 한 줄 + 예상 기술부채
**도메인**: 간편결제 플랫폼 PAYCO. 멤버십·포인트·청구서·금융을 통합 제공하며 모바일·PC 결제와 PG 연동 결제 서버를 운영하는 핀테크.

1. **PG 연동 다양성·장애 전파** — 다수 PG·카드사 연동으로 외부 장애가 결제 전체로 전파될 위험. 외부 호출 단일 추상화 + 서킷브레이커 + 폴백 패턴 적용 가능
