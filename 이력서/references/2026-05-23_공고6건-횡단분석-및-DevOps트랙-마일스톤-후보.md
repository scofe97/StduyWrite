1. **미디어로그 자바 클라이언트** (1순위) — 필수 스택 5/5 매칭. 외부 게이트웨이 호출 추상화 + 멱등 + 비동기 흐름이 본인 사고 회로 그대로. JVM 외 언어 공백·트래픽 메트릭 공백이 모두 영향 없음. **누적 공백을 거의 다 우회하는 정합 공고.**

### 6.2 후보 마일스톤 카드 (3 개)

**후보 A. Jenkins 멀티 인스턴스 운영 표준화 (SRE/Platform 트랙)**

기준점: 외부 시스템에 강하게 의존하는 모듈의 운영은 코드와 함께 형상관리되어야 한다. 인프라 변경이 코드 리뷰를 거치지 않으면 운영 장애가 사람에 의존한다.

- 문제 (가정): 사내 다중 Jenkins 인스턴스가 늘면서 Job 이관·플러그인 설치·webhook 등록이 인스턴스마다 수기 작업으로 처리되고 있었습니다.
- 해결: Jenkins 컨트롤러를 Helm 차트로 K8s 배포, `cloudbees-folder` 플러그인을 LTS 기본 이미지에 명시 설치, `webhook-listener.groovy` 스크립트로 빌드 완료 이벤트를 메시지 브로커로 직접 발행하도록 표준화했습니다.
- 해결: Job 이관 파이프라인(`config.xml` 추출 → 폴더 생성 → 재전송) 을 자동화해 인스턴스 추가/이관 비용을 한 사이클 안에 끝나도록 했습니다.
- 결과: 이미 RESUME 본문에 노출된 "부수적 Jenkins 운영 기여" 4 줄을 별도 카드로 승격. 정량은 본인이 사내 데이터로 확정해야 함 — "이관 소요 시간 N 시간 → N 분" 또는 "신규 인스턴스 N 분 안에 합류" 같은 한 줄이 필요.

**Skill Keywords**: Jenkins, Helm, Kubernetes Plugin, cloudbees-folder, Groovy, Webhook

**후보 B. 개인 GCP 3 노드 K8s 운영 자동화 (DevOps 사이드 트랙)**

기준점: 사내에서 처음 도입하는 기술은 사용자 트래픽이 닿기 전에 한 번은 직접 운영해 봐야 한다. 운영 함정은 개인 환경에서 사전에 흡수한다.

- 문제: Ubuntu 24.04 kubeadm 환경에서 containerd 충돌·conntrack 누락 등 네 가지 함정이 사내 K8s 이관 시 같은 형태로 재현될 가능성이 있었습니다.
- 해결: 개인 클러스터에서 함정 4 종을 사전 재현·해결한 뒤 단계별 회피 절차를 문서화했습니다. ArgoCD + Helm 으로 Redpanda·Schema Registry·Connect·Console·Jenkins·Nexus·Grafana·Loki·Tempo·GitLab CE 를 NodePort 기반 GitOps 로 통합 운영합니다.
- 해결: OpenTelemetry + Grafana + Loki + Tempo 스택으로 자가 관찰성을 구성해 본인 배포의 P95 응답·에러율·로그 상관관계를 1 분 안에 추적 가능합니다.
- 결과: 사내 K8s 이관 시 동일 함정 4 종 회피, 신규 도구 도입 의사결정의 위험 사전 흡수. 정량 보강 필요(§5 권고 "둘째" 의 k6 부하 결과를 그대로 끌어올릴 수 있음).

**Skill Keywords**: Kubernetes (CKA), kubeadm, ArgoCD, Helm, OpenTelemetry, Grafana, Loki, Tempo, GitOps

**후보 C. 외부 의존 장애 자동 회복 (SRE 트랙)**

기준점: 외부 시스템·사용자·시간 모두 시스템 흐름을 멈출 수 있다. 세 경우 모두 자동 회복 경로가 있어야 한다.

- 문제: 트럼본 결재 흐름은 외부 API 호출 실패, 사용자 반려, 결재 방치(시간 초과) 세 가지가 결재를 정지시키는데, 각 경우의 회복 절차가 운영자 수기 작업이었습니다.
- 해결: 실패·반려 케이스에 대한 재상신 자동화, 외부 호출 실패 시 자동 재상신 알람, 스케줄러로 만료 결재 자동 반려 + 만료 후 결재 이어가기 옵션을 추가했습니다.
- 해결: Jenkins 빌드 콜백을 webhook 으로 수신하되 5 분 폴링 fallback 을 함께 두어 webhook 유실에도 상태 추적이 끊기지 않게 했습니다. 24 시간 이상 응답 없는 좀비 Job 은 타임아웃 안전망으로 자동 종료 상태 전이됩니다.
- 결과: 운영자 수기 회복 작업이 시스템 자동 처리로 이동. 정량은 본인이 운영 데이터로 보강해야 함 — "수기 회복 N건/월 → 자동 회복 N건/월" 같은 한 줄이 필요.

**Skill Keywords**: Webhook, Polling Fallback, Scheduler, Idempotency, Timeout Safety Net, Self-Healing
