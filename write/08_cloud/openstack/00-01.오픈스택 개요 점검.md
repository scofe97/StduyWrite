# 오픈스택 개요 점검

> 본 개요의 참고용 점검 질문입니다. 서비스 내부가 아니라 "IaaS 란 무엇이고, AWS·Kubernetes 와 어떤 관계인가" 를 확인합니다.



## 점검 질문

> 층과 대응 관계를 스스로 설명할 수 있는지 확인합니다.

### Q1. 오픈스택을 한 문장으로 정의하면?

내가 소유한 데이터센터에 AWS 같은 클라우드를 직접 구축하는 오픈소스 IaaS 플랫폼입니다. 물리 서버를 묶어 VM·네트워크·스토리지를 REST API 로 찍어냅니다.

### Q2. 오픈스택과 Kubernetes 는 경쟁 관계인가?

아닙니다. 위아래 층입니다. 오픈스택(IaaS)이 VM 을 만들면, 그 VM 안에 Kubernetes 가 설치돼 컨테이너를 굴립니다. 오픈스택은 인프라를 *만드는* 아래층, K8s 는 그 위에서 앱을 *굴리는* 층입니다. 회사 CMP 는 이 K8s 위에서 돕니다.

### Q3. Nova·Neutron·Cinder·Keystone 을 AWS 로 바꾸면?

각각 EC2(VM)·VPC(네트워크)·EBS(블록 스토리지)·IAM(인증)입니다. 새 이름을 외우기보다 익숙한 AWS 서비스에 대응시키면 역할이 바로 잡힙니다.

### Q4. 개발자가 이 바닥 인프라 층을 알아야 하는 이유는?

내 앱이 왜 이 노드에 배치됐는지, 네트워크가 왜 이렇게 격리됐는지, 스토리지 성격이 왜 이런지는 결국 IaaS 층이 어떻게 깔렸는지에 좌우되기 때문입니다. 매니지드 K8s(EKS·GKE)에서는 감춰지지만, 사내 오픈스택 위에서는 이 층이 그대로 드러납니다.
