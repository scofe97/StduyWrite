# 멀티클러스터와 멀티테넌시 실습

> 등록된 클러스터와 destination 제한 구조를 최소 절차로 확인한다.

## 실습 절차
> cluster 목록과 secret 기반 등록 모델을 본다.

```bash
argocd cluster list
kubectl get secret -n argocd -l argocd.argoproj.io/secret-type=cluster
kubectl describe appproject default -n argocd
```

## 확인 포인트
> cluster secret과 AppProject destination 제한이 함께 가야 한다.

- 어떤 클러스터가 등록돼 있는지 본다.
- Project가 허용한 destination 범위를 확인한다.
