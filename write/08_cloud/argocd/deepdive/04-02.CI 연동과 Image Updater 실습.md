# CI 연동과 Image Updater 실습

> Image Updater annotation과 Git write-back 구조를 확인하는 실습이다.

## 실습 절차
> 실제 반영보다 설정 구조를 점검한다.

```bash
kubectl get application -n argocd -o yaml | rg 'argocd-image-updater|image-list|write-back'
kubectl get pods -A | grep image-updater || true
```

## 확인 포인트
> annotation, registry polling, write-back target이 함께 맞아야 동작한다.

- 이미지 목록 annotation이 있는지 본다.
- Image Updater Pod 또는 Deployment가 있는지 본다.
- write-back target이 환경별로 분리돼 있는지 확인한다.
