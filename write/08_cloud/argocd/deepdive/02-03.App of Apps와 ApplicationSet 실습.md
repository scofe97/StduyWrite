# App of Apps와 ApplicationSet 실습

> 부모 Application 하나로 자식 Application을 생성하고, ApplicationSet이 반복 생성을 어떻게 다루는지 확인한다.

## 실습 절차
> App of Apps와 ApplicationSet을 최소 예시로 본다.

```bash
kubectl apply -n argocd -f app-of-apps.yaml
kubectl get applications -n argocd
kubectl apply -n argocd -f applicationset.yaml
kubectl get applicationsets -n argocd
kubectl get applications -n argocd
```

## 확인 포인트
> 부모 앱이 자식 앱을 만들고, ApplicationSet이 규칙으로 여러 앱을 만든다는 차이를 본다.

- App of Apps는 Git 경로에 있는 자식 Application YAML을 반영한다.
- ApplicationSet은 generator 결과로 Application을 만든다.
