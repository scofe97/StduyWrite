# ArgoCD 접근과 선언적 설정 실습

> UI 로그인, CLI 로그인, ConfigMap/Secret 확인을 한 번에 점검하는 실습이다.

## 실습 절차
> 사람용 접근과 선언적 리소스 확인을 함께 본다.

```bash
kubectl port-forward svc/argocd-server -n argocd 8080:443
argocd login localhost:8080 --username admin --password <PASSWORD> --insecure
argocd account get-user-info
kubectl get configmap -n argocd argocd-cm -o yaml
kubectl get secret -n argocd argocd-secret -o yaml | head -40
```

## 확인 포인트
> Application과 AppProject는 결국 Kubernetes 리소스로 관리된다는 점을 기억한다.

- UI와 CLI가 모두 동작하는지 본다.
- `argocd-cm`과 Secret 구성이 분리되어 있는지 본다.
- 이후 self-management를 할 때 이 리소스들을 Git에 둘 수 있다는 점을 연결한다.
