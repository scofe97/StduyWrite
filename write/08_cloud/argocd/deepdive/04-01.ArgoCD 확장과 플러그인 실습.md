# ArgoCD 확장과 플러그인 실습

> CMP sidecar 구성을 보고 discover/init/generate 흐름을 확인하는 실습이다.

## 실습 절차
> 실제 배포보다 구성 확인에 집중한다.

```bash
kubectl get deployment -n argocd argocd-repo-server -o yaml | sed -n '1,220p'
kubectl get configmap -n argocd | grep plugin || true
```

## 확인 포인트
> repo-server 확장은 main container 오염보다 sidecar 분리가 더 안전하다.

- plugin configmap이 있는지 본다.
- repo-server sidecar 방식인지 확인한다.
