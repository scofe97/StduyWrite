# GitOps 성숙도와 미래 고려사항 점검

> 저장소 구조와 프로모션 전략을 질문 중심으로 다시 점검한다.

## 점검 질문
> GitOps는 도구보다 저장소 구조에서 성숙도가 갈린다.

### Q1. monorepo와 polyrepo 중 무엇이 정답인가?

조직 구조와 변경 패턴에 따라 다르다. 중요한 것은 진실 공급원과 환경 경계를 명확히 하는 것이다.

### Q2. Image Updater와 환경 프로모션을 함께 쓸 때 주의점은?

dev 자동 갱신과 prod 승인 경계를 분리해야 한다. 같은 branch나 values 파일을 공유하면 의도하지 않은 promotion이 일어난다.

### Q3. Rendered manifests는 언제 고려할 만한가?

강한 감사, 승인, diff 가시성이 필요한 조직에서다. 기본값으로 쓰면 저장소 복잡도가 커질 수 있다.
