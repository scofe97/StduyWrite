# Argo RolloutsмҷҖ л°°нҸ¬ м „лһө
---
> ArgoCDк°Җ Git мғҒнғңлҘј нҒҙлҹ¬мҠӨн„°м—җ л§һм¶”лҠ” лҸ„кө¬лқјл©ҙ, Argo RolloutsлҠ” к·ё вҖңл§һм¶”лҠ” мҲңк°„вҖқмқ„ м–ҙл–»кІҢ мһҳкІҢ мӘјк°ң м•Ҳм „н•ҳкІҢ м§„н–үн• м§ҖлҘј мұ…мһ„м§ҖлҠ” лҸ„кө¬лӢӨ. Blue-Greenкіј CanaryлҠ” лӢЁмҲң Deployment көҗмІҙлҘј мӢңк°„В·нҠёлһҳн”Ҫ лӢЁмң„лЎң л¶„н•ҙн•ҳлҠ” л‘җ к°Җм§Җ лҢҖн‘ң л°©мӢқмқҙлӢӨ.



## н•ҷмҠө лӘ©н‘ң
> Rolloutsк°Җ ArgoCDмҷҖ м–ҙл–»кІҢ л¶„м—…н•ҳлҠ”м§Җ, Blue-Greenкіј Canaryк°Җ мӢӨм ңлЎң м–ҙл–Ө K8s лҰ¬мҶҢмҠӨлҘј л§Ңл“ңлҠ”м§Җ ліёлӢӨ.

мқҙ мһҘм—җм„ң нҷ•мқён•  лӘ©н‘ңлҠ” лӢӨмқҢкіј к°ҷлӢӨ:

1. Deployment көҗмІҙмҷҖ `Rollout` лҰ¬мҶҢмҠӨмқҳ м°ЁмқҙлҘј м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.
2. Blue-Greenкіј Canary м „лһөмқҳ лҸҷмһ‘ лӢЁкі„лҘј K8s лҰ¬мҶҢмҠӨ ліҖнҷ” кҙҖм җм—җм„ң м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.
3. AnalysisTemplateмқҙ мһҗлҸҷ мҠ№кІ©/лЎӨл°ұм—җ м–ҙл–»кІҢ лҒјм–ҙл“ңлҠ”м§Җ м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.
4. ArgoCD `Application`кіј `Rollout`мқҙ reconciliation нқҗлҰ„м—җм„ң м–ҙл–»кІҢ л§һл¬јлҰ¬лҠ”м§Җ м„ӨлӘ…н•  мҲҳ мһҲлӢӨ.



## 1. мҷң Deploymentл§ҢмңјлЎңлҠ” л¶ҖмЎұн•ңк°Җ
> Kubernetesмқҳ кё°ліё DeploymentлҠ” RollingUpdateлҘј м ңкіөн•ҳм§Җл§Ң, нҠёлһҳн”Ҫ л¶„лҰ¬мҷҖ мһҗлҸҷ кІҖмҰқмқҖ л№ м ё мһҲлӢӨ.

кё°ліё DeploymentлҠ” мғҲ ReplicaSetмқ„ л§Ңл“Өкі  м җм§„м ҒмңјлЎң PodлҘј көҗмІҙн•ңлӢӨ. мқҙ нқҗлҰ„мқҖ лӢЁмҲңн•ҳм§Җл§Ң, вҖңмғҲ лІ„м „мқ„ 5%л§Ң л°ӣм•„ ліҙкі  5л¶„ лҸҷм•Ҳ м—җлҹ¬мңЁмқҙ лӮ®мқҖм§Җ нҷ•мқён•ң л’Ө 25%лЎң лҠҳлҰ°лӢӨвҖқмІҳлҹј мӢңк°„кіј нҠёлһҳн”Ҫ л№„мңЁмқ„ лӘ…мӢңм ҒмңјлЎң лӢӨлЈЁкё° м–ҙл өлӢӨ. мҡҙмҳҒм—җм„ң мӮ¬кі к°Җ лӮҳлҠ” мӢңм җмқҖ ліҙнҶө вҖңм „мІҙ көҗмІҙк°Җ лҒқлӮң м§Ғнӣ„вҖқмқёлҚ°, к·ёл•ҢлҠ” мқҙлҜё лҗҳлҸҢлҰ¬кё°м—җ лҠҰлӢӨ.

Argo RolloutsлҠ” мқҙ л№Ҳ мһҗлҰ¬лҘј л©”мҡҙлӢӨ. `Rollout` CRDлҠ” DeploymentлҘј лҢҖмІҙн•ҳлҠ” мӣҢнҒ¬лЎңл“ң лҰ¬мҶҢмҠӨлЎң, лӢЁкі„лі„ л№„мңЁ, мқјмӢң м •м§Җ, л©”нҠёлҰӯ кё°л°ҳ мһҗлҸҷ мҠ№кІ©, мһҗлҸҷ лЎӨл°ұмқ„ specмңјлЎң н‘ңнҳ„н•ңлӢӨ. ArgoCDлҠ” к·ё Rolloutмқ„ Gitм—җм„ң syncн•ҙ нҒҙлҹ¬мҠӨн„°м—җ м Ғмҡ©н•  лҝҗмқҙкі , мӢӨм ң вҖңлӢЁкі„лі„ көҗмІҙвҖқлҠ” `argo-rollouts-controller`к°Җ мұ…мһ„м§„лӢӨ.



## 2. ArgoCDмҷҖ Rolloutsмқҳ л¶„м—…
> л‘җ м»ЁнҠёлЎӨлҹ¬к°Җ лҸҷмӢңм—җ н•ң мӣҢнҒ¬лЎңл“ңлҘј л§Ңм§Җм§Җл§Ң мұ…мһ„ кІҪкі„лҠ” л¶„лӘ…н•ҳлӢӨ.

ArgoCD `application-controller`лҠ” Gitмқҳ `Rollout` л§ӨлӢҲнҺҳмҠӨнҠёмҷҖ нҒҙлҹ¬мҠӨн„° мғҒнғңлҘј л№„көҗн•ҙ OutOfSync/InSyncлҘј нҢҗм •н•ңлӢӨ. мҰү вҖңм§ҖкёҲ нҒҙлҹ¬мҠӨн„°м—җ мқҙ specмқҙ мһҲм–ҙм•ј н•ҳлҠ”к°ҖвҖқлҘј ліёлӢӨ. л°ҳл©ҙ `argo-rollouts-controller`лҠ” к·ё spec м•Ҳмқҳ лӢЁкі„ м •ліҙ(`steps:`)лҘј мқҪм–ҙ ReplicaSet л‘җ к°ңлҘј лқ„мҡ°кұ°лӮҳ, нҠёлһҳн”Ҫмқ„ м җм§„ мқҙлҸҷмӢңнӮӨкұ°лӮҳ, мқјмӢң м •м§Җн•ҳкұ°лӮҳ, AnalysisRunмқ„ л§Ңл“ лӢӨ.

м—¬кё°м„ң нқ”нһҲ н—·к°ҲлҰ¬лҠ” нҸ¬мқёнҠёлҠ” вҖңsync м§Ғнӣ„м—җ Applicationмқҙ Healthyк°Җ лҗҳм§Җ м•ҠлҠ”лӢӨвҖқлҠ” м җмқҙлӢӨ. ArgoCDк°Җ syncлҠ” м„ұкіөмӢңмј°м§Җл§Ң Rolloutмқҙ м•„м§Ғ м№ҙлӮҳлҰ¬ мӨ‘к°„ лӢЁкі„м—җ л©Ҳм¶° мһҲмңјл©ҙ, ArgoCDлҠ” к·ё мғҒнғңлҘј к·ёлҢҖлЎң вҖңProgressingвҖқмңјлЎң ліҙм—¬ мӨҖлӢӨ. кіөмӢқ л¬ём„ң кё°мӨҖмңјлЎң ArgoCDлҠ” `Rollout` health hookмқ„ к°Җм§Җкі  мһҲм–ҙ Rolloutмқҙ мҷ„м „нһҲ мҠ№кІ©лҗҳкұ°лӮҳ abortлҸјм•ј final statusк°Җ мһЎнһҢлӢӨ.

## 4. Blue-Green м „лһө
> л‘җ ReplicaSetмқ„ лҸҷмӢңм—җ лқ„мҡҙ л’Ө Service selectorлҘј н•ң лІҲм—җ м „нҷҳн•ңлӢӨ.

Blue-GreenмқҖ вҖңкё°мЎҙ лІ„м „(blue)кіј мғҲ лІ„м „(green)мқ„ лҸҷмӢңм—җ лқ„мҡ°кі , кІҖмҰқмқҙ лҒқлӮҳл©ҙ нҠёлһҳн”Ҫмқ„ н•ң лІҲм—җ greenмңјлЎң мҳ®кёҙлӢӨвҖқлҠ” нҢЁн„ҙмқҙлӢӨ. Serviceмқҳ selectorлҘј л°”кҫёлҠ” л°©мӢқмқҙлқј нҠёлһҳн”Ҫ л¶„лҰ¬лҠ” K8s Service н•ң к°ңмҷҖ лҜёлҰ¬ліҙкё°мҡ© Service н•ң к°ңлЎң кө¬нҳ„лҗңлӢӨ.

лӢӨмқҢ л§ӨлӢҲнҺҳмҠӨнҠёлҠ” лҜёлҰ¬ліҙкё° Service(`preview`)мҷҖ мӢӨм ң Service(`active`)лҘј л¶„лҰ¬н•ң мөңмҶҢ кө¬м„ұмқҙлӢӨ.

```yaml
# rollout-bluegreen.yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: web-bluegreen
spec:
  replicas: 4
  strategy:
    blueGreen:
      activeService: web-active
      previewService: web-preview
      autoPromotionEnabled: false       # мҡҙмҳҒмһҗк°Җ мҲҳлҸҷ promote
      scaleDownDelaySeconds: 300        # мҳӣ ReplicaSet 5л¶„ мң м§Җ
      prePromotionAnalysis:
        templates:
          - templateName: success-rate
        args:
          - name: service-name
            value: web-preview
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
        - name: web
          image: harbor.dev.trombone-v2.okestro.cloud/middleware/web:20260425-101500
          ports:
            - containerPort: 8080
```

мқҙ specмқ„ м Ғмҡ©н•ҳл©ҙ лӢӨмқҢ лӢЁкі„к°Җ мқјм–ҙлӮңлӢӨ.

1. green ReplicaSetмқҙ мғқм„ұлҗҳкі  Podк°Җ Readyк°Җ лҗҳл©ҙ `web-preview` Serviceк°Җ greenмқ„ к°ҖлҰ¬нӮЁлӢӨ.
2. `prePromotionAnalysis`мқҳ `success-rate` н…ңн”ҢлҰҝмқҙ 5xx л№„мңЁмқ„ кІҖмӮ¬н•ңлӢӨ. мһ„кі„м№ҳлҘј нҶөкіјн•ҳл©ҙ promote нӣ„ліҙк°Җ лҗңлӢӨ.
3. мҡҙмҳҒмһҗк°Җ `kubectl argo rollouts promote web-bluegreen`мқ„ мӢӨн–үн•ҳкұ°лӮҳ ArgoCD UIм—җм„ң promoteлҘј лҲ„лҘёлӢӨ.
4. `web-active` Serviceмқҳ selectorк°Җ greenмңјлЎң мҳ®кІЁм§Җкі , 5л¶„ л’Ө blue ReplicaSetмқҙ 0мңјлЎң мҠӨмјҖмқјлӢӨмҡҙлҗңлӢӨ.

`autoPromotionEnabled: false`лҘј л‘” мқҙмң лҠ” вҖңкІҖмҰқмқҖ мһҗлҸҷ, мҠ№кІ©мқҖ мӮ¬лһҢмқҳ мқҳмӮ¬кІ°м •вҖқмңјлЎң мұ…мһ„мқ„ лӮҳлҲ„кё° мң„н•ҙм„ңлӢӨ.



## 5. Canary м „лһө
> к°ҷмқҖ Service м•Ҳм—җм„ң нҠёлһҳн”Ҫ л№„мңЁмқ„ м җм§„ мқҙлҸҷмӢңнӮЁлӢӨ.

CanaryлҠ” мғҲ лІ„м „ Pod л№„мңЁмқ„ м җм§„м ҒмңјлЎң лҠҳлҰ¬л©° мқјм • лӢЁкі„л§ҲлӢӨ л©Ҳм¶° кІҖмҰқн•ҳлҠ” л°©мӢқмқҙлӢӨ. Blue-Greenкіј лӢ¬лҰ¬ Service selectorлҘј л°”кҫём§Җ м•Ҡкі , нҠёлһҳн”Ҫ лқјмҡ°нҢ… лҸ„кө¬(Ingress, Service Mesh)мҷҖ нҳ‘л Ҙн•ҙ л№„мңЁмқ„ мЎ°м •н•ңлӢӨ.

```yaml
# rollout-canary.yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: web-canary
spec:
  replicas: 10
  strategy:
    canary:
      canaryService: web-canary-svc
      stableService: web-stable-svc
      trafficRouting:
        nginx:
          stableIngress: web-ingress
      steps:
        - setWeight: 5
        - pause: { duration: 5m }
        - analysis:
            templates:
              - templateName: success-rate
            args:
              - name: service-name
                value: web-canary-svc
        - setWeight: 25
        - pause: { duration: 10m }
        - setWeight: 50
        - pause: { duration: 10m }
        - setWeight: 100
  selector:
    matchLabels:
      app: web
  template:
    spec:
      containers:
        - name: web
          image: harbor.dev.trombone-v2.okestro.cloud/middleware/web:20260425-101500
```

м—¬кё°м„ң `nginx.stableIngress`лҠ” Rolloutsк°Җ мһҗлҸҷмңјлЎң м№ҙлӮҳлҰ¬ м „мҡ© Ingress мӮ¬ліёмқ„ л§Ңл“Өкі  `nginx.ingress.kubernetes.io/canary-weight` annotationмңјлЎң л№„мңЁмқ„ мЎ°м •н•ҳкІҢ н•ңлӢӨлҠ” лң»мқҙлӢӨ. Service Mesh(Istio, Linkerd)лӮҳ ALB кё°л°ҳ нҷҳкІҪмқҙл©ҙ `trafficRouting` н•ҳмң„ нӮӨл§Ң л°”лҖҗлӢӨ.

к°Ғ лӢЁкі„к°Җ лҒқлӮҳлҠ” мЎ°кұҙмқҖ лӘ…мӢңм ҒмқҙлӢӨ. `setWeight`лҠ” weight мЎ°м • нӣ„ мҰүмӢң лӢӨмқҢ лӢЁкі„лЎң, `pause` мӨ‘ `duration`мқҙ м—Ҷмңјл©ҙ мӮ¬лһҢмқҙ promoteн•  л•Ңк№Ңм§Җ л¬ҙн•ң лҢҖкё°, `analysis`лҠ” AnalysisRunмқҙ Successfulмқҙ лҗ  л•Ңк№Ңм§Җ лӢӨмқҢ лӢЁкі„лЎң м•Ҳ л„ҳм–ҙк°„лӢӨ.



## 6. AnalysisTemplateмңјлЎң мһҗлҸҷ кІҖмҰқВ·лЎӨл°ұ
> Rollout лӢЁкі„ мӮ¬мқҙм—җ мӢӨм ң л©”нҠёлҰӯмқ„ лҒјмӣҢ л„Јм–ҙ вҖңм•Ҳм „ мһ„кі„м№ҳвҖқлҘј specмңјлЎң н‘ңнҳ„н•ңлӢӨ.

AnalysisTemplateмқҖ Prometheus, Datadog, New Relic, Wavefront, мһҗмІҙ Web/Job к°ҷмқҖ providerм—җ мҝјлҰ¬лҘј лҚҳм ё кІ°кіјлҘј л№„көҗн•ңлӢӨ. Rolloutмқҙ м–ҙл–Ө лӢЁкі„м—җм„ң мқҙ н…ңн”ҢлҰҝмқ„ нҳём¶ңн•ҳл©ҙ AnalysisRun мқёмҠӨн„ҙмҠӨк°Җ мғқм„ұлҗҳкі , кІ°кіјк°Җ `Successful`мқҙ м•„лӢҲл©ҙ Rollout м „мІҙк°Җ abortлҸј мһҗлҸҷмңјлЎң stableлЎң нҡҢк·Җн•ңлӢӨ.

```yaml
# analysis-success-rate.yaml
apiVersion: argoproj.io/v1alpha1
kind: AnalysisTemplate
metadata:
  name: success-rate
spec:
  args:
    - name: service-name
  metrics:
    - name: success-rate
      interval: 30s
      count: 5                          # 30s Г— 5 = 2.5л¶„ кҙҖмёЎ
      successCondition: result[0] >= 0.99
      failureLimit: 1
      provider:
        prometheus:
          address: http://prometheus.trb-mgm.svc:9090
          query: |
            sum(rate(http_requests_total{service="{{args.service-name}}",code!~"5.."}[1m]))
              /
            sum(rate(http_requests_total{service="{{args.service-name}}"}[1m]))
```

мқҙ н…ңн”ҢлҰҝмқҖ вҖңм§ҖлӮң 1л¶„к°„ 5xxк°Җ м•„лӢҢ мҡ”мІӯ л№„мңЁмқҙ 99% мқҙмғҒмқём§ҖвҖқлҘј 30мҙҲ к°„кІ©мңјлЎң 5нҡҢ ліёлӢӨ. н•ң лІҲмқҙлқјлҸ„ 99% лҜёл§Ңмқҙл©ҙ `failureLimit: 1`м—җ кұёл Ө Rollout abort. мқҙ spec н•ң мһҘмқҙ вҖңмҲҳлҸҷ лӘЁлӢҲн„°л§Ғ вҶ’ мҠ¬лһҷ вҶ’ мҡҙмҳҒмһҗ нҢҗлӢЁвҖқ нқҗлҰ„мқ„ мһҗлҸҷнҷ”н•ңлӢӨ.

мҡҙмҳҒ кҙҖм җм—җм„ң мЈјмқҳн•  м җмқҖ вҖңмҝјлҰ¬м—җ мғҲ лІ„м „ нҠёлһҳн”Ҫмқҙ м¶©л¶„нһҲ мһЎнҳҖ мһҲм–ҙм•ј н•ңлӢӨвҖқлҠ” м җмқҙлӢӨ. `setWeight: 1` м§Ғнӣ„ кі§л°”лЎң л¶„м„қмқ„ лҸҢлҰ¬л©ҙ н‘ңліёмқҙ л„Ҳл¬ҙ мһ‘м•„ кІ°кіјк°Җ нқ”л“ӨлҰ°лӢӨ. ліҙнҶө `setWeight` лӢӨмқҢ м§§мқҖ `pause`лЎң нҠёлһҳн”Ҫмқҙ м•Ҳм •лҗң л’Ө analysisлҘј л‘”лӢӨ.



## 7. Helm м°ЁнҠё кё°мӨҖ вҖ” м–ҙл–Ө нҢҢмқјмқҙ м–ҙл–Ө лҰ¬мҶҢмҠӨлҘј л§Ңл“ңлҠ”к°Җ
> Argo RolloutsлҠ” мһҗмІҙ Helm м°ЁнҠё(`argo-rollouts`)лҘј м ңкіөн•ҳкі , мӣҢнҒ¬лЎңл“ң м°ЁнҠёлҠ” к·ё мң„м—җ `Rollout` н…ңн”ҢлҰҝмқ„ м–№лҠ”лӢӨ.

м„Өм№ҳ м°ЁнҠё(`argoproj/argo-rollouts`)мҷҖ мӣҢнҒ¬лЎңл“ң м°ЁнҠё(мҳҲ: `tps-helm`)лҠ” м—ӯн• мқҙ лӢӨлҘҙлӢӨ. м„Өм№ҳ м°ЁнҠёлҠ” м»ЁнҠёлЎӨлҹ¬ мһҗмІҙлҘј лқ„мҡ°кі , мӣҢнҒ¬лЎңл“ң м°ЁнҠёлҠ” к°Ғ л§ҲмқҙнҒ¬лЎңм„ңл№„мҠӨмқҳ `Rollout` л§ӨлӢҲнҺҳмҠӨнҠёлҘј л§Ңл“ лӢӨ.

м„Өм№ҳ м°ЁнҠёмқҳ л””л үнҶ лҰ¬ кө¬мЎ°мҷҖ `helm install` мӢң л§Ңл“Өм–ҙм§ҖлҠ” K8s лҰ¬мҶҢмҠӨлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

```
argo-rollouts/                          # Helm chart лЈЁнҠё
в”ңв”Җв”Җ Chart.yaml                          # name, version, appVersion
в”ңв”Җв”Җ values.yaml                         # controller.replicas, dashboard.enabled, вҖҰ
в””в”Җв”Җ templates/
    в”ңв”Җв”Җ controller/
    в”Ӯ   в”ңв”Җв”Җ deployment.yaml             вҶ’ Deployment/argo-rollouts
    в”Ӯ   в”ңв”Җв”Җ serviceaccount.yaml         вҶ’ ServiceAccount
    в”Ӯ   в”ңв”Җв”Җ clusterrole.yaml            вҶ’ ClusterRole (Rollout/AnalysisRun к¶Ңн•ң)
    в”Ӯ   в””в”Җв”Җ clusterrolebinding.yaml
    в”ңв”Җв”Җ crds/                           вҶ’ CustomResourceDefinition Г— 6
    в”Ӯ   в”ңв”Җв”Җ rollout-crd.yaml            (Rollout)
    в”Ӯ   в”ңв”Җв”Җ analysisrun-crd.yaml        (AnalysisRun)
    в”Ӯ   в”ңв”Җв”Җ analysistemplate-crd.yaml   (AnalysisTemplate)
    в”Ӯ   в”ңв”Җв”Җ clusteranalysistemplate-crd.yaml
    в”Ӯ   в”ңв”Җв”Җ experiment-crd.yaml
    в”Ӯ   в””в”Җв”Җ plugin-crd.yaml
    в”ңв”Җв”Җ dashboard/                      вҶ’ Deployment + Service (м„ нғқ)
    в””в”Җв”Җ notifications/                  вҶ’ ConfigMap (м„ нғқ)
```

`helm template argo-rollouts argoproj/argo-rollouts -n argo-rollouts`мқ„ лҸҢлҰ¬л©ҙ мң„ нҠёлҰ¬к°Җ K8s лҰ¬мҶҢмҠӨ 6мў…(Deployment, ServiceAccount, ClusterRole, ClusterRoleBinding, CRD 6к°ң, м„ нғқм Ғ Dashboard Service)мңјлЎң нҺјміҗм§„лӢӨ. мӣҢнҒ¬лЎңл“ң мһ…мһҘм—җм„ң мқҳлҜё мһҲлҠ” кІ°кіјл¬јмқҖ Rollout/AnalysisTemplate CRD л“ұлЎқмқҙлӢӨ.

мӣҢнҒ¬лЎңл“ң м°ЁнҠё(`tps-helm` к°ҷмқҖ мҡ°мӮ° м°ЁнҠё)м—җм„ңлҠ” ліҙнҶө лӢӨмқҢ кө¬мЎ°лЎң `Rollout` н…ңн”ҢлҰҝмқ„ л‘”лӢӨ.

```
tps-helm/
в”ңв”Җв”Җ Chart.yaml
в”ңв”Җв”Җ values.yaml                         # кіөнҶө кё°ліёк°’
в”ңв”Җв”Җ values/
в”Ӯ   в”ңв”Җв”Җ values-ppp.yaml                 # 305P нҷҳкІҪ override
в”Ӯ   в”ңв”Җв”Җ values-dev.yaml
в”Ӯ   в”ңв”Җв”Җ values-prd.yaml
в”Ӯ   в””в”Җв”Җ values-bok.yaml
в””в”Җв”Җ templates/
    в”ңв”Җв”Җ _helpers.tpl
    в”ңв”Җв”Җ auth-api/
    в”Ӯ   в”ңв”Җв”Җ rollout.yaml                вҶ’ Rollout/auth-api
    в”Ӯ   в”ңв”Җв”Җ service-active.yaml         вҶ’ Service/auth-api
    в”Ӯ   в”ңв”Җв”Җ service-preview.yaml        вҶ’ Service/auth-api-preview
    в”Ӯ   в””в”Җв”Җ analysistemplate.yaml       вҶ’ AnalysisTemplate/auth-api-success-rate
    в””в”Җв”Җ pipeline-api/
        в””в”Җв”Җ вҖҰ
```

`values-ppp.yaml`мқҙ `auth-api.image.tag: 20260425-101500`мқ„ к°–кі  мһҲмңјл©ҙ, `helm template`мқҖ `templates/auth-api/rollout.yaml`мқҳ `{{ .Values.auth-api.image.tag }}` мһҗлҰ¬лҘј мұ„мӣҢ мөңмў… Rollout л§ӨлӢҲнҺҳмҠӨнҠёлҘј м¶ңл Ҙн•ңлӢӨ. ArgoCD `Application`мқҖ к·ё кІ°кіјлҘј нҒҙлҹ¬мҠӨн„°м—җ applyл§Ң н•ңлӢӨ.



## 8. ArgoCD ApplicationмңјлЎң Rollouts лҒјмӣҢ л„Јкё°
> Application н•ң к°ң м•Ҳм—җ Deployment лҢҖмӢ  Rolloutмқҙ мһҲлҠ” нҳ•нғңлӢӨ.

```yaml
# application-web.yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: web
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: default
  source:
    repoURL: https://bitbucket.org/okestrolab/tps_manifest.git
    targetRevision: main
    path: helm-charts/web
    helm:
      valueFiles:
        - values/values-ppp.yaml
  destination:
    server: https://kubernetes.default.svc
    namespace: trb-app
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=true
      - ApplyOutOfSyncOnly=true
```

`automated.selfHeal`мқҙ мјңм ё мһҲм–ҙлҸ„ Rollout м§„н–ү мӨ‘мқё лӢЁкі„к№Ңм§Җ мһҗлҸҷ syncк°Җ лҗҳлҸҢлҰ¬м§ҖлҠ” м•ҠлҠ”лӢӨ. ArgoCDлҠ” вҖңGitмқҳ specкіј нҒҙлҹ¬мҠӨн„°мқҳ specмқҙ к°ҷмқҖк°ҖвҖқлҘј ліҙм§Җ, вҖңнҠёлһҳн”Ҫ л№„мңЁмқҙ м–ҙлҠҗ лӢЁкі„мқёк°ҖвҖқк№Ңм§Җ л№„көҗн•ҳм§Җ м•Ҡкё° л•Ңл¬ёмқҙлӢӨ. мқҙ л¶„м—…мқҙ мһҗлҸҷ syncмҷҖ м җм§„ л°°нҸ¬лҘј лҸҷмӢңм—җ к°ҖлҠҘн•ҳкІҢ н•ңлӢӨ.



## 9. 305P мӢӨл¬ҙ мӮ¬лЎҖ
> 305P DEV(`dev.trombone-v2.okestro.cloud`)м—җлҠ” нҳ„мһ¬ Argo Rolloutsк°Җ лӘЁл“  м„ңл№„мҠӨм—җ м Ғмҡ©лҸј мһҲм§ҖлҠ” м•ҠлӢӨ. ліё м ҲмқҖ м Ғмҡ©н–Ҳмқ„ л•Ңмқҳ н‘ңмӨҖ нҢЁн„ҙмқ„ м •лҰ¬н•ңлӢӨ.

305P нҷҳкІҪмқҳ ArgoCDлҠ” `trb-oss/argocd`(v2.12.4)м—җ мң„м№ҳн•ҳкі , л§ӨлӢҲнҺҳмҠӨнҠё м ҖмһҘмҶҢлҠ” Bitbucket(`tps_manifest`)мқҙлӢӨ. нҸҗмҮ„л§қ мҡҙмҳҒ мқҙкҙҖмқ„ к°Җм •н•ҙ лӘЁл“  мқҙлҜём§ҖлҠ” Harbor `middleware` н”„лЎңм қнҠё(`harbor.dev.trombone-v2.okestro.cloud/middleware/*`) лҳҗлҠ” Nexus н”„лЎқмӢң(`nexus.okestro-k8s.com:55000/trombone/*`)м—җм„ңл§Ң лҒҢм–ҙмҳЁлӢӨ.

мқҙ нҷҳкІҪм—җ RolloutsлҘј лҸ„мһ…н•  л•Ң н‘ңмӨҖмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

1. `argo-rollouts` м°ЁнҠёлҠ” `trb-oss/argo-rollouts` л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ ArgoCD `Application`мңјлЎң лі„лҸ„ л¶ҖнҠёмҠӨнҠёлһ©н•ңлӢӨ(м»ЁнҠёлЎӨлҹ¬лҘј мһҗкё° мһҗмӢ мқҙ кҙҖлҰ¬н•ҳм§Җ м•ҠкІҢ н•ҳкё° мң„н•Ё).
2. к°Ғ л§ҲмқҙнҒ¬лЎңм„ңл№„мҠӨ(`auth-api`, `pipeline-api`, `workflow-api` л“ұ 12к°ң) Helm н…ңн”ҢлҰҝм—җм„ң `kind: Deployment`лҘј `kind: Rollout`мңјлЎң л°”кҫёкі , Canary 5%/25%/50%/100% steps + Prometheus(`prometheus.trb-mgm.svc:9090`) кё°л°ҳ AnalysisTemplateмқ„ л¶ҷмқёлӢӨ.
3. мқҙлҜём§Җ нғңк·ёлҠ” кё°мЎҙ Image Updater нқҗлҰ„мқ„ мң м§Җн•ңлӢӨ. Updaterк°Җ `values-ppp.yaml`м—җ мғҲ нғңк·ёлҘј commitн•ҳл©ҙ ArgoCDк°Җ syncн•ҳкі , Rolloutsк°Җ мһҗлҸҷмңјлЎң лӢЁкі„лі„ progressлҘј м§„н–үн•ңлӢӨ.
4. Notifications м»ЁнҠёлЎӨлҹ¬лЎң `RolloutAborted` мқҙлІӨнҠёлҘј мҠ¬лһҷ/л©”мқјлЎң м•ҢлҰ°лӢӨ(мһҗлҸҷ лЎӨл°ұмқҙ мқјм–ҙлӮ¬мқҢмқ„ мӮ¬лһҢмқҙ л№ лҘҙкІҢ мқём§Җн•ҳлҸ„лЎқ).

л№„л°ҖлІҲнҳё/нҶ нҒ° к°ҷмқҖ лҜјк°җ м •ліҙлҠ” `bitbucket-creds`, `harbor-creds`, `argocd-image-updater-secret`мңјлЎң л¶„лҰ¬н•ҙ ліёл¬ём—җм„ңлҠ” л§ҲмҠӨнӮ№н•ңлӢӨ. мӢӨм ң к°’мқҖ мқён”„лқј мҠӨнӮ¬ л¬ём„ңм—җм„ң кҙҖлҰ¬н•ңлӢӨ вҖ” `tps/infra/SKILL.md`, `references/14-v305p-environment.md`, `references/07-argocd-image-updater.md`лҘј м°ёмЎ°н•ңлӢӨ.



## 10. мҡҙмҳҒ мӢң мЈјмқҳ нҸ¬мқёнҠё
> м№ҙлӮҳлҰ¬лҠ” м•Ҳм „ мһҘм№ҳм§Җл§Ң мһҳлӘ» м„Өкі„н•ҳл©ҙ вҖңлҠҗлҰ¬кІҢ л§қн•ҳлҠ”вҖқ л°°нҸ¬к°Җ лҗңлӢӨ.

мҡҙмҳҒм—җм„ң к°ҖмһҘ нқ”нһҲ л§Ңл“ңлҠ” н•Ём •мқҖ лӢӨмқҢ м„ё к°Җм§ҖлӢӨ.

1. AnalysisTemplateмқҙ л„Ҳл¬ҙ лҠҗмҠЁн•ҳкІҢ мһЎнҳҖ мһҲм–ҙ мӢӨм ң мһҘм• м—җлҸ„ вҖңSuccessfulвҖқмқҙ л–Ём–ҙм§ҖлҠ” кІҪмҡ°. мһ„кі„м№ҳлҠ” м •мғҒ нҠёлһҳн”Ҫмқҳ лІ мқҙмҠӨлқјмқёмқ„ мёЎм •н•ң л’Ө кІ°м •н•ңлӢӨ.
2. `pause: {}`(л¬ҙн•ң лҢҖкё°)лҘј мҡҙмҳҒ мӨ‘к°„м—җ л‘җкі  мӮ¬лһҢмқҙ promoteлҘј мһҠлҠ” кІҪмҡ°. л¬ҙн•ң pauseлҠ” hotfix к°ҷмқҖ мҳҲмҷё лӢЁкі„м—җл§Ң м“°кі , мқјл°ҳ stepмқҖ `duration`мқ„ лӘ…мӢңн•ңлӢӨ.
3. ArgoCD `selfHeal: true`мҷҖ Rollout мқјмӢңм •м§Җк°Җ м¶©лҸҢн•ҳлҠ” кІғмІҳлҹј ліҙмқҙлҠ” кІҪмҡ°. мӢӨм ңлЎңлҠ” м¶©лҸҢн•ҳм§Җ м•Ҡм§Җл§Ң, мҡҙмҳҒмһҗк°Җ нҒҙлҹ¬мҠӨн„°м—җм„ң м§Ғм ‘ `kubectl edit rollout`мңјлЎң лӢЁкі„ л№„мңЁмқ„ л°”кҫёл©ҙ к·ё ліҖкІҪмқҖ Git кё°мӨҖ мғҒнғңмҷҖ лӢ¬лқј OutOfSyncк°Җ лҗңлӢӨ. лӢЁкі„ ліҖкІҪмқҖ н•ӯмғҒ Gitм—җм„ң н•ҳкі , мқјмӢң promote/abortлҠ” `kubectl argo rollouts` CLIлЎңл§Ң н•ңлӢӨ.
