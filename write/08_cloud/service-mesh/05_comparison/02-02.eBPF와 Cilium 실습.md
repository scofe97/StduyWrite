# eBPF와 Cilium 실습

> 본 장의 개념을 손으로 확인한다. 현재 실습 환경은 개인 GCP K8s 클러스터이며, 환경이 바뀌면 `## 실습 환경` 섹션만 갈아 끼우면 된다.


## 실습 환경` 섹션만 갈아 끼우면 된다.


## 실습 환경

현재 실습 대상은 `gcp` 스킬에 등록된 개인 클러스터다.

| 항목 | 값 | 역할 |
|------|-----|-----|
| dev-server | 4 vCPU / 8GB · asia-northeast3-a | Control Plane |
| dev-server-2 | 4 vCPU / 8GB | Worker |
| dev-server-3 | 4 vCPU / 8GB | Worker (주 워크로드) |
| K8s | kubeadm v1.31.14 | — |
| CNI | Flannel (10.244.0.0/16) | — |

접속은 `gcp` 스킬의 SSH 절차를 따른다. 상세는 [`.claude/skills/gcp/SKILL.md`](../../../../.claude/skills/gcp/SKILL.md) 참조.

> 주의: 현재 클러스터 CNI가 Flannel이면 Cilium CNI 관련 실습은 별도 테스트 클러스터에서 진행해야 한다. eBPF 개념 검증 및 Hubble 관측성 실습은 환경에 무관하게 수행 가능하다.


## 1. eBPF 환경 점검

### 1.1 커널 버전 및 eBPF 지원 확인

```bash
# 모든 노드의 커널 버전 조회
kubectl get nodes -o jsonpath='{range .items[*]}{.metadata.name}: {.status.nodeInfo.kernelVersion}{"\n"}{end}'

# 노드에 SSH 접속 후 eBPF 필수 기능 확인
uname -r

# eBPF 관련 커널 설정 확인
grep -E 'CONFIG_BPF|CONFIG_BPF_SYSCALL|CONFIG_BPF_JIT|CONFIG_CGROUP_BPF' /boot/config-$(uname -r)
# 모두 =y 또는 =m 이어야 함
```

### 1.2 cilium-preflight 사전 점검

Cilium 설치 전 필수 커널 기능 지원 여부를 확인한다.

```bash
# cilium-preflight DaemonSet 배포 (Cilium CLI가 있는 경우)
cilium install --dry-run-helm-values | grep -A5 preflight

# 또는 직접 preflight 체크 실행
kubectl apply -f https://raw.githubusercontent.com/cilium/cilium/main/install/kubernetes/quick-install.yaml --dry-run=client

# 수동 커널 기능 체크
cat /proc/sys/net/core/bpf_jit_enable
# 0이면 JIT 비활성화, 1 이상이어야 성능 발휘
```


## 2. eBPF 맵 조회 실습

### 2.1 bpftool로 로드된 eBPF 프로그램 확인

노드에 SSH 접속 후 실행한다.

```bash
# 로드된 eBPF 프로그램 목록 (Cilium이 설치된 노드)
sudo bpftool prog show

# 특정 프로그램의 바이트코드 확인
sudo bpftool prog dump xlated id <prog-id>

# 로드된 eBPF 맵 목록
sudo bpftool map show

# 특정 맵의 엔트리 조회
sudo bpftool map dump id <map-id>
```

### 2.2 iptables vs eBPF 규칙 수 비교

```bash
# 현재 iptables 규칙 수 확인 (kube-proxy 사용 시)
sudo iptables -L | wc -l
sudo iptables -t nat -L | wc -l

# 서비스 수 확인
kubectl get svc --all-namespaces | wc -l

# Cilium 사용 시 eBPF 맵 엔트리 수
# cilium bpf lb list | wc -l
```

규칙 수가 많을수록 iptables의 선형 평가 비용이 증가하는 반면, Cilium의 eBPF 해시 맵은 O(1) 조회를 유지한다.


## 3. Cilium 상태 확인 실습

Cilium이 설치된 환경에서 실행한다.

### 3.1 Cilium 에이전트 상태 확인

```bash
# Cilium 전체 상태 확인
cilium status

# 상세 상태 (컨트롤러, API 서버 연결 등)
cilium status --verbose

# 모든 노드의 Cilium 에이전트 파드 확인
kubectl get pods -n kube-system -l k8s-app=cilium -o wide

# 특정 에이전트 로그 확인
kubectl logs -n kube-system -l k8s-app=cilium --tail=50
```

### 3.2 엔드포인트 및 정책 상태

```bash
# 모든 엔드포인트 목록
cilium endpoint list

# 특정 엔드포인트의 정책 확인
cilium bpf policy get <endpoint-id>

# 연결 추적 테이블 확인
cilium bpf ct list global | head -20

# 서비스 로드밸런싱 맵 확인
cilium bpf lb list
```

### 3.3 연결성 테스트

```bash
# Cilium 기본 연결 테스트 (모든 기능 검증)
cilium connectivity test

# 특정 네임스페이스만 테스트
cilium connectivity test --namespace ch24-test

# 테스트 결과에서 실패 항목 확인
cilium connectivity test 2>&1 | grep -E 'FAIL|ERROR'
```


## 4. CiliumNetworkPolicy 실습

### 4.1 L4 정책 적용

```bash
# 테스트 네임스페이스 및 파드 생성
kubectl create namespace ch24-test

kubectl apply -n ch24-test -f - <<EOF
apiVersion: apps/v1
kind: Deployment
metadata:
  name: backend
spec:
  replicas: 1
  selector:
    matchLabels:
      app: backend
  template:
    metadata:
      labels:
        app: backend
    spec:
      containers:
      - name: nginx
        image: nginx:alpine
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: backend
spec:
  selector:
    app: backend
  ports:
  - port: 80
EOF

# L4 정책 적용 (checkout 레이블만 허용)
kubectl apply -n ch24-test -f - <<EOF
apiVersion: cilium.io/v2
kind: CiliumNetworkPolicy
metadata:
  name: allow-checkout-only
spec:
  endpointSelector:
    matchLabels:
      app: backend
  ingress:
    - fromEndpoints:
        - matchLabels:
            app: checkout
      toPorts:
        - ports:
            - port: "80"
              protocol: TCP
EOF
```

### 4.2 L7 정책 적용 및 Envoy 활성화 확인

```bash
# L7 규칙 포함 정책 (Envoy가 활성화됨)
kubectl apply -n ch24-test -f - <<EOF
apiVersion: cilium.io/v2
kind: CiliumNetworkPolicy
metadata:
  name: allow-api-only
spec:
  endpointSelector:
    matchLabels:
      app: backend
  ingress:
    - fromEndpoints:
        - matchLabels:
            app: checkout
      toPorts:
        - ports:
            - port: "80"
              protocol: TCP
          rules:
            http:
              - method: GET
                path: /api/.*
EOF

# L7 정책 적용 후 Envoy 프록시 확인
# L7 규칙이 있는 엔드포인트는 노드 Envoy를 경유함
cilium endpoint list | grep -i proxy
```

### 4.3 정책 연결성 확인

```bash
# checkout 레이블 파드로 접근 테스트 (허용)
kubectl run checkout-test -n ch24-test --rm -it \
  --labels="app=checkout" \
  --image=curlimages/curl:latest -- \
  curl -s http://backend/api/items

# 허용되지 않은 파드로 접근 테스트 (차단)
kubectl run unauthorized-test -n ch24-test --rm -it \
  --labels="app=other" \
  --image=curlimages/curl:latest -- \
  curl -s --connect-timeout 3 http://backend/
# Connection refused 또는 타임아웃 발생해야 함
```


## 5. Hubble 관측성 실습

### 5.1 Hubble 활성화 및 CLI 설치

```bash
# Hubble 활성화 (Cilium이 이미 설치된 경우)
cilium hubble enable

# Hubble CLI 포트포워딩
cilium hubble port-forward &

# Hubble 상태 확인
hubble status
```

### 5.2 트래픽 흐름 관측

```bash
# 특정 파드의 모든 트래픽 실시간 관측
hubble observe --pod ch24-test/backend --follow

# HTTP 5xx 에러만 필터링
hubble observe --http-status 5xx --last 100

# 서비스 간 흐름 (네임스페이스/서비스 형식)
hubble observe \
  --from-pod ch24-test/checkout-test \
  --to-pod ch24-test/backend

# 차단된 트래픽만 확인
hubble observe --verdict DROPPED --last 50

# JSON 출력으로 L7 상세 확인
hubble observe --pod ch24-test/backend \
  --protocol HTTP \
  --output json | jq '.flow.l7'
```

### 5.3 Hubble UI 접속

```bash
# Hubble UI 포트포워딩
cilium hubble ui

# 브라우저에서 http://localhost:12000 접속
# 서비스 의존성 그래프, 실시간 트래픽 흐름 확인
```


## 6. eBPF 디버깅 실습

### 6.1 패킷 드롭 원인 분석

```bash
# cilium monitor로 eBPF 이벤트 실시간 확인
cilium monitor --type drop

# 드롭 이유별 통계
cilium monitor --type drop 2>&1 | grep "drop reason" | sort | uniq -c | sort -rn

# Hubble로 드롭 필터링
hubble observe --verdict DROPPED --last 100

# 특정 드롭 이유 설명
# DROP_REASON_POLICY: 정책에 의해 차단
# DROP_NO_ENDPOINT: 목적지 엔드포인트 없음
# DROP_INVALID_PACKET: 잘못된 패킷
```

### 6.2 연결 추적 테이블 디버깅

```bash
# 특정 소스 IP의 연결 추적 확인
cilium bpf ct list global | grep <pod-ip>

# 연결 추적 만료 설정 확인
cilium config | grep ct-global

# bpftool로 맵 직접 조회 (노드 SSH)
sudo bpftool map dump pinned /sys/fs/bpf/cilium_ct4_global | head -20
```
