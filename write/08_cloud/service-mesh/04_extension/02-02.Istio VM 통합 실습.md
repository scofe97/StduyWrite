# Istio VM 통합 실습

> 본 장의 개념을 손으로 확인한다. 현재 실습 환경은 개인 GCP K8s 클러스터이며, 환경이 바뀌면 `## 실습 환경` 섹션만 갈아 끼우면 된다.


## 실습 환경` 섹션만 갈아 끼우면 된다.


## 실습 환경

현재 실습 대상은 `gcp` 스킬에 등록된 개인 클러스터다. 구성은 다음과 같다.

| 항목 | 값 | 역할 |
|------|-----|-----|
| dev-server | 4 vCPU / 8GB · asia-northeast3-a | Control Plane |
| dev-server-2 | 4 vCPU / 8GB | Worker |
| dev-server-3 | 4 vCPU / 8GB | Worker (주 워크로드) |
| K8s | kubeadm v1.31.14 | — |
| CNI | Flannel (10.244.0.0/16) | — |

VM 통합 실습은 GCP VM 인스턴스(dev-server-2 또는 별도 VM)를 Istio 메시에 편입시키는 방식으로 진행한다. VM이 없으면 실습 1~3은 리소스 정의 검증 수준으로 수행하고, 실습 4(실제 sidecar 설치)는 VM 준비 후 진행한다. 상세는 [`.claude/skills/gcp/SKILL.md`](../../../../.claude/skills/gcp/SKILL.md) 참조.


## 실습 1. WorkloadEntry와 ServiceEntry 정의

### 1-1. 네임스페이스 및 서비스 계정 준비

```bash
kubectl create namespace ch21-vm

kubectl apply -n ch21-vm -f - <<EOF
apiVersion: v1
kind: ServiceAccount
metadata:
  name: legacy-api-sa
EOF
```

### 1-2. WorkloadGroup 생성

```bash
kubectl apply -n ch21-vm -f - <<EOF
apiVersion: networking.istio.io/v1beta1
kind: WorkloadGroup
metadata:
  name: legacy-api-group
spec:
  metadata:
    labels:
      app: legacy-api
  template:
    serviceAccount: legacy-api-sa
    network: vm-network
  probe:
    httpGet:
      port: 8080
      path: /health
    initialDelaySeconds: 5
    periodSeconds: 10
EOF

kubectl get workloadgroup -n ch21-vm
```

### 1-3. ServiceEntry 생성

```bash
kubectl apply -n ch21-vm -f - <<EOF
apiVersion: networking.istio.io/v1beta1
kind: ServiceEntry
metadata:
  name: legacy-api-se
spec:
  hosts:
  - legacy-api.ch21-vm.svc.cluster.local
  ports:
  - number: 8080
    name: http
    protocol: HTTP
  resolution: STATIC
  workloadSelector:
    labels:
      app: legacy-api
EOF

kubectl get serviceentry -n ch21-vm
```


## 실습 2. VM용 설정 파일 생성

### 2-1. istioctl로 VM 온보딩 파일 생성

```bash
mkdir -p /tmp/vm-files

istioctl x workload entry configure \
  -f <(kubectl get workloadgroup -n ch21-vm legacy-api-group -o yaml) \
  -o /tmp/vm-files \
  --clusterID Kubernetes \
  --namespace ch21-vm

ls /tmp/vm-files/
# 출력: cluster.env  hosts  istio-token  root-cert.pem
```

### 2-2. 생성된 파일 내용 확인

```bash
cat /tmp/vm-files/cluster.env
# ISTIO_SERVICE_ACCOUNT, ISTIO_PILOT_PORT, ISTIO_INBOUND_PORTS 등 확인
---

cat /tmp/vm-files/hosts
# K8s 서비스 DNS 엔트리 확인
```


## 실습 3. WorkloadEntry 수동 등록

autoRegistration 없이 WorkloadEntry를 수동으로 등록하는 방법이다.

```bash
kubectl apply -n ch21-vm -f - <<EOF
apiVersion: networking.istio.io/v1beta1
kind: WorkloadEntry
metadata:
  name: vm-legacy-api-001
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "15090"
    prometheus.io/path: "/stats/prometheus"
spec:
  address: "10.178.0.3"
  labels:
    app: legacy-api
    version: v1
  serviceAccount: legacy-api-sa
  network: vm-network
EOF

kubectl get workloadentry -n ch21-vm
```


## 실습 4. VM에 Istio sidecar 설치 (VM 준비 시)

GCP VM에 SSH 접속 후 아래 절차를 수행한다.

```bash
# GCP VM (dev-server-2 또는 별도 인스턴스)에서 실행
---

# K8s 클러스터에서 생성한 파일 복사 (로컬에서 실행)
gcloud compute scp /tmp/vm-files/* <VM_NAME>:/tmp/ --zone=asia-northeast3-a

# VM에서 실행
# Istio sidecar 패키지 다운로드 (버전은 클러스터 Istio 버전과 맞춤)
ISTIO_VERSION=$(istioctl version --remote=false -o json | python3 -c "import json,sys; print(json.load(sys.stdin)['clientVersion']['version'])")
curl -LO "https://storage.googleapis.com/istio-release/releases/${ISTIO_VERSION}/deb/istio-sidecar.deb"
sudo dpkg -i istio-sidecar.deb

# 설정 파일 배치
sudo cp /tmp/cluster.env /var/lib/istio/envoy/
sudo cp /tmp/root-cert.pem /etc/certs/root-cert.pem
sudo mkdir -p /var/run/secrets/tokens
sudo cp /tmp/istio-token /var/run/secrets/tokens/istio-token
sudo cat /tmp/hosts >> /etc/hosts

# sidecar 서비스 시작
sudo systemctl start istio
sudo systemctl status istio

# 인증서 발급 확인
sudo openssl x509 -in /etc/certs/cert-chain.pem -noout -subject
# URI:spiffe://cluster.local/ns/ch21-vm/sa/legacy-api-sa 확인
```


## 실습 5. K8s ↔ VM 양방향 통신 검증

### 5-1. K8s에서 VM으로

```bash
# K8s Pod 배포
kubectl apply -n ch21-vm -f - <<EOF
apiVersion: v1
kind: Pod
metadata:
  name: client-pod
  labels:
    app: client
spec:
  containers:
  - name: curl
    image: curlimages/curl:latest
    command: ["/bin/sleep", "infinity"]
EOF

# VM의 서비스 DNS로 호출
kubectl exec -n ch21-vm client-pod -- \
  curl -s http://legacy-api.ch21-vm.svc.cluster.local:8080/health
```

### 5-2. mTLS 적용 확인

```bash
# PeerAuthentication을 STRICT로 설정
kubectl apply -n ch21-vm -f - <<EOF
apiVersion: security.istio.io/v1beta1
kind: PeerAuthentication
metadata:
  name: default
spec:
  mtls:
    mode: STRICT
EOF

# VM과의 mTLS 상태 확인
istioctl proxy-config secret client-pod.ch21-vm
```

### 5-3. 실습 정리

```bash
kubectl delete namespace ch21-vm
```
