# JSONPath와 kubectl 고급 조회 점검

> 본 장의 심화 점검 질문입니다. 자주 쓰는 JSONPath 패턴과 `jq`와의 경계를 운영 질문으로 다시 정리했습니다.



## 점검 질문
> 사람이 읽는 출력과 스크립트용 출력을 구분하는 습관을 점검합니다.

### Q1. JSONPath로 가장 먼저 익혀야 할 패턴은 무엇인가?

반복 조회용 `range ... end`, 탭 구분자, 줄바꿈 세 가지입니다. 이것만 익혀도 Pod 목록, 노드 IP, 재시작 횟수 같은 조회를 빠르게 만들 수 있습니다.

### Q2. JSONPath가 안 되는 일은 무엇인가?

공식 문서 기준으로 정규식은 지원하지 않습니다. 그래서 이름 패턴 매칭이나 복잡한 필터링이 필요하면 `-o json | jq ...`가 더 적합합니다.

### Q3. 필드 경로를 모를 때 어떻게 찾는가?

`kubectl get ... -o yaml`로 실제 구조를 보고, `kubectl explain`으로 필드 의미를 확인한 뒤 JSONPath를 짜는 편이 가장 빠릅니다. 처음부터 머리로 경로를 맞히려 하면 오히려 오래 걸립니다.
