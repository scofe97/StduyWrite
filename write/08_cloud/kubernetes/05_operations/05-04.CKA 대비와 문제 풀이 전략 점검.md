# CKA 대비와 문제 풀이 전략 점검

> 본 장의 심화 점검 질문이다. CKA를 실무 문서와 연결해 준비할 때 어디에 시간을 먼저 써야 하는지 Q&A 형태로 정리했다.



## 점검 질문
> 시험 준비를 별도 암기집이 아니라 운영 루틴 압축으로 다시 점검한다.

### Q1. CKA에서 가장 먼저 빨라져야 하는 것은 무엇인가?

`kubectl get`, `describe`, `logs`, `exec`, `apply`, `edit` 같은 기본 명령이다. 문제를 많이 아는 것보다 기본 명령을 빠르게 쓰는 쪽이 점수에 더 직접적으로 연결된다.

### Q2. JSONPath를 시험 준비에 왜 넣는가?

반복 조회를 줄이고 필요한 필드만 빠르게 뽑을 수 있기 때문이다. 다만 JSONPath 자체를 암기 과목으로 보기보다, 실무에서 자주 쓰는 짧은 스니펫 몇 개를 손에 익히는 편이 더 효율적이다.

### Q3. 이 문서 세트로 어떻게 복습하는가?

핵심 워크로드, 네트워킹, 스토리지, 보안, 운영 심화 순으로 한 번 돌리고, 각 deepdive에서 명령과 체크리스트만 다시 보는 방식이 가장 자연스럽다.
