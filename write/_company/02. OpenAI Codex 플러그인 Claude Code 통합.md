# OpenAI Codex 플러그인 — Claude Code 통합

> 작성일: 2026-03-31
> 출처: YouTube 자막 정리

---

## 1. 배경

OpenAI가 Claude Code 안에서 GPT 모델을 바로 사용할 수 있는 공식 플러그인을 출시했다. 레포지토리는 OpenAI 공식 계정 아래 `codex-plugin-cc`라는 이름으로 공개되어 있으며, Claude Code 마켓플레이스에 정식 등록되어 있다.

OpenAI 직원들이 직접 만든 플러그인이라는 점이 흥미롭다. 공식 입장은 "어차피 Claude Code 쓰면서 GPT도 같이 쓰고 싶어 했으니, 번거롭게 직접 플러그인 만들지 말고 우리가 공식으로 지원하겠다"는 것이다. Claude Code가 코딩 에이전트 시장을 장악한 상황에서, 그 생태계에 편승하여 시장을 확장하려는 전략으로 읽힌다. 자사 제품인 Codex의 존재감을 Claude Code 사용자에게 노출시키는 효과도 있다.

## 2. 주요 기능

현재 플러그인이 제공하는 기능은 다음과 같다. 향후 추가 기능이 계속 늘어날 것으로 예상된다.

- **review**: Codex에게 현재 코드 리뷰를 요청한다. 백그라운드 태스크로 실행되며, 완료 후 결과를 공유한다.
- **rescue**: 작업 복구 기능이다.
- **status**: 백그라운드에서 돌고 있는 Codex 작업의 진행 상황을 확인한다.
- **result**: 완료된 작업의 결과를 가져온다.
- **cancel**: 진행 중인 작업을 취소한다.

## 3. 설치 및 설정

설치 과정은 세 단계로 구성된다.

첫째, 마켓플레이스 등록이다. Claude Code에서 `/plugins` 명령으로 마켓플레이스에 접근하면 OpenAI Codex 플러그인이 보인다. 공식 레포지토리에서 제공하는 설치 명령어를 복사하여 `claude permissions` 설정에 붙여넣으면 마켓플레이스에 등록된다.

둘째, 플러그인 설치다. `/plugin install`로 Codex 플러그인을 설치한다. 설치 범위는 두 가지 중 선택할 수 있다. `install for you`는 개인 스코프로 설치하는 방식이고, `install for all collaborators in this repo`는 프로젝트 스코프로 설치하여 커밋에 포함시키는 방식이다. 프로젝트 스코프를 선택하면 다른 환경에서는 사용할 수 없으므로 상황에 맞게 결정한다.

셋째, 인증 확인이다. `/codex setup`을 실행하면 "ready to go" 메시지가 나온다. 기존에 Codex를 설치해 둔 상태라면 해당 API 키를 그대로 활용하므로 별도 로그인이 필요 없다. OAuth 방식이 아닌 공식 지원이기 때문에 밴 걱정도 없다.

## 4. 실전 데모: Golden Gate Bridge 3D

데모에서는 Claude Opus로 Golden Gate Bridge 3D 모델링을 생성한 뒤, Codex로 리뷰하여 품질을 개선하는 워크플로를 보여준다.

Opus에게 maximum effort 모드로 3D 모델링을 요청하자 22분간 이터레이션을 돌며 결과를 만들어냈다. 물리 엔진에 문제가 있어 자동차가 브릿지 중간을 뚫고 지나가는 현상이 있었지만, 바다의 텍스처 표현은 상당히 사실적이었다.

이 상태에서 `/codex review`를 실행하여 Codex에게 리뷰를 요청했다. "Golden Gate Bridge 3D 모델링을 했는데 더 리얼하게 개선할 수 있는지, 잘못된 부분은 없는지 확인해 달라"는 프롬프트를 보냈다. 리뷰는 백그라운드 태스크로 실행되었고, `/codex status`로 진행 상황을 확인할 수 있었다.

리뷰 결과는 Critical, Major, Minor 등급으로 분류된 총 16건의 수정 사항이었다. 이를 다시 Opus에게 전달하여 수정을 요청하자, 물리 엔진 버그가 해결되어 자동차가 브릿지를 정상적으로 통과하게 되었다. 비행 컨트롤 기능이 추가되었고, 바다 색이 더 밝아지는 등 전반적인 품질이 향상되었다.

## 5. 시장 전략 분석

이 플러그인은 OpenAI의 실용적인 시장 전략을 보여준다. Claude Code가 코딩 에이전트 도구 시장에서 지배적 위치를 차지한 현실을 인정하고, 경쟁하는 대신 그 생태계 안에서 GPT의 활용도를 높이는 방향을 택한 것이다.

실제 사용 관점에서 보면, 하나의 모델이 생성한 결과물을 다른 모델이 리뷰하는 워크플로는 "이거 진짜 맞아?"라는 검증 질문을 던질 수 있는 상대가 생기는 셈이다. GPT 구독을 이미 보유한 사용자라면 추가 비용 없이 크로스 모델 리뷰를 활용할 수 있어 실질적인 가치가 있다.

## 6. 활용 시나리오

Codex 플러그인의 가치는 단순히 "GPT도 쓸 수 있다"가 아니라, 서로 다른 모델의 블라인드스팟을 보완하는 크로스 모델 워크플로에 있다.

가장 강력한 기능은 **Review Gate**다. `/codex:setup --enable-review-gate`로 활성화하면, Claude가 응답을 완성하기 전에 Codex가 자동으로 리뷰하고 문제가 있으면 응답을 차단한다. Claude와 Codex 사이에 피드백 루프가 자동으로 형성되는 것이다. 단, 양쪽 모델의 사용량이 빠르게 소진되므로 적극적으로 모니터링하는 세션에서만 활성화해야 한다.

`/codex:adversarial-review`는 일반 리뷰와 결이 다르다. 단순히 "버그가 있는가"를 넘어서 설계 결정, 트레이드오프, 숨겨진 가정을 도전적으로 질문한다. 같은 모델의 셀프 리뷰와 구조적으로 다른 관점을 제공하므로 아키텍처 결정 검증에 효과적이다.

`/codex:rescue`는 버그 조사나 수정 시도를 Codex에게 백그라운드 태스크로 위임한다. Claude로 메인 작업을 계속하면서 동시에 Codex가 별도 작업을 수행하는 병렬 워크플로가 가능하다.

적합한 시나리오와 그렇지 않은 시나리오를 구분하면 다음과 같다.

| 시나리오 | 적합 여부 | 이유 |
|---------|----------|------|
| 출시 전 코드 리뷰 | 적합 | 다른 모델 관점으로 블라인드스팟 검출 |
| 복잡한 버그 디버깅 | 적합 | 하나가 막히면 다른 모델에 위임 가능 |
| 보안 감사 | 적합 | 모델 간 합의가 단일 모델보다 신뢰도 높음 |
| 아키텍처 결정 검증 | 적합 | adversarial review로 설계 도전 |
| 단순 CRUD 구현 | 비적합 | 비용 대비 이점 없음 |
| 빠른 프로토타이핑 | 비적합 | 지연 시간이 이터레이션 속도를 떨어뜨림 |
| 모든 코딩 루프 적용 | 비적합 | 양쪽 사용량 과다 소진 |

## 7. OpenCode와의 비교

OpenCode는 하나의 터미널에서 여러 LLM 제공자를 교체하며 사용하는 오픈소스 CLI 도구였다. Go로 작성되었고, Claude, GPT, Gemini, Groq 등을 설정으로 전환할 수 있었다. 현재는 프로젝트가 아카이브되어 Charmbracelet 팀의 **Crush**로 계승되었다.

핵심 차이는 "멀티 프로바이더 전환"과 "협업 구조"의 구분이다. OpenCode는 한 번에 하나의 모델을 선택하여 사용하는 구조다. GPT로 작업하다가 Claude로 전환하는 식이다. 반면 Codex 플러그인은 Claude가 메인이고 Codex가 리뷰어 또는 보조 역할을 맡는 협업 구조다. 두 모델이 동시에 동작한다.

| 비교 항목 | OpenCode/Crush | Codex 플러그인 |
|----------|---------------|---------------|
| 구조 | 멀티 프로바이더 전환 | Claude 메인 + Codex 보조 |
| 동시 사용 | 한 번에 하나 | 두 모델 동시 병렬 |
| 생태계 | 독립 CLI 도구 | Claude Code 플러그인 체계 |
| 리뷰 게이트 | 없음 | Stop Hook으로 자동 리뷰 |
| 라이선스 | MIT (오픈소스) | 오픈소스 (GitHub 공개) |

OpenCode처럼 "GPT로 전환"하는 것이 아니라, Claude가 주도하면서 GPT에게 특정 역할을 맡기는 협업 모델이라는 점이 근본적인 차이다.
