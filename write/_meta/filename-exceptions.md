# 파일명 규칙 의도된 예외
---
> 하네스 §10의 회귀 세트 검증에서 걸리는 파일 중, 카테고리 성격상 규칙을 완화하는 예외를 여기에 기록한다.

## 완화 대상 디렉토리

- `write/_archive/**` — 역사적 기록. 파일명 규칙 완화 (이미 하네스 §1이 정의)
- `write/_company/**` — `.gitignore` 제외. 규칙 적용 안 함
- `write/99_ETC/**` — 3개월 보류 공간. 재배치 전까지 원본 파일명 유지 허용

## Step D-2 시점 (2026-04-19) 예외 집계

정규 패턴 `^[0-9]{2}-[0-9]{2}\.`와 불일치하지만 의도적으로 유지하는 파일 총 55개.

### 08_devops/02_Jenkins/ — 27개

세 가지 하위 패턴이 혼재한다.

첫째, 본편 + 심화 시리즈 구조를 따르는 `05-0N` + `05-0Na/b/c` 패턴이다. `05-04.젠킨스 빌드 실행·큐 API 스펙.md`가 본편이고, `05-04a.…모델과 TPS 패턴.md`, `05-04b.…Queue 적재 이후`, `05-04c.…큐 내부 흐름`이 심화다. 원래 하네스는 `-x` 접미사를 금지하지만 `-a/b/c`는 명시 규정이 없다. Jenkins API처럼 한 주제에서 원본 스펙·현대화·실전 패턴을 나란히 두는 경우 이 방식이 장별 증분 번호보다 관계를 잘 드러낸다. 추후 하네스 §5에 "심화 시리즈 접미사 `-a/b/c` 허용" 조항을 정식화할 후보다.

둘째, `NN-점검.핵심 질문과 답.md` 형식의 복습 문서 9개가 있다. 각 장의 끝에서 Q&A 형식으로 개념을 검증하는 역할이다. 현재는 절 번호가 없어 규칙과 충돌한다. 다음 단계로 `NN-99.점검.md` 또는 `NN-00.점검.md`로 개명하는 것이 적절해 보이지만, 이번 단계에서는 Jenkins 영역 전체를 흔들지 않기로 결정했다.

셋째, `09-0Na` 패턴(예: `09-01a.공유 라이브러리 실전 패턴.md`) — 첫 번째 패턴과 같은 취지다.

### 99_ETC/ — 18개

분류 보류 공간이므로 의도적으로 규칙을 완화한다. 대부분은 원래 카테고리가 정해지면 함께 개명될 예정이다.

- `01-jenkins-analysis.md` 등 하이픈 구분 영문 제목 파일 8개 — TPS 분석 문서. 재배치 후보지는 `08_devops/02_Jenkins/_archive/` 또는 별도 분석 카테고리.
- `03-reconciliation-genealogy.md`, `06-harness-engineering.md` — 메타 성격. `_meta/`로 이관 후보.
- `manifest.md`, `TodoList.md`, `message-lib 스펙.md` — 번호 prefix 자체가 없다. 재배치 시 번호 부여 필요.
- `분석/` 하위 8개 — 배치 분석 시리즈. 재배치 대상이 확정되지 않아 보류.

### 05_messaging/ — 10개

두 하위 폴더에 `0x-XX` 형식의 초안이 남아 있다.

- `02_DAG/0x-01.DAG 엔진 아키텍처와 구현.md` 등 4개
- `03_SpringPractice/0x-0x.trace-id와 traceparent.md` 등 6개

`0x`는 "절 번호가 아직 결정되지 않음"을 표시하는 내부 컨벤션이다. 작성 중 흐름이 자주 바뀌어 고정 번호를 내리지 못하는 초안에 사용한다. `status: draft`와 함께 정규 번호 부여가 완료되면 예외에서 빠진다.
