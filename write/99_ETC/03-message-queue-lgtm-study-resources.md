# 메시지 큐 & LGTM 모니터링 학습 자료 조사

메시지 큐(Kafka/Redpanda)를 활용한 미들웨어 통신 관리와 LGTM(Loki+Grafana+Tempo+Mimir) 모니터링 스택은 현대 분산 시스템의 두 축이다. 메시지 큐는 서비스 간 순서 보장, 재시도, 실패 로깅을 담당하고, LGTM 스택은 로그·메트릭·트레이스를 통합 수집하여 시스템 상태를 관측한다. 이 문서는 두 영역의 한국어 학습 자료를 조사하여 정리한 결과다.

---

## 1. 메시지 큐 관련 도서

### 1.1 입문서

**카프카 핵심 가이드**(Kafka: The Definitive Guide, 2판)는 Confluent 공동 창립자들이 집필한 공식 레퍼런스에 가깝다. 프로듀서·컨슈머 내부 구조부터 파티션 전략, 트랜잭션까지 폭넓게 다루며 한국어 번역판이 한빛미디어에서 출간되어 있다. Kafka를 처음 접하는 개발자가 아키텍처 전반을 잡기에 적합하다.

**실전 카프카 개발부터 운영까지**(고승범 저, 책만)는 국내 저자가 쓴 실무 중심 입문서다. 로컬 클러스터 구축부터 프로듀서·컨슈머 예제, 카프카 커넥트, 스키마 레지스트리까지 한 권에 담았다. 번역서가 아니라 한국 실무 환경에 맞춘 설명이 특징이다.

### 1.2 심화서

**카프카 스트림즈와 ksqlDB**(Mitch Seymour 저, 한빛미디어)는 스트림 처리에 집중한다. Kafka Streams DSL과 Processor API, ksqlDB를 활용한 실시간 파이프라인 구축을 단계별로 안내한다. 메시지 큐 기초를 넘어 이벤트 스트리밍 아키텍처로 확장하려는 단계에서 읽으면 좋다.

**이벤트 기반 마이크로서비스 구축**(Adam Bellemare 저, 한빛미디어)은 메시지 큐를 기반으로 한 마이크로서비스 아키텍처를 다룬다. 이벤트 소싱, CQRS, Saga 패턴 등 메시지 큐 위에 올라가는 설계 패턴을 체계적으로 설명한다.

### 1.3 아키텍처 레퍼런스

**데이터 중심 애플리케이션 설계**(Martin Kleppmann 저, 위키북스)는 메시지 큐 전용서는 아니지만, 분산 시스템에서 메시지 전달 보장(at-least-once, exactly-once), 로그 기반 메시지 브로커의 원리를 깊이 있게 다룬다. 11장 "스트림 처리"는 Kafka의 설계 철학을 이해하는 데 필수 참고 자료다.

## 2. LGTM 모니터링 관련 도서

### 2.1 Grafana 중심

**그라파나 — 오픈소스 기반 모니터링 시스템**(이병용 외 저, 에이콘)은 국내에서 Grafana를 전면적으로 다루는 몇 안 되는 도서다. 대시보드 설계, 데이터소스 연동, 알림 설정을 실습 위주로 진행한다. LGTM 스택의 시각화 계층을 먼저 익히는 데 적합하다.

### 2.2 관측 가능성(Observability) 전반

**Observability Engineering**(Charity Majors 외 저, O'Reilly)은 한국어 번역판이 아직 없지만, 관측 가능성의 세 기둥(로그·메트릭·트레이스)을 정의하고 실무 적용 전략을 다루는 표준 레퍼런스다. LGTM 스택을 "왜" 도입하는지 맥락을 잡는 데 도움이 된다.

**프로메테우스 인프라스트럭처 모니터링**(조엘 바스토스 외 저, 에이콘)은 Prometheus 중심이지만 Grafana 연동, 알림 파이프라인 설계를 함께 다룬다. Mimir가 Prometheus 호환 장기 저장소이므로, Prometheus 기초를 먼저 잡으면 Mimir 학습이 자연스럽게 이어진다.

### 2.3 로깅·분산 추적

LGTM 스택에서 Loki(로그)와 Tempo(트레이스)를 전면으로 다루는 한국어 도서는 2026년 3월 기준 출간되지 않았다. Loki는 LogQL 쿼리, 라벨 설계, 청크 스토리지 구조가 핵심이고, Tempo는 OpenTelemetry Collector와의 연동이 관건인데, 둘 다 공식 문서와 블로그가 주요 학습 경로다.

## 3. 블로그 및 온라인 자료

### 3.1 메시지 큐

- **Confluent Blog** — Kafka 창시팀이 운영하는 공식 블로그로, 신규 기능 해설과 아키텍처 패턴 글이 꾸준히 올라온다
- **Redpanda Blog** — Kafka 호환 브로커 관점에서 성능 비교, WASM 변환, 스키마 레지스트리 활용법을 다룬다
- **카카오 기술 블로그** — "카프카" 검색 시 대규모 클러스터 운영기, 파티션 리밸런싱, 모니터링 사례가 나온다
- **라인 엔지니어링** — 대용량 메시지 처리, Kafka Streams 실전 적용기를 한국어로 공유한다
- **우아한형제들 기술 블로그** — 이벤트 기반 아키텍처 전환기, Kafka 기반 주문 시스템 사례가 있다

### 3.2 LGTM 모니터링

- **Grafana Labs Blog** — Loki, Tempo, Mimir 각 컴포넌트의 공식 튜토리얼과 릴리스 노트를 제공한다
- **Grafana Labs 공식 문서** — "Get started with LGTM" 가이드가 Docker Compose 기반 올인원 환경 구축을 안내한다
- **44bits.io** — 인프라·DevOps 주제의 한국어 기술 블로그로, Prometheus/Grafana 실습 시리즈가 있다
- **Kubernetes 한국 사용자 모임(KANS)** — 모니터링 세션에서 Prometheus+Grafana 구축기를 공유한다

### 3.3 유튜브·강의

- **Grafana Labs YouTube** — "Grafana LGTM Stack" 시리즈가 각 컴포넌트를 30분 단위로 설명한다
- **CloudNet@ 스터디** — 가시다님이 운영하는 스터디 그룹으로, Kubernetes 모니터링(Prometheus+Grafana) 핸즈온 영상을 공개한다

## 4. 추천 학습 순서

메시지 큐와 LGTM은 독립적으로 학습할 수 있지만, 실무에서는 메시지 큐의 관측 가능성을 LGTM으로 확보하는 형태로 합류한다. 아래는 병렬 학습 경로다.

### 4.1 메시지 큐 경로

첫 단계로 **실전 카프카 개발부터 운영까지**를 읽고 로컬 클러스터를 직접 구축한다. 프로듀서·컨슈머의 동작 원리를 코드로 확인하는 것이 목적이다. 이후 **카프카 핵심 가이드** 2판으로 내부 아키텍처(ISR, 컨트롤러, 트랜잭션)를 보강한다.

스트림 처리가 필요해지면 **카프카 스트림즈와 ksqlDB**로 넘어간다. 이벤트 기반 설계 전반을 잡으려면 **이벤트 기반 마이크로서비스 구축**을 병행하면 된다. **데이터 중심 애플리케이션 설계**는 어느 단계에서든 참조서로 활용한다.

### 4.2 LGTM 경로

Grafana 대시보드 조작에 먼저 익숙해지는 것이 효율적이다. **그라파나** 도서나 공식 "Get started" 가이드로 시각화 계층을 잡는다. 이후 **프로메테우스 인프라스트럭처 모니터링**으로 메트릭 수집·쿼리(PromQL)를 학습하면, Mimir는 Prometheus의 장기 저장 확장이므로 자연스럽게 이어진다.

Loki는 Grafana Labs 공식 문서의 LogQL 튜토리얼로 시작한다. Tempo는 OpenTelemetry Collector 설정이 선행 지식이므로, OpenTelemetry 공식 문서를 먼저 읽고 Tempo 연동을 진행한다. 네 컴포넌트가 모두 갖춰지면 Docker Compose로 LGTM 올인원 환경을 구축하여 통합 관측 가능성을 실습한다.

### 4.3 합류 지점

메시지 큐 클러스터의 메트릭(브로커 지연, 컨슈머 랙, 파티션 상태)을 Mimir로 수집하고, 애플리케이션 로그를 Loki로, 분산 추적을 Tempo로 보내는 구성이 합류 지점이다. 이 지점에 도달하면 "메시지가 유실되었는가?"라는 질문에 로그·메트릭·트레이스 세 축으로 답할 수 있게 된다.
