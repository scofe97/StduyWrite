# CKA 대비와 문제 풀이 전략

> CKA는 암기 시험이 아니라 명령줄에서 실제 작업을 수행하는 시험이다. 그래서 문서 구조를 시험 범위에 맞춰 다시 보는 것보다, 평소 실습 흐름을 압축하는 편이 더 효과적이다.



## 학습 목표
> 시험 정보를 최신 기준으로 짚되, 준비 방식은 실무 문서와 연결한다.

이 장에서 확인할 목표는 다음과 같다:

1. CKA가 performance-based 시험이라는 점을 이해할 수 있다.
2. kubectl, kubeadm, troubleshooting, networking, storage, security를 어떤 우선순위로 복습할지 정리할 수 있다.
3. 실무 문서를 시험 준비 체크리스트로 재활용하는 방식을 설명할 수 있다.



## 1. 최신 시험 정보에서 봐야 할 것
> 세부 문항보다 시험 성격과 기준 버전을 먼저 확인해야 한다.

공식 페이지 기준으로 CKA는 2시간의 performance-based 시험이고, 명령줄에서 여러 Kubernetes 작업을 수행하는 형식이다. 또한 시험 환경은 최신 Kubernetes minor 버전에 맞춰 일정 기간 안에 갱신된다. 따라서 오래된 블로그 요약본보다 공식 시험 버전과 커리큘럼을 먼저 확인하는 편이 안전하다.



## 2. 준비 우선순위
> 시험은 넓게 묻지만, 실제 점수는 자주 쓰는 작업의 속도에서 갈린다.

우선순위는 다음 순서가 현실적이다.

1. `kubectl get/describe/logs/exec/edit/apply`
2. Deployment, Service, Ingress, NetworkPolicy
3. PVC, StorageClass, StatefulSet
4. RBAC, ServiceAccount, context
5. kubeadm 업그레이드, 인증서, etcd 개념

여기에 JSONPath, `kubectl explain`, `drain`, `cordon`, `rollout undo` 같은 반복 명령이 붙는다.



## 3. 이 문서 세트로 준비하는 방법
> 새 요약집을 만들기보다 이미 정리한 본문과 점검편을 시험 루틴으로 재활용한다.

- 핵심 워크로드: Pod / Deployment / Service / Namespace
- 네트워킹: DNS / Ingress / Gateway / NetworkPolicy
- 스토리지와 상태: PVC / StorageClass / StatefulSet
- RBAC과 보안: ServiceAccount / 권한 / context
- 운영 심화: kubeadm / etcd / JSONPath

즉 이 카테고리 자체를 CKA용 실무형 노트로 재배열하면 된다.
