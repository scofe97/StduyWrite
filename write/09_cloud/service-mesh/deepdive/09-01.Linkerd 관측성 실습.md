# Linkerd 관측성 실습

> 본 장의 개념을 손으로 확인한다. 실습은 개인 GCP K8s 클러스터(kubeadm 기반 3노드) 위에서 진행하지만, 본문 명령은 환경에 무관한 `kubectl`/`linkerd` 범용 명령으로 쓴다.


## 실습 환경
> kubeadm 기반 3노드 GCP K8s 클러스터의 구성과 접속 방법을 안내한다.


| 항목 | 값 | 역할 |
|------|-----|-----|
| dev-server | 4 vCPU / 8GB · asia-northeast3-a | Control Plane |
| dev-server-2 | 4 vCPU / 8GB | Worker |
| dev-server-3 | 4 vCPU / 8GB | Worker (주 워크로드) |
| K8s | kubeadm v1.31.14 | — |
| 서비스 메시 | Linkerd (사이드카 주입 네임스페이스) | — |

접속은 `gcp` 스킬의 SSH 절차를 따른다. 상세는 [`.claude/skills/gcp/SKILL.md`](../../../../../.claude/skills/gcp/SKILL.md) 참조.


## 실습 1: Linkerd Viz 설치와 확인
> Viz 확장을 설치하고 구성 요소를 확인하며 외부 Prometheus 연동 옵션을 다룬다.


### 목표

Linkerd Viz 확장을 설치하고 기본 메트릭을 확인한다.

```bash
# Viz 확장 설치
linkerd viz install | kubectl apply -f -

# 설치 확인
linkerd viz check

# 구성 요소 확인
kubectl get pods -n linkerd-viz
```

외부 Prometheus가 있다면 내장 Prometheus를 비활성화한다.

```bash
linkerd viz install \
  --set prometheus.enabled=false \
  --set prometheusUrl=http://prometheus.monitoring:9090 \
  | kubectl apply -f -
```


## 실습 2: RED Method 메트릭 확인
> emojivoto 앱에서 linkerd viz stat으로 Rate·Error·Duration 세 황금 메트릭을 확인한다.


### 목표

emojivoto 앱에서 Rate·Error·Duration 세 황금 메트릭을 각 명령어로 확인한다.

```bash
# emojivoto 배포 (없다면)
curl -sL https://run.linkerd.io/emojivoto.yml | kubectl apply -f -

# Rate + Error + Duration 집계 보기
linkerd viz stat deploy -n emojivoto

# 특정 배포 상세 (source 필터)
linkerd viz stat -n emojivoto deploy/web \
  --from deploy/vote-bot

# 출력 예시:
# NAME   MESHED  SUCCESS    RPS   LATENCY_P50   LATENCY_P95   LATENCY_P99
# web    1/1     95.50%   1.0rps         1ms           2ms           5ms
```

SUCCESS 컬럼이 오류율(1 - 성공률), RPS가 Rate, LATENCY_P99가 Duration의 p99다. p99가 p50보다 10배 이상 높으면 긴 꼬리 지연 문제를 의심한다.


## 실습 3: linkerd viz top — 실시간 요청 스트림
> linkerd viz top으로 엔드포인트별 요청 분포를 실시간으로 관찰한다.


```bash
# 네임스페이스 전체 실시간 요청 관찰
linkerd viz top -n emojivoto

# 특정 배포만 필터
linkerd viz top -n emojivoto deploy/web

# 출력에서 확인할 것:
# - ROUTE별 요청 수와 성공률
# - 각 백엔드 서비스별 분포
```

`top`은 엔드포인트별 집계를 보여주는 반면, `tap`은 개별 요청의 헤더와 상태 코드까지 상세히 보여준다.


## 실습 4: linkerd viz tap — 실시간 요청 상세
> linkerd viz tap으로 개별 요청의 헤더와 상태 코드를 확인하고 민감한 헤더 제외 방법을 다룬다.


```bash
# 특정 배포로 들어오는 요청 실시간 관찰
linkerd viz tap -n emojivoto deploy/web

# 특정 출발지에서 특정 목적지로 필터
linkerd viz tap -n emojivoto deploy/web \
  --from deploy/vote-bot \
  --method GET

# 출력 예시:
# req id=0:0 proxy=in src=10.0.1.2:45678 dst=10.0.1.5:8080 ...
#   :method=GET :authority=web-svc :path=/api/list
# rsp id=0:0 proxy=in src=10.0.1.2:45678 dst=10.0.1.5:8080 ...
#   :status=200 latency=1234µs
```

프로덕션에서 tap 사용 시 주의사항이 있다. `Authorization` 헤더가 출력에 노출될 수 있으므로 민감한 헤더를 제외해야 한다.

```bash
# 민감한 헤더 제외 어노테이션 (Pod에 적용)
# config.linkerd.io/tap-ignore-header: "Authorization"
```


## 실습 5: Prometheus 쿼리로 SLO 메트릭 확인
> Prometheus에서 Rate·Error·Duration PromQL 쿼리를 직접 실행해 SLO 메트릭을 확인한다.


Viz가 설치된 상태에서 Prometheus에 직접 쿼리한다.

```bash
# Prometheus 포트 포워딩
kubectl port-forward -n linkerd-viz \
  svc/prometheus 9090:9090

# 브라우저에서 http://localhost:9090 접속 후 쿼리:
```

```promql
# 서비스별 초당 요청률
sum(rate(request_total{direction="inbound",
  namespace="emojivoto"}[1m])) by (deployment)

# 성공률 (1 - 오류율)
sum(rate(response_total{classification="success",
  namespace="emojivoto"}[1m])) by (deployment)
/
sum(rate(response_total{namespace="emojivoto"}[1m])) by (deployment)

# p99 지연시간 (밀리초)
histogram_quantile(0.99,
  sum(rate(response_latency_ms_bucket{
    namespace="emojivoto"}[1m])) by (le, deployment)
)
```


## 실습 6: linkerd viz edges — mTLS 토폴로지
> linkerd viz edges로 네임스페이스 내 연결의 mTLS 보안 상태를 확인한다.


```bash
# 네임스페이스 내 모든 연결 mTLS 상태
linkerd viz edges -n emojivoto

# 특정 배포의 연결
linkerd viz edges -n emojivoto deploy/web
```

`SECURED` 컬럼이 `✔`이면 mTLS 적용 연결이다. `✘`이면 비메시 워크로드(예: Ingress Controller)에서 오는 연결이다.


## 실습 7: Viz 대시보드 접속
> linkerd viz dashboard로 서비스 토폴로지와 골든 시그널을 시각적으로 확인한다.


```bash
# 브라우저에서 Viz 대시보드 열기
linkerd viz dashboard

# 또는 포트 포워딩으로 직접 접속
kubectl port-forward -n linkerd-viz svc/web 8084:8084
```

대시보드에서 서비스 토폴로지 그래프, 네임스페이스별 골든 시그널, 개별 서비스 드릴다운을 시각적으로 확인할 수 있다.


## 정리
> 실습에 사용한 리소스를 정리하는 명령어를 제공한다.


```bash
kubectl delete namespace emojivoto
# Viz 제거가 필요하면
linkerd viz uninstall | kubectl delete -f -
```
