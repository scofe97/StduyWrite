# CI 연동과 Image Updater 점검

> 자동 이미지 갱신이 만들 수 있는 운영 함정을 질문 중심으로 점검한다.

## 점검 질문
> 편의성보다 Git 경계와 승인 흐름이 더 중요하다.

### Q1. Image Updater를 모든 앱에 기본 적용해도 되는가?

아니다. 배포 승인 절차가 엄격한 앱이나 인프라 컴포넌트에는 오히려 수동 태그 고정이 더 안전할 수 있다.

### Q2. 가장 흔한 구조적 실수는?

여러 환경이 같은 values 파일이나 같은 write-back 경로를 공유하는 것이다. 한 환경의 자동 태그 갱신이 다른 환경 배포 기준을 바꿔 버릴 수 있다.

### Q3. Git write-back이 왜 보통 더 자연스러운가?

Git이 단일 진실 공급원이라는 원칙을 유지하기 쉽기 때문이다. 대신 저장소 구조와 승인 흐름이 먼저 정리돼 있어야 한다.
