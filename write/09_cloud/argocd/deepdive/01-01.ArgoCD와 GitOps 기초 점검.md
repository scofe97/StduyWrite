# ArgoCD와 GitOps 기초 점검

> Push 배포와 Pull 기반 GitOps의 차이를 질문 중심으로 다시 점검한다.

## 점검 질문
> GitOps는 배포 자동화보다 기준 상태 관리에 더 가깝다.

### Q1. CI가 직접 `kubectl apply`하는 구조와 무엇이 다른가?

GitOps에서는 클러스터 안의 컨트롤러가 Git 상태를 끌어온다. 외부 CI가 클러스터 자격증명을 직접 쥐지 않아도 된다.

### Q2. drift 감지가 왜 중요한가?

Git과 클러스터 상태가 어긋난 것을 빨리 드러내기 때문이다. 수동 수정이 장기 방치되면 장애 추적이 어려워진다.
