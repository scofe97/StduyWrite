# 인증·인가와 AppProject 점검

> SSO, RBAC, AppProject를 팀 경계 관점에서 점검한다.

## 점검 질문
> AppProject를 단순 폴더처럼 보면 안 된다.

### Q1. AppProject가 왜 보안 경계인가?

허용된 source repo, destination cluster/namespace, 리소스 범위를 제한하기 때문이다. 팀별로 무엇을 어디에 배포할 수 있는지 여기서 막는다.

### Q2. RBAC만 잘 짜면 AppProject가 없어도 되는가?

아니다. RBAC은 “누가 API를 호출할 수 있는가”에 가깝고, AppProject는 “그 앱이 실제로 어떤 범위에 배포될 수 있는가”를 제한한다.

### Q3. admin 계정을 계속 써도 되는가?

학습 초기 외에는 권장하지 않는다. SSO와 역할 분리가 없으면 누가 무엇을 했는지 추적성과 최소 권한 원칙이 무너진다.
