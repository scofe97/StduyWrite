# ArgoCD 설치와 아키텍처 실습

> ArgoCD 설치 직후 어떤 컴포넌트가 떠야 하는지 직접 확인하는 실습이다.

## 실습 환경
> 개인 GCP K8s 클러스터 또는 로컬 kind/minikube 기준으로 진행한다.

## 실습 절차
> 설치, 확인, 초기 진입 순으로 진행한다.

```bash
kubectl create namespace argocd
kubectl apply -n argocd --server-side -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
kubectl get pods -n argocd
kubectl get deployment,statefulset -n argocd
kubectl get crd | grep argoproj.io
kubectl get crd | grep gateway.networking.k8s.io || true
kubectl get deployment -n argocd | grep applicationset
kubectl get secret -n argocd argocd-initial-admin-secret -o jsonpath='{.data.password}' | base64 -d && echo
```

## 확인 포인트
> 개별 Pod 이름보다 역할 단위로 보는 편이 좋다.

- `argocd-server`, `argocd-repo-server`, `argocd-application-controller`가 떠 있는지 확인한다.
- `applicationset-controller`가 보이는지 확인한다.
- 초기 admin 비밀번호 조회가 가능한지 확인한다.
