## Q1: JCasC에서 UI 변경과 YAML 충돌이 발생하면 어떻게 되는가?

JCasC는 두 가지 적용 모드를 제공한다:

| 모드 | 동작 |
|------|------|
| `MERGE` | YAML에 명시된 항목만 덮어쓰고 나머지 UI 설정은 유지 |
| `REPLACE` | YAML 기준으로 전체 재설정 → UI 변경값 전부 사라짐 |

- 운영에서는 `REPLACE` 모드가 권장된다. YAML이 진실의 원천이 되어야 재현성이 보장된다.
- 팀 운영 정책으로 "JCasC 적용 후 UI 직접 변경 금지"를 수립해야 충돌을 방지할 수 있다.

## Q2: Secret Source를 선택하는 기준은 무엇인가?

배포 환경, 보안 요구 수준, 팀 규모에 따라 적합한 Secret Source가 달라진다:

- 환경변수: 로컬 개발/PoC에만 사용한다. Jenkins 관리자에게 노출되므로 운영 환경에서는 부적합하다.
- Docker Secrets: Docker Compose 환경에서 파일 기반으로 자동 매핑한다. 운영 복잡도가 낮아 소규모 팀에 적합하다.
- K8s Secrets: Kubernetes 환경의 표준이다. 볼륨 마운트로 `/run/secrets/`에 파일로 노출한다.
- HashiCorp Vault: 엔터프라이즈 환경에서 중앙 집중식 시크릿 관리가 필요할 때 사용한다. AppRole, Token, UserPass 세 가지 인증 방식을 지원한다.
- AWS Secrets Manager: AWS 환경에서 네이티브 통합이 필요할 때 선택한다.

선택 기준은 배포 환경(Docker/K8s/클라우드) + 보안 요구 수준 + 팀 규모의 조합으로 결정한다.

## Q3: 다중 YAML 파일 간 오버라이드 제약은 무엇인가?

`CASC_JENKINS_CONFIG`에 디렉토리를 지정하면 모든 `.yml`/`.yaml` 파일을 재귀 수집한다. 파일 간 같은 키를 선언하면 `ConfiguratorException`이 발생한다. 오버라이드가 아니라 보충(supplementary) 관계이기 때문이다.

충돌 방지 전략은 루트 키(`jenkins`, `tool`, `unclassified`, `credentials`) 단위로 파일을 분리하는 것이다.

## Q4: Reload API 방식 간 차이는 무엇인가?

용도에 따라 네 가지 방식 중 하나를 선택한다:

- Token 기반 (`/reload-configuration-as-code/?casc-reload-token=`): 사용자 인증이 불필요하다. CI/webhook 자동화에 최적이다.
- API Token 기반 (`/configuration-as-code/reload`): 사용자 인증이 필요하다. 수동 reload에 적합하다.
- YAML POST (`/configuration-as-code/configure`): 외부 YAML을 직접 전송한다. Git에 없는 임시 설정 테스트 시 유용하다.
- CLI (`reload-jcasc-configuration`): Jenkins CLI 환경에서 사용한다. 스크립트 자동화가 가능하다.

## Q5: export → diff 드리프트 감지의 한계는 무엇인가?

`export`는 현재 Jenkins 상태 전체를 출력하므로, `jenkins.yaml`에 명시하지 않은 기본값도 포함된다. 이로 인해 diff에 노이즈가 발생한다. 관리 대상 섹션만 필터링하는 전략이 필요하다.

export된 YAML의 키 순서가 `jenkins.yaml`과 다를 수 있어, 단순 diff보다 YAML 파서 기반 비교가 정확하다.

## Q6: Strict Secret Resolution은 언제 활성화해야 하는가?

기본 동작에서 미해결 `${VAR}`은 빈 문자열로 대체된다. 관리자 비밀번호가 빈 문자열이 되면 누구나 로그인할 수 있으므로 심각한 보안 문제로 이어진다.

`CASC_STRICT_SECRET_RESOLUTION=true`를 활성화하면 시크릿이 해결되지 않을 때 reload 자체가 중단된다. 운영 환경에서는 반드시 활성화해야 하며, 개발 환경에서도 활성화를 권장한다. `${VAR:-default}` 형식의 기본값은 strict 모드에서도 정상 동작한다.
