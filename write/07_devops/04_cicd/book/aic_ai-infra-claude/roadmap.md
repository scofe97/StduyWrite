# AI 인프라 — Claude로 — 장·절 TOC 원문 기록

---

> 이 문서는 단행본 《AI 인프라 — Claude로》(저자 sysnet4admin)의 **장·절 목차를 스크린샷에서 읽은 그대로** 옮긴 기록입니다. 정독 노트의 범위를 잡는 SSOT이며, 본문 요약이 아니라 "각 절이 원래 무엇을 다루는가"의 원문 보존이 목적입니다. 카테고리 경계·장 요약·톤 규약은 [README.md](README.md)가 맡습니다.
>
> **책은 총 9개 장 구성**입니다(§1.4.1 전체 흐름표로 확정). 하나의 가상 스타트업 **Notiflex**(B2B 알림 SaaS, Go 표준 라이브러리, GKE)를 9개 장에 걸쳐 성장시키며, 클로드 코드와 함께 인프라를 만드는 **GitAIOps**(GitOps + AI)가 책의 핵심 개념입니다.
>
> **판독 상태**: Ch1·Ch2(전체 절)와 Ch3~Ch7은 판독 완료. Ch8 §8.1~§8.3, Ch9 전체는 `(판독 필요)`로 비워 둡니다 — 추측으로 채우지 않습니다.

## 책의 공통 패턴

모든 장은 끝에서 두 절이 다음 형태로 닫힙니다.

```text
§N.(끝-1)  마무리: <그 장에서 Claude Code와 협업한 산출물>
§N.(끝)    N장 가드레일 살펴보기
```

마무리 절의 산출물은 장을 거치며 발전합니다: `/update-docs` 스킬(2장) → `CLAUDE.md` 행동규칙(3장) → 메모리에 작업 컨텍스트(4장) → 아키텍처 결정 기록(5장) → `claude-context/`(6장) → `settings.local.json` 권한 분리(7장) → `command-guardrails/`(8장). 9장은 그동안 쌓인 코드·설정·문서를 AI로 분석해 "살아있는 운영 표준"으로 정리합니다.
