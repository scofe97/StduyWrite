## Q1: Pipeline as Code가 Freestyle Job보다 나은 이유는?

Jenkinsfile은 Git 히스토리로 변경의 저자·시점·이유를 남기므로 Freestyle처럼 UI에서 누가 무엇을 바꿨는지 추적 불가능한 상태가 되지 않는다. 파이프라인 변경이 PR 리뷰에 들어오므로 위험한 설정 변경을 사전에 잡을 수 있고, Jenkins 서버가 날아가도 Git에서 clone하면 즉시 복원된다. 브랜치마다 Job을 따로 관리할 필요가 없는 것은 결과적으로 따라오는 부수 효과다.

상세 비교: `02-01.md` §1.

## Q2: Declarative와 Scripted Pipeline의 차이는?

Declarative는 정해진 DSL을 강제하므로 실행 전에 문법 구조 검증이 가능하고 실패한 stage부터 재시작할 수 있다. Scripted는 `node {}` 안의 순수 Groovy라 자유도는 높지만 오류는 실행 시점에야 드러나고 재시작은 처음부터 해야 한다. 실무 90%는 Declarative로 충분하며, 복잡한 로직만 `script {}` 블록으로 들여놓는다.

비교 표: `02-02.md` §1.

## Q3: post actions 종류와 실무 활용 패턴은?

`always`(워크스페이스 정리·테스트 결과 수집), `success`(알림·CD 트리거), `failure`(@here 멘션), `changed`(상태 전환 알림 — "이전까지 성공하던 빌드가 실패하기 시작했습니다") 네 가지가 핵심이다. 실행 순서는 결과별 조건이 먼저, `always`가 마지막이며 이 순서를 알면 "왜 알림이 두 번 가는가" 같은 의문이 풀린다. `post`는 pipeline 레벨과 stage 레벨 둘 다에서 정의 가능하다.

조건별 표: `02-02.md` §4.

## Q4: when {} 조건을 Git Flow에 매핑하는 방법은?

브랜치 이름 패턴(`feature/*`, `develop`, `release/*`, `main`, `hotfix/*`)에 따라 빌드 강도와 배포 대상을 분기하는 것이 기본이다. 두 가지 옵션이 가용성·비용에 결정적이다. `beforeAgent true`는 비싼 노드(GPU 등)를 점유하기 전에 조건을 평가해 무의미한 점유를 막고, `changeset "backend/**"`는 모노레포에서 변경된 부분만 빌드해 CI 시간을 줄인다.

브랜치별 동작 표: `02-03.md` §2.

## Q5: 병렬 실행의 이점과 주의사항은?

독립적인 작업(단위/통합 테스트, 보안 스캔, 린트)을 동시에 실행하면 전체 시간이 가장 긴 작업으로 수렴한다. 함정은 셋이다: ① 동일 워크스페이스 공유 시 파일 쓰기 레이스 — `agent` 분리 또는 `dir()`로 격리. ② 4 병렬 = 에이전트 슬롯 4개 점유로 다른 Job 지연 가능. ③ `failFast true`는 첫 실패에 나머지를 끊고, 기본값은 모두 끝까지 실행해 한 번에 모든 결과를 본다.

## Q6: 수동 승인 게이트 구현 시 에이전트 점유 문제는?

`input`이 `steps` 블록 안에 있으면 승인 대기 시간 동안 에이전트가 점유되어 리소스가 낭비된다. 해당 stage에 `agent none`을 지정해 에이전트를 풀고, `timeout(time: 2, unit: 'HOURS')`로 무한 대기를 방지하며, `parameters` 옵션으로 승인 사유를 입력받아 누가 언제 왜 승인했는지 감사 로그를 남긴다. 이 세 가지가 빠지면 운영 사고로 직결된다.

상세: `02-04.md` §1.

## Q7: Multibranch Pipeline이 일반 Pipeline Job과 다른 점은?

Branch Source Plugin이 SCM을 주기적으로 스캔해 `Jenkinsfile`이 있는 모든 브랜치를 자동 탐지하고, 각 브랜치마다 독립 하위 Job을 생성한다. 일반 Pipeline은 브랜치당 수동 생성, 삭제 시 수동 정리, PR 빌드 별도 설정이 필요하지만 Multibranch는 Orphaned Item Strategy로 자동 정리하고 GitHub/GitLab 통합으로 PR을 머지 상태로 빌드한다.

기능 비교 표: `02-03.md` §1.
