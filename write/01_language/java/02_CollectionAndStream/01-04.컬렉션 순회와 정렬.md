# 컬렉션 순회와 정렬
---
> 자바의 순회와 정렬 API는 Iterator에서 시작해 Comparable·Comparator를 거쳐 Java 21의 SequencedCollection으로 진화했다. 각 계층의 역할을 이해하면 어떤 컬렉션에서도 일관된 방식으로 처리할 수 있다.

## 1. Iterator 패턴

*Iterator*는 컬렉션의 내부 구조를 노출하지 않고 요소를 순서대로 접근하는 패턴이다. 배열 기반이든 노드 기반이든 같은 인터페이스로 순회할 수 있다.

```java
List<String> list = List.of("A", "B", "C");

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String element = it.next();
    System.out.println(element);
}
```

`Iterator`는 세 가지 메서드로 구성된다. `hasNext()`는 다음 요소 존재 여부를 반환하고, `next()`는 현재 요소를 반환하며 커서를 전진시킨다. `remove()`는 현재 요소를 안전하게 제거한다. 일반 for-loop에서 `list.remove()`를 호출하면 `ConcurrentModificationException`이 발생하는데, `Iterator.remove()`는 이를 피할 수 있다.

`ListIterator`는 `Iterator`를 확장해 양방향 순회와 요소 추가·교체를 지원한다.

```java
ListIterator<String> lit = list.listIterator(list.size());
while (lit.hasPrevious()) {
    System.out.println(lit.previous()); // 역방향 순회
}
```

## 2. enhanced for-loop와 Iterable

enhanced for-loop(for-each)는 `Iterable<T>` 인터페이스를 구현한 모든 객체에 사용할 수 있다. `Iterable`은 `iterator()` 메서드 하나를 선언하며, 컴파일러가 이를 `Iterator` 기반 while-loop으로 변환한다.

```java
// 이 코드는
for (String s : list) {
    System.out.println(s);
}

// 컴파일러가 아래로 변환한다
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
```

배열도 for-each를 지원하지만, 배열은 `Iterable`을 구현하지 않는다. 컴파일러가 인덱스 기반 for-loop로 직접 변환한다. for-each 안에서 컬렉션을 수정하면 `ConcurrentModificationException`이 발생하므로, 수정이 필요하면 `Iterator.remove()` 또는 `removeIf()`를 사용한다.

## 3. Comparable과 Comparator

자바에서 정렬 기준은 두 인터페이스로 표현된다.

*Comparable*은 클래스 자신이 정렬 기준을 내장하는 방식이다. `compareTo(T other)`를 구현하며, 음수·0·양수를 반환해 작다·같다·크다를 나타낸다. `String`, `Integer`, `LocalDate` 등 기본 타입들이 이미 구현하고 있다.

```java
public class Student implements Comparable<Student> {
    private String name;
    private int score;

    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.score, other.score); // 점수 오름차순
    }
}
```

*Comparator*는 외부에서 정렬 기준을 주입하는 방식이다. 클래스를 수정할 수 없거나 상황에 따라 다른 기준으로 정렬해야 할 때 사용한다.

```java
Comparator<Student> byName  = Comparator.comparing(Student::getName);
Comparator<Student> byScore = Comparator.comparingInt(Student::getScore).reversed();

// 성 먼저, 같으면 점수 역순
Comparator<Student> combined = byName.thenComparing(byScore);
```

## 4. Collections.sort()와 List.sort()

`Collections.sort(list)`는 `List<T>` 전체를 정렬하는 정적 유틸리티다. `List.sort(comparator)`는 인스턴스 메서드로 동일한 기능을 제공하며, Java 8부터 권장하는 방식이다. 두 메서드 모두 내부적으로 Timsort 알고리즘을 사용해 O(n log n) 성능을 보장하며, 안정 정렬(stable sort)이다.

```java
List<String> names = new ArrayList<>(List.of("Charlie", "Alice", "Bob"));

// 방법 1: Collections.sort()
Collections.sort(names);

// 방법 2: List.sort() — Java 8부터 권장
names.sort(Comparator.naturalOrder());
names.sort(Comparator.reverseOrder());

// 방법 3: Stream.sorted() — 원본 불변
List<String> sorted = names.stream()
    .sorted(Comparator.naturalOrder())
    .toList();
```

`Collections.sort()`와 `List.sort()`는 원본 리스트를 제자리에서(in-place) 정렬한다. 원본을 유지하려면 복사본을 만들거나 Stream을 사용한다.

## 5. Collections 유틸리티 메서드

`Collections` 클래스는 컬렉션을 감싸거나 변환하는 정적 팩토리 메서드를 제공한다.

```java
List<String> list = new ArrayList<>(List.of("A", "B", "C"));

// 수정 불가 래퍼 (원본과 같은 객체를 공유함)
List<String> unmodifiable = Collections.unmodifiableList(list);

// 동기화 래퍼 (단일 원소 잠금, Vector보다 나은 선택)
List<String> synced = Collections.synchronizedList(list);

// 단일 요소 불변 리스트
List<String> singleton = Collections.singletonList("only");

// 빈 불변 컬렉션
List<String> empty = Collections.emptyList();

// 요소 빈도수
int freq = Collections.frequency(list, "A");

// 역순 정렬
Collections.sort(list, Collections.reverseOrder());

// 셔플
Collections.shuffle(list);
```

`unmodifiableList()`는 `List.copyOf()`와 달리 원본과 내용을 공유한다. 원본이 변경되면 래퍼를 통해서도 변경 내용이 보인다. 진정한 불변 복사가 필요하면 `List.copyOf()`를 사용한다.

멀티스레드 환경에서 `synchronizedList()`는 개별 메서드 호출을 동기화하지만, 복합 연산(size 확인 후 add 등)의 원자성은 보장하지 않는다. 높은 동시성이 필요하면 `CopyOnWriteArrayList`나 `ConcurrentHashMap`을 사용한다.

## 6. Java 21 — SequencedCollection 인터페이스 계층

Java 21은 `SequencedCollection`, `SequencedSet`, `SequencedMap` 인터페이스를 추가해 첫 번째·마지막 요소 접근과 역순 뷰를 컬렉션 계층에 통합했다.

```
Iterable
  └─ Collection
       ├─ SequencedCollection   ← Java 21 추가
       │    ├─ List (ArrayList, LinkedList)
       │    └─ SequencedSet
       │         ├─ LinkedHashSet
       │         └─ SortedSet → TreeSet
       └─ ...
Map
  └─ SequencedMap               ← Java 21 추가
       ├─ LinkedHashMap
       └─ SortedMap → TreeMap
```

`SequencedCollection`이 제공하는 메서드는 다음과 같다.

```java
SequencedCollection<String> sc = new ArrayList<>(List.of("A", "B", "C"));

sc.getFirst();    // "A"
sc.getLast();     // "C"
sc.addFirst("Z"); // 앞에 삽입
sc.addLast("W");  // 뒤에 삽입
sc.removeFirst(); // 첫 요소 제거
sc.removeLast();  // 마지막 요소 제거

SequencedCollection<String> rev = sc.reversed(); // 역순 뷰
```

이전에는 `list.get(0)`과 `list.get(list.size() - 1)`으로 작성하거나, `Deque`를 써야 했던 코드가 표현적으로 간결해졌다. `SequencedMap`은 `firstEntry()`, `lastEntry()`, `reversed()` 등을 제공해 `LinkedHashMap`과 `TreeMap`을 통일된 인터페이스로 다룰 수 있다.
