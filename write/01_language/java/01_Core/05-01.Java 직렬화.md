# Java 직렬화

---

> 직렬화(Serialization)는 객체를 바이트 스트림으로 변환해 파일이나 네트워크로 전송하는 메커니즘이다. 편리하지만 보안 취약점이 크기 때문에 현대 시스템에서는 JSON이나 Protocol Buffers가 직렬화를 대체한다.

## 1. Serializable 인터페이스

Java의 직렬화는 `java.io.Serializable` 인터페이스를 구현하는 것에서 시작한다. 이 인터페이스는 메서드가 없는 *마커 인터페이스(Marker Interface)*로, JVM에게 해당 클래스가 직렬화 가능하다는 사실을 알리는 역할만 한다. `ObjectOutputStream`으로 객체를 쓰고, `ObjectInputStream`으로 복원한다.

```java
import java.io.*;

public class User implements Serializable {
    private static final long serialVersionUID = 1L;

    private String name;
    private int age;
    private transient String password; // 직렬화 제외

    public User(String name, int age, String password) {
        this.name = name;
        this.age = age;
        this.password = password;
    }
}
```

직렬화/역직렬화 실행 예제:

```java
// 직렬화: 객체 → 파일
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.ser"))) {
    oos.writeObject(new User("Alice", 30, "secret"));
}

// 역직렬화: 파일 → 객체
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.ser"))) {
    User user = (User) ois.readObject();
    System.out.println(user.getName()); // Alice
    System.out.println(user.getPassword()); // null (transient)
}
```

## 2. serialVersionUID

**`serialVersionUID`**는 직렬화된 클래스의 버전을 식별하는 고유 식별자다. 역직렬화 시 저장된 UID와 현재 클래스의 UID가 다르면 `InvalidClassException`이 발생한다. 명시적으로 선언하지 않으면 JVM이 클래스 구조를 기반으로 자동 생성하는데, 필드 추가만 해도 UID가 바뀌어 기존 직렬화 데이터를 읽지 못하게 된다. 이런 이유로 직접 `private static final long serialVersionUID = 1L;`을 선언하는 것이 필수적이다.

## 3. transient 키워드

*`transient`*는 특정 필드를 직렬화 대상에서 제외하도록 지시하는 키워드다. 비밀번호, 세션 토큰처럼 저장하면 안 되는 민감 정보나, 재계산 가능한 캐시 값에 사용한다. `transient`로 선언된 필드는 역직렬화 시 기본값(`null`, `0`, `false`)으로 복원된다.

## 4. 커스텀 직렬화

기본 직렬화 동작을 재정의하려면 `writeObject()`와 `readObject()` 메서드를 직접 구현한다. 이를 통해 민감 정보를 암호화하거나, 직렬화 전후에 유효성 검사를 추가할 수 있다.

```java
public class SecureUser implements Serializable {
    private static final long serialVersionUID = 1L;

    private String name;
    private transient String password;

    private void writeObject(ObjectOutputStream oos) throws IOException {
        oos.defaultWriteObject();
        // 암호화해서 저장 (예시)
        oos.writeObject(encrypt(password));
    }

    private void readObject(ObjectInputStream ois)
            throws IOException, ClassNotFoundException {
        ois.defaultReadObject();
        // 복호화해서 복원 (예시)
        this.password = decrypt((String) ois.readObject());
    }

    private String encrypt(String value) { return "enc:" + value; }
    private String decrypt(String value) { return value.replace("enc:", ""); }
}
```

## 5. Externalizable

`Externalizable` 인터페이스는 `Serializable`보다 더 완전한 제어권을 제공한다. `writeExternal()`과 `readExternal()`을 구현해 직렬화할 필드를 직접 지정한다. 기본 직렬화 메커니즘을 전혀 사용하지 않으므로 성능이 중요한 상황에서 유리하다. 단, 기본 생성자가 반드시 존재해야 역직렬화가 가능하다는 제약이 있다.

```java
public class FastUser implements Externalizable {
    private String name;
    private int age;

    public FastUser() {} // 필수 기본 생성자

    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeUTF(name);
        out.writeInt(age);
    }

    @Override
    public void readExternal(ObjectInput in) throws IOException {
        this.name = in.readUTF();
        this.age = in.readInt();
    }
}
```

## 6. 직렬화 프록시 패턴

*직렬화 프록시 패턴(Serialization Proxy Pattern)*은 Effective Java에서 권장하는 방어적 기법이다. 원본 클래스 대신 내부 정적 프록시 클래스를 직렬화하여, 역직렬화 시 불변식(invariant)이 깨지는 것을 방지한다.

```java
public final class Period implements Serializable {
    private final Date start;
    private final Date end;

    private static class SerializationProxy implements Serializable {
        private static final long serialVersionUID = 1L;
        private final Date start;
        private final Date end;

        SerializationProxy(Period p) {
            this.start = p.start;
            this.end = p.end;
        }

        private Object readResolve() {
            return new Period(start, end); // 생성자를 통해 불변식 보장
        }
    }

    private Object writeReplace() {
        return new SerializationProxy(this);
    }

    private Object readObject(ObjectInputStream stream) throws InvalidObjectException {
        throw new InvalidObjectException("프록시만 사용");
    }
}
```

## 7. 역직렬화 보안 위협

Java 직렬화의 가장 심각한 문제는 **역직렬화 공격(Deserialization Attack)**이다. 신뢰할 수 없는 소스에서 받은 바이트 스트림을 역직렬화하면, 공격자가 의도한 클래스의 메서드가 실행될 수 있다. Apache Commons Collections 라이브러리를 악용한 원격 코드 실행(RCE) 취약점이 대표적인 사례다.

대응 방법은 다음과 같다:

- 신뢰할 수 없는 데이터를 절대 역직렬화하지 않는다
- Java 9+의 `ObjectInputFilter`로 허용 클래스를 화이트리스트로 제한한다
- 가능하면 Java 직렬화 자체를 사용하지 않는다

```java
// ObjectInputFilter로 허용 클래스 제한 (Java 9+)
ObjectInputStream ois = new ObjectInputStream(inputStream);
ois.setObjectInputFilter(info -> {
    if (info.serialClass() == User.class) return ObjectInputFilter.Status.ALLOWED;
    return ObjectInputFilter.Status.REJECTED;
});
```

## 8. 대안: JSON과 Protocol Buffers

현대 시스템에서 Java 직렬화는 거의 사용하지 않는다. 보안 취약점, 버전 관리 어려움, Java 종속성이라는 세 가지 이유 때문이다.

| 항목 | Java 직렬화 | JSON (Jackson) | Protocol Buffers |
|------|-------------|----------------|-----------------|
| 언어 호환성 | Java 전용 | 언어 무관 | 언어 무관 |
| 보안 | 취약 | 안전 | 안전 |
| 가독성 | 불가 | 가능 | 불가 (바이너리) |
| 성능 | 보통 | 보통 | 빠름 |
| 스키마 진화 | 어려움 | 유연 | 명시적 |

Jackson을 사용한 JSON 직렬화는 다음과 같다:

```java
import com.fasterxml.jackson.databind.ObjectMapper;

ObjectMapper mapper = new ObjectMapper();

// 직렬화: 객체 → JSON 문자열
String json = mapper.writeValueAsString(new User("Alice", 30));

// 역직렬화: JSON 문자열 → 객체
User user = mapper.readValue(json, User.class);
```

Jackson은 `@JsonIgnore`, `@JsonProperty` 같은 애노테이션으로 세밀한 제어가 가능하고, 스키마 변경 시에도 `@JsonIgnoreProperties(ignoreUnknown = true)`로 하위 호환성을 유지할 수 있다.
