# Enum 활용
---
> enum은 단순한 상수 집합이 아니다. 필드, 메서드, 생성자를 가진 완전한 타입이며, 전략 패턴부터 싱글톤까지 실무 설계에 직접 활용할 수 있다.

## 1. 기본 문법과 내장 메서드

`enum` 타입은 클래스의 일종이다. 컴파일러가 `Enum<T>` 추상 클래스를 자동으로 상속시키며, 몇 가지 유용한 메서드를 제공한다:

```java
public enum Direction {
    NORTH, SOUTH, EAST, WEST
}
```

내장 메서드의 동작은 다음과 같다:

- `Direction.values()` — 선언 순서대로 모든 상수를 배열로 반환한다.
- `Direction.valueOf("NORTH")` — 이름으로 상수를 찾는다. 없으면 `IllegalArgumentException`을 던진다.
- `direction.name()` — 상수의 선언 이름을 문자열로 반환한다.
- `direction.ordinal()` — 선언 순서의 0-기반 인덱스를 반환한다.

`ordinal()`은 주의해서 써야 한다. 상수 순서가 바뀌면 저장된 ordinal 값의 의미가 달라지므로, DB 컬럼이나 직렬화에 ordinal을 직접 쓰면 안 된다. 이름(`name()`) 또는 별도 코드 필드를 쓰는 것이 안전하다.

## 2. 필드, 생성자, 메서드 추가

enum 상수에 데이터를 붙이려면 생성자와 필드를 정의한다. 생성자는 항상 `private`이다(컴파일러가 강제):

```java
public enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    VENUS  (4.869e+24, 6.0518e6),
    EARTH  (5.976e+24, 6.37814e6),
    MARS   (6.421e+23, 3.3972e6);

    private final double mass;    // 질량(kg)
    private final double radius;  // 반지름(m)

    Planet(double mass, double radius) {
        this.mass = mass;
        this.radius = radius;
    }

    static final double G = 6.67300E-11;

    public double surfaceGravity() {
        return G * mass / (radius * radius);
    }

    public double surfaceWeight(double otherMass) {
        return otherMass * surfaceGravity();
    }
}
```

이렇게 정의한 enum은 `Planet.EARTH.surfaceWeight(75.0)` 처럼 동작한다. 데이터와 로직이 상수 안에 캡슐화되어 switch 없이도 다형적으로 동작한다.

## 3. 추상 메서드를 가진 enum (상수별 메서드 구현)

각 상수가 같은 메서드를 다르게 구현해야 할 때, enum에 추상 메서드를 선언하고 상수별로 구현한다. 컴파일러가 모든 상수가 구현을 제공했는지 강제한다:

```java
public enum Operation {
    PLUS("+") {
        @Override
        public double apply(double x, double y) { return x + y; }
    },
    MINUS("-") {
        @Override
        public double apply(double x, double y) { return x - y; }
    },
    TIMES("*") {
        @Override
        public double apply(double x, double y) { return x * y; }
    },
    DIVIDE("/") {
        @Override
        public double apply(double x, double y) { return x / y; }
    };

    private final String symbol;

    Operation(String symbol) { this.symbol = symbol; }

    public abstract double apply(double x, double y);

    @Override
    public String toString() { return symbol; }
}
```

호출부는 `op.apply(x, y)`로 단순하다. switch 분기 없이 상수 자체가 행동을 결정한다.

## 4. EnumSet과 EnumMap

`java.util` 패키지의 `EnumSet`과 `EnumMap`은 enum 전용 고성능 컬렉션이다. 내부적으로 비트 벡터와 배열을 사용하므로 일반 `HashSet`/`HashMap`보다 훨씬 빠르다.

**EnumSet** — 비트 플래그 대신 사용하는 타입 안전한 집합이다:

```java
public enum Style { BOLD, ITALIC, UNDERLINE, STRIKETHROUGH }

// 특정 스타일 집합 생성
EnumSet<Style> styles = EnumSet.of(Style.BOLD, Style.ITALIC);

// 여집합
EnumSet<Style> others = EnumSet.complementOf(styles);

// 모든 상수
EnumSet<Style> all = EnumSet.allOf(Style.class);
```

**EnumMap** — enum을 키로 사용하는 맵이다:

```java
EnumMap<Direction, String> labels = new EnumMap<>(Direction.class);
labels.put(Direction.NORTH, "북쪽");
labels.put(Direction.SOUTH, "남쪽");

String label = labels.get(Direction.NORTH); // "북쪽"
```

선언 순서에 따라 내부 배열에 직접 접근하므로 `HashMap`보다 빠르고 메모리 효율도 좋다. enum을 키로 쓰는 맵이라면 항상 `EnumMap`을 우선 선택한다.

## 5. enum 싱글톤 패턴

Effective Java(3판) Item 3에서 Joshua Bloch가 권장한 방식이다. 직렬화와 리플렉션 공격에도 안전한 싱글톤을 단 한 줄로 구현한다:

```java
public enum AppConfig {
    INSTANCE;

    private final String dbUrl = System.getenv("DB_URL");
    private final int maxConnections = 10;

    public String getDbUrl() { return dbUrl; }
    public int getMaxConnections() { return maxConnections; }
}
```

사용 시 `AppConfig.INSTANCE.getDbUrl()`로 접근한다. JVM이 클래스 로딩 시 단 하나의 인스턴스만 생성하고, `Serializable`을 구현해도 역직렬화 시 새 인스턴스가 생기지 않는다.

## 6. enum을 활용한 전략 패턴

전략 패턴의 전형적인 구현은 인터페이스 + 여러 구현 클래스다. enum의 상수별 메서드 구현을 이용하면 클래스 파일 없이 하나의 enum으로 모든 전략을 표현할 수 있다:

```java
public enum DiscountPolicy {
    NONE {
        @Override
        public int discount(int price) { return price; }
    },
    TEN_PERCENT {
        @Override
        public int discount(int price) { return (int) (price * 0.9); }
    },
    FLAT_1000 {
        @Override
        public int discount(int price) { return Math.max(price - 1000, 0); }
    };

    public abstract int discount(int price);
}
```

주문 처리 코드에서 `DiscountPolicy`를 필드로 받으면, 어떤 할인 정책이든 동일한 `discount()` 호출로 처리된다. 새 정책을 추가할 때 기존 코드를 건드리지 않는다.

람다를 이용한 더 간결한 변형도 가능하다:

```java
public enum DiscountPolicy {
    NONE        (price -> price),
    TEN_PERCENT (price -> (int) (price * 0.9)),
    FLAT_1000   (price -> Math.max(price - 1000, 0));

    private final IntUnaryOperator strategy;

    DiscountPolicy(IntUnaryOperator strategy) { this.strategy = strategy; }

    public int discount(int price) { return strategy.applyAsInt(price); }
}
```

## 7. Java 21 switch pattern matching에서 enum 활용

Java 21의 `switch` 표현식은 enum 상수를 타입 패턴 없이 값으로 직접 매칭한다. 컴파일러는 모든 상수가 처리됐는지 검사한다:

```java
public enum TrafficLight { RED, YELLOW, GREEN }

String instruction = switch (light) {
    case RED    -> "정지";
    case YELLOW -> "감속 준비";
    case GREEN  -> "진행";
    // 모든 상수를 처리했으므로 default 불필요
};
```

Guarded pattern으로 조건을 추가할 수도 있다:

```java
public enum HttpStatus {
    OK(200), CREATED(201), BAD_REQUEST(400), NOT_FOUND(404), SERVER_ERROR(500);

    public final int code;
    HttpStatus(int code) { this.code = code; }
}

String category = switch (status) {
    case OK, CREATED                  -> "성공";
    case BAD_REQUEST, NOT_FOUND       -> "클라이언트 오류";
    case SERVER_ERROR                 -> "서버 오류";
};
```

Sealed Class와 결합하면 더 강력해진다. enum을 Sealed Interface의 구현 상수로 쓰거나, enum 타입 자체를 Sealed 계층의 일원으로 구성하면 `switch` 분기가 컴파일 타임에 완전히 검증된다.

## 8. 정리: enum 선택 기준

enum은 세 가지 조건이 맞을 때 가장 빛난다:

- 인스턴스 수가 컴파일 타임에 고정된다.
- 각 상수가 동일한 구조(필드 집합)를 공유한다.
- 싱글톤 의미론(같은 상수는 항상 동일 객체)이 필요하다.

상수마다 구조가 다른 데이터를 담아야 한다면 Sealed Class + record 조합이 더 적합하다. enum과 Sealed Class는 경쟁 관계가 아니라 서로 다른 문제를 푸는 도구다.
