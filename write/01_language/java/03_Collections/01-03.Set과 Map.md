# Set과 Map
---
> TreeSet과 TreeMap은 레드-블랙 트리로 정렬된 접근을 제공하고, Map의 현대적 API는 null 처리와 기본값 로직을 선언적으로 표현한다.

## 1. TreeSet — 정렬된 집합

`TreeSet`은 레드-블랙 트리(Red-Black Tree)로 구현된 정렬 집합이다. 레드-블랙 트리는 자가 균형 이진 탐색 트리로, 삽입·삭제 후에도 트리 높이를 O(log n)으로 유지한다. 덕분에 검색·삽입·삭제 모두 O(log n)을 보장한다.

요소를 저장할 때 크기를 비교해 정렬 순서를 결정한다. 기본적으로 `Comparable` 인터페이스의 `compareTo()`를 사용하고, 생성자에 `Comparator`를 전달하면 외부 기준으로 정렬할 수 있다.

```java
// 자연 순서 (Comparable)
TreeSet<String> treeSet = new TreeSet<>();
treeSet.add("C");
treeSet.add("A");
treeSet.add("B");
System.out.println(treeSet); // [A, B, C]

// 역순 정렬 (Comparator)
TreeSet<String> reversed = new TreeSet<>(Comparator.reverseOrder());
reversed.add("C");
reversed.add("A");
reversed.add("B");
System.out.println(reversed); // [C, B, A]
```

## 2. TreeMap — 정렬된 키-값 저장소

`TreeMap`은 `TreeSet`과 동일한 레드-블랙 트리 구조로 키를 정렬해 저장한다. `SortedMap` 및 `NavigableMap` 인터페이스를 구현하므로 범위 검색 API를 제공한다.

```java
TreeMap<String, Integer> scores = new TreeMap<>();
scores.put("Charlie", 80);
scores.put("Alice", 95);
scores.put("Bob", 70);

System.out.println(scores.firstKey()); // "Alice"
System.out.println(scores.lastKey());  // "Charlie"

// "Alice" 이상 "Charlie" 미만
NavigableMap<String, Integer> sub = scores.subMap("Alice", "Charlie");
```

## 3. HashMap vs TreeMap vs LinkedHashMap 비교

| 구현체 | 내부 구조 | 순서 보장 | 검색·삽입·삭제 | null 키 허용 |
|--------|----------|----------|--------------|-------------|
| `HashMap` | 해시 테이블 | 없음 | O(1) 평균 | 1개 허용 |
| `LinkedHashMap` | 해시 테이블 + 연결 리스트 | 삽입 순서 | O(1) 평균 | 1개 허용 |
| `TreeMap` | 레드-블랙 트리 | 키 정렬 순서 | O(log n) | 불허 |

순서가 필요 없고 빠른 접근이 우선이면 `HashMap`, 삽입 순서가 중요하면 `LinkedHashMap`, 키 순서로 정렬·범위 검색이 필요하면 `TreeMap`을 선택한다.

## 4. Map의 현대적 메서드

Java 8 이후 `Map` 인터페이스는 null 처리와 조건부 저장을 표현하는 메서드를 추가했다.

```java
Map<String, Integer> wordCount = new HashMap<>();

// getOrDefault — 키 없으면 기본값 반환
int count = wordCount.getOrDefault("apple", 0);

// putIfAbsent — 키가 없을 때만 저장
wordCount.putIfAbsent("apple", 1);

// computeIfAbsent — 키 없을 때 계산하여 저장 (반환값 활용 가능)
wordCount.computeIfAbsent("banana", k -> 0);

// merge — 기존 값과 새 값을 결합 (단어 빈도수 계산에 유용)
wordCount.merge("apple", 1, Integer::sum);

// compute — 키에 대해 조건 없이 계산·저장
wordCount.compute("cherry", (k, v) -> v == null ? 1 : v + 1);
```

`merge(key, value, remappingFunction)`은 키가 없으면 value를 저장하고, 있으면 기존 값과 새 값을 함수로 결합한다. 단어 빈도수처럼 누적 카운팅 패턴에서 자주 사용된다.

## 5. 불변 Map — Map.of()와 Map.copyOf()

`Map.of()`는 Java 9에서 추가된 불변 맵 팩토리 메서드다. 인자로 키-값 쌍을 교대로 전달하며, 최대 10쌍까지 지원한다. 10쌍 이상이면 `Map.ofEntries()`를 사용한다.

```java
// Map.of() — 최대 10쌍
Map<String, Integer> config = Map.of(
    "timeout", 30
    , "retry", 3
    , "maxConn", 100
);

// Map.ofEntries() — 10쌍 초과
Map<String, Integer> large = Map.ofEntries(
    Map.entry("a", 1)
    , Map.entry("b", 2)
    // ...
);

// Map.copyOf() — 기존 맵을 불변 복사
Map<String, Integer> mutableMap = new HashMap<>(config);
Map<String, Integer> immutable = Map.copyOf(mutableMap);
```

불변 맵은 `put()`, `remove()` 호출 시 `UnsupportedOperationException`을 던진다. `null` 키와 값을 허용하지 않으며, 반복 순서를 보장하지 않는다.

## 6. EnumSet과 EnumMap — 열거형 특수 구현체

열거형(enum) 타입을 다룰 때는 `EnumSet`과 `EnumMap`이 일반 `HashSet`/`HashMap`보다 훨씬 효율적이다. `EnumSet`은 비트 벡터(bit vector)로 구현되어 각 enum 상수를 비트 하나로 표현한다. 교집합·합집합 연산이 비트 AND·OR 하나로 끝난다.

```java
enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

// EnumSet — 비트 벡터 기반, 매우 빠름
EnumSet<Day> weekdays = EnumSet.range(Day.MON, Day.FRI);
EnumSet<Day> weekend  = EnumSet.of(Day.SAT, Day.SUN);
EnumSet<Day> all      = EnumSet.allOf(Day.class);

// EnumMap — 배열 기반, enum 순서로 저장
EnumMap<Day, String> schedule = new EnumMap<>(Day.class);
schedule.put(Day.MON, "회의");
schedule.put(Day.FRI, "배포");
```

`EnumMap`은 내부적으로 enum 선언 순서에 따라 인덱싱된 배열을 사용한다. 해시 계산이 필요 없어 `HashMap`보다 빠르고 메모리도 적게 쓴다. enum 타입을 키로 사용하는 모든 경우에 `EnumMap`을 우선 고려한다.
