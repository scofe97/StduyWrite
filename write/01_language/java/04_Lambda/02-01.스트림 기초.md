# 스트림 기초

---

> 스트림(Stream)은 데이터 처리 연산을 지원하도록 소스에서 추출된 연속된 요소다. 컬렉션이 데이터를 어떻게 저장할지를 다루는 반면, 스트림은 데이터를 어떻게 처리할지를 선언적으로 표현한다.

## 스트림이란 무엇인가

스트림은 자바 8에서 도입된 데이터 처리 추상화다. SQL 질의처럼 "300칼로리 이하인 요리 이름을 알파벳순으로 3개만 가져와라"를 코드로 표현할 때, 스트림을 사용하면 반복문과 임시 변수 없이 선언형으로 기술할 수 있다. 멀티코어 아키텍처에서 병렬 처리를 투명하게 활용할 수 있다는 점도 도입 배경 중 하나다.

스트림은 세 가지 핵심 속성을 가진다.

- **연속된 요소**: 특정 타입으로 이루어진 값의 집합 인터페이스를 제공한다
- **소스**: 컬렉션, 배열, I/O 자원 등 데이터 제공 소스로부터 데이터를 소비한다
- **데이터 처리 연산**: filter, map, reduce 같은 함수형 연산과 순차·병렬 실행을 모두 지원한다

```java
List<String> names = menu.stream()
        .filter(dish -> dish.getCalories() > 300)
        .map(Dish::getName)
        .limit(3)
        .collect(Collectors.toList());
```

## 컬렉션 vs 스트림

컬렉션과 스트림의 가장 큰 차이는 **데이터를 언제 계산하느냐**에 있다. 컬렉션은 현재 자료구조에 포함된 모든 값을 메모리에 저장하며, 요소를 추가하거나 삭제하기 전에 이미 계산이 완료되어 있어야 한다. 스트림은 요청할 때만 요소를 계산하는 지연(lazy) 자료구조로, 요소 추가나 삭제가 불가능하다.

반복 방식도 다르다. 컬렉션은 for-each처럼 개발자가 직접 요소를 반복하는 **외부 반복(external iteration)**을 사용한다. 스트림은 내부에서 알아서 반복을 처리하는 **내부 반복(internal iteration)**을 사용하므로, 병렬 처리나 최적화된 순서를 투명하게 적용할 수 있다.

스트림은 **일회용**이다. 한 번 탐색한 스트림은 소비된 것으로 간주하며, 다시 탐색하려면 원본 소스에서 새 스트림을 만들어야 한다.

| 구분 | 컬렉션 | 스트림 |
|------|--------|--------|
| 계산 시점 | 즉시 (전체 저장) | 요청 시 (지연 계산) |
| 반복 방식 | 외부 반복 | 내부 반복 |
| 재사용 | 가능 | 일회용 |
| 데이터 수정 | 가능 | 불가 |

## 스트림 생성

스트림은 다양한 소스로부터 생성할 수 있다.

```java
// 값으로 생성
Stream<String> s1 = Stream.of("Java", "Stream", "Lambda");

// 배열로 생성
int[] arr = {1, 2, 3, 4, 5};
IntStream s2 = Arrays.stream(arr);

// 범위로 생성 (IntStream.range vs rangeClosed)
IntStream s3 = IntStream.range(1, 5);       // 1, 2, 3, 4
IntStream s4 = IntStream.rangeClosed(1, 5); // 1, 2, 3, 4, 5

// iterate: 이전 값을 기반으로 무한 스트림 생성
Stream<Integer> evens = Stream.iterate(0, n -> n + 2); // 0, 2, 4, 6, ...

// generate: Supplier로 무한 스트림 생성
Stream<Double> randoms = Stream.generate(Math::random);
```

`iterate`와 `generate`는 무한 스트림을 생성하므로, 반드시 `limit` 같은 쇼트서킷 연산과 함께 사용해야 한다.

## 중간 연산

중간 연산(intermediate operation)은 스트림을 반환하므로 체인으로 연결할 수 있다. 핵심 특성은 **지연 평가(lazy evaluation)**로, 최종 연산이 호출되기 전까지는 실제로 아무 연산도 수행하지 않는다.

```java
// filter: Predicate를 만족하는 요소만 선택
Stream<Dish> vegetarian = menu.stream()
        .filter(Dish::isVegetarian);

// map: 각 요소를 다른 타입으로 변환
Stream<String> names = menu.stream()
        .map(Dish::getName);

// flatMap: Stream<Stream<T>>를 Stream<T>로 평탄화
List<String> uniqueChars = words.stream()
        .map(word -> word.split(""))
        .flatMap(Arrays::stream)
        .distinct()
        .collect(Collectors.toList());

// sorted: 정렬 (Comparator 생략 시 자연 순서)
Stream<String> sorted = menu.stream()
        .map(Dish::getName)
        .sorted();

// peek: 파이프라인 중간에서 요소를 확인 (디버깅용, 요소 변경 없음)
Stream<Dish> peeked = menu.stream()
        .filter(d -> d.getCalories() > 300)
        .peek(d -> System.out.println("filtered: " + d.getName()));
```

`flatMap`은 중첩 스트림을 납작하게 만드는 연산이다. `map`이 각 요소를 `Stream<String[]>`으로 변환한다면, `flatMap`은 그 스트림들을 하나의 `Stream<String>`으로 합쳐준다.

## 최종 연산

최종 연산(terminal operation)은 파이프라인을 실행하고 스트림을 닫는다. List, Integer, void 등 스트림이 아닌 결과를 반환한다.

```java
// forEach: 각 요소에 동작 적용 (반환값 없음)
menu.stream().forEach(d -> System.out.println(d.getName()));

// collect: 스트림 요소를 컬렉션으로 수집
List<String> list = menu.stream()
        .map(Dish::getName)
        .collect(Collectors.toList());

// count: 요소 개수 반환
long count = menu.stream().filter(Dish::isVegetarian).count();

// reduce: 누적 연산으로 단일 값 도출
Optional<Integer> sum = Stream.of(1, 2, 3, 4, 5)
        .reduce((a, b) -> a + b);

// findFirst: 첫 번째 요소 반환 (쇼트서킷)
Optional<Dish> first = menu.stream()
        .filter(Dish::isVegetarian)
        .findFirst();

// anyMatch: 하나라도 조건을 만족하면 true (쇼트서킷)
boolean hasVeg = menu.stream().anyMatch(Dish::isVegetarian);
```

## 스트림 파이프라인의 지연 평가

스트림은 최종 연산이 호출될 때까지 중간 연산을 실행하지 않는다. 이 지연 평가 덕분에 두 가지 최적화가 가능하다.

**쇼트서킷(short-circuiting)**: `limit(3)`을 사용하면 3개를 찾은 순간 나머지 요소에 대한 연산을 중단한다. 전체 컬렉션을 처리하지 않아도 결과를 도출할 수 있다.

**루프 퓨전(loop fusion)**: `filter`와 `map`처럼 여러 중간 연산이 하나의 패스(pass)로 합쳐진다. 각 연산마다 별도로 전체 컬렉션을 순회하지 않고, 하나의 요소가 파이프라인을 통과하는 방식으로 처리된다.

```java
// 파이프라인 예시: filter + map + limit이 하나의 패스로 처리됨
List<String> result = menu.stream()
        .filter(d -> {
            System.out.println("filtering: " + d.getName()); // 실제 호출 횟수 확인
            return d.getCalories() > 300;
        })
        .map(d -> {
            System.out.println("mapping: " + d.getName());
            return d.getName();
        })
        .limit(3)
        .collect(Collectors.toList());
// 전체 메뉴를 순회하지 않고, 조건을 만족하는 3개를 찾으면 즉시 종료된다
```
