# 스트림 활용과 수집

---

> 스트림의 진가는 최종 연산에서 드러난다. Collectors가 제공하는 다양한 수집기를 조합하면 그룹화, 분할, 통계를 선언형으로 표현할 수 있고, Java 9의 컬렉션 팩토리는 불변 컬렉션 생성 코드를 대폭 단축시킨다.

## Collector, Collect, Collectors의 구분

세 용어는 각기 다른 역할을 담당한다.

- **Collect**: `Stream`의 최종 연산 메서드. `Collector`를 인수로 받아 파이프라인을 실행한다
- **Collector**: 수집 방법을 정의하는 인터페이스. `supplier`, `accumulator`, `combiner`, `finisher`를 명세한다
- **Collectors**: 미리 구현된 `Collector`를 팩토리 메서드로 제공하는 유틸리티 클래스

```java
// 통화별로 트랜잭션을 그룹화
Map<Currency, List<Transaction>> result =
        transactions.stream()
                .collect(Collectors.groupingBy(Transaction::getCurrency));
//              ↑ Collect  ↑ Collectors의 팩토리 메서드 반환 Collector
```

## 기본 수집기: toList, toSet, toMap

가장 자주 쓰이는 수집기다.

```java
// toList: 순서 보장, 중복 허용
List<String> names = menu.stream()
        .map(Dish::getName)
        .collect(Collectors.toList());

// toUnmodifiableList: Java 10+, 수정 불가 리스트
List<String> immutable = menu.stream()
        .map(Dish::getName)
        .collect(Collectors.toUnmodifiableList());

// toSet: 중복 제거, 순서 미보장
Set<Dish.Type> types = menu.stream()
        .map(Dish::getType)
        .collect(Collectors.toSet());

// toMap: 키-값 쌍으로 수집. 키 중복 시 예외 발생
Map<String, Integer> calorieMap = menu.stream()
        .collect(Collectors.toMap(
                Dish::getName
                , Dish::getCalories
        ));
```

`toMap`은 동일한 키가 두 번 이상 등장하면 `IllegalStateException`을 던진다. 키 충돌을 허용하려면 세 번째 인수로 병합 함수를 제공해야 한다.

```java
// 키 충돌 시 새 값으로 덮어쓰기
Map<Dish.Type, Integer> maxCalByType = menu.stream()
        .collect(Collectors.toMap(
                Dish::getType
                , Dish::getCalories
                , Integer::max
        ));
```

## joining: 문자열 연결

```java
// 구분자, 접두사, 접미사를 지정할 수 있다
String menuStr = menu.stream()
        .map(Dish::getName)
        .collect(Collectors.joining(", ", "[", "]"));
// 결과: [pork, beef, chicken, french fries, ...]
```

## 요약 연산: counting, summingInt, averagingInt, summarizingInt

집계 통계를 한 번에 계산할 수 있다.

```java
long dishCount = menu.stream().collect(Collectors.counting());

int totalCalories = menu.stream()
        .collect(Collectors.summingInt(Dish::getCalories));

double avgCalories = menu.stream()
        .collect(Collectors.averagingInt(Dish::getCalories));

// summarizingInt: count, sum, min, max, average를 한 번에 반환
IntSummaryStatistics stats = menu.stream()
        .collect(Collectors.summarizingInt(Dish::getCalories));
// IntSummaryStatistics{count=9, sum=4300, min=120, average=477.7, max=800}
```

## groupingBy: 그룹화

`groupingBy`는 분류 함수를 기준으로 스트림을 Map으로 묶는다. SQL의 GROUP BY에 대응한다.

```java
// 단순 그룹화
Map<Dish.Type, List<Dish>> byType =
        menu.stream().collect(Collectors.groupingBy(Dish::getType));

// 분류 로직이 복잡할 때 람다 사용
Map<String, List<Dish>> byCaloricLevel = menu.stream()
        .collect(Collectors.groupingBy(dish -> {
            if (dish.getCalories() <= 400) return "DIET";
            else if (dish.getCalories() <= 700) return "NORMAL";
            else return "FAT";
        }));
```

### 다운스트림 수집기

`groupingBy`의 두 번째 인수로 **다운스트림 수집기(downstream collector)**를 지정하면, 각 그룹 내에서 추가 수집 연산을 적용할 수 있다.

```java
// 타입별 요리 수
Map<Dish.Type, Long> countByType =
        menu.stream().collect(Collectors.groupingBy(Dish::getType, Collectors.counting()));

// 타입별 최고 칼로리 요리
Map<Dish.Type, Optional<Dish>> maxCalByType =
        menu.stream().collect(Collectors.groupingBy(
                Dish::getType
                , Collectors.maxBy(Comparator.comparingInt(Dish::getCalories))
        ));

// collectingAndThen: Optional 언래핑
Map<Dish.Type, Dish> mostCaloricByType = menu.stream()
        .collect(Collectors.groupingBy(
                Dish::getType
                , Collectors.collectingAndThen(
                        Collectors.maxBy(Comparator.comparingInt(Dish::getCalories))
                        , Optional::get
                )
        ));

// 다수준 그룹화: 타입별 → 칼로리 등급별
Map<Dish.Type, Map<String, List<Dish>>> twoLevel = menu.stream()
        .collect(Collectors.groupingBy(
                Dish::getType
                , Collectors.groupingBy(dish -> {
                    if (dish.getCalories() <= 400) return "DIET";
                    else if (dish.getCalories() <= 700) return "NORMAL";
                    else return "FAT";
                })
        ));
```

## partitioningBy: 분할

`partitioningBy`는 `Predicate`를 분류 함수로 사용하는 특수한 그룹화다. 항상 `true`/`false` 두 키를 가진 Map을 반환한다.

```java
// 채식 여부로 분할
Map<Boolean, List<Dish>> partitioned =
        menu.stream().collect(Collectors.partitioningBy(Dish::isVegetarian));
// {false=[pork, beef, chicken, ...], true=[french fries, rice, ...]}

// 다운스트림 수집기와 조합
Map<Boolean, Map<Dish.Type, List<Dish>>> vegByType =
        menu.stream().collect(Collectors.partitioningBy(
                Dish::isVegetarian
                , Collectors.groupingBy(Dish::getType)
        ));
```

`filter`로도 같은 결과를 얻을 수 있지만, `partitioningBy`는 참과 거짓 양쪽 그룹을 동시에 유지한다는 점이 다르다. 조건을 만족하지 않는 그룹도 함께 필요할 때 유용하다.

## reduce vs collect

두 연산 모두 스트림 요소를 누적해서 단일 결과를 만들지만, 의미와 용도가 다르다.

```java
// reduce로 리스트 만들기 - 잘못된 사용
List<Integer> wrong = Stream.of(1, 2, 3).reduce(
        new ArrayList<>()
        , (list, e) -> { list.add(e); return list; }  // 누적자가 가변
        , (l1, l2) -> { l1.addAll(l2); return l1; }
);
// 병렬 실행 시 여러 스레드가 같은 리스트를 수정 → 데이터 손상

// collect로 리스트 만들기 - 올바른 사용
List<Integer> correct = Stream.of(1, 2, 3).collect(Collectors.toList());
```

`reduce`는 두 값을 하나로 합치는 **불변형 연산**이다. 컨테이너(리스트, 맵)를 만들 때 `reduce`를 사용하면 병렬 실행에서 공유 가변 상태 문제가 발생한다. 컬렉션 생성에는 반드시 `collect`를 사용해야 한다.

| 구분 | reduce | collect |
|------|--------|---------|
| 목적 | 두 값을 하나로 | 컨테이너에 누적 |
| 상태 | 불변 | 가변 컨테이너 |
| 병렬 안전성 | 독립 연산이면 안전 | 설계상 안전 |

## Collection API 팩토리 (Java 9)

Java 9 이전에는 소규모 불변 컬렉션을 만들기 위해 `new ArrayList<>()` + `add` 반복 또는 `Collections.unmodifiableList(Arrays.asList(...))` 같은 우회 코드가 필요했다. Java 9는 팩토리 메서드를 도입했다.

```java
// List.of: 불변 리스트
List<String> friends = List.of("Raphael", "Olivia", "Thibaut");
// friends.add("x"); → UnsupportedOperationException

// Set.of: 불변 집합 (중복 요소 전달 시 IllegalArgumentException)
Set<String> friendSet = Set.of("Raphael", "Olivia", "Thibaut");

// Map.of: 10쌍 이하일 때 (키-값 교차 나열)
Map<String, Integer> ageMap = Map.of(
        "Raphael", 30
        , "Olivia", 25
        , "Thibaut", 26
);

// Map.ofEntries: 10쌍 초과이거나 가독성이 중요할 때
Map<String, Integer> ageMap2 = Map.ofEntries(
        Map.entry("Raphael", 30)
        , Map.entry("Olivia", 25)
        , Map.entry("Thibaut", 26)
);
```

이 팩토리가 반환하는 컬렉션은 **완전한 불변**이다. `set`, `put`, `add`, `remove` 모두 `UnsupportedOperationException`을 던진다. 수정이 필요하다면 `new ArrayList<>(List.of(...))` 처럼 가변 컬렉션으로 복사해서 사용한다.

`toUnmodifiableList()`와 `List.of()`의 차이는 소스에 있다. `List.of()`는 리터럴 값으로 직접 생성하고, `toUnmodifiableList()`는 스트림 파이프라인의 결과를 불변 리스트로 수집할 때 사용한다.

```java
// 스트림 결과를 불변 리스트로 수집
List<String> unmodifiable = menu.stream()
        .map(Dish::getName)
        .collect(Collectors.toUnmodifiableList());
```
