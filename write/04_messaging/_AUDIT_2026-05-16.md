## 3. 번호 체계 위반

> `04-09`가 결번이다. 04 시리즈는 `04-01~08, 04-10` 9개 문서로 구성돼 있고 9번이 비어 있다.

second-brain-harness §5.0은 "번호는 단순 작성 순서가 아니라 주제 묶음 기준"임을 명시하고, §5.1은 두 자리 절 번호 강제와 결번 회피 의도를 시사한다. 현재 결번이 어떤 의도였는지 README에는 설명이 없다.

선택지는 두 가지다.

**옵션 A — `04-10`을 `04-09`로 재번호한다.**
이점은 정렬 안정성과 인지 비용 절감이다. 비용은 인바운드 링크 갱신이다. `04-10`은 `04-08`의 `related`에 들어가 있을 가능성이 있고, `spring/` 하위 5개 문서가 `04-10`을 `related`로 잡고 있다(`04-10` 프론트매터 역참조로 확인됨). 재번호 시 최소 6개 파일 동시 수정.

**옵션 B — `04-10`을 유지하고 README에 결번 사유를 명시한다.**
이점은 변경 비용 0. 비용은 미래 본인이 "왜 9번이 비어 있지?" 묻는 인지 비용을 README 한 줄로 흡수한다는 점. 결번이 의도된 경우(예: `04-09 Kafka 트랜잭션 API`가 추후 추가될 자리) 이 옵션이 자연스럽다.

**리포트의 추천**: 옵션 B. 사유는 다음과 같다. 결번이 우연이라면 옵션 A가 맞지만, `04-08 Exactly-once`와 `04-10 Kafka 공통 정책 스타터` 사이에는 "Kafka 트랜잭션 API와 Producer idempotent 옵션"이 들어갈 자연스러운 자리가 있다. 미래 추가 가능성이 있는 자리를 비워 두는 편이 학습 트리의 확장에 유리하다.

## 7. README MOC 점검

> README는 카테고리 인덱스로서 기능하지만, 학습 흐름이 산문 한 단락에 묻혀 있어 처음 진입하는 독자가 한 번에 잡기 어렵다.

현재 README의 "학습 흐름" 섹션은 다음과 같다.

```markdown
01 EDA & 통합 → 02 메시지 모델·계약 → 03 토픽 설계 → 04 브로커 아키텍처
→ 05 일관성 패턴 → 06 스트림 처리 → 07 CQRS·이벤트 소싱 → (심화) 09_advanced
```

문제는 두 가지다. 하나, 화살표가 산문 안에 있어 "어디서 어디로 가는지"가 시각적으로 약하다. 둘, 각 그룹 내부의 진입점 문서(01-01, 02-01 등)가 무엇인지가 표 없이 분산돼 있다.

권장 변경은 Mermaid `graph TB`로 흐름을 시각화하고, 각 그룹의 진입점 문서명을 노드 라벨에 명시하는 것이다. writing SKILL.md 규칙 15는 세로 방향(TD/TB)을 기본으로 한다. 노드 수가 7개이므로 가로로 그리면 잘린다.

이 변경은 본 리포트의 §8 P1으로 둔다. 메타데이터 백필이 끝난 다음 단계에서 처리한다.

추가로 README의 "실 사용 코드" 섹션은 `okestro/tps-gitlab2` 경로를 가리키는데, 이는 사내 자료 경계에 가까운 정보다. second-brain-harness §11 민감정보 센서 관점에서 외부 공유 시 점검 대상으로 표시해 둔다. `_company/`로 이동하지는 않아도 되지만, 외부 발표·블로그 추출 시 이 단락은 제외하는 것이 안전하다.

## 10. 검증 노트

이 리포트의 수치는 다음 명령으로 재현 가능하다.

- `tags: []` 26개: `grep -l "^tags: \[\]$" *.md` 결과 26줄 (README 포함). 본문 기준 25개.
- `related: []` 26개: `grep -l "^related: \[\]$" *.md` 결과 26줄 (README 포함). 본문 기준 25개.
- `status: final` 1개: `grep -l "^status: final$" *.md` 결과 `04-08`과 `README.md`. 본문 기준 1개.
- `source` 누락 32개: `head -15`에서 `^source:` 미발견. `05-08`만 source 존재.
- 본문 cross-link 0건 22개: `grep -cE "\]\([0-9]{2}-[0-9]{2}" *.md` 결과 0.
- 번호 결번: `ls 04-*.md` 결과 `04-01~08, 04-10`. `04-09` 부재.

수치가 시간이 지나 어긋날 수 있으므로 다음 세션에서 작업 전 위 명령을 한 번 더 돌려 갱신한다.
