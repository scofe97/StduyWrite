# 책 CH01 — 좋은 소프트웨어 설계란 무엇이고, 왜 필요한가?
---
> ⏳ **사용자 입력 대기 중**. 본문은 사용자가 책 1장을 읽고 *핵심 문장·예시·인상적이었던 한 줄* 을 전달해 주면 어시스턴트가 정리합니다. 절 제목만 보고 추측해서 채우지 않습니다.

## 작성 절차 (본 시리즈 공통)

1. 사용자가 책의 해당 장을 읽고 다음 형태로 핵심을 전달:
   - 장의 **핵심 주장** 1~3줄
   - **인상적이었던 문장** 직접 인용 (페이지 표기)
   - 저자가 든 **예시·비유** 1~2개
   - 사용자가 *이해가 모호한 지점* 또는 *동의/반대* 하는 지점
2. 어시스턴트는 사용자가 던진 내용을 두 갈래로 분류:
   - 기존 자매 문서에 *이미 있는* 개념 → 본 파일에는 한 줄 + cross-link 만
   - 자매 문서에 *없는* 개념 → 본 파일에서 정리
