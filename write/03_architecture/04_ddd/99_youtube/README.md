# 99_youtube — DDD 영상 자료 모음집
---
> 외부 유튜브 강의 등 영상 자료에서 정리한 DDD 학습 노트를 모읍니다. 교과서(Evans/Vernon) 기반 16편과 분리해, 실무자의 현장 경험·코드 데모·AI 도구 실연을 별도로 보관합니다.

이 폴더는 `04_ddd/` 의 교과서 시리즈(01-01 ~ 04-04)와 결이 다른 자료를 담습니다. 교과서 시리즈가 Evans/Vernon 원전을 SSOT 로 삼는다면, 이곳은 외부 영상 강의를 출처로 합니다. 그래서 두 갈래를 섞지 않고, 영상 자료만 따로 모아 출처와 결을 명확히 구분합니다.

현재 두 갈래의 영상을 정리했습니다. 01~03 은 한 SI 20년 경력 실무자의 **5-Step 아키텍처 시리즈**로, 빈약한 모델 진단 → 상태 모델 구현 → AI 설계 자동화 순으로 잇습니다. 04~05 는 서로 다른 화자의 **단독 발표**로, DDD 를 현장에 도입하는 전략(박재성, 우아콘)과 Modular Monolith 를 코드로 구현하는 방법(Adesso, Spring IO)을 다룹니다.

## 교과서 16편과의 관계

각 노트는 교과서 시리즈에 이미 정리된 개념(Entity/VO 정의·Aggregate 경계·유비쿼터스 언어 등)을 재서술하지 않고 해당 편으로 링크합니다. 영상 고유의 실무 관점만 본문으로 다룹니다. 특히 03편은 교과서 [04-04 AI 코딩 시대의 DDD](../04-04.AI%20코딩%20시대의%20DDD.md)(원칙)와 짝을 이루는 실전 워크플로우이고, 04·05편은 [03-04 모놀리스에서 마이크로서비스로](../03-04.모놀리스에서%20마이크로서비스로%20—%20언제%2C%20왜.md) 의 분리 의사결정과 강하게 이어집니다.

## 파일명 규칙

이 폴더는 2단 중첩(`04_ddd/99_youtube/`)이라 내부 파일명을 교과서 시리즈의 `NN-NN.` 패턴 대신 단순 순번 `NN.` 으로 둡니다. 의도된 예외입니다.

> 각 화자의 방법론(5-Step 아키텍처·ORM 비판·그릴미, "오스트랄로피테쿠스 패턴"·"마이크로서비스 프리미엄" 같은 명명)은 표준 DDD 절차가 아니라 발표자의 해석입니다. 각 노트가 발표자 주장과 원전 정의를 구분해 표기합니다.
