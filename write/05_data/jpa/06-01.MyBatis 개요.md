# MyBatis 개요
---
> MyBatis 는 SQL Mapper — JPA 와 JdbcTemplate 의 중간. SQL 을 직접 쓰지만 ResultSet 매핑과 파라미터 바인딩의 반복은 줄여준다. SQL 통제권을 유지하면서 보일러플레이트만 제거하고 싶을 때 선택.

## 1. 위치 — JDBC, JdbcTemplate, MyBatis, JPA

> 네 단계의 추상화 — 각각이 어떤 책임을 가져가는가.

| 도구 | 책임 |
|------|------|
| JDBC | 전부 직접 (Connection, PS, RS) |
| JdbcTemplate | SQL 직접, 매핑·예외 자동 |
| MyBatis | SQL 직접 (XML 분리), 매핑 자동, 동적 SQL 강력 |
| JPA | SQL 자동 생성, 객체 그래프 매핑 |

오른쪽으로 갈수록 추상도 증가, 통제 감소.

### 1-1. 어떤 도구를 언제

| 상황 | 권장 |
|------|------|
| SQL 이 도메인 자산 (튜닝·최적화 중심) | MyBatis |
| 객체 그래프 중심 도메인 | JPA |
| 단순 SQL 다수 + 동적 쿼리 거의 없음 | JdbcTemplate |
| 동적 쿼리 + SQL 통제 + JPA 회피 | MyBatis |

## 2. JDBC vs MyBatis

> 같은 SELECT 가 어떻게 줄어드는가.

### JDBC

```java
public Member findById(Long id) throws SQLException {
    String sql = "SELECT * FROM member WHERE id = ?";
    try (Connection conn = dataSource.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setLong(1, id);
        try (ResultSet rs = ps.executeQuery()) {
            if (rs.next()) {
                Member m = new Member();
                m.setId(rs.getLong("id"));
                m.setName(rs.getString("name"));
                return m;
            }
            return null;
        }
    }
}
```

### MyBatis

```java
@Mapper
public interface MemberMapper {
    Member findById(Long id);
}
```

```xml
<mapper namespace="com.example.MemberMapper">
    <select id="findById" resultType="Member">
        SELECT * FROM member WHERE id = #{id}
    </select>
</mapper>
```

호출:
```java
Member m = memberMapper.findById(1L);
```

## 3. XML 파일 경로와 특수문자

> SQL 을 XML 에 두기 — 코드와 SQL 분리.

```yaml
mybatis:
  mapper-locations: classpath:mapper/**/*.xml
  type-aliases-package: com.example.domain
```

### 3-1. XML 특수문자

XML 은 `<`, `>`, `&` 를 그대로 못 쓴다. 두 가지 해결:

1. **엔티티 참조** — `&lt;`, `&gt;`, `&amp;`.
2. **CDATA 구문** — SQL 통째로 감싸기.

```xml
<select id="findRecent" resultType="Member">
    <![CDATA[
        SELECT * FROM member
        WHERE created_at < #{cutoff}
          AND age > 20
    ]]>
</select>
```

CDATA 안에서는 XML 파싱이 적용되지 않아 SQL 그대로 가능.

## 4. 동적 쿼리 — MyBatis 의 강점

> JPA·JdbcTemplate 의 약점 — MyBatis 가 가장 자연스럽게 처리.

```xml
<select id="search" resultType="Member">
    SELECT * FROM member
    <where>
        <if test="name != null">
            AND name = #{name}
        </if>
        <if test="minAge != null">
            AND age >= #{minAge}
        </if>
        <if test="status != null">
            AND status = #{status}
        </if>
    </where>
    <if test="orderBy != null">
        ORDER BY ${orderBy}
    </if>
</select>
```

| 태그 | 용도 |
|------|------|
| `<if>` | 조건부 절 |
| `<choose>`/`<when>`/`<otherwise>` | switch |
| `<where>` | WHERE + 첫 AND 자동 제거 |
| `<set>` | UPDATE 의 SET + 끝 쉼표 자동 제거 |
| `<foreach>` | IN 절 등 반복 |

`#{}` 는 PreparedStatement 바인딩 (안전), `${}` 는 문자열 치환 (SQL Injection 위험 — 정렬 컬럼 등 통제된 값에만).

### 4-1. foreach 의 IN 절

```xml
<select id="findByIds" resultType="Member">
    SELECT * FROM member
    WHERE id IN
    <foreach item="id" collection="ids" open="(" separator="," close=")">
        #{id}
    </foreach>
</select>
```

## 5. ResultMap — 복잡한 매핑

> 컬럼 이름과 필드 이름이 다르거나 연관 객체를 함께 매핑할 때.

```xml
<resultMap id="memberWithTeam" type="Member">
    <id property="id" column="member_id"/>
    <result property="name" column="member_name"/>
    <association property="team" javaType="Team">
        <id property="id" column="team_id"/>
        <result property="name" column="team_name"/>
    </association>
</resultMap>

<select id="findWithTeam" resultMap="memberWithTeam">
    SELECT m.id AS member_id, m.name AS member_name,
           t.id AS team_id, t.name AS team_name
    FROM member m JOIN team t ON m.team_id = t.id
</select>
```

`<association>` 으로 N:1, `<collection>` 으로 1:N.

## 6. 애너테이션 기반 (XML 없이)

> 간단한 SQL 은 XML 없이 애너테이션만.

```java
@Mapper
public interface MemberMapper {
    @Select("SELECT * FROM member WHERE id = #{id}")
    Member findById(Long id);

    @Insert("INSERT INTO member (name) VALUES (#{name})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    void insert(Member m);
}
```

복잡한 동적 SQL 은 여전히 XML 이 가독성 좋음. 짧은 SQL 만 애너테이션.
