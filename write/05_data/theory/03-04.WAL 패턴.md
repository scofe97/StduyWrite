# 📝 Write-Ahead Log (WAL) 패턴

*분산 시스템에서 데이터 내구성을 보장하기 위한 핵심 패턴*

---

## 🎯 패턴 개요

**Write-Ahead Log**는 데이터를 메모리에 쓰기 전에 먼저 디스크에 로그를 기록하여 시스템 장애 시에도 데이터 내구성을 보장하는 패턴입니다. 이 패턴은 PostgreSQL, Apache Kafka, Apache Cassandra 등 수많은 분산 시스템에서 핵심적으로 사용됩니다.

### 핵심 개념

- **내구성 (Durability):** 모든 변경 사항을 디스크에 먼저 기록하여 시스템 장애에도 데이터 손실 방지
- **순차 쓰기 (Sequential Write):** 로그 파일에 순차적으로 append하여 디스크 I/O 성능 최적화
- **복구 (Recovery):** 시스템 재시작 시 로그를 재생하여 이전 상태 복원
- **원자성 (Atomicity):** 모든 작업이 완전히 수행되거나 전혀 수행되지 않음을 보장

---

## 💡 왜 필요한가?

분산 시스템에서 데이터를 안전하게 저장하려면 메모리에 쓰기 전에 디스크에 먼저 기록해야 합니다. 시스템이 갑자기 중단되더라도 디스크에 기록된 로그를 통해 데이터를 복구할 수 있습니다.

- 데이터베이스 트랜잭션: 커밋 전 로그 기록으로 ACID 보장
- 분산 합의 알고리즘: Raft, Paxos에서 복제 로그로 활용
- 메시지 큐: 메시지 손실 방지를 위한 영속성 보장
- 이벤트 소싱: 모든 상태 변경을 이벤트로 기록하여 재구성 가능

## 🏗️ 구현 구조

### LogEntry 구조

Java 25의 record를 활용한 불변 데이터 구조:

- `index`: 로그 항목의 순서 (long)
- `term`: 선거 기간 (Generation Clock) (long)
- `data`: 실제 데이터 (byte[])
- `timestamp`: 타임스탬프 (long)

### WriteAheadLog 클래스

**주요 메서드:**

- `append(LogEntry entry)`: 로그 항목 추가 및 fsync를 통한 디스크 동기화
- `readAll()`: 모든 로그 항목 읽기 및 역직렬화
- `serializeEntry(LogEntry entry)`: 로그 항목을 바이트 배열로 직렬화
- `deserializeEntry(ByteBuffer buffer)`: 바이트 배열에서 로그 항목으로 역직렬화

### 직렬화 형식

총 28+N 바이트 구조:

- `dataLength (4 bytes)`: 데이터 길이
- `index (8 bytes)`: 로그 인덱스
- `term (8 bytes)`: 선거 기간
- `timestamp (8 bytes)`: 타임스탬프
- `data (N bytes)`: 실제 데이터

## 💻 핵심 코드 예제

### append() 메서드

로그 항목을 추가하고 디스크에 동기화:

- `직렬화:` serializeEntry()로 LogEntry를 바이트 배열로 변환
- `쓰기:` fileChannel.write(버퍼)로 파일에 쓰기
- `fsync:` fileChannel.force(true)로 디스크 동기화 - 내구성 보장
- 반환: currentIndex를 증가시키고 현재 index 반환

### readAll() 메서드

파일에서 모든 로그 항목 읽기:

- 파일 크기 확인 후 ByteBuffer 할당
- 파일 내용을 버퍼로 읽기
- `버퍼에 데이터가 남아있는 동안` deserializeEntry()로 항목 복원
- 모든 항목을 List로 반환

### 직렬화/역직렬화

**`serializeEntry()`**:

- ByteBuffer.allocate(28 + data.length) - 고정 28바이트 + 가변 데이터 크기
- putInt(데이터길이) → putLong(index) → putLong(term) → putLong(timestamp) → put(data)

**`deserializeEntry()`**:

- getInt()로 데이터 길이 읽기
- getLong()으로 index, term, timestamp 읽기
- get(data배열)로 실제 데이터 읽기
- 새로운 LogEntry 객체 반환

## 🌐 실전 활용 사례

### PostgreSQL

- **디렉토리:** `pg_wal/`
- 트랜잭션 커밋 전에 WAL에 모든 변경사항 기록
- 크래시 시 WAL을 사용하여 데이터베이스 복구
- 복제(replication)에도 WAL 활용

### Apache Kafka

- **파일 형식:** `*.log`
- 모든 메시지를 순차적으로 로그에 기록
- 컴팩션 및 세그먼트 관리로 효율적인 저장공간 활용
- 브로커 재시작 시 로그를 통한 빠른 복구

### Apache Cassandra

- **디렉토리:** `commitlog/`
- 모든 쓰기 작업을 CommitLog에 먼저 기록
- SSTable로 flush되기 전까지 내구성 보장
- 노드 재시작 시 CommitLog를 통한 데이터 복구

### Redis AOF (Append-Only File)

- 모든 쓰기 명령을 AOF 파일에 기록
- 재시작 시 AOF 파일을 재생하여 데이터 복원
- fsync 정책을 통해 내구성과 성능 균형 조정

## 📊 핵심 요약

### 장점

- **내구성:** 크래시 상황에서도 데이터 손실 방지
- **성능:** 순차 쓰기로 랜덤 I/O보다 빠른 성능
- **원자성:** 트랜잭션의 전체 또는 전무 보장
- **복구:** 빠른 시스템 복구 및 재시작

### 단점

- **디스크 공간:** 로그 파일이 계속 누적되며 추가 저장공간 필요
- **복잡도:** 로그 로테이션, 컴팩션 등 추가 관리 필요
- **fsync 비용:** 디스크 동기화 시 성능 오버헤드 발생

### LogEntry.java

```java
package com.patterns.distributed.common;

/**
 * 분산 시스템의 로그 항목을 나타내는 기본 클래스
 * Write-Ahead Log, Replicated Log 등 다양한 패턴에서 사용됨
 */
public record LogEntry(
    long index,           // 로그 인덱스 (순서)
    long term,            // 선출 주기 (Generation Clock)
    byte[] data,          // 실제 데이터
    long timestamp        // 타임스탬프
) {
    public LogEntry {
        if (index < 0) {
            throw new IllegalArgumentException("Index must be non-negative");
        }
        if (term < 0) {
            throw new IllegalArgumentException("Term must be non-negative");
        }
        if (data == null) {
            throw new IllegalArgumentException("Data cannot be null");
        }
    }

    /**
     * 데이터 크기를 바이트 단위로 반환
     */
    public int size() {
        return data.length;
    }

    /**
     * 로그 항목의 간단한 문자열 표현
     */
    @Override
    public String toString() {
        return "LogEntry{" +
            "index=" + index +
            ", term=" + term +
            ", size=" + data.length +
            ", timestamp=" + timestamp +
            '}';
    }
}
```

### WriteAheadLog.java

```java
package com.patterns.distributed.chapter03;

import com.patterns.distributed.common.LogEntry;
import java.io.*;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;
import java.util.ArrayList;
import java.util.List;

/**
 * Write-Ahead Log (WAL) 패턴 구현
 *
 * 데이터를 메모리에 쓰기 전에 먼저 디스크에 로그를 기록하여
 * 시스템 장애 시에도 데이터 내구성을 보장합니다.
 *
 * 사용 예시: Apache Kafka, PostgreSQL, Cassandra
 */
public class WriteAheadLog implements AutoCloseable {

    private final Path logPath;
    private final FileChannel fileChannel;
    private long currentIndex = 0;

    public WriteAheadLog(Path logPath) throws IOException {
        this.logPath = logPath;
        this.fileChannel = FileChannel.open(
            logPath,
            StandardOpenOption.CREATE,
            StandardOpenOption.APPEND,
            StandardOpenOption.WRITE
        );
    }

    public synchronized long append(LogEntry entry) throws IOException {
        ByteBuffer buffer = serializeEntry(entry);
        fileChannel.write(buffer);
        fileChannel.force(true);  // fsync로 내구성 보장
        return currentIndex++;
    }

    public List<LogEntry> readAll() throws IOException {
        List<LogEntry> entries = new ArrayList<>();
        try (FileChannel readChannel = FileChannel.open(logPath, StandardOpenOption.READ)) {
            ByteBuffer buffer = ByteBuffer.allocate(8192);
            while (readChannel.read(buffer) > 0) {
                buffer.flip();
                while (buffer.remaining() >= 4) {
                    entries.add(deserializeEntry(buffer));
                }
                buffer.clear();
            }
        }
        return entries;
    }
```

```java
    private ByteBuffer serializeEntry(LogEntry entry) {
        byte[] data = entry.data();
        ByteBuffer buffer = ByteBuffer.allocate(4 + 8 + 8 + 8 + data.length);
        buffer.putInt(data.length);
        buffer.putLong(entry.index());
        buffer.putLong(entry.term());
        buffer.putLong(entry.timestamp());
        buffer.put(data);
        buffer.flip();
        return buffer;
    }

    private LogEntry deserializeEntry(ByteBuffer buffer) {
        int dataLength = buffer.getInt();
        long index = buffer.getLong();
        long term = buffer.getLong();
        long timestamp = buffer.getLong();
        byte[] data = new byte[dataLength];
        buffer.get(data);
        return new LogEntry(index, term, data, timestamp);
    }

    @Override
    public void close() throws IOException {
        if (fileChannel != null && fileChannel.isOpen()) {
            fileChannel.close();
        }
    }

    public long getCurrentIndex() {
        return currentIndex;
    }
}
```

```java
@Test
void testReadAllEntries() throws IOException {
    // Given
    int entryCount = 3;
    for (int i = 0; i < entryCount; i++) {
        byte[] data = ("test data " + i).getBytes();
        LogEntry entry = new LogEntry(i, 1, data, System.currentTimeMillis());
        wal.append(entry);
    }

    // When
    List<LogEntry> entries = wal.readAll();

    // Then
    assertEquals(entryCount, entries.size());
    for (int i = 0; i < entryCount; i++) {
        LogEntry entry = entries.get(i);
        assertEquals(i, entry.index());
        assertEquals(1, entry.term());
        assertEquals("test data " + i, new String(entry.data()));
    }
}
```

테스트 4: 내구성 테스트 (시스템 재시작 후 데이터 복구)

```java
@Test
void testDurabilityAfterReopen() throws IOException {
    // Given - 첫 번째 WAL에 데이터 쓰기
    byte[] data = "persistent data".getBytes();
    LogEntry entry = new LogEntry(0, 1, data, System.currentTimeMillis());
    wal.append(entry);
    wal.close();

    // When - 새로운 WAL 인스턴스로 같은 파일 열기 (시스템 재시작 시뮬레이션)
    WriteAheadLog newWal = new WriteAheadLog(logPath);
    List<LogEntry> entries = newWal.readAll();

    // Then - 데이터가 보존되어 있어야 함
    assertEquals(1, entries.size());
    assertEquals("persistent data", new String(entries.get(0).data()));

    newWal.close();
}
```
