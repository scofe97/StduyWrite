# GroupBy OrderBy 함정 — SQL 표준 모드의 오류
---
> "SELECT 절에 GROUP BY 에 없는 컬럼이 있다" 의 에러는 MySQL 의 `ONLY_FULL_GROUP_BY` 모드가 표준 SQL 을 강제하기 때문이다. MySQL 5.7+ 의 기본 동작이고, 이 함정은 옛 코드 마이그레이션에서 자주 만난다.

## 1. 에러 메시지의 의미

> `Expression #N of SELECT list is not in GROUP BY clause and contains nonaggregated column...` 의 정확한 의미.

```sql
-- 에러 발생
SELECT dept_id, name, AVG(salary)
FROM employees
GROUP BY dept_id;
```

이 쿼리에서 `dept_id` 가 같은 행이 여러 개일 때 `name` 은 어느 것을 보여줄지가 불분명하다. SQL 표준은 이런 모호함을 허용하지 않는다.

### 1-1. ONLY_FULL_GROUP_BY 모드

```sql
SELECT @@sql_mode;
-- "ONLY_FULL_GROUP_BY,STRICT_TRANS_TABLES,..."
```

이 모드가 켜져 있으면 SELECT 절의 모든 비집계 컬럼이 GROUP BY 에 있어야 한다. MySQL 5.7+ 의 기본.

## 2. 세 가지 해결

> 같은 의도를 표현하는 세 가지 방법 — 의도에 맞는 것을 고른다.

### 2-1. GROUP BY 에 추가

```sql
SELECT dept_id, name, AVG(salary)
FROM employees
GROUP BY dept_id, name;
```

`(dept_id, name)` 의 조합으로 그룹화. "같은 부서 + 같은 이름" 이 한 그룹.

### 2-2. 집계 함수로 감싸기

```sql
SELECT dept_id, MAX(name), AVG(salary)
FROM employees
GROUP BY dept_id;
```

각 부서에서 *임의 한 이름* (사전 순 최대) 만. 보통은 의도가 아닐 가능성이 높음.

### 2-3. 윈도우 함수로 재작성

```sql
SELECT DISTINCT dept_id, name, AVG(salary) OVER (PARTITION BY dept_id) AS dept_avg
FROM employees;
```

행을 그룹화하지 않고 *각 행 옆에* 부서 평균을 붙임. 가장 자연스러운 의도일 때가 많다.

여기서 질문 하나 — 어떤 결과를 원하는지 모를 때 어떻게 결정하나? "한 부서당 한 행" 을 원하면 2-1·2-2, "각 직원의 행에 부서 평균을 곁들이기" 를 원하면 2-3. 비즈니스 요구가 정답을 정한다.

## 3. ORDER BY 와의 상호작용

> `ORDER BY` 컬럼도 SELECT 절 또는 GROUP BY 절에 있어야 한다.

```sql
-- 에러
SELECT dept_id, COUNT(*)
FROM employees
GROUP BY dept_id
ORDER BY hire_date;     -- hire_date 가 SELECT 에도 GROUP BY 에도 없음
```

해결:

```sql
SELECT dept_id, COUNT(*), MIN(hire_date) AS earliest_hire
FROM employees
GROUP BY dept_id
ORDER BY earliest_hire;
```

집계 함수로 ORDER BY 의 정렬 키를 만든다.

## 4. ONLY_FULL_GROUP_BY 끄기 — 비추

> 호환성 위해 끌 수는 있지만 *데이터 모호함을 허용* 하는 결정이다.

```sql
SET SESSION sql_mode = (SELECT REPLACE(@@sql_mode, 'ONLY_FULL_GROUP_BY', ''));
```

비추하는 이유:

1. MySQL 8.0+ 는 표준 강제가 기본 방향.
2. 다른 DBMS (PostgreSQL, Oracle) 는 항상 표준 강제 — 마이그레이션 시 동일 에러.
3. 모호한 쿼리는 옵티마이저 변경 시 결과가 바뀔 수 있음.

옛 코드 마이그레이션이 끝나는 동안만 한시적으로 끄고, 본질적 해결은 §2 의 세 가지 방법으로.

## 5. 실전 패턴 — "각 그룹의 최신 행"

> 가장 자주 만나는 패턴 — 부서별 가장 최근 입사자.

### 잘못된 시도

```sql
SELECT dept_id, name, MAX(hire_date)
FROM employees
GROUP BY dept_id;
```

`name` 과 `MAX(hire_date)` 가 *다른 행에서 올 수 있다* — MySQL 의 비표준 동작에서.

### 올바른 방법 1 — 서브쿼리

```sql
SELECT e.dept_id, e.name, e.hire_date
FROM employees e
JOIN (
    SELECT dept_id, MAX(hire_date) AS latest
    FROM employees
    GROUP BY dept_id
) t ON e.dept_id = t.dept_id AND e.hire_date = t.latest;
```

### 올바른 방법 2 — 윈도우 함수

```sql
SELECT dept_id, name, hire_date
FROM (
    SELECT dept_id, name, hire_date,
           ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY hire_date DESC) AS rn
    FROM employees
) t
WHERE rn = 1;
```

윈도우 함수가 가독성·성능 모두 우위. MySQL 8.0+ 또는 다른 모든 DBMS 에서 사용 가능.

## 면접에서 받을 만한 질문

> 본문을 읽었다면 다음 질문에 답할 수 있어야 한다.

1. `ONLY_FULL_GROUP_BY` 모드가 SELECT 절의 비집계 컬럼을 막는 이유는 무엇이며, SQL 표준이 이 모호함을 허용하지 않는 근거는 무엇인가? — §1·§1-1
2. GROUP BY 위반을 푸는 세 방법(GROUP BY 추가·집계 함수·윈도우 함수)은 각각 어떤 의도에 맞는가? — §2
3. `ORDER BY` 절에 GROUP BY 에 없는 컬럼을 쓰면 왜 에러가 나며, 어떻게 해결하는가? — §3
4. `ONLY_FULL_GROUP_BY` 를 끄는 것을 비추하는 이유 세 가지는 무엇인가? — §4
5. "각 그룹의 최신 행" 패턴에서 단순 `MAX()` GROUP BY 가 틀리는 이유와 올바른 두 방법은 무엇인가? — §5
