# JdbcTemplate (기본 + NamedParameter + SimpleJdbcInsert)
---
> JdbcTemplate 은 JDBC 의 반복 코드 (`try-with-resources` · ResultSet 매핑 · 예외 변환) 를 한 줄로 줄여 줍니다. JPA 가 과한 경우, MyBatis 가 무거운 경우의 가벼운 선택입니다.

## 1. JDBC 직접 사용의 반복

> JDBC API 는 모든 SQL 호출마다 같은 보일러플레이트를 요구합니다.

```java
public Member findById(Long id) {
    String sql = "SELECT * FROM member WHERE id = ?";
    try (Connection conn = dataSource.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setLong(1, id);
        try (ResultSet rs = ps.executeQuery()) {
            if (rs.next()) {
                Member m = new Member();
                m.setId(rs.getLong("id"));
                m.setName(rs.getString("name"));
                return m;
            }
            return null;
        }
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}
```

매 메서드마다 반복되는 것:

1. Connection 획득·반환.
2. PreparedStatement 생성·파라미터 바인딩.
3. ResultSet → 객체 매핑.
4. SQLException 처리.

## 2. JdbcTemplate 으로 줄이기

> 위 메서드를 한 줄로 — 보일러플레이트는 JdbcTemplate 내부로 들어갑니다.

```java
@Repository
public class MemberRepository {
    private final JdbcTemplate jdbcTemplate;

    public MemberRepository(DataSource dataSource) {
        this.jdbcTemplate = new JdbcTemplate(dataSource);
    }

    private final RowMapper<Member> memberRowMapper = (rs, n) -> {
        Member m = new Member();
        m.setId(rs.getLong("id"));
        m.setName(rs.getString("name"));
        return m;
    };

    public Member findById(Long id) {
        return jdbcTemplate.queryForObject(
            "SELECT * FROM member WHERE id = ?", memberRowMapper, id);
    }

    public List<Member> findAll() {
        return jdbcTemplate.query("SELECT * FROM member", memberRowMapper);
    }

    public int update(Member member) {
        return jdbcTemplate.update(
            "UPDATE member SET name = ? WHERE id = ?", member.getName(), member.getId());
    }
}
```

핵심 API:

| 메서드 | 용도 |
|--------|------|
| `queryForObject(sql, rowMapper, args)` | 단건 조회 |
| `query(sql, rowMapper, args)` | 다건 조회 |
| `update(sql, args)` | INSERT/UPDATE/DELETE |
| `batchUpdate(sql, batchArgs)` | 배치 |

## 3. KeyHolder — INSERT 후 생성된 PK 얻기

> AUTO_INCREMENT PK 를 INSERT 직후 받아야 할 때.

```java
public Long insert(Member member) {
    KeyHolder keyHolder = new GeneratedKeyHolder();
    jdbcTemplate.update(conn -> {
        PreparedStatement ps = conn.prepareStatement(
            "INSERT INTO member (name) VALUES (?)",
            Statement.RETURN_GENERATED_KEYS);
        ps.setString(1, member.getName());
        return ps;
    }, keyHolder);
    return keyHolder.getKey().longValue();
}
```

`Statement.RETURN_GENERATED_KEYS` 플래그가 핵심.

## 4. 동적 쿼리의 한계

> JdbcTemplate 은 *문자열 SQL* 이라 조건이 가변일 때 String 조작이 번거롭습니다.

```java
// 안티패턴 — 조건이 늘면 if 가 폭증
public List<Member> search(String name, Integer minAge) {
    StringBuilder sql = new StringBuilder("SELECT * FROM member WHERE 1=1");
    List<Object> args = new ArrayList<>();
    if (name != null) { sql.append(" AND name = ?"); args.add(name); }
    if (minAge != null) { sql.append(" AND age >= ?"); args.add(minAge); }
    return jdbcTemplate.query(sql.toString(), memberRowMapper, args.toArray());
}
```

동적 쿼리가 잦은 도메인은 MyBatis 또는 QueryDSL 이 자연스러운 선택입니다. JdbcTemplate 은 *정적 SQL 중심* 인 곳에서 빛납니다.

## 5. NamedParameterJdbcTemplate — 이름 지정 바인딩

> `?` 의 위치 의존을 이름으로 — 가독성·실수 방지.

```java
private final NamedParameterJdbcTemplate namedTemplate;

public Member findByCriteria(String name, int minAge) {
    String sql = "SELECT * FROM member WHERE name = :name AND age >= :minAge";
    Map<String, Object> params = Map.of("name", name, "minAge", minAge);
    return namedTemplate.queryForObject(sql, params, memberRowMapper);
}
```

파라미터 전달의 세 가지 방식:

| 방식 | 예 | 적합 |
|------|------|------|
| `Map<String, Object>` | `Map.of("name", "alice")` | 즉석 단순 호출 |
| `MapSqlParameterSource` | `new MapSqlParameterSource("name", "alice")` | 조건부 파라미터 추가 |
| `BeanPropertySqlParameterSource` | `new BeanPropertySqlParameterSource(member)` | DTO 객체 통째 (권장) |

`BeanPropertySqlParameterSource` 가 가장 깔끔 — 객체의 getter 와 SQL 의 `:fieldName` 이 자동 매칭.

## 6. SimpleJdbcInsert — INSERT 전용 헬퍼

> 컬럼 이름 + 값을 Map 으로 던지면 SQL 을 자동 생성.

```java
private final SimpleJdbcInsert insertMember;

public MemberRepository(DataSource ds) {
    this.insertMember = new SimpleJdbcInsert(ds)
        .withTableName("member")
        .usingGeneratedKeyColumns("id");
}

public Long insert(Member m) {
    Map<String, Object> params = Map.of("name", m.getName(), "age", m.getAge());
    return insertMember.executeAndReturnKey(params).longValue();
}
```

테이블 메타데이터를 시작 시 한 번 읽어 컬럼을 자동 매칭. INSERT 가 단순한 경우 표준 선택.
