## 2. 판단의 재설계

감지 -> 분석 -> 판단 -> 실행

데이터를 사람이 아닌 에이전트가 판단한다.

- 데이터 기반, AI 파이프라인
- 90%+ 상황을 자동처리 하고 24/7 무중단 일관된 판단

운영으로 증명된 시스템

사람의 역할이 어떻게 변경되었는가?

- 운영자에서 감독자로
  - 시스템 판단 결과 검증 및 확인
  - 반복 업무 90% 자동처리
  - 예외 상황, 전략 업무 집중

전략적 파트너

- 기술 선도력: Amazon Bedrock 기반 멀티 에이전트
- 실행력: Preserve 아키텍쳐 협업: 통계 기반 정보 증류부터 멀티 에이전트 오케트스레이션
- 생태계: 한번 구축으로 끝나는게 아닌 기술이 진화할때 함께 진화

운영방식 표준화

IAC -> AgentHub -> 워크플로우 통합 -> 개인 에이전트

## 4. ContextLake (LLM 시대의 DataLake)

DataLake 

- 데이터가 흐른다.
- 시계열 / 로그 테이블 / 지표 JSON

ContextLake

- 의미가 흐른다.
- 자연어 맥락 / 판정 근거 / 관계 맥락

추론결과의 컨텍스트 패키징 (점수, 배경, 비즈니스)

- 컨텍스트 파이프라인

LLM은 숫자에 약하지만 맥락만 있다면 전문가가 된다.

실시간 의미가 흐르는 레이크, 동시다발전 상황을 종합 분석 및 제안
