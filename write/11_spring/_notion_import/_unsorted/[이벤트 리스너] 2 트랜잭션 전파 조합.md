# 트랜잭션 전파와 조합

---

> *AFTER_COMMIT 리스너는 이전 트랜잭션이 이미 커밋된 후에 실행되므로, 이전 트랜잭션과 완전히 독립적입니다!*
> 

## 죽은 트랜잭션 문제

```java
// ❌ DANGER: DB 쓰기가 저장되지 않음!
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRED)  // 기본값
public void saveLog(UserCreatedEvent event) {
    logRepository.save(new Log());  // 저장 안됨!
}

// ✅ CORRECT: 새 트랜잭션 생성으로 정상 저장
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRES_NEW)
public void saveLog(UserCreatedEvent event) {
    logRepository.save(new Log());  // 정상 저장됨!
}
```

```java
1. UserService.saveUser() → @Transactional
   ├─ INSERT user
   └─ 메서드 종료 → 트랜잭션 커밋 시작

2. PlatformTransactionManager.commit()
   ├─ JDBC Connection.commit() 호출
   ├─ Connection.setAutoCommit(true)
   └─ 트랜잭션 상태 → COMMITTED

3. AFTER_COMMIT 리스너 실행
   └─ saveLog() 호출

4. saveLog() 내부 @Transactional(propagation = REQUIRED)
   ├─ TransactionSynchronizationManager.isActualTransactionActive() → true
   ├─ 기존 TransactionStatus 재사용 시도
   └─ 하지만 Connection은 이미 커밋됨 → **쓰기 무시됨**
```

- AFTER_COMMIT 리스너는 트랜잭션 커밋 후, `cleanupAfterCompletion()` 전에 실행됩니다.
- `TransactionSynchronizeManager.isActualTransationActive()`가 true가 반환되며, Propagation.REQUIRED는 기존 트랜잭션에 “참여”하려고 시도합니다.
- 하지만 그 트랜잭션은 이미 “죽은 트랜잭션”이며, DB 쓰기가 실행되지만 커밋되지 못합니다.

## REQUIRES_NEW

`AFTER_COMMIT`에서 DB 쓰기가 필요하면 `REQUIRE_NEW`를 조합해서 사용해야 합니다.

```java
@Service
public class UserService {
    @Transactional  // ← 이전 트랜잭션
    public User createUser(String name) {
        User user = userRepository.save(new User(name));
        eventPublisher.publishEvent(new UserCreatedEvent(user));
        return user;
        // ← 여기서 트랜잭션 커밋
    }
}

@Component
public class UserEventListener {
    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void afterCommit(UserCreatedEvent event) {
        // ← 여기는 이전 트랜잭션이 이미 커밋된 후
        // 이전 트랜잭션과 완전히 독립적
    }
}
```

- 일반적인 경우에 `@Trnsactional`의 기본 propagation은 `REQUIRED`이며, 대부분의 트랜잭션 사용시 기존 트랜잭션에 참여하는것이 올바른 동작입니다.
- `@TransactionEventListener`도 위의 이유로 `@Transactional`에 대해 `REQUIRED`가 발생하며, 이러한 특수 케이스가 생기는 경우, `REQUIRES_NEW`를 사용하는게 좋습니다.

[https://github.com/spring-projects/spring-framework/pull/33470](https://github.com/spring-projects/spring-framework/pull/33470)

```java
// ❌ 이제 런타임 에러 발생!
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional  // REQUIRED가 기본값
public void saveLog(Event e) {
    // 에러: "@TransactionalEventListener method must not be annotated with
    //       @Transactional unless when declared as REQUIRES_NEW or NOT_SUPPORTED"
}

// ✅ 명시적으로 REQUIRES_NEW 지정 - 정상 동작
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRES_NEW)
public void saveLog(Event e) {
    // 정상 실행
}

// ✅ NOT_SUPPORTED도 허용 - 트랜잭션 없이 실행
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.NOT_SUPPORTED)
public void readOnly(Event e) {
    // 정상 실행 (읽기 전용)
}
```

```java
Failed to process @EventListener annotation on bean with name 'dangerousListener': 
@TransactionalEventListener method must not be annotated with @Transactional unless when declared as REQUIRES_NEW or NOT_SUPPORTED: public void com.example.transaction.listener.DangerousListener.handleUserEvent(com.example.transaction.event.UserEvent)
```

- 스프링 프레임워크 팀은 이 문제를 인식하고 최신 버전에서 검증 로직을 추가했습니다.
- 스프링 부트 기준 3.5.0 이상, 스프링 프레임워크 기준 6.2기준 이상

| 항목 | 변경 전 | 변경 후 |
| --- | --- | --- |
| **검증 범위** | 메서드 레벨 `@Transactional`만 | 클래스 레벨 `@Transactional`도 포함 (전체 커버) |
| **오류 처리** | Silent failure (런타임 버그) | Fail-fast (스타트업 시 에러, 조기 발견) |
| **트랜잭션 격리** | 깨질 수 있음 (이벤트 조기 처리) | 보장 (새 트랜잭션이나 비트랜잭션 실행) |
| **개발자 경험** | 디버깅 어려움 | 명확한 에러 메시지로 미스컨피그레이션 즉시 알림 |

# **@TransactionalEventListener + @Transactional 조합**

---

| **전파 레벨** | **트랜잭션 상태** | **isActive** | **DB 쓰기 가능 여부** | **권장 사용** |
| --- | --- | --- | --- | --- |
| **없음** | 커밋된 TX 컨텍스트 활성 | true | ❌ 불가 (죽은 트랜잭션) | 읽기 전용 작업 |
| **REQUIRED** | 커밋된 TX에 참여 | true | ❌ 불가 (죽은 트랜잭션) | ⚠️ 사용 금지 |
| **REQUIRES_NEW** | 새 독립 TX 생성 | true | ✅ 가능 | **DB 쓰기 필수** |
| **NOT_SUPPORTED** | TX 없음 | false | ❌ 불가 | 캐시/외부 API |

## **REQUIRED (기본값)**

```java
// ❌ DANGER: 죽은 트랜잭션에 참여 - DB 쓰기가 저장되지 않음!
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRED)
public void afterCommitWithRequired(UserCreatedEvent event) {
    // ⚠️ 이미 커밋된 트랜잭션 컨텍스트에 "참여"하게 됨
    // ⚠️ 이 저장은 실제로 커밋되지 않습니다!
    userLogRepository.save(new UserLog(event.getUser().getId()));  // 저장 안됨!
}
```

- 이미 커밋된 트랜잭션 컨텍스트에 참여를 시도하며, 죽은 트랜잭션 문제가 발생합니다.
- DB 쓰기가 실제로 커밋되지 않습니다.

```java
[Service Transaction]
  └─ 커밋 완료 ★
      [TransactionSynchronizationManager는 아직 활성]

[AFTER_COMMIT Listener - REQUIRED]
  ├─ 기존 커밋된 트랜잭션 컨텍스트에 참여 시도
  ├─ userLogRepository.save()  ← 실행되지만 커밋 안됨!
  └─ ❌ DB 쓰기 손실!
```

## **REQUIRES_NEW**

```java
// ✅ CORRECT: 새로운 독립 트랜잭션 생성 - DB 쓰기가 정상적으로 저장됨
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRES_NEW)
public void afterCommitWithRequiresNew(UserCreatedEvent event) {
    // 완전히 새로운 독립 트랜잭션 생성
    auditLogRepository.save(new AuditLog(event.getUser().getId()));  // 정상 저장!
}
```

- REQUIRES_NEW는 항상 새로운 트랜잭션을 사용합니다.
- DB쓰기가 정상적으로 커밋됩니다.

## BEFORE_COMMIT

```java
@TransactionalEventListener(phase = TransactionPhase.BEFORE_COMMIT)
@Transactional(propagation = Propagation.REQUIRED)
public void beforeCommitWithRequired(UserCreatedEvent event) {
    // 발행자의 트랜잭션에 참여
    statisticsRepository.increment("user_count");
}
```

- 발행자와 동일한 트랜잭셔에 참여합니다.

## 외부 API 호출

```java
// Service
@Transactional
public User createUser(String name) {
    User user = userRepository.save(new User(name));
    eventPublisher.publishEvent(new UserCreatedEvent(user));
    return user;
    // ← TX1 커밋 완료
}

// Listener
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
public void callExternalApi(UserCreatedEvent event) {
    // TX1과 완전히 독립
    // 외부 API 실패해도 User 데이터는 안전
    externalApi.notify(event.getUser());
}
```

## 로그 저장(새 트랜잭션)

```java
// Service
@Transactional
public User createUser(String name) {
    User user = userRepository.save(new User(name));
    eventPublisher.publishEvent(new UserCreatedEvent(user));
    return user;
    // ← TX1 커밋 완료
}

// Listener
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
@Transactional(propagation = Propagation.REQUIRES_NEW)
public void saveLog(UserCreatedEvent event) {
    // TX2 시작 (TX1과 독립)
    logRepository.save(new Log(event.getUser().getId()));
    // TX2 커밋
}
```
