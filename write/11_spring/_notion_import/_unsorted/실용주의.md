# 2. 객체 종류

## VO

소프트웨어 설계자 입장에서 불변성, 동등성, 자가 검증이라는 특징을 만족한다면 VO이다.

- 불변성
    - 값은 변하지 않는다. (1은 변하지 않는 값)
    - 불변성은 시스템 복잡도를 낮추고, 신뢰할 수 있게 만들어준다.
    - 불변성을 추구하고, 불확실성을 제거하는 편이 좋으나 모든 객체와 함수를 불변으로 만들수는 없다.
- 동등성
    - 어떤 객체가 값이고, 상태가 모두 같다면 같은 객체로 봐야한다.
    - VO는 식별자가 없어야한다. (VO자체가 값이기 때문)
- 자가 검증
    - 클래스 상태로 유요한지 검증할 수 있어야 한다.
    - 아무리 불변이고, 동등성을 보장하더라도 잘못된 값이 들어간다면 신뢰가 불가능하다.

## DTO

DTO는 데이터 전송에 사용되는 객체를 의미한다.

- 사용이유
    - 다른 객체의 메서드를 호출하거나, 시스템을 호출할 때 매개변수 나열이 힘들기 때문
    - 다른 객체 & 시스템에 데이터를 구조적으로 만들어 전달하기 위함

DTO는 객체라 보기 애매하다. 오롯이 데이터를 효과적으로 전달하는데만 집중해야 한다.

- DTO에 데이터 읽기/쓰기 이외의 로직이 들어가서는 안된다.
- DTO는 그저 데이터 나열이 힘들어 전달하기 위해 만들어진 객체

## DAO

DAO는 데이터베이스 접근과 관련된 역할을 지닌 객체를 의미한다.

1.  데이터베이스와의 연결 관리
2. 데이터베이스에 연결해 데이터에 대한 CRUD 연산 수행
3. 보안 취약성을 고려한 쿼리 작성

데이터에 접근하기 위해 만들어진 객체 → 복잡하고 번거로운 DB 접근 관련 로직을 전문적으로 처리하기 위해 만들어진 객체
