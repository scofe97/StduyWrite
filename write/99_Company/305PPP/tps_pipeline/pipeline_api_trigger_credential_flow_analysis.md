## 2. 빠른 결론
핵심 결론은 다음과 같다:

- 트리거 실행 시점에는 Jenkins Job 정의를 다시 반영하는 로직이 있다.
- 파이프라인 생성 시 credential을 자동으로 함께 추가하는 로직은 없다.
- 일반 파이프라인 생성과 수정은 둘 다 Jenkins 반영이 먼저이고 DB 저장이 뒤따른다.
- 실행 이후 상태 동기화, 로그 수집, 예약 실행 후처리는 `ppln-logging-api`가 담당한다.

## 3. 문서 선택 가이드
질문 목적에 따라 다음 문서를 보면 된다:

- "실행할 때 Jenkins가 실제로 수정되나?"를 보려면 1번 문서
- "credential이 자동으로 붙나?"를 보려면 2번 문서
- "create/update 시퀀스가 어떻게 되나?"를 보려면 3번 문서
- "실행, 중지, 예약, 동기화까지 한 번에 보고 싶다"면 4번 문서
- "일반 실행과 트리거 실행의 차이를 보고 싶다"면 5번 문서
- "`ftl` 파일이 무엇인지, 왜 필요한지"를 보려면 5번 문서
- "`ftl` 말고 다른 템플릿 형식이 있는지, 2026-03 기준 권장안이 무엇인지"를 보려면 6번 문서
