# AWS LG U+ 에이전틱 AI 기반 대규모 마이그레이션

## 배경

대규모 애플리케이션 환경 - 100개 이상의 애플리케이션이 다양한 기술 스택으로 운영되는 환경

온프레미스 기반의 분산된 운영구조: 서비스별 인프라, 로그, 보안이 독립적으로 관리

SI 기반 개발체계: 서비스마다 다른 외주 개발사 중심의 개발 및 운영

### 문제점

- 분산된 운영구조로 인해 인프라와 운영현황을 통합적으로 모니터링 어려움
- 보안 대응 한계
- 혁신속도 저하

## 단순 마이그레이션이 아닌 AI 기반혁신

운영 표준화와 보안강화 - 분산된 온프레미스 환경을 한번에 관리

마이그레이션 속도와 품질 확보 - 100개 이상의 애플리케이션을 안정적이고 빠르게 마이그레이션

AX 기반 서비스 혁신 가속화 - AI 빠르게 도입해서 서비스 혁신에 속도를 높인다.

### 복잡성

- 표준화되지 않은 애플리케이션 환경 (다양한 아키텍쳐, 기술 스택)
- 레거시 기술 및 코드 기반
- 대규모 전환에 따른 검증 부단

### 해결책

마이그레이션 4가지 단계

1. 진단: 현황 분석과 리스크 파악을 AI로 진행
2. 설계: AI분석결과 기반으로 아키텍쳐 및 리팩토링 도출
3. 구현및 이전: 실제 변환 수행
4. 확산(스케일 아웃): 반복되는건 자동화 해서 제외

## 4단계 상세

### Assessment

전환 과정에서 발생할 수 있는 리스크를 사전에 식별하여 최적의 전환 솔루션과 방법

- 산출물 분석 -> 아키텍쳐 다이어그램  / 연동 명세서 / 배포-운영 가이드 / DB 스키마
- 애플리케이션 인벤토리 구성, 인터뷰 에이전트 진행 (UCMP)
  - ex) DAU가 몇명인지 등
  - ex) 비즈니스 목적
- 최종적으로 마크다운 형식의 리포트와 json형식의 데이터 제공

인터뷰 에이전트 효과

1. 인터뷰 품질 표준화: 모든 애플리케이션에 동일한 기준으로 질문
2. 정보 누락 최소화: 마이그레이션에 필요한 핵심 질문 세트와 재질문
3. 표준화된 결과물: AI가 분석 가능한 형태로 결과를 축적하여 설계 및 액션 아이템 도출

코드 분석

- AWS의 수많은 마이그레이션을 경험으로 진행
- Transformation Definition -> 코드 분석 기준과 결과 형식
