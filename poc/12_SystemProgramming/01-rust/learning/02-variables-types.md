# 02. 변수, 타입, 함수

Rust의 변수는 기본적으로 불변(immutable)입니다. 타입 시스템은 강타입(strongly-typed)이며, 컴파일러가 대부분의 타입을 추론할 수 있습니다. 함수는 표현식 기반이며, 마지막 표현식의 값이 자동으로 반환됩니다.

## 명령어 요약

| 키워드/구문 | 설명 |
|-------------|------|
| `let x = 5;` | 불변 변수 선언 |
| `let mut x = 5;` | 가변 변수 선언 |
| `let x: i32 = 5;` | 타입 명시 |
| `let x = 5; let x = "str";` | 섀도잉 (타입 변경 가능) |
| `i8, i16, i32, i64, i128` | 부호 있는 정수 |
| `u8, u16, u32, u64, u128` | 부호 없는 정수 |
| `f32, f64` | 부동소수점 |
| `bool` | true/false |
| `char` | 유니코드 문자 ('가', '😊') |
| `&str` | 문자열 슬라이스 |
| `(i32, f64, bool)` | 튜플 |
| `[i32; 5]` | 고정 크기 배열 |
| `fn name(param: Type) -> ReturnType { }` | 함수 정의 |
| `{ expr }` | 블록 표현식 |

## 체크포인트

1. **let x = 5; x = 6; 이 컴파일 실패하는 이유는?**
   - x가 불변 변수이기 때문. `let mut x = 5;`로 선언해야 재할당 가능

2. **섀도잉과 mut의 차이는?**
   - 섀도잉: 새로운 변수 생성, 타입 변경 가능, 불변성 유지
   - mut: 같은 변수 수정, 타입 변경 불가, 가변

3. **표현식과 문장의 차이는?**
   - 표현식: 값을 반환 (예: `5 + 6`, `if x { 1 } else { 2 }`)
   - 문장: 값을 반환하지 않음, 세미콜론으로 종료 (예: `let x = 5;`)

4. **함수의 마지막 표현식에 세미콜론을 붙이면?**
   - 문장이 되어 값을 반환하지 않음 (반환 타입 `()`)
   - `return` 키워드 사용 시 세미콜론 필요

5. **i32와 isize의 차이는?**
   - i32: 항상 32비트
   - isize: 플랫폼에 따라 32비트 또는 64비트 (포인터 크기와 동일)

6. **배열과 튜플의 근본 차이는?**
   - 배열: 같은 타입의 요소들, 고정 크기, 인덱스 접근
   - 튜플: 다른 타입의 요소들, 고정 크기, 패턴 매칭 또는 `.0`, `.1` 접근
