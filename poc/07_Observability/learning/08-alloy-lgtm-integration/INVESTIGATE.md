# Ch06. Alloy-Loki-Tempo Integration - 점검 질문

## Q1. 로그와 트레이스를 연결할 때 가장 중요한 공통 키는 무엇인가?

**핵심 포인트**:
- `trace_id`
- 하지만 Loki 고정 라벨로 올릴지는 신중해야 함

## Q2. 연결 키와 검색 키를 왜 구분해야 하는가?

**핵심 포인트**:
- 건별 식별자와 집계용 필드의 특성이 다름
- 비용과 탐색성의 균형이 필요함

## Q3. 현재 PoC 구조를 Alloy-Loki-Tempo 구조로 어떻게 재해석할 수 있는가?

**핵심 포인트**:
- 현재 collector 중심 구조를 이해
- backend와 로그 경로를 확장해서 상상할 수 있어야 함

## Q4. 애플리케이션이 직접 Loki와 Tempo에 각각 보내지 않고 Alloy를 두는 이유는 무엇인가?

**핵심 포인트**:
- 설정 단순화
- 처리와 라우팅 중앙화
- backend 교체 용이성

## Q5. Grafana Explore에서 장애 분석을 시작할 때 일반적인 흐름은 무엇인가?

**핵심 포인트**:
- 로그에서 이상 징후
- trace 열기
- span 병목 확인
- 필요 시 다시 로그로 돌아오기

## Q6. 통합이 안 되는 팀에서 자주 보이는 징후는 무엇인가?

**핵심 포인트**:
- 로그에는 trace_id 없음
- 서비스명 표준이 맞지 않음
- 라벨 전략이 일관되지 않음
