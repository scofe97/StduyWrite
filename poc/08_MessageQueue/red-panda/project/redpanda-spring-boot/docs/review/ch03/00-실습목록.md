## I5 구현 범위: 토픽 유형별 설정 (Entity/Event/Command)

**근거**: 11-topic-design.md §7에서 Entity/Event/Command 토픽 유형별 `NewTopic` 설정 차이가 추가됨. 기존 I1은 기본 NewTopic 빈 생성만 다루며, cleanup.policy/retention 등 유형별 설정은 다루지 않음.

### 구현 항목
- **Entity Topic**: `cleanup.policy=compact` + `delete.retention.ms` + tombstone 발행/소비
- **Event Topic**: `cleanup.policy=delete` + `retention.ms` + `compression.type`
- **Command Topic**: 짧은 retention + 단일 컨슈머 패턴
- 3종 `NewTopic` Bean 생성 (`ch02.entity.users`, `ch02.event.orders`, `ch02.command.process`)
- `rpk topic describe`로 설정 확인
- 기존 `chapter2.orders`(Event 유형)와 비교하여 차이 체감

### I1과의 차이
- I1: NewTopic 빈 생성 방법, 파티션/레플리카 기본 설정
- I5: 토픽 **용도별** cleanup.policy, retention, compression 설정 — "왜 다르게 설정하는가"에 초점
