## 각 실습의 구현 범위

### 1. Avro 스키마 설계 (`AvroSchemaDesigner`)

- `src/main/avro/ch10/SchemaEvolutionV1.avsc` 읽기 — 기본 필드 구성 이해
- `src/main/avro/ch10/SchemaEvolutionV2.avsc` 읽기 — 필드 추가 시 기본값 패턴 확인
- `src/main/avro/ch10/NestedRecord.avsc` 읽기 — 중첩 레코드(array, record, union) 타입 이해
- `AvroSchemaDesigner.java`에서 스키마 설계 패턴(필드 타입 선택, union null 처리)을 설명하는 TODO 구현

### 2. Schema Registry 헬스체크 (`SchemaRegistryHealthCheck`)

- `SchemaRegistryHealthCheck.java`에서 Schema Registry REST API(`/subjects`, `/config`) 호출 로직 구현
- 연결 실패 시 예외 처리 및 로그 출력 패턴 작성
- `Ch10TopicConfig.java`의 Schema Registry URL 설정값과 연동 확인

### 3. 호환성 테스트 (`CompatibilityTester`)

- `CompatibilityTester.java`에서 V1 스키마 등록 → V2 스키마 호환성 검증 요청 시퀀스 구현
- BACKWARD 호환 성공 케이스(기본값 있는 필드 추가)와 실패 케이스(기본값 없는 필드 추가) 모두 작성
- Schema Registry `/compatibility/subjects/{subject}/versions/latest` 엔드포인트 활용

### 4. 스키마 진화 데모 (`SchemaEvolutionDemo`)

- `SchemaEvolutionDemo.java`에서 V1 Producer → V2 Consumer 메시지 수신 시나리오 구현
- V1 메시지를 V2 스키마로 역직렬화할 때 기본값이 채워지는 과정 로그로 확인
- `auto.register.schemas` 설정을 true/false로 전환하며 동작 차이 실험

### 5. 통합 테스트 (`Ch10SchemaRegistryTest`)

- `Ch10SchemaRegistryTest.java`에서 SchemaRegistryHealthCheck → CompatibilityTester → SchemaEvolutionDemo 순서로 전체 흐름 테스트
- @EmbeddedKafka 또는 docker-compose 로컬 인프라 기반으로 Schema Registry 포함 실행
- 호환성 위반 시 예외가 발생하는지, 호환 시 정상 역직렬화되는지 assert 검증
