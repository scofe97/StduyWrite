# Ch09 실습 #7: 통합 테스트

## 목적

Practice #1~#6 전체를 Testcontainers 기반 End-to-End 테스트로 검증한다. 정상/실패/보상/타임아웃/멱등성 5가지 시나리오를 커버한다.

## 테스트 인프라

```mermaid
graph LR
    subgraph Testcontainers
        R[Redpanda v25.3.6<br/>Kafka API + Schema Registry]
        P[PostgreSQL 16<br/>jdbc:tc URL 자동 기동]
    end

    subgraph Spring Boot Test
        O[Orchestrator]
        FH[FlightHandler]
        HH[HotelHandler]
        S[RecoveryScheduler]
    end

    O -- Command --> R
    R -- Event --> O
    R -- Command --> FH
    R -- Command --> HH
    O -- SagaState --> P
    FH -- ProcessedCommand --> P
    HH -- ProcessedCommand --> P
    S -- 조회/수정 --> P
```

- **Redpanda**: `@Container` + `@DynamicPropertySource`로 동적 주소 주입
- **PostgreSQL**: `jdbc:tc:postgresql:16:///saga` — Testcontainers JDBC URL 방식 (별도 `@Container` 불필요)
- **Awaitility**: 비동기 CTP 체인 완료를 폴링 검증 (최대 30초, 1초 간격)

## 테스트 시나리오 (5개)

### Test 1: 정상 플로우 → COMPLETED

```mermaid
sequenceDiagram
    participant T as Test
    participant O as Orchestrator
    participant F as FlightHandler
    participant H as HotelHandler
    participant DB as SagaState

    T->>O: startSaga(ICN→NRT, Tokyo Hotel)
    O->>DB: STARTED, FLIGHT_BOOKING
    O->>F: BookFlightCommand

    F->>O: FlightBooked(FLT-xxx)
    O->>DB: IN_PROGRESS, HOTEL_BOOKING
    O->>H: BookHotelCommand

    H->>O: HotelBooked(HTL-xxx)
    O->>DB: COMPLETED

    T->>DB: assertThat(status).isEqualTo(COMPLETED)
    T->>DB: assertThat(flightReservationId).startsWith("FLT-")
    T->>DB: assertThat(hotelReservationId).startsWith("HTL-")
```

**검증**: `SagaStatus.COMPLETED`, 양쪽 예약 ID 존재, `completedAt` not null.

### Test 2: Step 1 실패 → FAILED

```mermaid
sequenceDiagram
    participant T as Test
    participant O as Orchestrator
    participant F as FlightHandler
    participant DB as SagaState

    T->>O: startSaga(departure="FAIL")
    O->>DB: STARTED, FLIGHT_BOOKING
    O->>F: BookFlightCommand

    F->>O: FlightBookingFailed(reason)
    O->>DB: FAILED

    Note over O: Step 1 실패 → 보상할 대상 없음

    T->>DB: assertThat(status).isEqualTo(FAILED)
    T->>DB: assertThat(failedStep).isEqualTo("FLIGHT_BOOKING")
    T->>DB: assertThat(flightReservationId).isNull()
```

**검증**: 보상 없이 즉시 `FAILED`, `flightReservationId` null.

### Test 3: Step 2 실패 → 보상 → COMPENSATED

```mermaid
sequenceDiagram
    participant T as Test
    participant O as Orchestrator
    participant F as FlightHandler
    participant H as HotelHandler
    participant DB as SagaState

    T->>O: startSaga(hotelName="FAIL")
    O->>F: BookFlightCommand
    F->>O: FlightBooked(FLT-xxx)
    O->>DB: IN_PROGRESS

    O->>H: BookHotelCommand
    H->>O: HotelBookingFailed(reason)
    O->>DB: COMPENSATING

    O->>F: CancelFlightCommand(FLT-xxx)
    F->>O: FlightCancelled
    O->>DB: COMPENSATED

    T->>DB: assertThat(status).isEqualTo(COMPENSATED)
    T->>DB: assertThat(flightReservationId).startsWith("FLT-")
    T->>DB: assertThat(hotelReservationId).isNull()
```

**검증**: 항공 예약 후 취소 완료, `COMPENSATED`, `flightReservationId` 보존 (추적용).

### Test 4: 타임아웃 → 스케줄러 보상 → COMPENSATED

```mermaid
sequenceDiagram
    participant T as Test
    participant DB as SagaState
    participant S as Scheduler
    participant F as FlightHandler
    participant O as Orchestrator

    T->>DB: SagaState 직접 생성<br/>(HOTEL_BOOKING, stepStartedAt=3분전)

    T->>S: checkTimeouts() 수동 호출
    S->>DB: IN_PROGRESS SAGA 조회
    S->>S: stepStartedAt + 2분 < now → 타임아웃!
    S->>DB: COMPENSATING, failureReason="Timeout"
    S->>F: CancelFlightCommand(FLT-TIMEOUT-01)

    F->>O: FlightCancelled
    O->>DB: COMPENSATED

    T->>DB: assertThat(status).isEqualTo(COMPENSATED)
    T->>DB: assertThat(failureReason).contains("Timeout")
```

**핵심**: `stepStartedAt`을 과거(3분 전)로 설정하여 2분 타임아웃을 즉시 트리거. 실제 2분을 기다리지 않는다.

### Test 5: 멱등성 → ProcessedCommand 중복 방지

```mermaid
sequenceDiagram
    participant T as Test
    participant O as Orchestrator
    participant F as FlightHandler
    participant H as HotelHandler
    participant PC as ProcessedCommand

    T->>O: startSaga() → COMPLETED 완료

    Note over PC: BOOK_FLIGHT, BOOK_HOTEL 기록 존재

    T->>PC: tryAcquire(sagaId, "BOOK_FLIGHT")
    PC-->>T: 0 (이미 처리됨)

    T->>PC: tryAcquire(sagaId, "BOOK_HOTEL")
    PC-->>T: 0 (이미 처리됨)

    Note over T: 중복 Command가 도착해도<br/>Handler가 스킵하는 것을 증명
```

**검증**: 정상 플로우 완료 후 동일 `(sagaId, commandType)`으로 `tryAcquire()` 재호출 → 0 반환.
