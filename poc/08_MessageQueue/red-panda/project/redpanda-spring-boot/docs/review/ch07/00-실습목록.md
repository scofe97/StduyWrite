## 각 실습의 구현 범위

### 1. 트랜잭션 Producer 설정
- application.yml: transaction-id-prefix, idempotence, acks 설정
- Java Configuration: ProducerFactory에 transactionIdPrefix 설정
- 트랜잭션 없이 전송 vs 트랜잭션으로 전송 — 동작 차이 확인

### 2. executeInTransaction vs @Transactional 비교
- executeInTransaction: KafkaTemplate.executeInTransaction() 콜백 방식
- @Transactional: Spring 선언적 트랜잭션 (KafkaTransactionManager 빈 필요)
- 같은 로직을 두 방식으로 구현하여 코드 비교
- 실패 시 롤백 동작 차이 확인

### 3. CTP 패턴 구현
- @KafkaListener + @SendTo 조합으로 CTP 구현
- EOSMode.V2 설정 (Spring Kafka 3.x)
- Consumer isolation.level=read_committed 설정
- 변환 중 예외 발생 시 트랜잭션 롤백 확인

### 4. DB+Kafka 원자성
- 접근법 1: ChainedKafkaTransactionManager (JPA + Kafka 체이닝)
- 접근법 2: Transactional Outbox (OutboxEvent 엔티티 + 릴레이)
- 접근법 3: Transactional Inbox (Consumer 쪽 멱등 처리)
- 세 접근법 장단점 비교 문서 작성

### 5. 수동 커밋 심화
- enable-auto-commit: false + ack-mode: MANUAL 이중 설정
- MANUAL(배치 단위 커밋) vs MANUAL_IMMEDIATE(즉시 커밋) 동작 차이
- Acknowledgment.nack(index, sleepMs): 특정 레코드부터 재처리
- 배치 리스너(BatchAcknowledgingMessageListener)에서 수동 커밋

### 6. 실행 제어 패턴
- 병렬 실행: ConcurrentKafkaListenerContainerFactory concurrency=N
- 순차 실행: concurrency=1 + 단일 파티션
- 키 기반 순차: 같은 키 → 같은 파티션 → 순서 보장, 다른 키 → 병렬
- 각 패턴별 ContainerFactory 빈 분리 설정
