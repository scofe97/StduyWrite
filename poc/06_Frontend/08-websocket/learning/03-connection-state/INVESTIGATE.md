# INVESTIGATE: 연결 상태 관리

## 학습 목표
WebSocket의 연결 상태(readyState)를 이해하고, 상태에 따른 UI 분기를 구현합니다.

---

## 소크라테스 질문

### Q1. readyState 값의 의미

> readyState 0~3은 각각 무엇을 의미하는가?

**생각해볼 점:**
- 각 상태에서 `send()` 메서드를 호출하면 어떻게 되는가?
- 상태 전이(state transition)는 어떤 순서로 일어나는가?
- `UNINSTANTIATED` 상태는 언제 발생하는가?

---

### Q2. 상태별 UI 분기

> 연결 상태에 따라 UI를 어떻게 분기해야 하는가?

**생각해볼 점:**
- `CONNECTING` 상태에서 사용자에게 어떤 피드백을 줘야 하는가?
- `OPEN` 상태에서만 활성화해야 하는 UI 요소는?
- `CLOSED` 상태에서 재연결 버튼을 보여줘야 하는가?

---

### Q3. 상태 변경 감지

> 상태가 변경될 때 어떻게 감지하고 반응해야 하는가?

**생각해볼 점:**
- `readyState`가 변경되면 React 컴포넌트가 자동으로 리렌더링되는가?
- 상태 변경 시 특정 액션을 수행하려면 어떻게 해야 하는가?
- `useEffect`를 사용해야 하는 경우와 콜백을 사용해야 하는 경우는?

---

### Q4. 상태 시각화

> 연결 상태를 사용자에게 어떻게 시각적으로 전달해야 하는가?

**생각해볼 점:**
- 색상으로 상태를 표현한다면 어떤 색상이 적절한가?
- 아이콘은 어떤 것이 적절한가?
- 실시간 데이터가 오래된 경우 사용자에게 알려야 하는가?
