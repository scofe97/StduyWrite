# 02. 내비게이션 - 마우스 없이 이동하기

Vim의 생산성 향상은 내비게이션에서 시작됩니다. hjkl 같은 기본 이동부터 단어 단위, 줄 단위, 화면 단위, 파일 단위 이동까지, 상황에 맞는 이동 방법을 선택하면 마우스를 잡는 시간이 사라집니다. 핵심은 "가장 적은 키스트로크로 목표 위치에 도달하는 것"입니다.

---

## 목표

- [ ] hjkl로 자연스럽게 이동할 수 있다
- [ ] 단어/줄/화면/파일 단위 이동을 구사할 수 있다
- [ ] f/t 문자 점프를 활용할 수 있다
- [ ] 상대 줄번호를 활용한 이동을 할 수 있다

---

## 1. 기본 이동 (hjkl)

Vim의 가장 기본적인 이동은 hjkl 키입니다. 화살표 키를 사용하지 않는 것이 Vim 철학의 핵심입니다.

### hjkl 키 매핑

```
        k
        ↑
h ← 커서 → l
        ↓
        j
```

| 키 | 방향 | 의미 |
|---|------|------|
| `h` | ← 왼쪽 | left |
| `j` | ↓ 아래 | down (j의 아래 꼬리를 연상) |
| `k` | ↑ 위 | up |
| `l` | → 오른쪽 | right |

### 왜 hjkl인가?

Vim의 전신인 vi는 1976년 ADM-3A 터미널에서 개발되었습니다. 이 터미널의 키보드는 hjkl 위치에 방향키가 표시되어 있었습니다. 현대 키보드에는 별도 화살표 키가 있지만, hjkl을 사용하면 다음과 같은 장점이 있습니다.

**hjkl의 장점:**
1. **홈 포지션 유지**: 손이 홈 로우를 벗어나지 않음
2. **속도**: 화살표 키까지 손을 이동하는 시간 절약
3. **조합 가능**: `5j` (5줄 아래), `3l` (3칸 오른쪽) 등 숫자와 조합
4. **일관성**: 다른 Unix 도구(less, man 등)도 동일한 키 사용

### 왜 화살표 키를 쓰지 않아야 하는가?

화살표 키를 사용할 수 있지만, 다음 이유로 권장하지 않습니다.

1. **손목 이동**: 홈 포지션 → 화살표 키 → 홈 포지션 반복은 비효율적
2. **습관 형성 실패**: 화살표 키에 의존하면 Vim의 진정한 효율성을 경험할 수 없음
3. **고급 기능 접근 불가**: hjkl에 익숙해야 w/b/e 같은 단어 단위 이동이 자연스러워짐

**실습 팁:** 처음에는 답답하더라도 화살표 키를 완전히 봉인하세요. 2-3일이면 hjkl이 자연스러워집니다.

### 숫자와 조합

모든 이동 명령은 숫자를 앞에 붙여 반복할 수 있습니다.

```vim
5j    " 5줄 아래로
3k    " 3줄 위로
10l   " 10칸 오른쪽으로
2h    " 2칸 왼쪽으로
```

상대 줄번호(`set relativenumber`)를 활성화하면, 목표 줄까지의 거리가 숫자로 표시되어 정확한 이동이 가능합니다.

## 2. 단어 단위 이동

한 글자씩 이동하는 것보다 단어 단위로 이동하는 것이 훨씬 빠릅니다. Vim은 word와 WORD를 구분합니다.

### word vs WORD

- **word**: 영문자, 숫자, 언더스코어로 구성된 단위. 기호는 별도의 word로 취급
- **WORD**: 공백으로 구분된 단위. 기호 포함

**예시:**
```
let value = foo-bar.baz;
```

- word 구분: `let`, `value`, `=`, `foo`, `-`, `bar`, `.`, `baz`, `;` (9개 word)
- WORD 구분: `let`, `value`, `=`, `foo-bar.baz;` (4개 WORD)

### word 이동 (소문자)

| 키 | 동작 | 설명 |
|---|------|------|
| `w` | word 시작으로 앞으로 이동 | next word start |
| `b` | word 시작으로 뒤로 이동 | back to word start |
| `e` | word 끝으로 앞으로 이동 | end of word |
| `ge` | word 끝으로 뒤로 이동 | back to end of word |

### WORD 이동 (대문자)

| 키 | 동작 | 설명 |
|---|------|------|
| `W` | WORD 시작으로 앞으로 이동 | 공백까지 점프 |
| `B` | WORD 시작으로 뒤로 이동 | 공백까지 뒤로 |
| `E` | WORD 끝으로 앞으로 이동 | 공백 전까지 |

### 사용 팁

- **코드 편집**: `w`, `b`, `e` (기호를 개별 단위로 처리)
- **자연어 편집**: `W`, `B`, `E` (공백 기준으로 빠르게 이동)

**실습:**
```javascript
const userName = "John-Doe";
```
커서가 `u`(userName의 u)에 있을 때:
- `w` → `N` (userName의 N, camelCase의 대문자를 word로 인식하지 않음, 실제로는 `=`로 이동)
- `2w` → `"` (다음 단어 시작)
- `e` → `e` (userName의 마지막 e)
- `b` → `u` (이전 word 시작)

## 3. 줄 단위 이동

줄의 특정 위치나 파일의 특정 줄로 빠르게 이동하는 명령입니다.

### 줄 내부 이동

| 키 | 동작 | 설명 |
|---|------|------|
| `0` | 줄의 맨 앞 (0번째 컬럼) | 공백 포함 |
| `^` | 줄의 첫 번째 문자 (공백 제외) | 들여쓰기 후 첫 문자 |
| `$` | 줄의 맨 뒤 | 마지막 문자 |
| `g_` | 줄의 마지막 문자 (공백 제외) | trailing space 무시 |

**예시:**
```python
    print("Hello")
```
(앞에 공백 4칸, 뒤에 공백 2칸 있다고 가정)

- `0` → 줄의 맨 앞 (첫 번째 공백)
- `^` → `p` (print의 p)
- `$` → 마지막 공백
- `g_` → `)` (괄호)

### 파일 내 줄 이동

| 키 | 동작 | 설명 |
|---|------|------|
| `gg` | 파일의 첫 줄로 이동 | 1G와 동일 |
| `G` | 파일의 마지막 줄로 이동 | |
| `{n}G` | n번째 줄로 이동 | 예: `50G` (50번째 줄) |
| `{n}gg` | n번째 줄로 이동 | 예: `10gg` (10번째 줄) |
| `:{n}` | n번째 줄로 이동 (Command-line 모드) | 예: `:42` |

### 현재 위치 확인

| 키 | 동작 |
|---|------|
| `Ctrl+g` | 현재 위치 정보 표시 (파일명, 줄 번호, 전체 비율) |
| `:=` | 전체 줄 수 표시 |
| `:.=` | 현재 줄 번호 표시 |

**예시 출력:**
```
"main.go" 150L, 3520C                    [10%]
```
(파일명 main.go, 전체 150줄, 3520자, 현재 위치 10%)

## 4. 화면 단위 이동

파일이 길 때 화면 단위로 이동하면 빠르게 탐색할 수 있습니다.

### 화면 위치 이동

| 키 | 동작 | 설명 |
|---|------|------|
| `H` | 화면 최상단으로 이동 | High |
| `M` | 화면 중간으로 이동 | Middle |
| `L` | 화면 최하단으로 이동 | Low |

이 명령은 **화면에 보이는 범위 내에서만** 이동합니다. 파일의 전체 내용과는 무관합니다.

### 스크롤

| 키 | 동작 | 설명 |
|---|------|------|
| `Ctrl+d` | 반 페이지 아래로 스크롤 | down |
| `Ctrl+u` | 반 페이지 위로 스크롤 | up |
| `Ctrl+f` | 한 페이지 아래로 스크롤 | forward |
| `Ctrl+b` | 한 페이지 위로 스크롤 | backward |

**팁:** `Ctrl+d`와 `Ctrl+u`가 가장 실용적입니다. 한 페이지 전체보다 반 페이지가 맥락을 유지하기 좋습니다.

### 커서 중심 스크롤

| 키 | 동작 | 설명 |
|---|------|------|
| `zz` | 현재 줄을 화면 중앙으로 | 커서는 고정, 화면만 스크롤 |
| `zt` | 현재 줄을 화면 최상단으로 | top |
| `zb` | 현재 줄을 화면 최하단으로 | bottom |

이 명령은 커서 위치는 그대로 두고 화면만 스크롤합니다. 맥락을 유지하면서 현재 줄을 편한 위치에 배치할 때 유용합니다.

## 5. 문자 점프 (f/t)

줄 내에서 특정 문자로 빠르게 이동하는 강력한 기능입니다. 단어 이동보다 정확하고 빠릅니다.

### 기본 문자 점프

| 키 | 동작 | 설명 |
|---|------|------|
| `f{char}` | 앞으로 {char} 찾아 이동 | find forward |
| `F{char}` | 뒤로 {char} 찾아 이동 | find backward |
| `t{char}` | 앞으로 {char} 직전까지 이동 | till forward |
| `T{char}` | 뒤로 {char} 직후까지 이동 | till backward |

### 반복 이동

| 키 | 동작 |
|---|------|
| `;` | 같은 방향으로 반복 |
| `,` | 반대 방향으로 반복 |

**예시:**
```javascript
const result = calculate(x, y, z);
```

커서가 `c`(const의 c)에 있을 때:
- `f(` → `(` (괄호로 이동)
- `t)` → `z` (닫는 괄호 직전까지)
- `F=` → `=` (뒤로 등호 찾기)
- `fc` → `c` (calculate의 c로 이동)
- `;` → 다음 `c` (없으면 이동 안 함)

### f vs t 선택 기준

- **f**: 해당 문자를 포함하여 삭제/변경할 때 (예: `df,` → 쉼표까지 삭제)
- **t**: 해당 문자는 남기고 직전까지만 작업할 때 (예: `dt,` → 쉼표 전까지만 삭제)

**실전 활용:**
```javascript
function updateUser(userId, userName, userEmail) {
```

`userId` 부분만 삭제하려면:
1. 커서를 `u` 위치로 이동
2. `dt,` 입력 → `userId` 삭제, `,` 는 남음

## 6. 상대 줄번호 활용

상대 줄번호(`relativenumber`)를 활성화하면 현재 줄 기준으로 거리가 표시되어 정확한 이동이 가능합니다.

### 설정 방법

`~/.config/nvim/init.lua`에 추가:
```lua
vim.opt.relativenumber = true
```

또는 Normal 모드에서:
```vim
:set relativenumber
```

### 사용 예시

```
  3  function calculate() {
  2    let x = 10;
  1    let y = 20;
15    return x + y;    ← 현재 줄 (절대 줄번호)
  1  }
  2
  3  function main() {
```

현재 줄(15번)에서 `let x = 10;` 줄로 이동하려면:
- 상대 줄번호 확인: `2` (2줄 위)
- `2k` 입력 → 정확히 해당 줄로 이동

화면에 보이는 범위 내에서는 상대 줄번호를 보고 `{n}j` 또는 `{n}k`로 단번에 이동하는 것이 가장 빠릅니다.

## 체크포인트

### 1. w와 W의 차이는?

<details>
<summary>모범 답안 확인</summary>

`w`는 word 단위로 이동하며, word는 영문자, 숫자, 언더스코어로 구성된 단위입니다. 기호는 별도의 word로 취급됩니다. 반면 `W`는 WORD 단위로 이동하며, WORD는 공백으로만 구분됩니다. 예를 들어 `foo-bar.baz`에서 `w`는 `foo`, `-`, `bar`, `.`, `baz` 각각을 별도 word로 인식하지만, `W`는 전체를 하나의 WORD로 취급합니다. 코드를 편집할 때는 `w`가 유용하고, 자연어를 빠르게 이동할 때는 `W`가 효율적입니다.

</details>

### 2. 화면에서 커서를 움직이지 않고 스크롤하는 방법은?

<details>
<summary>모범 답안 확인</summary>

커서는 고정하고 화면만 스크롤하는 명령은 `zz`, `zt`, `zb`입니다. `zz`는 현재 줄을 화면 중앙으로, `zt`는 최상단으로, `zb`는 최하단으로 배치합니다. 커서의 줄 번호는 변하지 않고 화면만 스크롤되므로, 현재 작업 중인 줄을 편한 위치에 배치하여 맥락을 유지하면서 코드를 읽을 때 유용합니다. 예를 들어 함수 정의를 확인한 후 `zz`로 중앙에 배치하면 위아래 맥락을 동시에 볼 수 있습니다.

</details>

### 3. f와 t의 차이를 설명하세요

<details>
<summary>모범 답안 확인</summary>

`f{char}`는 해당 문자를 찾아 그 문자 위에 커서를 위치시킵니다(find). 반면 `t{char}`는 해당 문자를 찾되 그 직전까지만 이동합니다(till). 예를 들어 `const x = 10;`에서 커서가 `c`에 있을 때, `f;`를 실행하면 세미콜론 위로 이동하고, `t;`를 실행하면 `0` 위로 이동합니다. 삭제나 변경 연산과 조합할 때 차이가 명확합니다. `df,`는 쉼표를 포함하여 삭제하지만, `dt,`는 쉼표는 남기고 직전까지만 삭제합니다. 상황에 맞춰 선택하면 추가 조작 없이 원하는 결과를 얻을 수 있습니다.

</details>
