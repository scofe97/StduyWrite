## 1. 삽입 모드 진입점 (6가지)

Normal 모드에서 Insert 모드로 진입하는 방법은 여러 가지가 있으며, 각각 커서의 위치가 다릅니다. 상황에 맞는 진입점을 선택하면 불필요한 이동을 줄일 수 있습니다.

### 기본 삽입 명령어

| 명령어 | 진입 위치 | 설명 |
|--------|-----------|------|
| `i` | 커서 앞 | insert before cursor |
| `I` | 줄의 첫 문자 앞 | insert at line start |
| `a` | 커서 뒤 | append after cursor |
| `A` | 줄 끝 | append at line end |
| `o` | 현재 줄 아래에 새 줄 | open line below |
| `O` | 현재 줄 위에 새 줄 | open line above |

### 시각적 비교

```
원본 줄: hello world
         ^커서 위치 (l에 있음)

i 실행:  hel|lo world    (l 앞에서 Insert 모드)
I 실행:  |hello world    (첫 문자 h 앞에서 Insert 모드)
a 실행:  hell|o world    (l 뒤에서 Insert 모드)
A 실행:  hello world|    (줄 끝에서 Insert 모드)
o 실행:  hello world
         |              (아래 새 줄에서 Insert 모드)
O 실행:  |
         hello world    (위 새 줄에서 Insert 모드)
```

### 상황별 선택 가이드

**줄 끝에 무언가를 추가할 때:**
- 비효율: `$` (줄 끝 이동) + `a` (커서 뒤 삽입)
- 효율: `A` (한 번에 줄 끝으로 가서 삽입)

예시: 줄 끝에 세미콜론 추가
```
A;<Esc>              # 줄 끝에 ; 추가하고 Normal 모드로
```

**새 줄을 추가할 때:**
- 비효율: `$` (줄 끝) + `a` (삽입) + `<Enter>` (개행)
- 효율: `o` (아래 새 줄 생성 및 삽입)

**줄 시작 부분에 무언가를 추가할 때:**
- 비효율: `0` (줄 시작) + `i` (삽입)
- 효율: `I` (첫 문자 앞에서 삽입)

예시: 주석 추가
```
I# <Esc>             # 줄 시작에 # 추가
I// <Esc>            # 줄 시작에 // 추가
```

**단어 중간을 수정할 때:**
- `i` 사용 (현재 커서 위치에서 삽입)

**단어 뒤에 추가할 때:**
- `a` 사용 (커서 뒤에서 삽입)

이 여섯 가지 명령어만으로 모든 삽입 시나리오를 효율적으로 처리할 수 있습니다.

## 2. 삭제 명령어

Vim에서 삭제는 단순히 내용을 지우는 것이 아니라 **잘라내기**에 가깝습니다. 삭제된 텍스트는 자동으로 레지스터에 저장되므로, 나중에 붙여넣기가 가능합니다.

### 기본 삭제 명령어

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `x` | 커서 위치 문자 삭제 | delete character under cursor |
| `X` | 커서 앞 문자 삭제 | delete character before cursor |
| `dd` | 현재 줄 삭제 | delete line |
| `D` | 커서부터 줄 끝까지 삭제 | delete to end of line (d$와 동일) |
| `J` | 다음 줄을 현재 줄에 합침 | join lines (개행 문자 삭제) |

### 연산자 조합 삭제

앞 챕터에서 배운 연산자 + 모션 조합도 삭제에 활용됩니다:

| 명령어 | 동작 |
|--------|------|
| `dw` | 단어 삭제 (커서부터 단어 끝) |
| `diw` | 단어 전체 삭제 (커서 위치 무관) |
| `d$` 또는 `D` | 커서부터 줄 끝까지 삭제 |
| `d0` | 줄 시작부터 커서 앞까지 삭제 |
| `dG` | 커서부터 파일 끝까지 삭제 |
| `dgg` | 파일 시작부터 커서까지 삭제 |
| `di"` | 따옴표 안 내용 삭제 |
| `dap` | 단락 전체 삭제 |

### J (줄 합치기) 활용

`J` 명령은 개행 문자를 삭제하여 다음 줄을 현재 줄에 합칩니다. 공백 하나를 자동으로 추가합니다.

```
원본:
hello
world

J 실행 후:
hello world
```

`gJ`를 사용하면 공백을 추가하지 않고 합칩니다.

### 삭제 = 잘라내기

중요한 점은 Vim의 삭제는 클립보드에 저장된다는 것입니다:

```
dd                   # 줄 삭제 (레지스터에 저장됨)
p                    # 삭제한 줄 붙여넣기
```

이는 "잘라내기 + 붙여넣기" 작업을 매우 빠르게 만듭니다.

## 3. 변경(change) 명령어

변경(change)은 삭제와 삽입을 결합한 명령입니다. 수학적으로 표현하면 `c = d + i`입니다. 즉, 텍스트를 삭제하고 즉시 Insert 모드로 진입합니다.

### 기본 변경 명령어

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `cc` | 현재 줄 변경 | 줄 내용을 삭제하고 Insert 모드 (들여쓰기 유지) |
| `C` | 커서부터 줄 끝까지 변경 | change to end of line (c$와 동일) |
| `s` | 커서 문자 변경 | substitute character (xi와 동일) |
| `S` | 줄 전체 변경 | substitute line (cc와 동일) |

### 연산자 조합 변경

| 명령어 | 동작 |
|--------|------|
| `cw` | 단어 변경 (커서부터 단어 끝) |
| `ciw` | 단어 전체 변경 |
| `ci"` | 따옴표 안 내용 변경 |
| `ci(` | 괄호 안 내용 변경 |
| `ci{` | 중괄호 안 내용 변경 |
| `cit` | HTML 태그 안 내용 변경 |
| `caw` | 단어와 주변 공백 변경 |

### 삭제 vs 변경: 언제 무엇을 사용할까?

**삭제(`d`)를 사용하는 경우:**
- 텍스트를 제거만 하고 싶을 때
- 삭제 후 다른 작업을 할 때
- 예: `dap` (단락 삭제), `dd` (줄 삭제)

**변경(`c`)을 사용하는 경우:**
- 텍스트를 제거하고 즉시 새 내용을 입력할 때
- 예: `ciw` (단어를 새 단어로 교체), `ci"` (따옴표 안 내용 변경)

실전 예시:
```javascript
const oldName = "value";
```

`oldName`을 `newName`으로 변경하려면:
```
ciw                  # oldName 삭제 + Insert 모드
newName<Esc>         # 새 이름 입력
```

`d` + `i`를 각각 실행하는 것보다 `c` 하나가 더 효율적입니다.

## 4. 치환(replace)

치환은 텍스트를 삭제하지 않고 덮어쓰는 명령입니다. Insert 모드로 진입하지 않고 Normal 모드에서 빠르게 문자를 바꿀 수 있습니다.

### 단일 문자 치환

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `r{char}` | 커서 문자를 {char}로 치환 | replace character |
| `R` | Replace 모드 진입 | 덮어쓰기 모드 (Insert의 덮어쓰기 버전) |

### r (단일 문자 치환)

`r` 명령은 커서 아래 문자 하나를 교체하고 Normal 모드를 유지합니다.

```
원본: hello
      ^커서 (e 위)

rx 실행:
결과: hxllo
      ^커서 유지, Normal 모드
```

실전 활용:
- 오타 수정: `ra` (문자를 a로 변경)
- 문장 부호 변경: `r.` (문자를 .으로 변경)
- 괄호 종류 변경: `r]` (괄호를 ]로 변경)

### R (Replace 모드)

`R` 명령은 Replace 모드로 진입합니다. Insert 모드와 유사하지만, 입력한 문자가 기존 문자를 덮어씁니다.

```
원본: hello world
      ^커서

R 실행 후 "hi" 입력:
결과: hillo world
      ^커서 (Replace 모드)

<Esc> 실행:
결과: hillo world
      ^Normal 모드로 복귀
```

Replace 모드는 고정 길이 텍스트를 편집할 때 유용합니다 (예: 표, 고정폭 데이터).

## 5. 되돌리기와 다시하기

Vim의 undo 시스템은 대부분의 에디터보다 강력합니다. 단순한 선형 undo가 아니라 트리 구조로 모든 변경 이력을 보존합니다.

### 기본 명령어

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `u` | 마지막 변경 취소 | undo |
| `Ctrl+R` | 취소한 변경 다시 실행 | redo |
| `U` | 현재 줄의 모든 변경 취소 | undo all changes on line |

### Undo 트리 개념

대부분의 에디터는 선형 undo를 사용합니다:

```
A → B → C → D
        ↑ undo하면 C로 돌아가고, D는 사라짐
```

Vim은 트리 구조를 사용합니다:

```
    A
    |
    B
   / \
  C   E
  |
  D
```

예를 들어:
1. 텍스트 A 입력
2. 텍스트 B 입력
3. `u`로 undo (B 취소, A 상태)
4. 텍스트 E 입력 (새 가지 생성)
5. 여전히 B→C→D 가지도 보존됨

Vim은 모든 변경 이력을 보존하므로, 잘못된 undo 후에도 이전 상태를 복구할 수 있습니다.

### Undo 브랜치 탐색

플러그인(예: undotree)을 사용하면 undo 트리를 시각적으로 탐색할 수 있습니다. 기본 Vim에서는:

```
:earlier 1m          # 1분 전 상태로
:later 1m            # 1분 후 상태로
:earlier 5           # 5번의 변경 전으로
```

이 명령들은 시간 기반 undo를 가능하게 합니다.

## 6. 기타 유용한 편집

### 대소문자 변환

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `~` | 커서 문자의 대소문자 토글 | toggle case |
| `guu` | 현재 줄을 소문자로 | lowercase line |
| `gUU` | 현재 줄을 대문자로 | uppercase line |
| `guw` | 단어를 소문자로 | lowercase word |
| `gUw` | 단어를 대문자로 | uppercase word |

연산자 조합도 가능합니다:
```
guiw                 # 단어 전체를 소문자로
gUi"                 # 따옴표 안을 대문자로
gu$                  # 커서부터 줄 끝까지 소문자로
```

### 숫자 증감

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `Ctrl+A` | 커서 위치 숫자 증가 | increment number |
| `Ctrl+X` | 커서 위치 숫자 감소 | decrement number |

예시:
```
원본: count = 5;
      커서를 5 위에 두고 Ctrl+A 실행

결과: count = 6;
```

숫자 수식어와 조합:
```
10<Ctrl+A>           # 10 증가
5<Ctrl+X>            # 5 감소
```

이 기능은 버전 번호, 카운터, 인덱스 등을 빠르게 수정할 때 유용합니다.

### 들여쓰기

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `>>` | 현재 줄 들여쓰기 | indent line |
| `<<` | 현재 줄 내어쓰기 | dedent line |
| `==` | 현재 줄 자동 들여쓰기 | auto-indent line |

비주얼 모드에서 여러 줄 선택 후:
```
>                    # 선택 영역 들여쓰기
<                    # 선택 영역 내어쓰기
=                    # 선택 영역 자동 들여쓰기
```

## 체크포인트

<details>
<summary>1. c와 d의 차이는?</summary>

`c` (change)와 `d` (delete)는 모두 텍스트를 삭제하지만, 삭제 후 동작이 다릅니다.

`d` (delete)는 텍스트를 삭제한 후 Normal 모드를 유지합니다. 단순히 텍스트를 제거하고 다른 작업을 계속하려 할 때 사용합니다. 예를 들어, `dd`로 줄을 삭제한 후 다른 줄로 이동하거나, `diw`로 단어를 삭제한 후 다른 편집 작업을 수행합니다.

`c` (change)는 텍스트를 삭제한 후 즉시 Insert 모드로 진입합니다. 이는 `d` + `i`를 결합한 것과 같습니다. 텍스트를 제거하고 즉시 새 내용을 입력하려 할 때 사용합니다. 예를 들어, `ciw`로 단어를 삭제하고 바로 새 단어를 타이핑하거나, `ci"`로 따옴표 안 내용을 삭제하고 새 문자열을 입력합니다.

실전에서는 "삭제 후 입력"이 목적이면 `c`를, "삭제만"이 목적이면 `d`를 사용합니다.
</details>

<details>
<summary>2. A와 a는 각각 언제 사용하나요?</summary>

`a` (append)는 현재 커서 위치의 바로 뒤에서 Insert 모드로 진입합니다. 커서가 이미 원하는 위치에 있고, 그 뒤에 텍스트를 추가하고 싶을 때 사용합니다. 예를 들어, 단어 중간에 커서가 있고 단어 끝에 텍스트를 추가하고 싶다면 `ea` (단어 끝으로 이동 + append)를 사용할 수 있습니다.

`A` (append at end of line)는 커서의 현재 위치와 상관없이 줄 끝으로 즉시 이동하여 Insert 모드로 진입합니다. 줄 끝에 무언가를 추가하고 싶을 때 사용합니다. 예를 들어, 줄 끝에 세미콜론을 추가하려면 `A;<Esc>`를 사용합니다. 이는 `$a`보다 키 입력이 적고 더 직관적입니다.

실전 팁: 코드 작성 시 줄 끝에 세미콜론, 쉼표, 중괄호 등을 추가하는 경우가 매우 많으므로 `A`는 가장 자주 사용하는 명령어 중 하나입니다.
</details>

<details>
<summary>3. Vim의 undo가 다른 에디터와 다른 점은?</summary>

대부분의 에디터는 선형(linear) undo를 사용합니다. 변경 A → B → C를 한 후 undo로 B로 돌아가고 새로운 변경 D를 하면, C는 완전히 사라지고 되돌릴 수 없습니다. Undo/Redo는 일직선 위를 앞뒤로 이동하는 것과 같습니다.

Vim은 트리(tree) 구조의 undo를 사용합니다. 모든 변경이 영구적으로 보존됩니다. 위 시나리오에서 Vim은 두 개의 가지를 생성합니다: A → B → C 가지와 A → B → D 가지. C는 사라지지 않고 여전히 undo 트리에 존재하며, 적절한 명령으로 접근할 수 있습니다.

이를 활용하는 명령어:
- `:earlier 10m` - 10분 전 상태로 (시간 기반 undo)
- `:later 10m` - 10분 후 상태로
- `:undolist` - undo 트리 보기
- 플러그인(undotree)을 사용하면 시각적으로 트리를 탐색 가능

이 시스템 덕분에 Vim에서는 "실수로 undo를 너무 많이 했다가 새 변경을 해서 이전 작업을 잃어버림"이 발생하지 않습니다. 모든 것이 보존됩니다.
</details>
