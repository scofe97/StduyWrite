# 05. 레지스터와 클립보드 - 26개 이상의 클립보드

대부분의 에디터에는 클립보드가 하나뿐입니다. 복사한 내용을 새로 복사하면 이전 내용이 사라집니다. Vim은 이 문제를 레지스터(register)라는 시스템으로 해결합니다. 알파벳 26개를 포함하여 수십 개의 독립된 저장 공간이 있어, 여러 텍스트를 동시에 기억하고 필요할 때 꺼내 쓸 수 있습니다.

---

## 목표

- [ ] 이름 있는 레지스터에 텍스트를 저장하고 꺼낼 수 있다
- [ ] 시스템 클립보드와 Vim 레지스터를 연동할 수 있다
- [ ] 특수 레지스터의 용도를 이해한다

---

## 1. 레지스터 기본 사용법

레지스터는 Vim 내부의 저장 공간으로, 복사(yank), 삭제(delete), 변경(change) 작업의 결과를 저장합니다. 각 레지스터는 고유한 이름(문자)을 가지며, 따옴표(`"`)를 사용하여 접근합니다.

### 기본 문법

```
"{register}{operator}
```

또는

```
"{register}p
```

여기서 `{register}`는 레지스터 이름(a-z, 0-9, +, * 등)이고, `{operator}`는 y(yank), d(delete), c(change), p(paste) 등입니다.

### 예시

```
"ayy                 # 현재 줄을 레지스터 a에 복사
"ap                  # 레지스터 a의 내용을 붙여넣기
"bdiw                # 단어를 레지스터 b에 삭제(잘라내기)
"bp                  # 레지스터 b의 내용을 붙여넣기
```

### 레지스터 내용 확인

레지스터에 무엇이 저장되어 있는지 확인하려면:

```
:reg                 # 모든 레지스터 내용 보기
:registers           # 위와 동일
:reg a b c           # 특정 레지스터(a, b, c)만 보기
```

출력 예시:
```
--- Registers ---
"a   first line
"b   second line
""   last deleted text
"0   last yanked text
```

## 2. 레지스터 종류

Vim은 다양한 목적을 위해 여러 종류의 레지스터를 제공합니다. 각 레지스터는 특정 상황에서 자동으로 사용되거나, 명시적으로 지정하여 사용할 수 있습니다.

### 무명 레지스터 ("")

무명 레지스터는 레지스터를 명시하지 않았을 때 기본으로 사용됩니다. `d`, `c`, `x`, `y` 등의 작업 결과가 자동으로 여기에 저장됩니다.

```
yy                   # 줄 복사 → "" 레지스터에 저장
p                    # "" 레지스터에서 붙여넣기 (""p와 동일)
dd                   # 줄 삭제 → "" 레지스터에 덮어씀
```

### 이름 있는 레지스터 ("a ~ "z)

알파벳 소문자 26개를 사용자가 명시적으로 지정할 수 있습니다. 이 레지스터들은 자동으로 덮어쓰이지 않으므로, 여러 텍스트를 동시에 보관할 수 있습니다.

```
"ayy                 # 현재 줄을 레지스터 a에 복사
"bdiw                # 단어를 레지스터 b에 잘라내기
"cyap                # 단락을 레지스터 c에 복사

# 나중에 어디서든
"ap                  # 레지스터 a 붙여넣기
"bp                  # 레지스터 b 붙여넣기
"cp                  # 레지스터 c 붙여넣기
```

### 대문자 레지스터 ("A ~ "Z) - Append 모드

대문자를 사용하면 기존 내용을 덮어쓰는 것이 아니라 **추가(append)**합니다.

```
"ayy                 # 첫 줄을 레지스터 a에 복사
j                    # 다음 줄로 이동
"Ayy                 # 두 번째 줄을 레지스터 a에 추가
"ap                  # 레지스터 a 붙여넣기 → 두 줄 모두 출력
```

이는 여러 위치의 텍스트를 하나의 레지스터에 모을 때 유용합니다.

### Yank 레지스터 ("0)

`"0` 레지스터는 마지막 `y`(yank) 명령의 결과만 저장합니다. `d`(delete)나 `c`(change) 결과는 저장하지 않습니다.

```
yiw                  # 단어 복사 → "0에 저장
dd                   # 줄 삭제 → ""에 저장 ("0는 변하지 않음)
p                    # ""에서 붙여넣기 (삭제한 줄)
"0p                  # "0에서 붙여넣기 (복사한 단어)
```

이는 "복사한 내용을 삭제 후에도 안전하게 보존"하는 핵심 메커니즘입니다.

### 숫자 레지스터 ("1 ~ "9) - 삭제 히스토리

`"1`부터 `"9`까지는 삭제 히스토리를 저장합니다. `"1`이 가장 최근 삭제이고, 새로운 삭제가 발생할 때마다 이전 내용이 `"2`, `"3`, ...로 밀려납니다.

```
dd                   # 첫 줄 삭제 → "1에 저장
dd                   # 두 번째 줄 삭제 → "1에 저장, 이전 내용은 "2로 이동
dd                   # 세 번째 줄 삭제 → "1에 저장, 이전 내용은 "2, "3으로 이동

"1p                  # 가장 최근 삭제 붙여넣기
"2p                  # 두 번째로 최근 삭제 붙여넣기
"3p                  # 세 번째로 최근 삭제 붙여넣기
```

단, 한 줄 미만의 삭제(예: `x`, `diw`)는 이 히스토리에 저장되지 않습니다.

### 시스템 클립보드 레지스터 ("+)

`"+` 레지스터는 운영체제의 시스템 클립보드와 연동됩니다. 이를 통해 Vim과 다른 애플리케이션 간에 텍스트를 주고받을 수 있습니다.

```
"+yy                 # 현재 줄을 시스템 클립보드에 복사
"+p                  # 시스템 클립보드 내용 붙여넣기
"+dd                 # 현재 줄을 시스템 클립보드에 잘라내기
```

브라우저에서 텍스트를 복사한 후 Vim에서 `"+p`로 붙여넣거나, Vim에서 `"+yy`로 복사한 후 다른 애플리케이션에 붙여넣을 수 있습니다.

### 선택 클립보드 레지스터 ("*)

`"*` 레지스터는 X11 시스템의 선택 버퍼(primary selection)와 연동됩니다. Linux/Unix에서 마우스로 텍스트를 선택하면 자동으로 이 버퍼에 복사되고, 중간 마우스 버튼으로 붙여넣기가 가능합니다.

macOS와 Windows에서는 `"+`와 `"*`가 동일하게 작동합니다.

```
"*yy                 # 선택 클립보드에 복사 (Linux)
"*p                  # 선택 클립보드에서 붙여넣기
```

### 블랙홀 레지스터 ("_)

`"_` 레지스터는 "블랙홀"입니다. 여기에 저장된 내용은 즉시 사라지며, 다른 레지스터를 오염시키지 않습니다.

```
"_dd                 # 줄 삭제하되 레지스터에 저장하지 않음
"_diw                # 단어 삭제하되 레지스터에 저장하지 않음
```

이는 "삭제는 하되, 무명 레지스터를 보존하고 싶을 때" 유용합니다. 예를 들어, 복사한 텍스트를 여러 곳에 붙여넣으면서 중간에 불필요한 줄을 삭제해야 할 때:

```
yy                   # 줄 복사 → ""에 저장
/pattern             # 대상 위치 찾기
p                    # 붙여넣기
/other               # 다른 위치 찾기
"_dd                 # 불필요한 줄 삭제 (""를 오염시키지 않음)
p                    # 원래 복사한 내용 다시 붙여넣기
```

### 검색 레지스터 ("/")

`"/` 레지스터는 마지막 검색 패턴을 저장합니다. 이는 수동으로 설정하기보다는 읽기 용도로 사용됩니다.

```
/hello               # "hello" 검색 → "/에 저장
:s/<Ctrl+R>//world/g # "/의 내용(hello)을 world로 치환
```

### Dot 레지스터 (".)

`".` 레지스터는 마지막으로 삽입한 텍스트를 저장합니다. Insert 모드에서 입력한 내용이 여기에 저장됩니다.

```
i                    # Insert 모드 진입
hello world          # 텍스트 입력
<Esc>                # Normal 모드 복귀 → "hello world"가 ".에 저장

".p                  # 마지막 삽입 텍스트 붙여넣기
```

### 파일명 레지스터 ("%)

`"%` 레지스터는 현재 파일의 경로를 저장합니다.

```
"%p                  # 현재 파일 경로 붙여넣기
:echo @%             # 현재 파일 경로 출력
```

### 명령 레지스터 (":")

`":` 레지스터는 마지막으로 실행한 명령 라인 명령을 저장합니다.

```
:echo "hello"        # 명령 실행 → ":에 저장
@:                   # 마지막 명령 다시 실행
```

## 3. 실전 패턴

레지스터를 실전에서 어떻게 활용하는지 구체적인 패턴을 살펴봅니다.

### 패턴 1: 텍스트 교환 (Swap)

두 텍스트를 서로 바꾸는 전통적인 방법:

```
# A와 B를 교환하고 싶을 때
yiw                  # A를 복사 (""와 "0에 저장)
viwp                 # B를 선택하여 A로 붙여넣기 (B는 ""로 이동)
# 이제 ""에는 B가 저장됨
# 원래 A 위치로 이동
viw                  # A 선택 (현재 A가 있는 위치)
"0p                  # "0에 있는 원래 A를 붙여넣기
```

이 패턴의 핵심은 `"0` 레지스터가 yank한 내용을 보존한다는 점입니다.

### 패턴 2: 복사 후 안전한 붙여넣기

복사한 내용을 여러 곳에 붙여넣으면서 중간에 삭제 작업을 할 때:

**문제 상황:**
```
yy                   # 줄 복사
p                    # 붙여넣기
dd                   # 불필요한 줄 삭제 → "" 레지스터 오염
p                    # 삭제한 줄이 붙여넣어짐 (원하는 동작 아님)
```

**해결책 1: "0 사용**
```
yy                   # 줄 복사 → "0에 저장
p                    # 붙여넣기
dd                   # 줄 삭제 → ""에 저장 ("0는 변하지 않음)
"0p                  # 원래 복사한 줄 붙여넣기
```

**해결책 2: 블랙홀 레지스터 사용**
```
yy                   # 줄 복사
p                    # 붙여넣기
"_dd                 # 줄 삭제 (""를 오염시키지 않음)
p                    # 원래 복사한 줄 다시 붙여넣기
```

### 패턴 3: 여러 텍스트 수집

여러 위치의 텍스트를 하나의 레지스터에 모으기:

```
"ayy                 # 첫 번째 줄을 레지스터 a에 복사
/next                # 다음 위치 찾기
"Ayy                 # 두 번째 줄을 레지스터 a에 추가
/another             # 또 다른 위치 찾기
"Ayy                 # 세 번째 줄을 레지스터 a에 추가

# 모든 내용이 레지스터 a에 모임
"ap                  # 모든 줄을 한 번에 붙여넣기
```

### 패턴 4: 시스템 클립보드 연동

외부 애플리케이션과 텍스트 주고받기:

```
# 브라우저에서 코드 복사 → Ctrl+C
# Vim에서
"+p                  # 브라우저 코드 붙여넣기

# Vim에서 코드 복사하여 외부로
"+yy                 # 시스템 클립보드에 복사
# 다른 애플리케이션에서 Ctrl+V로 붙여넣기
```

### 패턴 5: 매크로와 레지스터 결합

매크로는 실제로 레지스터에 저장됩니다:

```
qa                   # 레지스터 a에 매크로 기록 시작
...작업...
q                    # 기록 종료

:reg a               # 매크로 내용 확인 (레지스터 a)
"ap                  # 매크로를 텍스트로 붙여넣기 (편집 가능)
# 텍스트 편집 후
"ayy                 # 수정된 매크로를 레지스터 a에 복사
@a                   # 수정된 매크로 실행
```

## 4. 클립보드 설정

Neovim이나 현대적인 Vim 설정에서는 시스템 클립보드를 기본으로 사용하도록 설정할 수 있습니다.

### clipboard 옵션 설정

```lua
-- Neovim (init.lua)
vim.opt.clipboard = "unnamedplus"
```

```vim
" Vim (vimrc)
set clipboard=unnamedplus
```

이 설정을 활성화하면:
- 모든 `y` (yank) 작업이 자동으로 시스템 클립보드(`"+`)에 복사됩니다.
- 모든 `d` (delete) 작업도 시스템 클립보드에 저장됩니다.
- `p` (paste)가 시스템 클립보드에서 가져옵니다.

### 장점

- Vim과 외부 애플리케이션 간 복사/붙여넣기가 매우 편리해집니다.
- `"+` 접두사를 매번 타이핑할 필요가 없습니다.
- 외부에서 복사한 내용을 즉시 `p`로 붙여넣을 수 있습니다.

### 단점

- Vim 내부의 레지스터 독립성을 일부 잃습니다.
- `d`로 삭제한 내용이 시스템 클립보드를 오염시킬 수 있습니다.
- 네트워크를 통한 원격 Vim 사용 시 클립보드 동기화가 느려질 수 있습니다.

### 권장 사항

초보자는 `clipboard=unnamedplus`를 설정하여 편의성을 높이고, 레지스터 시스템에 익숙해진 후 필요에 따라 비활성화하여 더 세밀한 제어를 할 수 있습니다.

3. 또 다음 단어로 이동 후 추가: `"Xyiw`
4. `:reg x`로 확인 (여러 단어가 모여 있음)
5. `"xp`로 붙여넣기

### 연습 3: Yank 레지스터 활용

3. `p` 실행 → 삭제한 줄이 붙여넣어짐
4. `"0p` 실행 → 복사한 단어가 붙여넣어짐

### 연습 4: 블랙홀 레지스터

1. 줄 복사: `yy`
2. 불필요한 줄을 블랙홀로 삭제: `"_dd`
3. `p`로 원래 복사한 줄 붙여넣기 (여전히 작동함)

### 연습 5: 시스템 클립보드

1. 외부 애플리케이션에서 텍스트 복사 (Ctrl+C 또는 Cmd+C)
2. Vim에서 `"+p`로 붙여넣기
3. Vim에서 줄 복사: `"+yy`
4. 외부 애플리케이션에서 붙여넣기 (Ctrl+V 또는 Cmd+V)

## 명령어 요약

| 명령어 | 설명 |
|--------|------|
| `"{register}{operator}` | 레지스터 지정 연산 |
| `:reg` | 모든 레지스터 내용 보기 |
| `:reg a b c` | 특정 레지스터만 보기 |
| `""` | 무명 레지스터 (기본) |
| `"a ~ "z` | 이름 있는 레지스터 (사용자 지정) |
| `"A ~ "Z` | Append 모드 (기존 내용에 추가) |
| `"0` | Yank 레지스터 (마지막 yank만) |
| `"1 ~ "9` | 삭제 히스토리 (1이 최근) |
| `"+` | 시스템 클립보드 |
| `"*` | 선택 클립보드 (X11) |
| `"_` | 블랙홀 레지스터 (버리기) |
| `"/` | 검색 레지스터 |
| `".` | 마지막 삽입 텍스트 |
| `"%` | 현재 파일명 |
| `":` | 마지막 명령 |
| `"ayy` | 줄을 레지스터 a에 복사 |
| `"ap` | 레지스터 a에서 붙여넣기 |
| `"Ayy` | 줄을 레지스터 a에 추가 |
| `"+yy` | 시스템 클립보드에 복사 |
| `"+p` | 시스템 클립보드에서 붙여넣기 |
| `"_dd` | 줄 삭제 (레지스터 오염 방지) |
| `"0p` | 마지막 yank 붙여넣기 |

## 체크포인트

<details>
<summary>1. yank한 텍스트를 삭제 후에도 안전하게 붙여넣는 두 가지 방법은?</summary>

**방법 1: "0 레지스터 사용**

`"0` 레지스터는 마지막 yank 작업의 결과만 저장하고, delete나 change 작업의 영향을 받지 않습니다. 따라서 yank 후 삭제 작업을 하더라도 `"0p`로 원래 복사한 내용을 붙여넣을 수 있습니다.

```
yiw                  # 단어 복사 → ""와 "0에 저장
dd                   # 줄 삭제 → ""에 저장 ("0는 변하지 않음)
"0p                  # "0에서 원래 복사한 단어 붙여넣기
```

**방법 2: 블랙홀 레지스터 사용**

삭제 작업을 블랙홀 레지스터(`"_`)로 수행하면 무명 레지스터(`""`)를 오염시키지 않습니다. 따라서 일반 `p` 명령으로도 원래 복사한 내용을 붙여넣을 수 있습니다.

```
yiw                  # 단어 복사 → ""에 저장
"_dd                 # 줄 삭제 (블랙홀로 → ""는 변하지 않음)
p                    # ""에서 원래 복사한 단어 붙여넣기
```

두 방법 모두 유효하며, 상황에 따라 편한 것을 선택하면 됩니다. `"0p`는 타이핑이 더 길지만 명시적이고, `"_dd`는 삭제 시점에 의도를 명확히 합니다.
</details>

<details>
<summary>2. "_ 레지스터의 용도는?</summary>

`"_` (블랙홀 레지스터)는 "데이터를 버리는" 특수 레지스터입니다. 이 레지스터에 저장된 내용은 즉시 사라지며, 어떤 레지스터도 오염시키지 않습니다.

**주요 용도:**

1. **무명 레지스터 보존**: 복사한 내용을 여러 곳에 붙여넣으면서 중간에 삭제 작업을 할 때, 삭제된 내용이 무명 레지스터(`""`)를 덮어쓰지 않도록 합니다.

2. **레지스터 클린**: 특정 텍스트를 삭제하되, 나중에 붙여넣을 필요가 전혀 없을 때 사용합니다. 예를 들어, 불필요한 디버그 로그를 삭제할 때 레지스터를 채울 필요가 없습니다.

3. **성능**: 대량의 텍스트를 삭제할 때 레지스터에 저장하지 않으므로 미세하게 빠릅니다.

**실전 예시:**
```
yy                   # 중요한 줄 복사
/pattern             # 대상 위치 찾기
p                    # 붙여넣기
/other               # 다른 위치 찾기
"_3dd                # 불필요한 3줄 삭제 (블랙홀로)
p                    # 원래 복사한 줄 다시 붙여넣기
```

블랙홀 레지스터가 없었다면 `3dd`가 무명 레지스터를 덮어써서 마지막 `p`가 삭제된 3줄을 붙여넣게 됩니다.
</details>

<details>
<summary>3. "+와 일반 레지스터의 차이는?</summary>

**일반 레지스터 (`""`, `"a-z` 등):**
- Vim 프로세스 내부에만 존재합니다.
- Vim을 종료하면 사라집니다 (viminfo 설정에 따라 일부 보존 가능).
- 다른 애플리케이션과 공유되지 않습니다.
- 여러 개의 독립된 저장 공간으로, 다양한 텍스트를 동시에 보관할 수 있습니다.

**시스템 클립보드 레지스터 (`"+`):**
- 운영체제의 시스템 클립보드와 연동됩니다.
- Vim과 다른 애플리케이션 간에 텍스트를 주고받을 수 있습니다.
- 브라우저, 편집기, 메신저 등 모든 애플리케이션에서 Ctrl+C/Ctrl+V로 접근하는 것과 동일한 클립보드입니다.
- 하나의 공유 공간으로, Vim 외부에서 복사하면 내용이 덮어쓰여집니다.

**사용 예시 비교:**

일반 레지스터:
```
"ayy                 # Vim 내부에만 저장
"ap                  # Vim에서만 접근 가능
```

시스템 클립보드:
```
"+yy                 # OS 클립보드에 저장
# 이제 Chrome, VSCode, Slack 등 어디서든 Ctrl+V로 붙여넣기 가능

# 반대로, 브라우저에서 Ctrl+C로 복사 후
"+p                  # Vim에서 붙여넣기
```

**언제 무엇을 사용할까:**
- Vim 내부에서만 작업: 일반 레지스터 (빠르고 독립적)
- 외부 애플리케이션과 연동: `"+` (시스템 클립보드)
- 설정으로 항상 시스템 클립보드 사용 가능: `set clipboard=unnamedplus`
</details>
