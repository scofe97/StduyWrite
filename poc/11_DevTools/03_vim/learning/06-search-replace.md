# 06. 검색과 치환 - 코드에서 원하는 것을 찾고 바꾸기

코드 베이스에서 특정 변수명을 찾거나, 패턴에 맞는 문자열을 일괄 변경하는 것은 일상적인 작업입니다. Vim의 검색은 정규식을 기본 지원하며, 치환 명령은 sed와 유사한 문법으로 강력한 텍스트 변환을 수행합니다. IDE의 Find & Replace와 같은 기능이지만, 키보드에서 손을 떼지 않고 수행할 수 있습니다.

---

## 목표

- [ ] / 검색과 n/N 탐색을 자유롭게 사용할 수 있다
- [ ] * 과 # 으로 단어를 빠르게 검색할 수 있다
- [ ] :%s 치환 명령의 문법과 플래그를 이해한다

---

## 1. 기본 검색

Vim의 검색 기능은 파일 내에서 패턴을 찾아 커서를 이동시킵니다. 정규식을 지원하므로 복잡한 패턴도 검색할 수 있습니다.

### 검색 명령어

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `/pattern` | 앞으로 검색 | search forward |
| `?pattern` | 뒤로 검색 | search backward |
| `n` | 다음 검색 결과 | next match |
| `N` | 이전 검색 결과 | previous match |
| `*` | 커서 아래 단어를 앞으로 검색 | search word forward |
| `#` | 커서 아래 단어를 뒤로 검색 | search word backward |
| `g*` | 커서 아래 단어를 부분 일치로 앞으로 검색 | |
| `g#` | 커서 아래 단어를 부분 일치로 뒤로 검색 | |

### 기본 사용법

```
/hello               # "hello" 검색 (앞으로)
<Enter>              # 검색 실행 → 첫 번째 일치로 이동
n                    # 다음 일치로 이동
N                    # 이전 일치로 이동
?world               # "world" 검색 (뒤로)
```

검색 후 `n`과 `N`을 반복하여 모든 일치 항목을 순회할 수 있습니다. 검색은 파일 끝에 도달하면 자동으로 파일 시작으로 순환합니다.

### 검색 하이라이트

검색 결과를 시각적으로 강조 표시하려면:

```vim
:set hlsearch        " highlight search - 검색 결과 하이라이트 활성화
:set nohlsearch      " 하이라이트 비활성화
:noh                 " :nohlsearch의 축약 - 일시적으로 하이라이트 제거
```

일반적으로 `hlsearch`는 항상 켜두고, 하이라이트가 방해될 때만 `:noh`를 실행합니다. 다음 검색을 하면 하이라이트가 다시 활성화됩니다.

**권장 설정:**
```vim
:set hlsearch        " 검색 하이라이트 활성화
:set incsearch       " incremental search - 타이핑하는 동안 실시간 검색
```

`incsearch`를 활성화하면 `/hel`까지만 타이핑해도 일치하는 첫 번째 항목으로 즉시 이동합니다.

### 정규식 패턴

Vim은 강력한 정규식 엔진을 제공합니다. 기본 패턴은 Vim의 "magic" 모드를 사용합니다.

| 패턴 | 의미 | 예시 |
|------|------|------|
| `.` | 임의의 문자 하나 | `/h.t` → "hat", "hot", "hit" |
| `*` | 앞 패턴 0회 이상 | `/ab*` → "a", "ab", "abb" |
| `\+` | 앞 패턴 1회 이상 | `/ab\+` → "ab", "abb" (a는 안 됨) |
| `\?` | 앞 패턴 0회 또는 1회 | `/colou\?r` → "color", "colour" |
| `^` | 줄의 시작 | `/^hello` → 줄 시작의 "hello" |
| `$` | 줄의 끝 | `/world$` → 줄 끝의 "world" |
| `\<` | 단어의 시작 | `/\<the` → "the", "them" (other는 안 됨) |
| `\>` | 단어의 끝 | `/ing\>` → "running" (ings는 안 됨) |
| `\(\)` | 그룹 | `/\(ab\)\+` → "ab", "abab" |
| `\|` | OR 연산 | `/cat\|dog` → "cat" 또는 "dog" |
| `[]` | 문자 클래스 | `/[aeiou]` → 모음 하나 |
| `[^]` | 부정 문자 클래스 | `/[^0-9]` → 숫자가 아닌 문자 |

**예시:**
```
/log.*Error          # "log"로 시작하고 "Error"를 포함하는 줄
/function\s\+\w\+    # "function" 뒤에 공백과 단어가 오는 패턴
/\d\{3\}-\d\{4\}     # 전화번호 패턴 (예: 123-4567)
```

### 대소문자 구분

| 설정 | 동작 |
|------|------|
| `:set ignorecase` | 대소문자 무시 |
| `:set noignorecase` | 대소문자 구분 (기본값) |
| `:set smartcase` | 검색 패턴에 대문자가 있으면 구분, 없으면 무시 |

**패턴 내 플래그:**
- `\c` - 대소문자 무시 (case insensitive): `/hello\c` → "hello", "Hello", "HELLO"
- `\C` - 대소문자 구분 (case sensitive): `/hello\C` → "hello"만

**권장 설정:**
```vim
:set ignorecase      " 기본적으로 대소문자 무시
:set smartcase       " 대문자가 포함되면 구분
```

이 조합은 다음과 같이 작동합니다:
- `/hello` → "hello", "Hello", "HELLO" 모두 검색 (소문자만 입력 → ignorecase)
- `/Hello` → "Hello"만 검색 (대문자 포함 → smartcase가 구분 모드로 전환)

## 2. 단어 검색 단축키

커서 아래 단어를 빠르게 검색하는 명령어는 변수명이나 함수명을 추적할 때 매우 유용합니다.

### * 과 # 명령어

| 명령어 | 동작 | 설명 |
|--------|------|------|
| `*` | 커서 아래 단어를 완전 일치로 앞으로 검색 | search word forward (whole word) |
| `#` | 커서 아래 단어를 완전 일치로 뒤로 검색 | search word backward (whole word) |
| `g*` | 커서 아래 단어를 부분 일치로 앞으로 검색 | search word forward (partial) |
| `g#` | 커서 아래 단어를 부분 일치로 뒤로 검색 | search word backward (partial) |

### 완전 일치 vs 부분 일치

**`*` (완전 일치):**
```javascript
const user = getUser();  // 커서를 "user"에 두고 * 실행
const userName = "";     // 검색되지 않음 (user는 userName의 일부)
const user_id = 123;     // 검색되지 않음
const user = {};         // 검색됨 ✓
```

`*`는 `/\<user\>`와 동일하게 작동합니다 (단어 경계 검색).

**`g*` (부분 일치):**
```javascript
const user = getUser();  // 커서를 "user"에 두고 g* 실행
const userName = "";     // 검색됨 ✓
const user_id = 123;     // 검색됨 ✓ (user 포함)
const user = {};         // 검색됨 ✓
```

`g*`는 `/user`와 동일하게 작동합니다.

### 실전 활용 패턴: 검색 + 변경 + 반복

이 패턴은 코드 리팩토링의 핵심입니다:

1. 변수명이나 함수명에 커서를 위치
2. `*` - 해당 단어를 검색
3. `ciw` - 단어를 변경
4. `newName<Esc>` - 새 이름 입력
5. `n` - 다음 검색 결과로 이동
6. `.` - 변경 반복 (또는 `n`으로 건너뛰기)
7. `n.n.n.` - 필요한 만큼 반복

이 패턴의 강점은 **선택적 치환**입니다. IDE의 "Replace All"과 달리, 각 일치 항목을 검토하고 변경할지 건너뛸지 결정할 수 있습니다.

**예시:**
```javascript
// "count"를 "totalCount"로 변경하되, 일부는 유지
const count = 0;        // 변경: totalCount
function increment(count) {  // 건너뛰기 (매개변수는 유지)
  return count + 1;     // 건너뛰기
}
const count = getCount();  // 변경: totalCount
```

순서:
1. 첫 번째 `count`에 커서
2. `*` (count 검색)
3. `ciw` → `totalCount<Esc>` (첫 번째 변경)
4. `n` (다음, function 매개변수)
5. `n` (건너뛰기, return의 count)
6. `n` (건너뛰기, 다음 const count로)
7. `.` (변경 반복)

## 3. 치환 명령 :%s

치환(substitute) 명령은 검색 패턴에 일치하는 텍스트를 새 텍스트로 변경합니다. sed와 유사한 문법을 사용하며, 정규식과 범위 지정을 지원합니다.

### 기본 문법

```
:[range]s/pattern/replacement/[flags]
```

**구성 요소:**
- `[range]` - 치환을 적용할 범위 (생략 시 현재 줄)
- `s` - substitute 명령
- `/pattern/` - 검색할 패턴 (정규식)
- `/replacement/` - 치환할 텍스트
- `[flags]` - 옵션 플래그

### 범위 지정

| 범위 | 의미 | 예시 |
|------|------|------|
| (생략) | 현재 줄만 | `:s/old/new/` |
| `%` | 전체 파일 | `:%s/old/new/g` |
| `1,10` | 1~10줄 | `:1,10s/old/new/g` |
| `.` | 현재 줄 | `:.s/old/new/` |
| `$` | 마지막 줄 | `:$s/old/new/` |
| `.,+5` | 현재 줄부터 5줄 | `:.,+5s/old/new/g` |
| `'<,'>` | 비주얼 선택 영역 | Visual 모드에서 `:` 입력 시 자동 삽입 |
| `1,$` | 전체 파일 | `:%`와 동일 |

### 플래그

| 플래그 | 의미 | 설명 |
|--------|------|------|
| `g` | global | 줄 내 모든 일치 항목 치환 (기본은 첫 번째만) |
| `c` | confirm | 각 치환마다 확인 요청 |
| `i` | ignore case | 대소문자 무시 |
| `I` | case sensitive | 대소문자 구분 |
| `n` | count only | 치환하지 않고 일치 개수만 표시 |

### 기본 예시

```vim
:s/old/new/          " 현재 줄의 첫 번째 "old"를 "new"로
:s/old/new/g         " 현재 줄의 모든 "old"를 "new"로
:%s/old/new/g        " 파일 전체의 모든 "old"를 "new"로
:%s/old/new/gc       " 파일 전체, 각 치환마다 확인
:5,20s/error/warning/g  " 5~20줄의 모든 "error"를 "warning"으로
```

### confirm 플래그 사용

`c` 플래그를 사용하면 각 일치 항목마다 확인 프롬프트가 표시됩니다:

```vim
:%s/old/new/gc
```

프롬프트 옵션:
- `y` - yes, 이 항목 치환
- `n` - no, 이 항목 건너뛰기
- `a` - all, 나머지 모두 치환 (확인 중단)
- `q` - quit, 치환 중단
- `l` - last, 이 항목을 치환하고 중단
- `^E` - 화면을 위로 스크롤
- `^Y` - 화면을 아래로 스크롤

이는 IDE의 "Replace with Confirmation"과 유사하지만, 키보드만으로 조작할 수 있습니다.

## 4. 실전 리팩토링 패턴

치환 명령의 강력함은 정규식과 결합할 때 드러납니다. 실전에서 자주 사용하는 패턴을 살펴봅니다.

### 패턴 1: 변수명 일괄 변경 (완전 일치)

단어 경계를 사용하여 부분 일치를 방지합니다:

```vim
:%s/\<oldName\>/newName/g
```

예시:
```javascript
// 변경 전
const oldName = 1;
const oldNameCount = 2;  // 변경되지 않음 (완전 일치 아님)
const name = oldName;

// 변경 후 (:%s/\<oldName\>/newName/g)
const newName = 1;
const oldNameCount = 2;  // 그대로 유지
const name = newName;
```

### 패턴 2: 함수 시그니처 변경 (정규식 그룹)

정규식 그룹(`\(\)`)과 참조(`\1`, `\2`)를 사용합니다:

```vim
:%s/function \(\w\+\)(/const \1 = (/g
```

예시:
```javascript
// 변경 전
function greet(name) {
function sayHello(msg) {

// 변경 후
const greet = (name) {
const sayHello = (msg) {
```

여기서 `\(\w\+\)`는 함수명을 캡처하고, `\1`로 참조합니다.

### 패턴 3: 줄 끝 공백 제거

줄 끝의 불필요한 공백을 제거하는 일반적인 작업:

```vim
:%s/\s\+$//g
```

- `\s\+` - 공백 문자 1개 이상
- `$` - 줄 끝
- `//` - 빈 문자열로 치환 (즉, 삭제)

### 패턴 4: console.log 일괄 삭제

디버그 로그를 제거할 때:

```vim
:g/console\.log/d
```

또는 치환으로:

```vim
:%s/.*console\.log.*\n//g
```

주의: `.`는 정규식에서 특수 문자이므로 `\.`로 이스케이프합니다.

### 패턴 5: 주석 추가/제거

줄 시작에 주석 추가:

```vim
:5,10s/^/# /         " 5~10줄의 시작에 "# " 추가
```

주석 제거:

```vim
:5,10s/^# //         " 5~10줄의 시작에서 "# " 제거
```

### 패턴 6: 대소문자 변환

치환의 특수 플래그를 사용:

```vim
:%s/\<\w/\u&/g       " 모든 단어를 대문자로 시작 (Title Case)
:%s/.*/\U&/g         " 모든 줄을 대문자로
:%s/.*/\L&/g         " 모든 줄을 소문자로
```

- `\u` - 다음 문자를 대문자로
- `\U` - 이후 모든 문자를 대문자로
- `\l` - 다음 문자를 소문자로
- `\L` - 이후 모든 문자를 소문자로
- `&` - 일치한 전체 텍스트

### 패턴 7: 빈 줄 제거

```vim
:g/^$/d              " 완전히 빈 줄 삭제
:g/^\s*$/d           " 공백만 있는 줄도 삭제
```

`:g` 명령은 다음 섹션에서 설명합니다.

## 5. :g(global) 명령

`:g` (global) 명령은 패턴에 매칭되는 모든 줄에 특정 명령을 실행합니다. 치환과는 다른 방식으로 강력한 텍스트 조작이 가능합니다.

### 기본 문법

```
:[range]g/pattern/command
```

- `[range]` - 범위 (생략 시 전체 파일)
- `g` - global 명령
- `/pattern/` - 검색 패턴
- `command` - 실행할 Ex 명령

### 기본 예시

```vim
:g/TODO/d            " TODO가 포함된 모든 줄 삭제
:g/^$/d              " 빈 줄 모두 삭제
:g/console\.log/d    " console.log가 있는 줄 모두 삭제
:g/error/p           " error가 포함된 줄 모두 출력
:g/function/normal A; " function이 있는 줄 끝에 세미콜론 추가
```

### :v (역 global)

`:v`는 `:g!`와 동일하며, 패턴에 매칭되지 **않는** 줄에 명령을 실행합니다.

```vim
:v/TODO/d            " TODO가 없는 줄 모두 삭제 (TODO만 남김)
:v/^#/d              " #으로 시작하지 않는 줄 삭제 (주석만 남김)
```

### :g vs :%s 비교

| 작업 | :g 명령 | :%s 명령 |
|------|---------|----------|
| 줄 삭제 | `:g/pattern/d` | (불가능) |
| 줄 복사 | `:g/pattern/t$` | (불가능) |
| 줄 이동 | `:g/pattern/m$` | (불가능) |
| 텍스트 치환 | (불편함) | `:%s/old/new/g` |
| Normal 명령 실행 | `:g/pattern/normal A;` | (불가능) |

**언제 무엇을 사용할까:**
- **줄 단위 작업** (삭제, 이동, 복사): `:g`
- **텍스트 치환** (문자열 변경): `:%s`
- **복잡한 편집**: `:g` + `normal` 조합

### 실전 예시

**패턴 1: 특정 줄만 추출**
```vim
:g/ERROR/t$          " ERROR가 포함된 줄을 파일 끝에 복사
```

**패턴 2: 조건부 들여쓰기**
```vim
:g/^function/normal >>  " function으로 시작하는 줄 들여쓰기
```

**패턴 3: 로그 필터링**
```vim
:v/\[INFO\]\|\[ERROR\]/d  " INFO나 ERROR가 아닌 로그 줄 삭제
```

**패턴 4: 중복 줄 제거 (정렬 후)**
```vim
:sort                " 먼저 정렬
:g/^\(.*\)$\n\1$/d   " 중복된 연속 줄 제거
```

<details>
<summary>1. :%s/foo/bar/gc에서 각 부분의 의미는?</summary>

이 명령은 여러 구성 요소로 이루어져 있습니다:

- `:` - Ex 명령 모드로 진입
- `%` - 범위 지정으로, 전체 파일(1부터 마지막 줄까지)을 의미합니다. `1,$`와 동일합니다.
- `s` - substitute(치환) 명령
- `/foo/` - 검색 패턴. "foo"를 찾습니다.
- `/bar/` - 치환 텍스트. 찾은 "foo"를 "bar"로 바꿉니다.
- `g` - global 플래그. 각 줄에서 첫 번째 일치만이 아니라 **모든** 일치 항목을 치환합니다. 이 플래그가 없으면 각 줄의 첫 번째 "foo"만 "bar"로 변경됩니다.
- `c` - confirm 플래그. 각 치환마다 사용자에게 확인을 요청합니다. `y`(yes), `n`(no), `a`(all), `q`(quit) 등의 옵션을 제공합니다.

**전체 의미:** 파일 전체에서 모든 "foo"를 "bar"로 치환하되, 각 치환마다 사용자에게 확인을 받습니다.

**실전 활용:** 변수명이나 함수명을 변경할 때, 일부는 변경하고 일부는 유지하고 싶을 때 이 명령을 사용합니다. `c` 플래그 덕분에 각 항목을 검토하고 선택적으로 치환할 수 있습니다.
</details>

<details>
<summary>2. *과 g*의 차이는?</summary>

두 명령 모두 커서 아래 단어를 검색하지만, **일치 조건**이 다릅니다.

**`*` (완전 일치):**
- 단어 경계를 고려하여 **완전히 일치하는** 단어만 검색합니다.
- 내부적으로 `/\<word\>` 패턴을 사용합니다.
- 예: "user"에 `*` 실행 → "user"만 검색, "userName"이나 "getUser"는 검색 안 됨

**`g*` (부분 일치):**
- 단어 경계를 무시하고 **포함하는** 모든 텍스트를 검색합니다.
- 내부적으로 `/word` 패턴을 사용합니다.
- 예: "user"에 `g*` 실행 → "user", "userName", "getUser" 모두 검색됨

**언제 무엇을 사용할까:**
- **변수명/함수명 추적**: `*` (완전 일치) - 정확히 해당 식별자만 찾기
- **문자열 패턴 찾기**: `g*` (부분 일치) - 해당 텍스트가 포함된 모든 곳 찾기

예를 들어, "count"라는 변수를 리팩토링하는데 "accountCount"는 건드리지 않고 싶다면 `*`를 사용해야 합니다. 반대로 "log"라는 단어가 들어간 모든 함수(log, logger, console.log 등)를 찾고 싶다면 `g*`가 유용합니다.
</details>

<details>
<summary>3. :g 명령과 :%s의 차이를 설명하세요</summary>

두 명령 모두 파일 전체를 대상으로 작업하지만, **작업 방식**과 **목적**이 다릅니다.

**`:%s` (substitute - 치환 명령):**
- **목적**: 텍스트 패턴을 다른 텍스트로 **치환**
- **작동 방식**: 패턴에 일치하는 **텍스트 조각**을 찾아 변경
- **단위**: 문자열/패턴
- **예시**: `:%s/old/new/g` - "old"라는 텍스트를 "new"로 변경

**`:g` (global - 줄 단위 명령):**
- **목적**: 패턴에 일치하는 **줄 전체**에 특정 명령 실행
- **작동 방식**: 패턴에 일치하는 **줄**을 찾아 Ex 명령이나 Normal 명령 실행
- **단위**: 줄
- **예시**: `:g/TODO/d` - "TODO"가 포함된 줄 전체를 삭제

**비교 예시:**

파일 내용:
```
const value = 10;  // TODO: refactor
function test() {  // TODO: add tests
  return value;
}
```

- `:%s/TODO/DONE/g` 실행 결과:
  ```
  const value = 10;  // DONE: refactor
  function test() {  // DONE: add tests
    return value;
  }
  ```
  (텍스트만 치환, 줄은 유지)

- `:g/TODO/d` 실행 결과:
  ```
    return value;
  }
  ```
  (TODO가 포함된 줄 전체가 삭제됨)

**언제 무엇을 사용할까:**
- 문자열/변수명/패턴을 다른 것으로 **변경**: `:%s`
- 특정 패턴이 있는 줄을 **삭제/이동/복사**: `:g`
- 조건부 편집 작업: `:g` + `normal` 명령 조합
</details>
