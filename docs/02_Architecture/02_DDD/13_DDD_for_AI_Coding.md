# DDD for AI Coding — AI 코딩 시대의 도메인 주도 설계

> 출처: "AI 코딩 잘 하고 있다고? DDD 모르면 지금 망하고 있는 겁니다" (YouTube)

## 핵심 전제

AI 코딩 도구(Cursor, Claude Code, Copilot 등)를 쓸 때 **DDD 구조가 없으면 AI가 만드는 코드의 품질이 급격히 낮아진다**. DDD는 AI와 협업하기 위한 "언어적 계약"이자 "구조적 가이드"다.

---

## 4가지 원칙

### 원칙 1: 구조가 패턴이 된다

AI는 기존 코드의 패턴을 인식하여 새 코드를 생성한다. DDD 구조로 도메인을 일관되게 작성하면, AI가 새 도메인을 만들 때 동일한 구조를 복제한다.

**구조 없는 경우**: AI가 매번 조금씩 다른 코드를 생성 → 유지보수 불가능
**구조 있는 경우**: 첫 도메인이 "기준 패턴" → AI가 이를 복제하여 일관성 유지

```
# 기준 패턴 (Order 도메인)
order/
├── domain/         # Entity, Value Object, Domain Event
├── application/    # Use Case, Service
├── infrastructure/ # Repository, External API
└── presentation/   # Controller, DTO

# AI가 동일 패턴으로 생성 (Payment 도메인)
payment/
├── domain/
├── application/
├── infrastructure/
└── presentation/
```

**실무 적용**: 첫 도메인은 직접 짜라. AI에게 구조를 잡아달라고 하면 기준이 없어 일관성 보장 불가.

### 원칙 2: 도메인 단위로 나누면 AI 품질이 올라간다

AI에게 전체 코드베이스를 보여주면 컨텍스트 오염이 발생한다. 도메인 단위로 필요한 코드만 보여주면 집중도와 정확도가 상승한다.

**안티패턴**: 전체 프로젝트를 열고 "결제 버그 고쳐줘"
**올바른 방법**: `payment/` 도메인 코드만 열고 "결제 금액 계산 오류 수정"

이것이 가능하려면 도메인 간 의존성이 낮아야 한다. DDD의 Bounded Context가 자연스럽게 이를 달성한다.

| 접근 방식 | AI 입력 토큰 | AI 집중도 | 환각 확률 |
|----------|------------|----------|----------|
| 전체 코드 제공 | 높음 | 낮음 | 높음 |
| 도메인 단위 제공 | 낮음 | 높음 | 낮음 |

### 원칙 3: 코드 이름 = 비즈니스 언어 (Ubiquitous Language)

코드의 클래스명, 메서드명, 변수명이 비즈니스 용어와 일치하면 AI가 정확한 위치를 찾는다. 불일치하면 AI가 추측하여 환각(hallucination)이 발생한다.

**불일치 예시**:
- 비즈니스: "주문 취소" → 코드: `updateStatus(ORDER_CANCEL)` — AI가 "주문 취소"를 검색해도 못 찾음
- 비즈니스: "결제 승인" → 코드: `processPayment()` — "승인"이 코드에 없어 AI가 추측

**일치 예시**:
- 비즈니스: "주문 취소" → 코드: `Order.cancel()` — AI가 정확히 찾음
- 비즈니스: "결제 승인" → 코드: `Payment.approve()` — 명확한 매핑

**핵심**: AI는 자연어 프롬프트와 코드 이름을 매칭한다. 이름이 일치할수록 정확도가 올라간다.

### 원칙 4: Hook/Trigger + 도메인 = 자동 컨텍스트 로딩

Claude Code의 훅(Hook)이나 트리거 키워드를 도메인과 연결하면, AI가 관련 코드를 자동으로 읽고 맥락을 파악한다.

**예시**: "결제" 키워드 → `payment/` 도메인의 SKILL.md 자동 로드 → 도메인 규칙/구조 파악 → 정확한 코드 생성

이 패턴이 제대로 작동하려면 **DDD 구조가 선행**되어야 한다. 구조 없이 트리거만 설정하면 AI가 읽어야 할 코드의 범위가 불명확해진다.

## 핵심 요약

1. **첫 도메인은 직접 짜라** — AI가 복제할 "기준 패턴"을 만들어야 한다
2. **도메인 단위로 보여줘라** — 전체 코드 노출은 AI 품질을 떨어뜨린다
3. **코드 이름 = 비즈니스 언어** — 이름이 일치할수록 AI 정확도가 올라간다
4. **구조 → 트리거 → 자동화** — DDD 구조가 선행되어야 AI 자동화가 작동한다
