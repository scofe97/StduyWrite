## 1. 개요

### 1.1 Claude Code + Chrome이란?

Claude Code CLI와 Chrome 확장 프로그램을 연결하여 **터미널에서 브라우저를 제어**하는 기능입니다.

```
┌─────────────────┐         ┌─────────────────┐
│   Claude Code   │ ←────→  │ Chrome Extension│
│   (터미널)       │  Native │   (브라우저)     │
│   명령 & 코드    │ Messaging│  실행 & 검증    │
└─────────────────┘   API   └─────────────────┘
```

### 1.2 Build → Test → Verify 워크플로우

```
Build (터미널)          Test (브라우저)         Verify (터미널)
─────────────          ──────────────          ──────────────
코드 작성/수정    →    브라우저에서 테스트   →    콘솔 로그 분석
                       폼 제출, UI 확인          에러 디버깅
                       API 응답 확인             수정 사항 적용
```

### 1.3 출시 정보

| 항목 | 내용 |
|------|------|
| 출시일 | 2025년 12월 18일 (Pro, Team, Enterprise) |
| Claude Code 버전 | v2.0.73 이상 |
| Chrome 확장 버전 | v1.0.36 이상 |
| 지원 플랜 | Pro, Team, Enterprise (유료) |
| 지원 브라우저 | Google Chrome (Brave, Arc 미지원) |
| 제한사항 | WSL 미지원, 헤드리스 모드 없음 |

## 2. 설정 방법

### 2.1 사전 요구사항 확인

```bash
# Claude Code 버전 확인
claude --version  # v2.0.73 이상 필요

# 업데이트 필요시
claude update
```

### 2.2 Chrome 확장 프로그램 설치

1. Chrome 웹스토어에서 "Claude in Chrome" 검색
2. 확장 프로그램 설치 (v1.0.36 이상)
3. Chrome 재시작

### 2.3 연결 시작

```bash
# Chrome 모드로 Claude Code 시작
claude --chrome

# 연결 상태 확인 (Claude Code 내에서)
/chrome
```

### 2.4 연결 옵션

| 옵션 | 설명 |
|------|------|
| Reconnect extension | 연결 재시도 |
| Enabled by default | 기본 활성화 (컨텍스트 사용량 증가) |
| Site permissions | 사이트별 권한 관리 |

## 3. 핵심 기능

### 3.1 브라우저 제어

| 기능 | 설명 | 예시 |
|------|------|------|
| 페이지 탐색 | URL 이동, 뒤로/앞으로 | "google.com 열어줘" |
| 요소 클릭 | 버튼, 링크 클릭 | "로그인 버튼 클릭해줘" |
| 폼 입력 | 텍스트 입력, 선택 | "이메일 필드에 test@test.com 입력" |
| 스크롤 | 페이지 스크롤 | "페이지 하단으로 스크롤" |
| 멀티탭 | 여러 탭 동시 제어 | "3개 탭을 탭 그룹으로 열어줘" |

### 3.2 개발 지원

| 기능 | 설명 |
|------|------|
| 콘솔 로그 읽기 | console.log, error, warning 모니터링 |
| 네트워크 요청 | API 호출 확인, 응답 분석 |
| DOM 상태 | 요소 상태, 스타일, 속성 검사 |
| 에러 디버깅 | JavaScript 에러 자동 감지 및 분석 |

### 3.3 자동화

| 기능 | 설명 |
|------|------|
| 폼 자동 작성 | CSV 데이터로 반복 입력 |
| 데이터 추출 | 웹페이지에서 구조화된 정보 수집 |
| 멀티사이트 워크플로우 | 여러 사이트 간 작업 연결 |
| GIF 녹화 | 브라우저 상호작용 기록 |

### 3.4 인증된 앱 접근

Chrome에 이미 로그인된 사이트는 재인증 없이 접근 가능:
- Google Docs, Gmail, Google Calendar
- Notion, Slack
- CRM, 내부 관리 도구
- 로컬 개발 서버 (localhost)

## 4. 실전 활용 시나리오

### 4.1 로컬 개발 테스트

```
프롬프트:
"localhost:3000/login에서 다음을 테스트해줘:
1. 이메일 필드 비워두고 제출 → 에러 메시지 확인
2. 잘못된 이메일 형식 입력 → 에러 메시지 확인
3. 올바른 값 입력 → 성공 확인"
```

**적합한 상황**:
- 폼 유효성 검사 테스트
- UI 반응 확인
- 사용자 흐름 검증

### 4.2 디버깅

```
프롬프트:
"대시보드 페이지를 열고 콘솔에 에러가 있는지 확인해줘.
있으면 에러 메시지와 발생 위치를 알려줘."
```

```
프롬프트:
"localhost:3000/api/users 호출 시 네트워크 탭에서:
- 요청 헤더 확인
- 응답 상태 코드 확인
- 응답 데이터 구조 확인"
```

### 4.3 데이터 입력 자동화

```
프롬프트:
"contacts.csv 파일의 각 행에 대해 CRM에 새 연락처를 추가해줘:
1. crm.example.com/contacts/new 이동
2. 이름 필드에 name 컬럼 값 입력
3. 이메일 필드에 email 컬럼 값 입력
4. 저장 버튼 클릭
5. 다음 행 반복"
```

### 4.4 데이터 추출 (스크래핑)

```
프롬프트:
"상품 목록 페이지에서 다음 정보를 추출해서 products.csv로 저장해줘:
- 상품명
- 가격
- 재고 상태
- 상품 URL"
```

### 4.5 문서 작업

```
프롬프트:
"최근 커밋 내역을 기반으로 프로젝트 업데이트 초안을 작성하고
docs.google.com/document/d/[ID]에 추가해줘"
```

### 4.6 데모 GIF 생성

```
프롬프트:
"장바구니에 상품 추가부터 결제 완료까지 전체 흐름을 GIF로 녹화해줘.
파일명은 checkout-flow.gif로 저장해줘."
```

### 4.7 멀티사이트 워크플로우

```
프롬프트:
"내 캘린더에서 내일 미팅 확인하고,
외부 참석자가 있는 미팅에 대해 LinkedIn에서 해당 회사 정보를 찾아
간단한 요약을 정리해줘."
```

## 5. 효과적인 프롬프트 패턴

### 5.1 구체적으로 작성

```
❌ 나쁜 예시:
"페이지 테스트해줘"

✅ 좋은 예시:
"localhost:3000/login에서 빈 이메일로 제출 시
'이메일을 입력하세요' 에러가 표시되는지 확인"
```

### 5.2 단계별 지시

```
✅ 좋은 예시:
"다음 단계를 수행해줘:
1. 대시보드 페이지 열기
2. '새 프로젝트' 버튼 클릭
3. 프로젝트 이름에 'Test Project' 입력
4. 저장 버튼 클릭
5. 목록에 새 프로젝트가 표시되는지 확인"
```

### 5.3 기대 결과 명시

```
✅ 좋은 예시:
"로그인 후 /dashboard로 리다이렉트되고,
사용자 이름이 헤더에 표시되는지 확인해줘"
```

### 5.4 템플릿 모음

**폼 테스트 템플릿**:
```
"[URL]에서 다음을 테스트해줘:
1. [필드명] 비워두고 제출 → [기대 에러] 확인
2. 잘못된 형식 입력 → [기대 에러] 확인
3. 올바른 값 입력 → [기대 결과] 확인"
```

**디버깅 템플릿**:
```
"[URL] 열고 다음을 확인해줘:
- 콘솔 에러/경고 여부
- [특정 API] 호출 상태
- [특정 요소] 렌더링 상태"
```

**데이터 추출 템플릿**:
```
"[URL]에서 다음 정보를 추출해서 [파일명]으로 저장해줘:
- 필드1: [설명]
- 필드2: [설명]
- 필드3: [설명]"
```

**E2E 테스트 템플릿**:
```
"다음 시나리오를 테스트해줘:
1. [시작 URL] 접속
2. [액션1] 수행
3. [액션2] 수행
4. [기대 결과] 확인
각 단계에서 콘솔 에러 없는지 확인"
```

## 6. 개발 워크플로우 통합

### 6.1 아침 루틴

```bash
# 1. Chrome 모드로 Claude Code 시작
claude --chrome

# 2. 개발 서버 상태 확인
"localhost:3000 열고 페이지 정상 로딩되는지 확인해줘"
```

### 6.2 개발 중 즉석 테스트

```
# 기능 개발 후
"방금 수정한 검색 기능을 테스트해줘:
- 검색어 입력 후 결과 표시 확인
- 빈 검색어로 검색 시 전체 목록 표시 확인"
```

### 6.3 버그 수정 후 확인

```
# 버그 재현 단계 실행
"다음 단계로 버그가 수정됐는지 확인해줘:
1. 상품 상세 페이지 열기
2. 수량을 0으로 변경
3. 장바구니 담기 버튼 클릭
4. 에러 메시지 표시 확인 (이전: 빈 페이지)"
```

### 6.4 PR 전 검증

```
"다음 페이지들이 정상 동작하는지 확인해줘:
1. /ticket/list - 목록 로딩
2. /ticket/create - 폼 표시
3. /workflow - 워크플로우 목록
각 페이지에서 콘솔 에러 없는지 함께 확인"
```

### 6.5 코드 리뷰 지원

```
"PR에서 수정한 컴포넌트들을 브라우저에서 확인하고,
UI가 디자인 명세대로 렌더링되는지 검증해줘"
```

## 7. 다른 도구와 비교

| 특성 | Claude Code + Chrome | Playwright MCP | Selenium |

| **자연어 지시** | ✅ | ✅ (Claude 연동) | ❌ |

| **헤드리스** | ❌ | ✅ | ✅ |
| **스크립트 재사용** | ❌ (대화형) | ✅ | ✅ |
| **CI/CD 통합** | ❌ | ✅ | ✅ |
| **학습 곡선** | 낮음 | 중간 | 높음 |

### 권장 사용 상황

| 도구 | 적합한 상황 |
|------|------------|
| **Claude Code + Chrome** | 즉석 테스트, 디버깅, 일회성 자동화, 빠른 검증 |

## 8. 문제 해결

### 8.1 연결 문제

| 문제 | 해결책 |
|------|--------|
| 확장 프로그램 연결 안됨 | `/chrome` → "Reconnect extension" |
| 버전 불일치 | `claude update` 및 확장 프로그램 업데이트 |
| Chrome 미실행 | Chrome 실행 후 재연결 |

### 8.2 브라우저 동작 문제

| 문제 | 해결책 |
|------|--------|
| 브라우저 응답 없음 | 모달 다이얼로그 닫기, 새 탭 요청 |
| 권한 오류 | 확장 프로그램 사이트 권한 확인 |
| 페이지 로딩 느림 | 구체적인 대기 조건 명시 |
| 요소 찾기 실패 | 더 구체적인 선택자 제공 |

### 8.3 일반적인 에러

```
# Native Messaging Host 오류
→ Chrome 재시작 후 재연결

# 권한 거부
→ 확장 프로그램 설정에서 사이트 권한 허용

# 타임아웃
→ 작업을 더 작은 단위로 분할
```

## 9. 베스트 프랙티스

### 9.1 해야 할 것 (DO)

- **구체적인 프롬프트**: URL, 요소, 기대 결과 명시
- **단계별 진행**: 복잡한 작업은 작은 단계로 분할
- **로컬 개발 우선**: localhost 테스트에 적극 활용
- **GIF 녹화**: 데모, 버그 리포트에 활용
- **콘솔 모니터링**: 개발 중 에러 즉시 확인

### 9.2 피해야 할 것 (DON'T)

| 상황 | 이유 | 대안 |
|------|------|------|
| CI/CD 자동화 | 헤드리스 미지원 | Playwright 사용 |
| 민감한 데이터 입력 | 보안 리스크 | 테스트 데이터만 사용 |
| 복잡한 반복 테스트 | 스크립트화 불가 | Playwright 스크립트 |
| 오래 걸리는 작업 | 세션 타임아웃 | 작업 분할 |
| 금융/은행 사이트 | 차단됨 | 해당 없음 |

### 9.3 보안 고려사항

```
✅ 권장
─────────────
- 신뢰할 수 있는 사이트에서만 사용
- 테스트 데이터/계정 사용
- 작업 결과 모니터링

❌ 금지
─────────────
- 실제 비밀번호/API 키 입력
- 금융 거래 자동화
- 무인 상태로 장시간 실행
```
