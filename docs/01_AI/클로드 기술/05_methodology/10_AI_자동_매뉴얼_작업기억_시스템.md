## 배경

09번 문서(하네스 엔지니어링)가 **이론 프레임워크**(Context, Constraints, Entropy Management 3대 축)를 정리했다면, 이 문서는 한 개발자가 6개월간 Claude Code를 실전 투입하며 구축한 **4개의 구체적 시스템**을 다룬다.

핵심 비유: AI = 능력은 뛰어나지만 30분마다 기억을 잃는 신입 직원. 매뉴얼, 체크리스트, 보고 체계를 잡아주면 에이스가 된다.

## 1. 자동 매뉴얼 시스템

### 문제
- 수천 줄의 가이드라인을 작성했지만 AI가 참조하지 않음
- "설명서 안 읽고 가구 조립하는 사람"과 동일한 현상

### 해결: Skills + Hooks

**2가지 시동 장치**:
1. **시작 전 알림 장치** (Pre-prompt Hook)
   - 사용자 지시 내용을 분석하여 관련 매뉴얼을 AI에게 자동 전달
   - 비유: 선배가 "이 업무는 매뉴얼 3번 참고해야 돼" 귀띔
2. **완료 후 체크 장치** (Post-completion Hook)
   - AI 작업 완료 시 결과물 점검
   - "에러 처리 추가했나요? 보안 확인했나요?" 상기

### 4가지 활성화 조건

| # | 조건 | 예시 |
|---|------|------|
| 1 | **키워드** | "백엔드", "서버", "API" → 백엔드 매뉴얼 활성화 |
| 2 | **의도 파악** | "새 기능 만들어 줘" → 해당 영역 매뉴얼 활성화 |
| 3 | **작업 위치** | 특정 폴더 파일 수정 중 → 해당 영역 매뉴얼 자동 활성화 |
| 4 | **파일 내용** | 파일 안 특정 패턴 감지 → 관련 매뉴얼 자동 활성화 |

### Hub + Chapter 최적화

**문제**: 처음에 스킬 매뉴얼 하나가 1,500줄 초과 → 비효율적

**해결**: 하나의 큰 매뉴얼을 **목차(Hub) + 상세 챕터 파일**로 분리
- 목차만 먼저 확인 → 필요한 챕터만 로드
- **결과**: AI 자원 소비 40~60% 감소

### 도입 전후 비교

| 항목 | Before | After |
|------|--------|-------|
| 매뉴얼 참조 | AI가 무시 | 자동 확인 |
| 작업 방식 | 매번 수동 알림 | 자동 활성화 |
| 결과물 품질 | 들쑥날쑥 | 일정 유지 |
| 수정 시간 | 많음 | 확 줄음 |

## 2. 작업 기억 시스템

### 문제
- AI의 최대 약점: "자신감은 하늘을 찌르는데 기억력이 금붕어"
- 대화가 길어지면 처음 세운 계획을 잊어버림

### 해결: 3문서 패턴

큰 작업 시작 시 무조건 3개 문서 생성:

| # | 문서 | 역할 | 건축 비유 |
|---|------|------|----------|
| 1 | **계획서** | 무엇을 만들 건지 처음~끝 | 설계도 |
| 2 | **맥락노트** | 왜 이렇게 결정했는지, 관련 자료 위치 | 시방서 |
| 3 | **체크리스트** | 뭘 끝냈고 뭐가 남았는지 추적 | 공정표 |

### 프로세스

1. **계획 세우기**: AI에게 전체 계획 수립 지시
2. **계획 검토**: 사람이 꼼꼼히 검토 (AI가 핵심을 오해하는 경우 多)
3. **문서 저장**: 승인 후 즉시 3문서로 저장 (바로 구현 시작 금지)
4. **단계별 실행**: 1~2개 작업씩 시키고 → 결과 확인 → 다음 작업
5. **체크리스트 갱신**: "방금 한 거 체크하고 다음에 뭘 해야 하는지 정리해"

### 핵심 원칙

- **"알아서 해"는 최악의 지시**: 반드시 계획 먼저
- **한 번에 전부 시키지 말 것**: 1~2개 작업 → 확인 → 다음
- **계획 승인 시 즉시 문서 저장**: AI가 바로 달려드는 것 방지

### 효과
- AI가 딴 길로 가는 일 거의 소멸
- 30분 전 계획이 2시간 뒤에도 유지
- 새 대화를 시작해도 문서를 읽으면 즉시 복구

## 3. 자동 품질 검사

### 문제
- AI가 "다 했습니다" → 실수 수두룩

### 해결: 3중 검사 체계

1. **수정 기록 장치**
   - AI가 파일 수정 시마다 어떤 파일을 건드렸는지 자동 기록 (CCTV 역할)
   - 처음엔 파일 하나마다 검사 → 비효율 (요리 중간에 맛보기와 같음)

2. **완료 후 검사 장치**
   - AI가 대답을 완전히 끝냈을 때 검사 실행
   - 수정 내역 확인 → 오류 체크 → 소량이면 AI가 즉시 수정 / 다량이면 전문 담당 추천

3. **셀프 체크 리마인더**
   - 작업 완료 시 자동 알림: "방금 수정한 파일에서 확인할 것: 오류 처리 추가? 보안 위험?"
   - 강제가 아닌 상기 → AI가 스스로 보완

### 전체 흐름

```
지시 → 매뉴얼 자동 전달 → 작업 수행 → 수정 기록 → 완료 후 검사 → 셀프 체크 → 오류 수정 → 깔끔한 결과물
```

## 4. 전문 에이전트 팀

### 문제
- 하나의 AI에게 모든 것을 시키면 품질이 불안정

### 해결: 역할 분담

| 팀 | 역할 |
|----|------|
| **품질 관리팀** | 코드 품질 검사 |
| **테스트팀** | 테스트 작성 및 실행 |
| **기획팀** | 계획 및 구조 설계 |
| **코드 리뷰팀** | AI 결과물을 다른 AI가 검토 |

### 핵심 팁: 상세 보고서 요구

- 초기: 에이전트가 "다 했습니다"로 끝냄
- 개선: "무엇을 발견했고, 무엇을 수정했고, 왜 그렇게 판단했는지 보고서로 정리해라"

### AI 코드 리뷰

- AI가 만든 결과물을 다른 AI 에이전트가 리뷰
- 자기가 만든 걸 자기가 리뷰 → 의외로 효과적
- 누락, 취약점, 일관성 문제를 잘 잡아냄

|--------|----------------|-------------|------|
| 자동 매뉴얼 | Skills + Hooks | CLAUDE.md + specs/ + skills/ | 구현됨 |
| 4가지 활성화 조건 | 키워드/의도/위치/내용 | description 트리거 + Forced eval hook | 구현됨 |
| Hub+Chapter 분리 | 큰 매뉴얼 → 목차+챕터 | learning/ 허브 패턴 (SKILL.md + references/) | 구현됨 |
| 품질 검사 | 수정 기록 + 완료 후 검사 | Git hooks (commit-msg, pre-commit) | 부분 구현 |
| 전문 에이전트 | 역할별 AI 팀원 | .claude/agents/ 정의 | 구현됨 |
