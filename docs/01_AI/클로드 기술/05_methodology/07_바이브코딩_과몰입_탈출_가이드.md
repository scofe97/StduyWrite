## 핵심 개념: 몰입 vs 과몰입

| 상태 | 특징 |
|------|------|
| **몰입** | 주도감, 회복, 리듬/피드백 세 축이 살아있는 상태 |
| **과몰입(터널)** | 주도감 상실, 에너지 고갈, 행동→확인 루프 단절 |

### 몰입(Flow)의 심리학

Csikszentmihalyi 이론: **도전과 기술이 균형 잡힌 순간**에 몰입 발생

| 측면 | 과거 (직접 코딩) | 현재 (AI 활용) |
|------|------------------|----------------|
| **역할** | 직접 코딩 | 프롬프트 작성, AI 검토 |
| **만족감** | 높음 | 낮아질 위험 |
| **생산성** | 중간 | 높음 |
| **참여도** | 능동적 | 수동적 위험 |

## 터널 감지 체크리스트

다음 질문에 "아니오"가 많다면 과몰입 상태:

1. 작업 중 신체 조정이 가능한가?
2. 완료 후 충만감을 느끼는가?
3. 정한 횟수/시간을 준수하는가?
4. 명확한 학습 목표를 인식하고 있는가?
5. 언제든 멈출 수 있는가?

---

## 과몰입 4가지 패턴

| 패턴 | 증상 |
|------|------|
| **무한 탐색** | 최선의 방법 찾기에 빠져 시작 못함 |
| **대화 착각** | 정보 수집을 진척으로 오인 |
| **증상 치료** | 근본 원인 무시한 반복 수정 |
| **결정 회피** | 선택 미루고 옵션만 축적 |

---

## 5단계 탈출 프로토콜

### 1. 시작 전 선언
- 목표, 산출물, 멈춤 기준을 기록

### 2. 루프 단위 운영
```
할 일 하나 → 질문 → 실행 → 확인 → 반복
```

### 3. 터널 신호 대응
- 신호 감지 시 즉각 상태 조정 행동 실행

### 4. 회복 블록 예약
- 산책, 스트레칭 등 작은 회복 활동 스케줄화

### 5. 실험 → 가설 판정
- 가설을 세우고 실험으로 검증하는 흐름 유지

## AI 활용 패턴

프롬프트 출력을 3줄로 고정:

```
1. 지금 할 일
2. 바로 확인
3. 다음 선택
```

이 구조로 질 높은 루프 구성 유도

## AI 코딩의 숨겨진 비용

### 문제 인식

생산성 향상이 개발자의 **창의성과 성취감을 대체하지 못함**

> "프롬프트 엔지니어링이 새로운 형태의 몰입이 될 수 있을까?"

### 균형 잡기 전략

| 전략 | 설명 |
|------|------|
| **의도적 비효율** | 일부러 직접 코딩하는 시간 유지 |
| **새로운 만족감** | 고수준 시스템 설계에 집중 |
| **역할 분담** | AI는 반복 작업, 창의성은 인간 |

## 핵심 관점

> **과몰입은 근성 문제가 아닌 운영 체계 문제**
>
> 시스템을 개선하면 해결할 수 있다.
> 운전대를 다시 잡는 순간부터 몰입이 안전해진다.
