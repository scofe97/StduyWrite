# Claude Code의 지속적인 학습

> 원본 영상: [Developers Digest - 클로드 코드의 지속적인 학습](https://www.youtube.com/watch?v=sWbsD-cP4rI)

---

## 개요

Claude Code는 세션 간에 프로젝트에 대한 지식을 유지하고 축적할 수 있습니다. 이를 통해 프로젝트를 더 깊이 이해하고, 반복적인 설명 없이도 효율적으로 작업할 수 있습니다.

---

## 1. 지속적 학습의 개념

### Claude Code의 메모리 시스템

```
┌─────────────────────────────────────────────────────────────┐
│                    Claude Code 메모리 계층                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 1. CLAUDE.md (프로젝트 레벨)                         │   │
│  │    - 프로젝트 개요, 아키텍처, 규칙                    │   │
│  │    - 매 세션 시작 시 자동 로드                        │   │
│  │    - 영구 저장 (Git 관리 가능)                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 2. Memory Bank / Specs (지식 저장소)                 │   │
│  │    - 리서치, 계획, 결정 기록                          │   │
│  │    - 도메인 지식 축적                                 │   │
│  │    - 문제 해결 히스토리                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 3. Skills (기능 패키지)                              │   │
│  │    - 반복 작업 자동화                                 │   │
│  │    - 팀 컨벤션 적용                                   │   │
│  │    - 워크플로우 표준화                                │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 세션 간 지식 유지

| 방식 | 설명 | 저장 위치 |
|------|------|----------|
| **CLAUDE.md** | 프로젝트 핵심 정보 | 프로젝트 루트 |
| **리서치 문서** | 코드베이스 분석 결과 | `specs/research/` |
| **계획 문서** | 구현 계획 및 결정 | `specs/plans/` |
| **스킬 파일** | 반복 작업 지침 | `.claude/skills/` |

## 2. CLAUDE.md 활용

### 프로젝트 두뇌 역할

CLAUDE.md는 **프로젝트의 두뇌** 역할을 합니다. 매 세션 시작 시 자동으로 로드되어 Claude가 프로젝트 컨텍스트를 즉시 파악할 수 있게 합니다.

### 효과적인 CLAUDE.md 구조

```markdown
# 프로젝트명 - Claude 가이드

## 프로젝트 개요
[한 문단으로 프로젝트 설명]

## 기술 스택
- Backend: [기술]
- Frontend: [기술]
- Database: [기술]

## 아키텍처
[아키텍처 설명 및 다이어그램]

## 디렉토리 구조
```
src/
├── domain/
├── application/
└── infrastructure/
```

## 코딩 컨벤션
- [규칙 1]
- [규칙 2]

## 주의사항
- [프로젝트 특이사항]

## 최근 업데이트
- [YYYY-MM-DD] [변경 내용]
```

### CLAUDE.md 업데이트 시점

| 시점 | 업데이트 내용 |
|------|-------------|
| 새 기능 추가 | 아키텍처 변경, 새 패턴 |
| 중요 결정 | 기술적 결정 및 근거 |
| 문제 해결 | 자주 발생하는 이슈 및 해결책 |
| 팀 규칙 변경 | 코딩 컨벤션, 워크플로우 |

## 3. 지식 축적 워크플로우

### 학습 → 저장 → 재사용 사이클

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   세션 1: 학습                                               │
│   ─────────────                                             │
│   - 코드베이스 분석                                          │
│   - 패턴 발견                                                │
│   - 문제 해결                                                │
│         ↓                                                   │
│   📝 문서로 저장                                             │
│   - specs/research/[topic].md                               │
│   - CLAUDE.md 업데이트                                       │
│                                                             │
│   ─────────────────────────────────────────────────────────  │
│                                                             │
│   세션 2: 재사용                                             │
│   ─────────────                                             │
│   - 저장된 문서 참조                                         │
│   - 반복 설명 불필요                                         │
│   - 바로 작업 시작                                           │
│         ↓                                                   │
│   📝 새로운 발견 추가                                        │
│                                                             │
│   ─────────────────────────────────────────────────────────  │
│                                                             │
│   세션 N: 누적된 지식                                        │
│   ──────────────────                                        │
│   - 깊은 프로젝트 이해                                       │
│   - 빠른 문제 해결                                           │
│   - 일관된 품질                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 지식 축적 영역

| 영역 | 저장 내용 | 예시 |
|------|----------|------|
| **아키텍처** | 시스템 구조, 패턴 | DDD 구조, 계층 분리 |
| **도메인** | 비즈니스 로직, 용어 | 워크플로우 상태, 결재 프로세스 |
| **기술** | 라이브러리 사용법, 설정 | MapStruct 매핑 규칙 |
| **문제 해결** | 발생한 이슈, 해결책 | Enum 변환 오류 해결 |

## 4. 효과적인 지속 학습 전략

### 전략 1: 리서치 문서 축적

```
specs/research/
├── architecture-overview.md      # 전체 아키텍처
├── domain-workflow.md            # 워크플로우 도메인
├── domain-approval.md            # 승인 도메인
├── pattern-mapper.md             # 매퍼 패턴
└── troubleshooting-enum.md       # Enum 이슈 해결
```

### 전략 2: 결정 기록 (ADR)

```markdown
# ADR-001: MapStruct 도입

**날짜**: 2025-01-03
**상태**: 승인됨

## 컨텍스트
기존 Converter 방식의 수동 매핑이 유지보수 어려움

## 결정
MapStruct를 사용한 컴파일 타임 매퍼 생성

## 결과
- 타입 안전성 향상
- 보일러플레이트 코드 감소
- 컴파일 시점 오류 발견
```

### 전략 3: 문제 해결 히스토리

```markdown
# Troubleshooting: Enum 변환 오류

**문제**: IllegalArgumentException: No enum constant

**원인**: MapStruct가 Enum.valueOf() 사용

**해결**:
```java
@Named("stringToEnumType")
public EnumType stringToEnumType(String code) {
    return EnumType.fromCode(code);
}
```

**적용 위치**: WorkflowMapperHelper.java
```

## 5. 자동 학습 유도 프롬프트

### 세션 시작 시

```
@CLAUDE.md

이전 세션에서 학습한 내용을 확인하고,
새롭게 발견한 패턴이나 규칙이 있다면 
CLAUDE.md 또는 관련 문서에 추가해주세요.
```

### 작업 완료 후

```
이번 작업에서 발견한 중요한 패턴이나 주의사항이 있다면
적절한 문서에 기록해주세요:
- 아키텍처 관련 → CLAUDE.md
- 특정 도메인 → specs/research/domain-*.md
- 문제 해결 → specs/troubleshooting/
```

### 정기 정리

```
프로젝트의 현재 상태를 분석하고,
CLAUDE.md와 specs/ 문서들이 최신 상태인지 확인해주세요.
업데이트가 필요한 부분을 수정해주세요.
```

## 6. 지속 학습의 효과

### Before: 매 세션 반복

```
세션 1: "이 프로젝트는 DDD 구조입니다..." (설명)
세션 2: "이 프로젝트는 DDD 구조입니다..." (또 설명)
세션 3: "이 프로젝트는 DDD 구조입니다..." (또또 설명)
```

### After: 누적 학습

```
세션 1: 프로젝트 분석 → CLAUDE.md 작성
세션 2: @CLAUDE.md 로드 → 바로 작업 시작
세션 3: @CLAUDE.md 로드 → 더 깊은 작업 가능
         + 새 발견 추가
```

### 정량적 효과

| 지표 | 개선 효과 |
|------|----------|
| **세션 시작 시간** | 50% 단축 (설명 불필요) |
| **코드 품질** | 일관성 향상 (규칙 축적) |
| **문제 해결** | 빠른 해결 (히스토리 참조) |
| **온보딩** | 신규 멤버 빠른 적응 |

├── CLAUDE.md                     # 🧠 프로젝트 두뇌
│
├── specs/                        # 📋 지식 저장소
│   ├── research/                 # 코드베이스 분석
│   │   ├── architecture.md
│   │   └── domain-*.md
│   ├── plans/                    # 구현 계획
│   ├── decisions/                # 기술 결정 (ADR)
│   └── troubleshooting/          # 문제 해결 히스토리
│
├── .claude/                      # Claude 설정
│   └── skills/                   # 스킬 정의
│
└── src/                          # 소스 코드
```

## 요약

### 핵심 원칙

```
1. 기록하라   → 발견한 것을 문서로 저장
2. 구조화하라 → 일관된 폴더 구조 유지
3. 참조하라   → 매 세션 시작 시 문서 로드
4. 업데이트하라 → 새 발견 시 문서 갱신
```

### 지속 학습 효과

| 항목 | 효과 |
|------|------|
| **효율성** | 반복 설명 제거, 빠른 작업 시작 |
| **품질** | 일관된 패턴, 축적된 노하우 |
| **확장성** | 팀 전체 지식 공유 |
| **유지보수** | 히스토리 추적, 빠른 온보딩 |
