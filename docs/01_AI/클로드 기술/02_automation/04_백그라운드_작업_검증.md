## 1. 긴 작업은 백그라운드에서

오래 걸리는 작업은 멍하니 보고 있지 말고, 백그라운드 에이전트에게 맡기기.

### 방법

1. **백그라운드 에이전트 활용**: 검증 작업을 별도 에이전트에 위임
2. **ralph-wiggum 플러그인**: 자동 권한 승인으로 중단 없는 실행

### ralph-wiggum 플러그인

```bash
# 샌드박스 환경에서 권한 승인 없이 실행
--permission-mode=dontAsk
```

AI가 권한 승인을 묻지 않고 계속 일하게 설정.

**GitHub**: [ralph-wiggum 플러그인](https://lnkd.in/g_rFKyDv)

## 2. 가장 중요한 팁: AI에게 '검증 수단'을 쥐어주세요

> 품질을 2~3배 높이는 비결

Claude가 코드를 짠 뒤 **스스로 검증할 수 있는 루프(Loop)**를 만들어 주는 것.

### 검증 루프 구조

```
┌─────────────────────────────────────────────────────────┐
│                    코드 생성                             │
└─────────────────────────────────────────────────────────┘
                          │
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    검증 수행                             │
│  - 테스트 실행                                           │
│  - 린트 검사                                             │
│  - 타입 검사                                             │
│  - 브라우저 테스트 (Chrome 확장)                         │
└─────────────────────────────────────────────────────────┘
                          │
              ┌───────────┴───────────┐
              ↓                       ↓
        ┌──────────┐           ┌──────────┐
        │  통과    │           │  실패    │
        └──────────┘           └──────────┘
              │                       │
              ↓                       ↓
        ┌──────────┐           ┌──────────┐
        │  완료    │           │ 수정 후  │
        │          │           │ 재시도   │
        └──────────┘           └──────────┘
                                      │
                                      ↓
                              (루프 반복)
```

### Claude Chrome 확장 프로그램

브라우저에서 직접 테스트하여 검증:

- UI 동작 확인
- API 응답 검증
- 실제 사용자 경험 테스트

**설치**: [Claude Chrome 확장 프로그램](https://lnkd.in/gkq6N88d)

## 검증 수단 예시

| 검증 유형 | 도구/방법 | 적용 시점 |
|----------|----------|----------|
| **단위 테스트** | Jest, pytest, JUnit | 함수/메서드 작성 후 |
| **타입 검사** | TypeScript, mypy | 코드 변경 시 |
| **린트** | ESLint, Prettier | 커밋 전 |
| **빌드** | npm build, gradle | 기능 완성 후 |
| **브라우저 테스트** | Chrome 확장 | UI 변경 후 |
| **API 테스트** | Postman, curl | 엔드포인트 구현 후 |

---

## 핵심 포인트

1. **자동 검증 루프**: Claude가 스스로 오류를 발견하고 수정
2. **백그라운드 실행**: 사용자가 기다리지 않아도 됨
3. **권한 자동 승인**: 중단 없는 연속 작업

---

## 적용 예시

```bash

3. 모든 테스트 통과할 때까지 반복
"

# 2. 검증 결과 확인
cat tasks/logs/verification.log
```
