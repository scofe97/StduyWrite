# Claude Code 자율 실행 가이드

> Claude Code를 몇 시간 동안 자율적으로 실행하는 방법

**원본 영상**: [Developers Digest - Claude Code를 몇 시간 동안 자율적으로 실행하는 방법](https://www.youtube.com/watch?v=o-pMCoVPN_k)

---

## 개요

Claude Code를 사람의 개입 없이 몇 시간 동안 자율적으로 실행하여 대규모 작업을 자동화하는 방법을 다룹니다.

---

## 1. 핵심 옵션: `--dangerously-skip-permissions`

### 기본 개념

Claude Code는 기본적으로 **파일 수정**이나 **명령 실행** 시 사용자에게 권한을 요청합니다. 자율 실행을 위해서는 이 권한 확인을 생략해야 합니다.

```bash
claude --dangerously-skip-permissions
```

### 동작 방식

```
┌─────────────────────────────────────────────────────────────┐
│                    일반 모드 vs 자율 모드                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  일반 모드:                                                  │
│  ─────────                                                  │
│  Claude: "파일을 수정해도 될까요?" → 사용자: "예"            │
│  Claude: "명령을 실행해도 될까요?" → 사용자: "예"            │
│  (매번 확인 필요 → 자율 실행 불가)                           │
│                                                             │
│  자율 모드 (--dangerously-skip-permissions):                │
│  ──────────────────────────────────────────                 │
│  Claude: 파일 수정 → 자동 실행                               │
│  Claude: 명령 실행 → 자동 실행                               │
│  (확인 없이 진행 → 자율 실행 가능)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 안전한 실행 환경 구성

### ⚠️ 필수 주의사항

> **`--dangerously-skip-permissions`는 Claude가 시스템 명령어를 마음대로 실행할 수 있게 합니다.**
> 
> 중요한 파일 삭제, 시스템 설정 변경 등의 위험이 있으므로 **반드시 격리된 환경**에서만 사용하세요.

### 권장 환경: Docker 컨테이너

```dockerfile
# Dockerfile 예시
FROM node:20-slim

# Claude Code 설치
RUN npm install -g @anthropic-ai/claude-code

# 작업 디렉토리 설정
WORKDIR /workspace

# 프로젝트 복사
COPY . .

# Claude Code 실행
CMD ["claude", "--dangerously-skip-permissions", "-p", "작업 내용"]
```

### Docker 실행

```bash
# 이미지 빌드
docker build -t claude-autonomous .

# 컨테이너 실행 (네트워크 격리)
docker run --rm \
  --network none \
  -v $(pwd):/workspace \
  claude-autonomous
```

### 격리 환경 체크리스트

- [ ] **네트워크 차단** (`--network none`)
- [ ] **볼륨 마운트 최소화** (필요한 디렉토리만)
- [ ] **읽기 전용 마운트** 고려 (`:ro`)
- [ ] **중요 파일 백업**
- [ ] **별도 사용자 계정** 사용

## 3. 효과적인 자율 실행을 위한 설정

### 3.1 CLAUDE.md 작성

자율 실행 시 Claude가 참조할 명확한 지침을 작성합니다.

```markdown
# 프로젝트 가이드

## 프로젝트 개요
[프로젝트 설명]

## 디렉토리 구조
[주요 폴더 설명]

## 코딩 컨벤션
[스타일 가이드]

## 자율 실행 시 규칙
- 테스트 실패 시 롤백
- 기존 패턴 준수
- 로그 파일 생성 필수

## 금지 사항
- 시스템 파일 수정 금지
- 외부 API 호출 금지
- 패키지 설치 금지
```

### 3.2 작업 범위 명확화

```bash
# 명확한 작업 지시와 함께 실행
claude --dangerously-skip-permissions -p "
다음 작업을 수행하세요:
1. src/components/ 폴더의 모든 컴포넌트에 TypeScript 타입 추가
2. 각 파일 수정 후 npm run typecheck 실행
3. 오류 발생 시 수정
4. 완료 후 summary.md 작성

제한사항:
- package.json 수정 금지
- 기존 로직 변경 금지
- 타입 추가만 수행
"
```

## 4. 실행 방법

### 4.1 단일 작업 실행

```bash
# -p 옵션으로 작업 내용 전달
claude --dangerously-skip-permissions -p "모든 TODO 주석을 찾아 이슈로 정리해주세요"
```

### 4.2 스크립트 파일로 실행

```bash
# task.txt 파일에 작업 내용 작성
cat > task.txt << 'EOF'
다음 작업을 순서대로 수행하세요:

1. 코드베이스 분석
   - 사용하지 않는 import 찾기
   - 중복 코드 식별

2. 리팩토링
   - 사용하지 않는 import 제거
   - 중복 코드 함수로 추출

3. 검증
   - npm run lint 실행
   - npm run test 실행

4. 보고서 작성
   - refactoring-report.md 생성
   - 변경 내용 요약
EOF

# 파일 내용으로 실행
claude --dangerously-skip-permissions -p "$(cat task.txt)"
```

### 4.3 백그라운드 실행

```bash
# nohup으로 백그라운드 실행
nohup claude --dangerously-skip-permissions -p "대규모 리팩토링 작업" > claude.log 2>&1 &

# 로그 모니터링
tail -f claude.log
```

### 4.4 스케줄 실행 (cron)

```bash
# crontab 편집
crontab -e

# 매일 새벽 2시에 코드 분석 실행
0 2 * * * cd /path/to/project && claude --dangerously-skip-permissions -p "일일 코드 분석 수행" >> /var/log/claude-daily.log 2>&1
```

## 5. 모니터링 및 로깅

### 로그 파일 설정

```bash
# 상세 로그와 함께 실행
claude --dangerously-skip-permissions -p "작업 내용" 2>&1 | tee -a claude-$(date +%Y%m%d-%H%M%S).log
```

### 진행 상황 체크포인트

작업 지시에 체크포인트를 포함시킵니다:

```
작업 수행 중 다음 파일에 진행 상황을 기록하세요:
- progress.md: 완료된 작업 목록
- errors.md: 발생한 오류 및 해결 방법
- summary.md: 최종 결과 요약
```

## 6. 실전 활용 예시

### 예시 1: 대규모 타입 마이그레이션

```bash
claude --dangerously-skip-permissions -p "
프로젝트를 JavaScript에서 TypeScript로 마이그레이션하세요.

단계:
1. .js 파일을 .ts로 변환
2. 기본 타입 추가 (any 허용)
3. tsconfig.json 설정
4. 컴파일 오류 수정
5. 테스트 실행 및 검증

각 단계 완료 후 migration-log.md에 기록하세요.
"
```

### 예시 2: 코드 품질 개선

```bash
claude --dangerously-skip-permissions -p "
코드 품질을 개선하세요.

작업:
1. ESLint 오류 모두 수정
2. 사용하지 않는 코드 제거
3. 함수 길이 50줄 이상인 것 분리
4. 주석 없는 복잡한 로직에 주석 추가

각 파일 수정 후 npm run lint 실행하여 검증하세요.
"
```

### 예시 3: 테스트 코드 생성

```bash
claude --dangerously-skip-permissions -p "
테스트 커버리지를 80% 이상으로 높이세요.

작업:
1. 현재 커버리지 확인 (npm run coverage)
2. 테스트가 없는 함수 식별
3. 각 함수에 대한 단위 테스트 작성
4. 엣지 케이스 포함
5. 최종 커버리지 확인

테스트 파일은 __tests__ 폴더에 작성하세요.
"
```

## 7. 주의사항 및 베스트 프랙티스

### ✅ 권장 사항

| 항목 | 설명 |
|------|------|
| **격리 환경 사용** | Docker, VM 등 격리된 환경에서 실행 |
| **백업 필수** | 작업 전 코드베이스 백업 |
| **명확한 지시** | 구체적이고 범위가 명확한 작업 지시 |
| **단계별 체크포인트** | 진행 상황 기록 요청 |
| **제한 사항 명시** | 하지 말아야 할 것 명확히 지정 |
| **테스트 검증** | 각 단계 후 테스트 실행 요청 |

### ❌ 주의 사항

| 항목 | 위험 |
|------|------|
| **프로덕션 환경** | 절대 프로덕션에서 실행 금지 |
| **네트워크 연결** | 격리 없이 네트워크 연결 상태로 실행 금지 |
| **무제한 권한** | root/admin 권한으로 실행 금지 |
| **모호한 지시** | 범위가 불명확한 작업 지시 금지 |

## 8. 트러블슈팅

### 문제: 작업이 중간에 멈춤

```bash
# 타임아웃 설정
timeout 3600 claude --dangerously-skip-permissions -p "작업"
```

### 문제: 메모리 부족

```bash
# Node.js 메모리 제한 증가
NODE_OPTIONS="--max-old-space-size=8192" claude --dangerously-skip-permissions -p "작업"
```

### 문제: 무한 루프

작업 지시에 탈출 조건을 포함:

```
최대 100개 파일까지만 처리하세요.
30분 이상 걸리는 작업은 중단하고 보고하세요.
동일한 오류가 3번 발생하면 해당 파일은 건너뛰세요.
```
