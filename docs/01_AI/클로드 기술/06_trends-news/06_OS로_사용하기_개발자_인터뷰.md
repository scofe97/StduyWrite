## 핵심 철학

### "내 OS는 이제 Mac이 아니라 Claude Code다"

Claude Code를 단순한 도구가 아닌 **운영체제 수준의 워크플로우 허브**로 활용하는 관점. 개발 작업의 시작과 끝을 Claude Code 내에서 처리.

### 추상화 레벨 상승

> "반복되는 작업을 워크플로우로, 워크플로우를 자동화로, 그리고 더 높은 수준의 문제에 집중"

1. 반복 작업 식별
2. 워크플로우로 변환
3. 자동화 구현
4. 고차원 문제에 집중

## 핵심 기술 인사이트

### Git Worktree 활용

Git worktree를 통해 병렬 개발 환경 구축:
- 격리된 폴더 인스턴스 생성
- 코드 충돌 없이 동시 기능 개발
- **Claude Code 멀티 인스턴스 활용**에 최적

```bash
# worktree 생성 예시
git worktree add ../feature-branch feature/new-feature
```

### Anthropic 생태계 장점

MCP, Skills, Plugins, Commands가 만드는 **오픈 표준**:
- 크로스 도구 통합 가능
- Codex, Gemini CLI를 Claude Code 내에서 호출 가능
- 확장성과 유연성 확보

## 시그니처 메타 워크플로우

### /wrap - 세션 분석 시스템

5개의 병렬 서브에이전트로 세션 종료 시 분석:

| 에이전트 | 역할 |
|----------|------|
| 문서 업데이트 | 변경사항 문서화 |
| 자동화 패턴 | 반복 패턴 식별 |
| 학습 추출 | 인사이트 도출 |
| 후속 작업 | 다음 단계 정리 |
| 중복 감지 | 중복 작업 식별 |

### /session-insights - 세션 시각화

히스토리 세션 분석 도구:
- 토큰 사용량 추적
- 대화 패턴 분석
- 스킬 변환 추천

## 실천 가이드

### 1. 워크트리 기반 병렬 개발

```bash
# 기능별 워크트리 생성
git worktree add ../feature-a feature/a
git worktree add ../feature-b feature/b

# 각 워크트리에서 Claude Code 인스턴스 실행
cd ../feature-a && claude
```

### 2. 세션 래핑 습관화

세션 종료 전 `/wrap` 패턴 실행:
- 무엇을 했는지 문서화
- 자동화 가능한 패턴 식별
- 다음 세션 준비

### 3. 반복 작업 스킬화

반복되는 워크플로우 발견 시:
1. 패턴 문서화
2. 스킬로 변환
3. 커뮤니티 공유
