## 커리어 인사이트

### 1. 잠재 수요(Latent Demand) 찾기

> "제품 개발에서 잠재 수요보다 중요한 것은 없다. 성공한 Facebook 제품들은 모두 이것을 정확히 포착했다."

**핵심**: 새로운 행동을 강요하지 말고, 사용자가 **이미 하고 있는 것**을 관찰하라.

### 2. 사이드 프로젝트의 가치

> "엔지니어의 슈퍼파워는 자동화다. 가장 지루한 것들을 자동화할 수 있다."

사이드 프로젝트가 엔지니어를 **제너럴리스트**로 만든다:
- 제품 기획
- 디자인
- 사용자 리서치

모든 역할을 직접 수행하게 됨.

### 3. 제너럴리스트가 이긴다

> "코딩하고, 제품을 생각하고, 디자인을 이해하고, 사용자와 대화하고 싶어하는 엔지니어와 일하는 것을 좋아한다."

**Anthropic 문화**: 모든 직군이 "Member of Technical Staff" 타이틀 사용.

### 4. 기술적 의견 충돌 해결법

**신뢰가 먼저**, 기술적 신뢰는 그 다음.

| 단계 | 행동 |
|------|------|
| 1 | 시니어의 결정에 따라 실행할 의지를 먼저 보여줌 |
| 2 | 결과가 틀렸을 때, 시니어가 오히려 옹호자가 됨 |

### 5. Unshipping 문화 (기능 제거)

Instagram에서 배운 교훈:

> "기능 추가는 사용하지 않는 대부분의 사용자에게 해가 된다. 기능 제거는 평균 경험을 개선한다."

## Claude Code 개발 인사이트

### Model-Ready Thinking

> "오늘의 모델이 아니라 **6개월 후 모델**을 위해 빌드하라."

| 시점 | AI 코드 작성 비율 |
|------|------------------|
| 초기 | 10% |
| 현재 (Sonnet/Opus 4) | **80-90%** |

### 생산성 향상

> "2025년 인원이 3배 증가했음에도, 엔지니어 생산성은 Claude Code 덕분에 **~70% 향상**되었다."

### AI 코딩 베스트 프랙티스

> "모델에게 검증 방법을 제공하라 - 브라우저 테스트, 테스트 스위트, 시뮬레이터. 최종 품질이 **2-3배 향상**된다."

| 검증 방법 | 효과 |
|----------|------|
| 브라우저 테스트 | 실제 동작 검증 |
| 테스트 스위트 | 자동화된 품질 보장 |
| 시뮬레이터 | 다양한 시나리오 검증 |

**중요**: 이 원칙은 AI 코드뿐 아니라 **인간 코드에도 동일하게 적용**.

## 적용 포인트

### Claude Code 활용

1. **검증 방법 제공**: 테스트 스위트와 함께 작업 요청
2. **미래 지향 설계**: 현재 모델 한계에 맞추지 말 것
3. **자동화 우선**: 반복 작업은 스킬/워크플로우로 변환

### 커리어 전략

1. **제너럴리스트 지향**: 코딩 + 제품 + 디자인 + 사용자 이해
2. **사이드 프로젝트**: 자동화 도구 만들기
3. **신뢰 구축**: 기술적 의견보다 실행 의지 먼저
