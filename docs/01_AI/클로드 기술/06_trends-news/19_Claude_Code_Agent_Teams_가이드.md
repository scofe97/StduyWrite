# Claude Code Agent Teams — 단축키부터 실전 세팅까지 5분 정리

> 아직은 비효율적이지만, 멀티 에이전트가 가는 방향을 이해하려면 직접 써봐야 한다

---

## 배경

Opus 4.6 출시와 함께 공개된 Claude Code Agent Teams. 개인적으로 아직은 비효율적이라고 생각한다. 토큰 소모도 크고 조율 비용도 만만치 않다.

그런데 Tom(@tomcrawshaw01)이 쓴 가이드를 읽다 보니 **구조 자체는 알아둘 가치가 있다.** 프리뷰 단계에서 미리 손에 익혀두면 정식 출시 때 차이가 날 것이라는 판단에 정리해봤다.

---

## 1. 혼자 일하던 에이전트가 팀을 꾸리기 시작했다

리드 에이전트가 작업을 쪼개서 여러 팀원에게 분배하고 **병렬로 동시에** 돌리는 구조다.

### 서브 에이전트와의 핵심 차이

| 구분 | 서브 에이전트 | Agent Teams |
|------|-------------|-------------|
| **통신 방식** | 결과만 보고 | 팀원끼리 직접 메시지 교환 |
| **협업 수준** | 일방향 위임 | 발견 공유 + 이의 제기 가능 |
| **관계** | 상하 관계 | 팀 구조 |

---

## 2. 세팅은 30초면 끝난다

실험 기능이라 기본값이 "off" 상태다. 딱 한 줄만 추가하면 된다.

### settings.json 설정

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

- 환경변수로 직접 잡아도 되지만, settings.json이 세션 간 유지되니까 더 편하다
- 공식 문서: https://lnkd.in/gxzb4xT4

## 3. 핵심 단축키

팀이 돌아가기 시작하면 리드와 팀원 사이를 오가면서 작업을 확인해야 한다.

| 단축키 | 기능 | 비고 |
|--------|------|------|
| **Shift+Up / Shift+Down** | 팀원 간 전환 | in-process 모드, 가장 많이 사용 |
| **Enter** | 선택한 팀원 세션 상세 보기 | |
| **Escape** | 팀원의 현재 턴 중단 | |
| **Ctrl+T** | 공유 태스크 리스트 토글 | |
| **Shift+Tab** | 위임(Delegate) 모드 토글 | 리드가 조율만 하도록 잠금 |
| **클릭** | 팀원 패널 직접 상호작용 | split-pane 모드에서만 |

> **Delegate 모드는 꼭 알아두자.** 켜지 않으면 리드가 위임 대신 자기가 직접 작업을 시작하는 경우가 생긴다.

## 4. 터미널마다 되는 게 다르다

### 터미널 호환성

| 터미널 | split-pane | in-process |
|--------|-----------|------------|
| **tmux** | O | O |
| **iTerm2** | O | O |
| **VS Code 통합** | X | O |
| **Windows Terminal** | X | O |
| **Ghostty** | X | O |

### split-pane 모드 사용법

Ghostty에서 작업하고 있다면 기본 in-process 모드에서 Shift+Up/Down으로 팀원을 전환하게 된다.

팀원 출력을 동시에 보고 싶으면 tmux 세션 안에서 Claude Code를 실행하는 방법이 있다:

```bash
tmux new -s {my-project}
claude --teammate-mode tmux
```

`teammateMode` 기본값이 "auto"라서 tmux 세션 안이면 **자동으로 split-pane으로 전환**된다.

## 5. 작업 분배와 태스크 관리

태스크 리스트 기반으로 작업이 흘러간다. 리드가 목록을 만들면 팀원이 알아서 가져가는 구조다.

### 작업 흐름

```
리드가 태스크 리스트 생성
  → 팀원이 자동으로 미할당 작업 가져감
  → 태스크 간 의존 관계 설정 가능 (선행 → 후속)
  → 특정 팀원 지정 배분 가능
  → 팀원 종료: 리드에게 shutdown 요청 → 확인 후 종료
  → 전체 정리: 모든 팀원 종료 → 리드에게 cleanup 지시
```

### 정교한 팀 구성 팁

간단하게 말해도 팀 호출을 명시하면 해당 기능이 자동 작동하지만, 아래를 정리하면 더 정교한 팀을 구성할 수 있다:

- 총 팀원 수
- 각 팀원의 이름과 역할
- 위임 범위
- 외부 파일로 정리한 task 간의 Dependency rule

## 6. 토큰 관리

팀원 하나가 독립된 Claude Code 세션이라 **병렬로 돌리면 비용이 급격히 올라간다.**

### 토큰 절약 규칙

| 규칙 | 이유 |
|------|------|
| **spawn 프롬프트를 구체적으로** | 리드 대화 이력은 팀원에게 안 넘어감. CLAUDE.md + MCP 서버 컨텍스트만 자동 로드 |
| **파일 단위로 확실히 분리** | 한 파일에 팀원 두 명이 손대면 덮어쓰기 발생 |
| **리서치/리뷰부터 시작** | 코드 작성보다 조율 복잡도가 낮음 |
| **작업 단위를 작게** | 함수 하나, 테스트 파일 하나처럼 명확한 결과물이 나오는 크기 |

---

## 7. 알아두면 좋은 한계

프리뷰 단계라 거친 부분이 꽤 있다. 미리 알아두면 도구 탓을 안 하게 된다.

| 한계 | 대응 |
|------|------|
| `/resume`이나 `/rewind` 사용 시 in-process 팀원 소멸 | 리드에게 새로 spawn 지시 |
| 태스크 상태 지연 (완료인데 미표시) | 직접 확인 후 리드에게 업데이트 요청 |
| 세션당 팀 1개만 가능 | 팀원이 하위 팀 생성 불가 |
| 리드 교체 불가 | 팀을 만든 세션이 끝까지 리드 |

## 정리

| 항목 | 내용 |
|------|------|
| **뭐냐** | 리드 + 팀원 구조의 병렬 멀티 에이전트 |
| **핵심 차이** | 서브 에이전트와 달리 팀원끼리 직접 통신 |
| **세팅** | settings.json에 환경변수 한 줄 추가 |
| **필수 단축키** | Shift+Up/Down (전환), Shift+Tab (위임 모드) |
| **권장 시작** | 코드 작성보다 리서치/리뷰부터 |
| **현재 상태** | 프리뷰, 비용 대비 효율 의문, 구조는 파악해둘 가치 |

아직 거친 프리뷰 단계이고 비용 대비 효율도 의문이 남는다. 그래도 멀티 에이전트가 가는 방향을 이해하려면 한번 직접 켜보는 게 가장 빠르다. **도구는 써봐야 판단할 수 있으니까.**
