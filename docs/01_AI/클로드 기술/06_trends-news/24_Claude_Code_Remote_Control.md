## 핵심 요약

Anthropic이 Claude Code에 **Remote Control** 기능을 공식 출시했다. 로컬 터미널에서 실행 중인 Claude Code 세션을 웹 브라우저나 모바일 기기에서 그대로 접속해 사용할 수 있는 기능이다.

왜 이 기능이 나왔을까? 기존에도 Claude Code Web을 통해 원격 작업이 가능했지만, API 비용제를 사용해야 했기 때문에 비용 부담이 컸다. 로컬 환경이 아니다 보니 슬래시 커맨드, 스킬, MCP 서버 같은 로컬 설정도 사용할 수 없었고, 디버깅도 불편했다. 결과적으로 특수한 경우가 아니면 잘 쓰이지 않는 기능이었다.

Remote Control은 이 문제를 근본적으로 해결한다. 로컬에서 실행되는 세션을 그대로 웹에 미러링하는 방식이라, 구독제(Pro/Max 플랜) 비용만으로 사용할 수 있고, 로컬의 모든 설정이 그대로 유지된다. Anthropic 입장에서 API 비용제라는 캐시카우를 포기한 셈인데, 이는 구독자 저변을 더 공격적으로 넓히겠다는 전략적 판단으로 보인다.

## 사용 방법

### 실행

두 가지 방법이 있다.

**새 세션으로 시작하는 경우**, 터미널에서 `--remote-control` 플래그와 함께 실행한다.

```bash
claude --remote-control
```

실행하면 "Connected"라고 표시되면서 프로젝트가 리모트로 연결됐다는 메시지가 나온다. 스페이스바를 누르면 QR코드가 표시된다.

**이미 실행 중인 세션에서 활성화하려면**, 세션 안에서 슬래시 커맨드를 입력한다.

```
/remote-control
```

역시 링크가 제공되며, 그 링크를 통해 외부 기기에서 접속할 수 있다.

### 연결 방식 3가지

| 방식 | 설명 | 상태 |
|------|------|------|
| **URL 복사** | 터미널에 표시된 링크를 복사해 브라우저에 붙여넣기 | 정상 작동 |
| **QR코드** | 스페이스바로 QR코드를 표시, 모바일로 스캔 | 정상 작동 |
| **Claude 앱** | iOS/Android 클로드 앱에서 직접 접속 | 아직 미지원 가능성 (조만간 업데이트 예정) |

QR코드로 Claude 앱에서 바로 접속하는 기능은 2026년 2월 시점에서 아직 작동하지 않을 수 있다. 한국 리전 배포 지연 때문일 가능성이 있으며, 웹 브라우저를 통한 접속은 문제 없이 동작한다.

URL을 복사하지 않아도 claude.ai 웹에 같은 계정으로 로그인하면 왼쪽 사이드바에 리모트 세션 목록이 표시된다. 거기서 클릭만으로 접속할 수 있다.

### 세션 이름 변경

여러 세션을 동시에 리모트로 열어놓을 때 구분이 필요하다면, `/rename`으로 세션 이름을 먼저 변경한 뒤 `/remote-control`을 실행하면 된다.

## 기존 Claude Code Web과의 차이

| 항목 | Remote Control | Claude Code Web |
|------|---------------|-----------------|
| **실행 위치** | 로컬 기기 | Anthropic 인프라 |

## 설정

매번 `--remote-control` 플래그를 붙이거나 `/remote-control` 커맨드를 치는 게 번거롭다면, `/config`에서 모든 세션에 자동 적용되도록 설정할 수 있다.

```
/config
→ remote_control = true
```

이렇게 설정하면 Claude Code를 실행할 때마다 자동으로 리모트 세션이 열린다. 외부에서 갑자기 접속하고 싶을 때를 대비해 항상 켜두는 것도 괜찮은 선택이다.

## 제약사항

Remote Control은 몇 가지 제약이 있다.

- **플랜 요구사항**: Pro 또는 Max 플랜이 필요하다. Team이나 Enterprise 플랜은 아직 미지원이며, 추후 지원 예정이다.
- **1세션 1커넥션**: 하나의 Claude Code 세션에는 하나의 리모트 커넥션만 연결할 수 있다. 다만 여러 세션을 동시에 열어서 각각 리모트로 연결하는 것은 가능하다.
- **터미널 유지 필수**: 로컬 터미널이 열려 있어야 한다. 컴퓨터가 슬립 모드에 들어가면 세션이 끊기지만, 컴퓨터가 다시 깨어나면 자동으로 세션이 복원된다.
- **10분 네트워크 타임아웃**: 네트워크 응답이 10분 이상 없으면 세션이 자동 종료된다. 이 경우 다시 실행해야 한다.
- **계정 일치 필수**: 로컬 Claude Code에 로그인한 계정과 웹/앱에서 접속하는 계정이 반드시 같아야 한다.

## 보안

Remote Control의 보안 모델은 비교적 단순하다. 로컬 Claude Code 세션은 **아웃바운드 HTTP 요청만** 수행하며, 인바운드 포트를 로컬 기기에서 열지 않는다. 외부에서 로컬 기기로 직접 들어오는 연결이 없기 때문에, Tailscale이나 SSH 터널링 같은 별도 네트워크 설정 없이도 안전하게 사용할 수 있다.

웹과 모바일 인터페이스는 로컬 세션의 "창구" 역할만 한다. 모든 실행은 로컬에서 이루어지며, 원격 인터페이스는 입출력을 중계하는 매개체일 뿐이다.

## 기존 모바일 원격개발 가이드(04번)와 비교

`04_모바일_원격개발_가이드.md`는 Tailscale VPN + SSH + tmux 조합으로 모바일에서 Claude Code에 접근하는 서드파티 우회 방법을 다뤘다. Remote Control이 나오기 전에는 이것이 유일한 방법이었다.

|------|---------------------|----------------------|
| **설정 난이도** | Tailscale + SSH + Termius 설치 필요 | 플래그 하나 또는 슬래시 커맨드 하나 |
| **인프라 비용** | Vultr VM 시간당 $0.29, 24시간 가동 시 월 ~$210 | 추가 비용 없음 (Pro/Max 구독만) |
| **네트워크 설정** | VPN 구성, nftables, fail2ban 등 보안 레이어 필요 | 별도 설정 없음 |
| **접속 방식** | SSH 클라이언트 필요 | 웹 브라우저만 있으면 됨 |
| **병렬 에이전트** | tmux + Git worktree로 최대 6개 | 로컬에서 여러 세션을 열면 가능 |
| **알림 시스템** | Poke 웹훅으로 푸시 알림 가능 | 웹 인터페이스에서 직접 확인 |
| **장점** | VM 기반이라 로컬 컴퓨터 꺼도 됨 | 설정이 간단하고 추가 비용 없음 |

대부분의 사용자에게는 Remote Control이 더 간편하고 비용 효율적이다. 다만 로컬 컴퓨터를 항상 켜둘 수 없거나, 알림 시스템이 필요하거나, 완전한 비동기 에이전트 운영이 필요한 경우에는 여전히 VM 기반 방식이 유리할 수 있다.

## 활용 시나리오

Remote Control은 로컬 환경을 그대로 원격에서 접근할 수 있기 때문에, 단순히 "모바일에서 코딩"하는 것 이상의 활용이 가능하다.

### 모바일 코드 리뷰 및 모니터링

이동 중에 빌드 상태를 확인하거나, 에이전트가 수행 중인 작업을 감시할 수 있다. 로컬에서 장시간 작업을 시켜놓고 외출한 뒤, 모바일로 진행 상황을 체크하는 패턴이 자연스럽게 가능해진다.

### 비동기 에이전트 관리

로컬에서 Claude Code 에이전트를 띄워놓고 외출한 뒤, 모바일로 접속해서 진행 상황을 확인하고 추가 지시를 내릴 수 있다. 에이전트가 질문을 하며 대기 중일 때 모바일로 답변해주는 것도 가능하다.

### 학습 스케줄러 원격 관리

기존에 설정해둔 learning-scheduler CLI를 모바일에서도 조회하고 실행할 수 있다. 로컬 환경이 그대로 노출되기 때문에, 터미널에서 할 수 있는 것은 리모트에서도 동일하게 할 수 있다.

### MCP 서버 원격 활용

로컬에 설정된 MCP 서버(파일시스템, 브라우저 자동화, LSP 등)를 모바일에서도 접근할 수 있다. 파일 검색, 코드 분석, 웹 자동화 등 로컬 MCP 도구의 전체 기능을 원격에서 활용 가능하다.

### 멀티 디바이스 개발

데스크탑에서 시작한 작업을 태블릿이나 다른 노트북에서 이어서 진행할 수 있다. 세션이 로컬에서 유지되고 있으므로, 어떤 기기에서 접속하든 동일한 컨텍스트를 이어받는다.

## FAQ

### 모바일에서 쓸 수 있는 기기는?

iOS, Android, 태블릿, 웹 브라우저를 지원하는 모든 기기에서 사용 가능하다. 같은 Anthropic 계정으로 로그인하는 것이 유일한 조건이다.

### 일정관리 앱처럼 쓸 수 있을까?

Claude Code는 범용 AI 코딩 에이전트이므로 전용 일정관리 앱(캘린더, 리마인더 등)과는 성격이 다르다. 하지만 MCP 서버와 커스텀 스킬을 활용하면 채팅 인터페이스를 통한 일정 조회, 등록, 수정은 충분히 가능하다. 기존 learning-scheduler CLI가 좋은 예시인데, "오늘 학습 스케줄 알려줘"같은 자연어 명령으로 로컬 스케줄러를 조작할 수 있다. 다만 푸시 알림이나 위젯 같은 네이티브 앱 기능은 제공하지 않으므로, "채팅으로 물어봐야 답해주는" 방식이라는 점을 이해해야 한다.

### 슬래시 커맨드가 자동 완성되나?

웹 인터페이스에서는 슬래시 커맨드 자동 완성이 지원되지 않는다. 하지만 직접 타이핑해서 입력하면 정상적으로 실행된다.
