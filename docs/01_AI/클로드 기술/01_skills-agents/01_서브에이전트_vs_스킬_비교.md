# Claude Code: 서브에이전트 vs 스킬 비교

> Claude Code의 두 가지 기능 확장 메커니즘 비교 분석

## 개요

Claude Code에서 AI의 기능을 확장하는 두 가지 주요 방법:
- **서브에이전트 (Subagent)**: 작업을 위임받는 독립적인 AI 비서
- **스킬 (Skill)**: 작업 수행 시 참조하는 지침/자산 모음

---

## 1. 서브에이전트 (Subagent)

### 정의

> **메인 에이전트가 작업을 위임할 수 있는 독립적인 AI 비서**

- 각 서브에이전트는 고유한 이름과 목적을 가짐
- 특정 역할/도메인에 특화된 전문가 에이전트
- 별도의 컨텍스트에서 독립적으로 실행

### 비유

```
서브에이전트 = "팀원"

┌─────────────────────────────────────────┐
│           메인 에이전트 (팀장)           │
│                   │                     │
│     ┌─────────────┼─────────────┐       │
│     ↓             ↓             ↓       │
│ ┌───────┐   ┌───────┐   ┌───────┐       │
│ │리서치  │   │ 코드  │   │ 리뷰  │       │
│ │ 팀원  │   │ 팀원  │   │ 팀원  │       │
│ └───────┘   └───────┘   └───────┘       │
│     ↓             ↓             ↓       │
│     └─────────────┼─────────────┘       │
│                   ↓                     │
│             결과 통합                    │
└─────────────────────────────────────────┘
```

### 사용 시기

| ✅ 사용하면 좋은 경우 |
|---------------------|
| 대량의 파일/문서를 분석해야 할 때 |
| 병렬 처리로 속도를 높이고 싶을 때 |
| 특정 전문 분야의 깊은 분석이 필요할 때 |
| 메인 에이전트의 컨텍스트를 절약하고 싶을 때 |

### 설정 방법

**위치**: `.claude/agents/[agent-name].md`

```yaml
---
name: code-reviewer
description: 코드 품질, 보안, 유지보수성 검토 전문가
tools: Read, Grep, Glob, Bash
model: inherit
---

# 코드 리뷰어

## 역할
코드 변경사항을 분석하고 품질/보안/유지보수성 관점에서 리뷰

## 체크리스트
1. 코딩 컨벤션 준수
2. 보안 취약점 확인
3. 성능 이슈 검토
4. 테스트 커버리지 확인
```

### 동작 예시

```
[사용자] "이 프로젝트의 인증 시스템을 분석해줘"

[메인 에이전트]
├─ research-agent 호출 → 인증 관련 50개 파일 탐색
├─ code-analyzer 호출 → 코드 구조 분석
└─ security-reviewer 호출 → 보안 취약점 검토

[병렬 실행 후 결과 통합]

[최종] 종합 분석 보고서 생성
```

## 2. 스킬 (Skill)

### 정의

> **Claude가 특정 작업 수행 시 참조하는 지침과 자산의 모음**

- 업무 매뉴얼/SOP와 유사한 개념
- 지침, 코드, 참고 자료를 폴더로 묶어 관리
- Claude가 필요 시 자동으로 발견하고 적용

### 비유

```
스킬 = "매뉴얼/SOP"

┌─────────────────────────────────────────┐
│              Claude                      │
│                 │                        │
│                 ↓                        │
│    ┌────────────────────────┐           │
│    │      스킬 라이브러리     │           │
│    │  ┌──────┐ ┌──────┐     │           │
│    │  │커밋  │ │리뷰  │ ... │           │
│    │  │매뉴얼│ │매뉴얼│     │           │
│    │  └──────┘ └──────┘     │           │
│    └────────────────────────┘           │
│                 │                        │
│                 ↓                        │
│         매뉴얼 참조하여 작업             │
└─────────────────────────────────────────┘
```

### 사용 시기

| ✅ 사용하면 좋은 경우 |
|---------------------|
| 반복적인 작업을 표준화하고 싶을 때 |
| 팀 컨벤션/규칙을 적용하고 싶을 때 |
| 특정 작업의 품질을 일관되게 유지하고 싶을 때 |
| 지식/노하우를 재사용하고 싶을 때 |

### 설정 방법

**위치**: `.claude/skills/[skill-name]/SKILL.md`

```
.claude/skills/commit-helper/
├── SKILL.md              # 핵심 지침 (필수)
├── SECURITY.md           # 보안 체크리스트 (선택)
├── STYLE.md              # 스타일 가이드 (선택)
└── scripts/              # 지원 스크립트 (선택)
    └── run-linters.sh
```

**SKILL.md 예시**:

```markdown
---
name: commit-helper
description: 팀 컨벤션에 맞는 커밋 메시지 생성
---

# 커밋 메시지 생성기

## 규칙
1. `git diff --staged`로 변경사항 분석
2. 이모지 prefix 사용
3. 한국어로 작성
4. 50자 이내 제목

## Prefix
- ✨ feat: 새 기능
- 🐛 fix: 버그 수정
- 📝 docs: 문서
- ♻️ refactor: 리팩토링
- ✅ test: 테스트
- 🔧 chore: 설정

## 예시
```
✨ feat: 사용자 인증 미들웨어 추가

- JWT 토큰 검증 구현
- 역할 기반 접근 제어 추가

Closes #123
```
```

### 동작 예시

```
[사용자] "커밋 메시지 작성해줘"

[Claude]
├─ .claude/skills/commit-helper/ 자동 발견
├─ SKILL.md 로드
├─ git diff --staged 실행
└─ 스킬 지침에 따라 메시지 생성

[결과]
✨ feat: 사용자 인증 미들웨어 추가

- JWT 토큰 검증 구현
- 역할 기반 접근 제어 추가

Closes #123
```

## 3. 핵심 비교

### 비교 테이블

| 항목 | 서브에이전트 | 스킬 |
|------|-------------|------|
| **본질** | 독립적인 AI 비서 | 지침/자산 모음 |
| **비유** | 팀원 | 매뉴얼/SOP |
| **위치** | `.claude/agents/` | `.claude/skills/` |
| **핵심 파일** | `[name].md` | `SKILL.md` |
| **호출 방식** | 메인 에이전트가 명시적 호출 | Claude가 자동 발견 |
| **실행 주체** | 서브에이전트 자체 | Claude (스킬 참조) |
| **컨텍스트** | 별도 (분리됨) | Claude 내 로드 |
| **병렬 처리** | ✅ 가능 | ❌ 불가 |

### 선택 가이드

```
┌─────────────────────────────────────────────────────────┐
│                    무엇을 선택할까?                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  "작업을 위임하고 싶다" → 서브에이전트                   │
│  ────────────────────────────────────                   │
│  • 대량 파일 분석                                        │
│  • 병렬 처리 필요                                        │
│  • 전문 분야 깊은 분석                                   │
│  • 컨텍스트 절약                                         │
│                                                         │
│  "규칙/절차를 적용하고 싶다" → 스킬                      │
│  ──────────────────────────────                         │
│  • 반복 작업 표준화                                      │
│  • 팀 컨벤션 적용                                        │
│  • 품질 일관성 유지                                      │
│  • 지식 재사용                                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 4. 실제 활용 예시

### 서브에이전트 활용

| 에이전트 | 역할 |
|---------|------|
| `research-agent` | 코드베이스 탐색, 관련 파일 발견 |
| `code-reviewer` | 코드 품질/보안/유지보수성 검토 |
| `test-generator` | 테스트 코드 생성 |
| `doc-writer` | 문서 작성 |
| `security-auditor` | 보안 취약점 분석 |

### 스킬 활용

| 스킬 | 용도 |
|------|------|
| `commit-helper` | 팀 컨벤션에 맞는 커밋 메시지 생성 |
| `code-review-guide` | 코드 리뷰 체크리스트 적용 |
| `deploy-checklist` | 배포 전 검증 항목 확인 |
| `api-doc-generator` | API 문서 형식 표준화 |
| `test-template` | 테스트 코드 템플릿 적용 |

## 5. 설정 퀵스타트

### 서브에이전트 생성

```bash
# 1. 폴더 생성
mkdir -p .claude/agents

# 2. 에이전트 정의 파일 생성
cat > .claude/agents/research-agent.md << 'EOF'
---
name: research-agent
description: 코드베이스 탐색 및 분석 전문가
tools: Read, Grep, Glob
model: inherit
---

# 리서치 에이전트

## 역할
- 대규모 코드베이스에서 관련 파일 탐색
- 코드 구조 및 패턴 분석
- 발견사항 요약 보고

## 출력 형식
- 관련 파일 목록
- 주요 발견사항
- 추천 사항
EOF
```

### 스킬 생성

```bash
# 1. 폴더 생성
mkdir -p .claude/skills/commit-helper

# 2. SKILL.md 작성
cat > .claude/skills/commit-helper/SKILL.md << 'EOF'
---
name: commit-helper
description: 팀 컨벤션에 맞는 커밋 메시지 생성
---

# 커밋 메시지 생성기

## 규칙
1. git diff --staged로 변경사항 분석
2. 이모지 prefix 사용
3. 한국어로 작성
4. 50자 이내 제목

## Prefix
- ✨ feat: 새 기능
- 🐛 fix: 버그 수정
- 📝 docs: 문서
EOF
```

## 6. 함께 사용하기

서브에이전트와 스킬은 **상호 보완적**으로 사용할 수 있습니다:

```
┌─────────────────────────────────────────────────────────┐
│                  복합 워크플로우 예시                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [사용자] "이 기능을 리뷰하고 커밋해줘"                  │
│                                                         │
│  [메인 에이전트]                                         │
│  ├─ code-reviewer (서브에이전트) 호출                   │
│  │   └─ 코드 품질/보안 분석 수행                        │
│  │                                                      │
│  ├─ 리뷰 결과 수신                                      │
│  │                                                      │
│  └─ commit-helper (스킬) 적용                           │
│      └─ 팀 컨벤션에 맞는 커밋 메시지 생성               │
│                                                         │
│  [결과] 리뷰 완료 + 커밋 메시지 생성                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 요약

| 구분 | 서브에이전트 | 스킬 |
|------|-------------|------|
| **한 줄 정의** | 작업을 위임받는 전문가 팀원 | 작업 수행 시 참조하는 매뉴얼 |
| **핵심 장점** | 병렬 처리, 컨텍스트 분리 | 표준화, 일관성, 재사용 |
| **설정 위치** | `.claude/agents/` | `.claude/skills/` |

**선택 기준**:
- 복잡한 분석/병렬 처리 → **서브에이전트**
- 반복 작업 표준화 → **스킬**
