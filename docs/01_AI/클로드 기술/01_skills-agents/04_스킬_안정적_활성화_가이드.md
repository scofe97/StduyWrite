# Claude Code Skills 안정적 활성화 가이드

## 개요

Claude Code skills는 설명(description) 기반으로 자동 활성화되어야 하지만, 실제로는 **20%** 성공률에 불과합니다. 200+ 테스트를 통해 **80-84%** 성공률을 달성하는 두 가지 방법을 발견했습니다.

## 문제점

- Skills가 "자율적으로" 활성화된다고 문서에 명시되어 있음
- 실제로는 Claude가 skills를 무시하고 직접 구현을 시작함
- 단순 instruction hook: **약 20% 성공률** (동전 던지기 수준)

## 테스트 환경

### 테스트 대상 Skills (SvelteKit 예시)
| Skill | 용도 |
|-------|------|
| svelte5-runes | $state, $derived, $effect 등 runes 시스템 |
| sveltekit-data-flow | 데이터 로딩, form actions, load 함수 |
| sveltekit-structure | 파일 기반 라우팅, 레이아웃, SSR |
| sveltekit-remote-functions | query(), command() 원격 함수 |

### 테스트 프롬프트 유형
1. **Form/Route Creation** - 다중 skill 필요 (가장 어려움)
2. **Data Loading** - 단일 skill
3. **Server Actions** - 2개 skill
4. **Remote Functions** - 단일 skill
5. **Svelte 5 Runes** - 단일 skill

## 4가지 Hook 유형 테스트 결과

| Hook 유형 | 성공률 | 특징 |
|-----------|--------|------|
| No hook | ~10% | 기준선 |
| **Simple instruction** | **20%** | 단순 지시 - 무시됨 |
| **Forced eval** | **84%** | 명시적 평가 강제 |
| **LLM eval** | **80%** | API 사전 평가 |

### 상세 결과 (프롬프트별)

| 프롬프트 유형 | Simple | LLM-Eval | Forced |
|---------------|--------|----------|--------|
| Form/Route Creation | 0% | 0% | 80% |
| Data Loading | 0% | 100% | 100% |
| Server Actions | 10% | 100% | 40% |
| Remote Functions | 90% | 100% | 100% |
| Svelte 5 Runes | 100% | 100% | 100% |

## 권장 방법 1: Forced Eval Hook (84%)

### 작동 원리

**Simple instruction이 실패하는 이유**: 수동적 제안 → Claude가 무시

**Forced eval이 성공하는 이유**: 3단계 강제 프로세스

```
Step 1 - EVALUATE: 각 skill에 대해 YES/NO 명시적 평가
Step 2 - ACTIVATE: Skill() 도구로 즉시 활성화
Step 3 - IMPLEMENT: 활성화 후에만 구현 시작

CRITICAL: 평가만 하고 활성화하지 않으면 무의미함
```

### 핵심 포인트
- **Commitment mechanism**: Claude가 "YES - 필요함"이라고 작성하면 활성화해야 함
- **공격적 언어**: "MANDATORY", "WORTHLESS", "CRITICAL" 등 강한 표현 사용
- **순차적 강제**: 평가 → 활성화 → 구현 순서 강제

### 설정 방법

1. Hook 스크립트 생성: `.claude/hooks/skill-forced-eval-hook.sh`
2. 실행 권한 부여: `chmod +x skill-forced-eval-hook.sh`
3. settings.json 설정:

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/skill-forced-eval-hook.sh"
          }
        ]
      }
    ]
  }
}
```

## 권장 방법 2: LLM Eval Hook (80%)

### 작동 원리
- Claude API를 사용해 프롬프트 전달 전 어떤 skills가 매칭되는지 사전 평가
- 비용: 프롬프트당 약 $0.0004 (Haiku 4.5 기준)

### 장점
- 10% 저렴 ($0.0606 vs $0.0673)
- 17% 빠름 (5.0s vs 5.4s)
- "스마트"한 관련 skill 추가 가능

### 단점
- **완전 실패 가능**: Form/Route Creation에서 0% (10회 모두 실패)
- API 키 및 외부 호출 필요
- 가변적 결과

## 비용/성능 비교 (Haiku 4.5 기준)

| Hook 유형 | 성공률 | 총 비용 | 테스트당 비용 | 평균 시간 |
|-----------|--------|---------|---------------|-----------|
| Forced | 84% | $0.3367 | $0.0067 | 7.2s |
| LLM-Eval | 80% | $0.3030 | $0.0061 | 6.0s |
| Simple | 20% | $0.2908 | $0.0058 | 6.7s |

## 선택 가이드

### Forced Eval 선택 시
- 가장 일관된 활성화 원할 때 (84%)
- 상세한 출력(verbose) 허용 가능할 때
- 순수 클라이언트 솔루션 원할 때 (API 호출 없음)

### LLM Eval 선택 시
- 비용/속도 최적화 원할 때
- 단순한 프롬프트 (단일 skill) 위주일 때
- 가끔의 완전 실패 허용 가능할 때
- Anthropic API 키 설정되어 있을 때

## 핵심 교훈

1. **Skills는 자동 활성화되지 않음** - 문서와 달리 실제로는 무시됨
2. **단순 instruction은 무의미** - 20% 성공률
3. **Commitment mechanism이 핵심** - 명시적 평가 + 활성화 강제
4. **84% vs 20%** - verbose 출력이 트레이드오프지만 충분히 가치 있음
