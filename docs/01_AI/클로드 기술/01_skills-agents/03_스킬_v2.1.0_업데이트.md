## 핵심 요약

Claude Code v2.1.0은 **1,096건의 커밋**이 반영된 대규모 업데이트로, 에이전트 생명주기 제어, 스킬 재사용성, 세션 이동성, 다국어 출력 등 인프라 수준의 기능이 대거 강화되었다.

---

## 스킬(Skills) - 이번 업데이트의 핵심

### 스킬이란?

**"폴더 단위의 능력 확장 시스템"**으로, Markdown 파일과 선택적 스크립트로 구성된 Claude의 능력을 확장하는 방법이다.

```
skills/
├── my-skill/
│   ├── SKILL.md          # 핵심 지침 (필수)
│   ├── config.yaml       # 메타데이터 (선택)
│   └── scripts/          # 실행 스크립트 (선택)
│       └── process.py
```

### 스킬 vs 기존 커스텀 커맨드

| 구분 | 커스텀 커맨드 | 스킬 |
|------|-------------|------|
| 구조 | 단일 Markdown | 폴더 + 여러 파일 |
| 스크립트 실행 | 불가 | **Python/JS 실행 가능** |
| 토큰 로딩 | 전체 로드 | 메타데이터만 로드 → 필요 시 전체 |
| 재사용성 | 프로젝트 단위 | 전역 사용 가능 |

### 토큰 효율성 (핵심 장점)

- 세션 시작 시 각 스킬의 **메타데이터만 수십 토큰으로 로드**
- 실제 필요할 때만 전체 내용을 불러옴
- MCP의 토큰 소모 문제를 해결

### 스킬 핫 리로드

세션 재시작 없이 **실시간으로 스킬 로직 수정 가능**
- 개발 중 빠른 테스트 가능
- 반복 작업 효율 극대화

## 주요 신규 기능

### 1. 에이전트/스킬 훅(Hook) 시스템

```
도구 사용 전 → Hook 실행 → 도구 실행 → Hook 실행 → 완료
```

- 상태 관리, 도구 제약, 감사 로깅을 세밀하게 제어
- 슬래시 명령어에도 훅 적용 가능

### 2. 포크된 서브 에이전트 컨텍스트

- 메인 에이전트 상태를 오염시키지 않고 **격리된 환경에서 테스트**
- 안전한 실험 환경 제공

### 3. 세션 텔레포테이션

- 터미널 ↔ 브라우저 간 세션 이동 가능
- 작업 연속성 보장

### 4. 비동기 서브 에이전트

- 병렬 태스크 실행으로 작업 속도 향상
- 독립적인 작업을 동시에 처리

## 개발 편의성 개선

| 기능 | 설명 |
|------|------|
| **Shift+Enter** | iTerm2, Kitty, Ghostty, WezTerm에서 별도 설정 없이 작동 |
| **Ctrl+B** | 에이전트와 셸 명령 동시 백그라운드 처리 |
| **Vim 모션** | 마우스 없이 빠른 작업 가능 |
| **/plan 명령어** | 플랜 모드 즉시 활성화 |
| **자동완성 개선** | 슬래시 명령어가 입력 중 어디서든 작동 |

## 스킬 생성 방법

### 1. 스킬 크리에이터 사용 (권장)

```
/skill-creator
```

대화형으로 워크플로를 물어보고 필요한 구조를 자동 생성

### 2. 수동 생성

```bash
mkdir -p .claude/skills/my-skill
touch .claude/skills/my-skill/SKILL.md
```

### 스킬 구조 예시

```markdown
# SKILL.md

## 트리거
- "보고서 생성", "리포트 만들어"

## 작업 지침
1. 데이터 수집
2. 분석 수행
3. Markdown 형식으로 출력

## 출력 형식
- 표 형식 사용
- 요약 먼저, 상세 내용은 접기 처리
```

## MCP vs Skills 비교

| 구분 | MCP | Skills |
|------|-----|--------|
| 복잡도 | 프로토콜 기반, 복잡 | **Markdown/YAML 기반, 단순** |
| 토큰 사용 | 항상 전체 로드 | 필요 시에만 로드 |
| 이식성 | Claude 전용 | 다른 모델에서도 활용 가능 |
| 공유 | 서버 설정 필요 | 파일 복사만으로 공유 |

## 버그 수정 (45건 이상)

- **보안**: 디버그 로그에 OAuth 토큰, API 키, 비밀번호 노출 문제 수정
