## 실제 스킬 활용 사례

### 영상 편집 자동화 (노토키 팀 - 1등 수상)

**문제**: 유튜브 영상 편집이 노동 집약적
- 여러 번 반복 촬영 중 잘한 부분만 남기기
- 무음 구간 제거
- 중복 발언 제거

**해결책: 2개의 스킬 개발**

#### 스킬 1: 무음 구간 자르기
- FFmpeg 바이너리를 활용
- 데시벨이 낮은 구간 자동 감지 및 제거
- 자막이 인식되지 않은 구간 제거

#### 스킬 2: 자막 기반 중복 제거
- 자막 분석으로 중복 발언 감지
- "안녕하세요. 네이버..."를 여러 번 말한 경우 → 자동으로 중복 제거

**결과**: 2분 촬영 → 약 30초로 자동 편집

### 9쿠 가게 자동 전화 에이전트 (1등)
- 가게에 자동으로 전화
- "얼마나 남았는지" 자료 조사
- 에이전트 스킬로 구현

### Agentic Workflow 빌더 (빌더 조시)
- 에이전틱 워크플로우를 빌드하는 스킬

## 기술 심화: Claude Skill 작성법

> 참고: [Claude Skills 공식 문서](https://code.claude.com/docs/ko/skills)

### SKILL.md 기본 구조

모든 스킬은 `SKILL.md` 파일이 필수입니다. YAML 프론트매터 + 마크다운 본문으로 구성됩니다.

```markdown
---
name: video-silence-remover
description: |
  영상에서 무음 구간을 자동으로 감지하고 제거하는 스킬.
  유튜브 편집 시 사용. FFmpeg 설치 필요.
---

# Video Silence Remover

## 실행 조건
- 입력: 영상 파일 경로
- 출력: 무음 구간 제거된 영상

## 처리 단계
1. FFmpeg silencedetect로 무음 구간 감지
2. 타임스탬프 추출
3. 무음 구간 제거 후 영상 재조합

## 예시
사용자: "이 영상에서 무음 구간 제거해줘"
→ 스킬 자동 활성화
```

### 핵심 필드 설명

| 필드 | 설명 |
|------|------|
| `name` | 소문자, 하이픈만 사용. 디렉터리명과 일치 권장 |
| `description` | **가장 중요**. 무엇을, 언제, 언제 쓰지 말지까지 명시 |
| `allowed-tools` | 스킬 활성화 시 사용 가능한 도구 제한 |

### 스킬 저장 위치

```
~/.config/claude/skills/     # 사용자 전역 스킬
.claude/skills/              # 프로젝트별 스킬
```

### 쉽게 만들기

```bash
# Claude Code에서 skill-creator 사용
/skill-creator
```

## 기술 심화: FFmpeg 무음 구간 제거

> 참고: [silencecut-ffmpeg](https://github.com/beenotung/silencecut-ffmpeg)

### FFmpeg silencedetect 원리

영상에서 무음 구간을 한 번에 제거하는 것은 불가능합니다. **2단계 접근**이 필요합니다.

#### Step 1: 무음 구간 감지

```bash
ffmpeg -i input.mp4 \
  -af silencedetect=n=-35dB:d=0.8 \
  -f null - 2>&1 | grep silencedetect
```

| 파라미터 | 설명 |
|----------|------|
| `n=-35dB` | 무음 기준 데시벨 (조용한 환경: -50dB, 시끄러운 환경: -30dB) |
| `d=0.8` | 최소 무음 지속 시간 (0.8초 이상 무음이면 감지) |

#### Step 2: 무음 구간 제거 (Jump Cuts)

감지된 타임스탬프로 영상 분할 후 재조합합니다.

```bash
# Python 스크립트로 자동화
# 1. silencedetect 출력 파싱
# 2. 비무음 구간 추출
# 3. ffmpeg concat으로 재조합
```

### 자동화 도구

- **silencecut-ffmpeg**: `npx silencecut-ffmpeg input.mp4 output.mp4`
- **Silence Speedup**: Electron 기반 GUI 앱
- **final-cut-it-out**: Final Cut Pro X 전용

## 기술 심화: Ralph Loop (자율 실행)

> 참고: [The Ralph Playbook](https://claytonfarr.github.io/ralph-playbook/)

### Ralph Loop란?

Ralph는 Claude를 **자율 반복 실행**하는 방법론입니다. 파일 기반 상태 관리로 컨텍스트를 유지하며 PRD가 완료될 때까지 반복합니다.

### 실행 패턴

```
읽기(plan) → 태스크 선택 → 구현 → 테스트 → 커밋 → 컨텍스트 초기화 → 반복
```

### 왜 작동하는가?

1. **Fresh Context**: 매 반복마다 새 컨텍스트로 AI "스마트 존" 유지
2. **File-based Memory**: specs, plan, agents 파일로 학습 지속
3. **Backpressure**: 테스트, 빌드 실패 시 자동 수정

### 주요 구현체

| 이름 | 특징 |
|------|------|
| [snarktank/ralph](https://github.com/snarktank/ralph) | PRD 완료까지 자동 반복, progress.txt로 상태 관리 |
| [frankbria/ralph-claude-code](https://github.com/frankbria/ralph-claude-code) | 지능형 종료 감지 포함 |
| [Ralphy](https://github.com/michaelshimeles/ralphy) | Claude, Codex, OpenCode, Cursor 등 다중 도구 지원 |

### 보안 주의사항

Ralph는 `--dangerously-skip-permissions` 플래그가 필요합니다. **반드시 샌드박스 환경에서 실행**하세요.

```bash
# Docker 샌드박스 권장
# 또는 Fly Sprites, E2B 등 원격 격리 환경
```

## 기술 심화: Oh My OpenCode

> 참고: [GitHub - oh-my-opencode](https://github.com/code-yeongyu/oh-my-opencode)

### 개요

Oh My OpenCode는 OpenCode를 위한 오케스트레이션 레이어입니다. Claude Code 호환성을 제공하며 hooks, commands, skills, agents, MCP를 지원합니다.

### 핵심 기능

- **Sisyphus Agent**: 메인 에이전트 컨텍스트를 가볍게 유지하며, 백그라운드에서 더 빠르고 저렴한 모델로 병렬 탐색
- **LSP 활용 리팩토링**: 결정론적이고 안전한 수술적 코드 변경
- **내장 MCP**: Exa (웹검색), Context7 (문서), grep_app (GitHub 검색)

### 특징

> "Claude Code is great. But if you're a hacker, you'll fall head over heels for OpenCode."

- 화면 깜빡임 없음
- 파일별 LSP, 린터, 포매터 자동 활성화
- 모델 믹스 앤 매치 가능

## 기술 심화: 디자인 시스템 + LLM

> 참고: [Claude For Code - Design Process](https://uxplanet.org/claude-for-code-how-to-use-claude-to-streamline-product-design-process-97d4e4c43ca4)

### 왜 디자인 시스템이 중요한가?

LLM은 일관되지 않은 디자인을 생성하는 경향이 있습니다. 이를 방지하려면 **디자인 시스템을 명시적으로 정의**해야 합니다.

### 권장 구조

```
design-system/
├── design-system.md       # 전체 가이드라인
├── colors.json           # 색상 팔레트
├── typography.json       # 폰트 시스템
├── spacing.json          # 간격 규칙
└── components.json       # 컴포넌트 명세
```

### 예시: colors.json

```json
{
  "primary": {
    "50": "#eff6ff",
    "500": "#3b82f6",
    "900": "#1e3a8a"
  },
  "semantic": {
    "success": "#22c55e",
    "error": "#ef4444",
    "warning": "#f59e0b"
  }
}
```

### Claude 활용법

1. **스타일가이드 → JSON 변환**: Claude가 디자인 명세를 JSON, CSS 변수, Tailwind 설정으로 변환
2. **UI 스크린샷 → 코드**: 스크린샷에서 HTML + TailwindCSS 프로토타입 생성
3. **일관성 유지**: 디자인 시스템 파일을 항상 참조하도록 설정

### 주의사항

Tailwind CSS 창시자 Adam Wathan의 경험에 따르면, 600개 컴포넌트에 다크 모드를 추가하는 작업에서 Claude Code가 LLM 없이 작업하는 것보다 **50% 느렸다**는 결과도 있습니다. 복잡한 반복 작업에서는 LLM이 항상 효율적이지 않을 수 있습니다.

## 참가자 인사이트

### 스킬 활용의 핵심

> "스킬 만드는 스킬을 가장 많이 쓴다"

- 스킬을 만들어두면 활용하게 됨
- 한번 써보고 "되네!" 확인 후 방치하는 경우도 많음
- **목표: 완성** (완성만 해도 큰 성과)

### 개발 생태계 변화

1. **빠른 제작**: 3시간 안에 모든 팀이 결과물 완성
2. **비개발자도 참여**: 개발자가 아닌 분들도 스킬로 아웃풋 생성
3. **코드를 안 보는 시대**: 어셈블리 → C → Python → Claude Code
4. **차세대 앱스토어**: 코드가 아닌 프롬프트(자연어)로 작성된 앱

### 디자이너의 AI 활용 팁 (빌더 조시)

**레퍼런스 선별이 핵심**

1. **좋은 레퍼런스 소스**
   - Framer: 잘 나가는 디자인 도구
   - Dribbble: 디자이너 인정 커뮤니티 (가입 제한으로 퀄리티 높음)
   - Mobbin: 웹사이트 레퍼런스

2. **원하는 스타일 찾기**
   - Pinterest 이미지 서치 기능 활용
   - 키워드 입력 → 이미지셋 → 유사 이미지 탐색으로 좁혀나가기

3. **일관된 디자인 유지**
   - 디자인 시스템 파일 (`.md`) 작성
   - JSON 파일로 쪼개서 LLM이 항상 참조하게 설정

4. **폰트 추천**
   - 한글: Pretendard 같은 검증된 폰트 사용

## 주최자 인사이트 (구봉 님)

### 스킬톤 기획 이유

1. **빠른 제작 가능**: 다른 해커톤과 달리 짧은 시간에도 임팩트 있게 완성
2. **다양한 영역 활용**: 마케팅, 비디오 생성, 개발 등
3. **생태계 빠른 성장**: 앱스토어에 가까운 느낌

### 향후 계획: 랄프 해커톤

- **콘셉트**: 1박 2일 해커톤 + 수면
- 랄프(자율 실행)를 돌려놓고 잠을 자는 해커톤
- 노트북 터치 금지, 다음날 아침에 결과 확인
- "얘가 다 돌아가게 하고 나는 잔다"
