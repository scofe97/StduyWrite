# 멀티 에이전트 동시 운영 전략

> 탑레벨 개발자들의 에이전트 활용법: 5개 이상 동시 운영

---

## 핵심 요약

- 에이전트 1개 운영은 **초급 레벨**
- 탑레벨 개발자는 **5개 이상** 동시 운영
- 서브에이전트까지 활용하면 **최대 20개** 상시 운영 가능
- 핵심은 **업무 할당 능력**과 **코드 리뷰 속도**

---

## 에이전트 Role 분담 전략

### 팀 구조처럼 역할 부여

하나의 제품에 여러 에이전트가 개입할 때, 팀원에게 일을 분담하듯 역할을 나눈다.

| Role | 담당 영역 |
|------|----------|
| **UI** | 프론트엔드, 컴포넌트 |
| **Backend** | API, 서버 로직 |
| **QA** | 테스트, 품질 보증 |
| **DevOps** | 배포, 인프라 |
| **Security** | 보안 검토 |
| **Feature** | 특정 기능 개발 |

### QA 에이전트 세분화

QA 역할도 더 세분화하여 병렬 처리 가능:

```
QA Agent
├── Unit Test Agent
├── Integration Test Agent
├── E2E Test Agent
├── Device Test Agent (Android/iOS)
└── Cross-Browser Test Agent
```

## 동시성 관리 전략

### 전략 1: 공용 문서 업데이트 (미봉책)

```
project/
└── docs/
    ├── __feature_auth/     # 에이전트 작업 중
    ├── __feature_payment/  # 에이전트 작업 중
    └── timeline.md         # 업데이트 타임라인
```

- `docs/` 폴더에 참조 문서 배치
- `__` 접두사로 현재 작업 중인 피처 표시
- 타임라인 기준으로 변경 이력 관리

**단점**: 사이클이 복잡하고 불안정

### 전략 2: 프록시 문서 (중간 조정자)

스키마 충돌 등의 문제를 방지하기 위한 **이벤트 전파 방식**:

```
시나리오: User 테이블 동시 수정

Agent 2: User 테이블 수정 (8:00 PM)
    ↓ 변경 기록 남김
Proxy Doc: "8:00 PM - User 테이블 확장 (Agent 2)"
    ↓
Agent 1: 작업 시작 전 Proxy Doc 확인 (8:15 PM)
    → 마지막 체킹 포인트(4:00 PM) 이후 변경사항 반영
```

**장점**: 스키마 충돌, 마이그레이션 꼬임 방지

### 전략 3: 물리적 워크스페이스 분리 (권장)

기존 Git Strategy를 물리적으로 모사:

```
local/
├── project-agent1/  (feature/auth 브랜치)
├── project-agent2/  (feature/payment 브랜치)
├── project-agent3/  (feature/dashboard 브랜치)
└── main-repo/       (메인 브랜치)
```

**워크플로우**:
1. 프로젝트 클론을 에이전트별로 생성
2. 각 에이전트에 독립 브랜치 할당
3. 피처 구현 완료 시 즉시 메인 브랜치에 병합
4. 다른 에이전트는 Git 히스토리 체킹 후 최신화

**병합 타이밍**: 피처 구현 완료 즉시 (바로바로)

## 서브에이전트 활용

```
Main Agent (5개)
├── Agent 1 → Sub-agent (백그라운드 2-3개)
├── Agent 2 → Sub-agent (백그라운드 2-3개)
├── Agent 3 → Sub-agent (백그라운드 2-3개)
├── Agent 4 → Sub-agent (백그라운드 2-3개)
└── Agent 5 → Sub-agent (백그라운드 2-3개)

총 가동: 최대 20개 에이전트
```

## 필수 역량

### 1. 코드 리뷰 속도

- 에이전트 컨펌 사이클: **1분 ~ 5분**
- Diff 확인 후 빠른 판단 필요
- Source Control에서 변경사항 즉시 파악

### 2. 업무 할당 능력

- 교차 작업으로 인한 병목 최소화
- 롤(Role) 세분화로 충돌 방지
- 디테일한 임무 설계

|----------|------------|------|
| 초급 | 1개 | 대기 시간 발생 |
| 중급 | 3~5개 | 일반적인 리드 개발자 |
| 고급 | 5개 + 서브에이전트 | 최대 20개 동시 가동 |

## 핵심 마인드셋

> "에이전트 하나 돌리면서 열심히 개발한다고 말하는 건 군맹이처럼 일하는 것"

- 개발자 → **리드/지휘자** 역할 전환
- 코드 작성 → **코드 리뷰 & 의사결정**
- 혼자 개발 → **팀(에이전트) 관리**
