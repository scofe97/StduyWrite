## 핵심 요약
> Claude Code는 터미널에서 실행되는 AI 도구로, 기획부터 개발, 디버깅, 자동화까지 지원한다.
> 로컬 환경에서만 동작하여 보안이 우수하며, git과 bash에 제약 없이 접근 가능하다.

## 학습 목표
이 내용을 읽고 나면:
- [ ] Claude Code의 설치 및 기본 실행 방법을 알 수 있다
- [ ] 핵심 사용팁(컨텍스트 참조, 권한 설정 등)을 적용할 수 있다
- [ ] 커스텀 명령어, 서브에이전트, SuperClaude 등 고급 기능을 활용할 수 있다

## 1. 설치 및 실행

Claude Code는 npm을 통해 전역으로 설치하고, 프로젝트 디렉토리에서 실행한다.

```bash
# 설치
npm install -g @anthropic-ai/claude-code

# 프로젝트로 이동 후 실행
cd your-project
claude
```

> **TIP**: IntelliJ 사용자는 플러그인 설치로 enhanced diff 보기와 컨텍스트 공유 기능을 활용할 수 있다.

---

## 2. 핵심 사용팁

| 기능 | 사용법 | 설명 |
|------|--------|------|
| 초기 설정 | `/init` | CLAUDE.MD 파일 생성 |
| 권한 자동화 | `--dangerously-skip-permissions` | 권한 확인 건너뛰기 |
| 작업 초기화 | `/clear` | 새 작업 전 컨텍스트 정리 |
| 컨텍스트 참조 | `@파일명` | 필요한 파일/정보 포함 |
| 이미지 삽입 | `Ctrl+V` | 클립보드 이미지 추가 |
| 멀티 작업 | 순차적 입력 | 여러 명령어를 큐에 등록 |

> **비유**: `@` 기호는 SNS에서 누군가를 태그하는 것처럼, Claude에게 특정 파일을 "주목하게" 만드는 기능이다.

## 3. 설정 커스터마이징

### 3.1 CLAUDE.md 파일

프로젝트별 컨텍스트 문서화:
- 명령어, 스타일 가이드, 워크플로우 정의
- 팀 협업을 위해 git에 커밋 가능

### 3.2 허용 도구 관리

보안을 위해 사전 승인 가능한 도구 지정

### 3.3 MCP 연동

커스텀 도구 및 외부 서비스 통합

### 3.4 슬래시 명령어

반복되는 워크플로우를 템플릿화

```
.claude/commands/
├── generate-test.md      # 테스트 자동 생성
├── create-pr.md          # PR 생성 자동화
└── code-review.md        # 코드 리뷰 자동화
```

## 4. 효과적인 워크플로우 패턴

### 4.1 기본 패턴

```
탐색 → 계획 → 구현 → 커밋
```

### 4.2 TDD 패턴

테스트 작성 먼저, 그 다음 구현

### 4.3 시각적 반복

스크린샷 기반으로 UI 개선 반복

### 4.4 Git/GitHub 자동화

커밋, PR, 이슈 관리 자동화

## 9. 고급 기능

### 9.1 서브에이전트 (Sub-agents)

특정 작업을 위한 사전 구성된 AI 인격체로, 자체 컨텍스트와 전문 도구를 사용한다.

> **비유**: 서브에이전트는 회사에서 특정 업무를 담당하는 전문 팀원과 같다.

### 9.2 SuperClaude

19개의 특수 명령어와 9개 페르소나로 기능을 확장한다.

| 명령어 | 기능 |
|--------|------|
| `/sc:implement` | 기능 구현 |
| `/sc:improve` | 코드 개선 |
| `/sc:build` | 빌드 관리 |

### 9.3 시스템 알림

```bash
npm install -g claude-notify
```

작업 완료 시 macOS 알림 수신

## 10. 비용 최적화

- 권장 비용: **약 $0.5-0.75/작업**
- `/clear`로 불필요한 컨텍스트 정리
- 복잡한 작업은 계획 수립 후 구현

---

## 실무 적용 포인트

### 이런 상황에서 사용하세요
- 새 프로젝트 시작 시 `/init`으로 기본 설정
- 반복적인 작업은 커스텀 명령어로 자동화
- 복잡한 작업은 서브에이전트에 위임

### 주의할 점
- ⚠️ `--dangerously-skip-permissions`는 신뢰할 수 있는 프로젝트에서만 사용
- ⚠️ 새 작업 전 `/clear`로 컨텍스트 정리 권장
- ⚠️ 민감한 정보가 포함된 파일은 `@` 참조 시 주의
