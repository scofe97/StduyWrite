## 디렉토리 구조

```
~/.claude/
├── skills/       # 워크플로우 정의 (복잡한 작업 단축)
├── commands/     # 슬래시 커맨드 (즉시 실행)
├── agents/       # 특화된 서브에이전트 정의
├── rules/        # 필수 베스트 프랙티스 가이드라인
└── config.json   # MCP 및 도구 설정
```

## 핵심 컴포넌트

### 1. Skills & Commands

**Skills**: 재사용 가능한 워크플로우를 마크다운으로 정의

```markdown
<!-- ~/.claude/skills/refactor-clean.md -->
# Refactor Clean Skill

복잡한 프롬프트를 단일 단축키로 결합
예: /refactor-clean, /tdd, /e2e
```

**Commands**: 여러 스킬을 순차적으로 호출 가능

```bash
# 단일 프롬프트로 리팩토링 → 테스트 → 커버리지 분석 자동화
/refactor-clean
```

---

### 2. Hooks - 이벤트 기반 자동화

4가지 생명주기 훅으로 지능적 자동화 구현:

| Hook | 실행 시점 | 활용 예시 |
|------|----------|----------|
| **PreToolUse** | 도구 실행 전 | 입력 검증, 긴 명령에 tmux 제안 |
| **PostToolUse** | 도구 완료 후 | prettier로 자동 포맷팅 |
| **UserPromptSubmit** | 메시지 제출 시 | 입력 전처리 |
| **Stop** | 응답 완료 시 | 최종 console.log 감사 |

**설정 예시**:
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": ["prettier --write $FILE"]
      }
    ]
  }
}
```

### 3. Subagents - 특화된 위임

> "제한된 범위와 도구 권한을 가진 특화 에이전트에 작업 위임"

**예시 에이전트**:
- `planner.md` - 계획 수립 전문
- `architect.md` - 아키텍처 설계 전문
- `security-reviewer.md` - 보안 검토 전문

**장점**: 에이전트 역량을 샌드박싱하여 컨텍스트 비대화 방지

---

### 4. Rules 모듈화

`~/.claude/rules/` 디렉토리에 도메인별 규칙 분리:

| 파일 | 내용 |
|------|------|
| `security.md` | 검증, 시크릿 관리 |
| `coding-style.md` | 불변성, 파일 제한 |
| `testing.md` | TDD 워크플로우, 80% 커버리지 |
| `git-workflow.md` | 커밋 형식, PR 프로세스 |
| `performance.md` | 모델 선택 전략 |

---

### 5. MCP 컨텍스트 관리

**핵심 원칙**:

> "20-30개 MCP를 등록하되, 프로젝트당 5-6개만 활성화"

**문제**: 너무 많은 활성 도구 → 컨텍스트 윈도우 과다 소비
- 200k 토큰이 70k 사용 가능 공간으로 축소됨

**전략**:
```json
// ~/.claude.json
{
  "disabledMcpServers": [
    "mcp-server-postgres",    // 이 프로젝트에서 불필요
    "mcp-server-kubernetes",  // 이 프로젝트에서 불필요
    ...
  ]
}
```

**권장**: 활성 MCP 총 10개 미만 유지

## 물리적 환경 설정

### Zed 에디터 통합

```
"Agent Panel을 통해 Claude 통합, 파일 수정 실시간 추적"
```

**권장 레이아웃**:
- 왼쪽: 터미널 (Claude Code)
- 오른쪽: Zed (시각적 파일 리뷰)

**필수 설정**: Zed에서 자동 저장 활성화 → Claude가 최신 파일 상태 접근

### Tmux 활용

긴 작업을 분리 가능한 세션에서 실행 → 병렬 워크로드 관리

```bash
# 새 세션 생성
tmux new -s claude-work

# 분리 (Ctrl+B, D)
# 재연결
tmux attach -t claude-work
```

## Status Line 모니터링

한 줄에서 모든 정보 추적:

```
username | directory | git-branch | context-% | model | time | todo-count
```

**설정 예시**:
```json
{
  "statusLine": {
    "show": true,
    "format": "%u | %d | %b | %c%% | %m | %t | TODO: %T"
  }
}
```
