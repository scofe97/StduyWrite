# A2UI 소개 (Introduction)

> **학습 목표**: A2UI가 무엇인지, 왜 필요한지, 누가 사용하는지 이해한다.

---

## 1. A2UI란?

**A2UI (Agent to UI)**는 AI 에이전트가 **선언적 JSON 메시지**를 통해 풍부한 사용자 인터페이스를 생성할 수 있게 해주는 **프로토콜**입니다.

```
에이전트 → JSON 메시지 → 클라이언트 → 네이티브 UI 렌더링
```

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **선언적** | 실행 코드 없이 JSON 데이터만 사용 |
| **LLM 친화적** | 평탄한 구조로 LLM이 쉽게 생성 가능 |
| **크로스 플랫폼** | 웹, 모바일, 데스크톱에서 동일 UI 렌더링 |
| **스트리밍** | 실시간 점진적 UI 업데이트 지원 |
| **보안** | 코드 실행 없이 데이터만 전송하므로 안전 |

---

## 2. 왜 A2UI가 필요한가?

### 기존 방식의 한계

**문제 1: 텍스트만의 상호작용**
```
사용자: "12월 25일 저녁 7시에 2명 예약해줘"
챗봇: "몇 시에 예약할까요?"
사용자: "저녁 7시"
챗봇: "몇 명인가요?"
사용자: "2명"
... (반복적인 대화)
```

**문제 2: 분산 시스템의 어려움**
- 다양한 서버에서 실행되는 에이전트들
- 원격으로 안전하게 UI를 조작하는 방법 부재
- HTML/JavaScript는 보안 위험 존재

### A2UI의 해결책

```json
{
  "surfaceUpdate": {
    "surfaceId": "reservation",
    "components": [
      {"id": "date", "component": {"DateTimeInput": {...}}},
      {"id": "guests", "component": {"TextField": {...}}},
      {"id": "submit", "component": {"Button": {...}}}
    ]
  }
}
```

→ 에이전트가 **폼, 날짜 선택기, 버튼**을 직접 생성하여 효율적인 상호작용!

## 3. 누가 A2UI를 사용하는가?

### 3.1 호스트 앱 개발자 (프론트엔드)

**역할**: 에이전트가 생성한 UI를 렌더링하는 앱 개발

**핵심 장점**:
- ✅ 브랜드 스타일링 권한 유지
- ✅ 로컬/원격/제3자 에이전트 모두 지원
- ✅ 선언적 데이터로 보안 확보
- ✅ 웹, 모바일, 데스크톱 지원

**예시 시나리오**:
```
1. 사내 AI 어시스턴트 앱 개발
2. 다중 에이전트 플랫폼 구축
3. 크로스플랫폼 AI 기반 앱
```

### 3.2 에이전트 개발자 (백엔드/AI)

**역할**: A2UI 메시지를 생성하는 AI 에이전트 개발

**핵심 장점**:
- ✅ LLM 친화적인 평탄한 구조
- ✅ 텍스트 이상의 풍부한 UI 생성
- ✅ 모든 A2UI 클라이언트와 호환
- ✅ 점진적 스트리밍 렌더링

**예시 시나리오**:
```
1. 레스토랑 예약 에이전트
2. 데이터 입력 폼 자동 생성
3. 승인 워크플로우 에이전트
```

### 3.3 플랫폼 빌더 (SDK 제작자)

**역할**: 에이전트 오케스트레이션 플랫폼, 프레임워크 개발

**핵심 장점**:
- ✅ A2A 및 다른 프로토콜과 표준 호환
- ✅ 커스텀 컴포넌트 카탈로그 확장
- ✅ Apache 2.0 오픈소스

---

## 4. A2UI가 아닌 것

| A2UI는... | A2UI가 아닌 것... |
|-----------|------------------|
| 프로토콜 (통신 규약) | 프레임워크 (개발 도구) |
| 선언적 UI 설명 | HTML 대체재 |
| 플랫폼 독립적 | 웹 전용 |
| 데이터 기반 | 강력한 스타일링 시스템 |

---

## 5. 실제 사용 사례

### 프로덕션 배포

| 서비스 | 설명 | 활용 방식 |
|--------|------|-----------|
| **Google Opal** | AI 미니앱 플랫폼 | 자연어로 UI 생성/편집 |
| **Gemini Enterprise** | 기업용 AI 에이전트 | 데이터 입력 폼, 승인 대시보드 |
| **Flutter GenUI SDK** | 크로스플랫폼 UI | 모바일/데스크톱/웹 동시 지원 |

### 파트너 통합

- **AG UI / CopilotKit**: React 기반 풀스택 에이전틱 프레임워크
- **Google 내부 AI 제품**: 다중 에이전트 워크플로우 표준화
