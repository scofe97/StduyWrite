## 각 실습의 구현 범위

### 1. Testcontainers 기본 통합 테스트
- RedpandaContainer 의존성 및 컨테이너 설정
- @DynamicPropertySource로 bootstrap-servers 동적 주입
- 기본 Producer → Consumer 왕복 테스트 (메시지 전송 + 수신 검증)
- @SpringBootTest + @Testcontainers 어노테이션 조합

### 2. SAGA 통합 테스트
- 정상 플로우: OrderCreated → InventoryReserved → PaymentCompleted → ShippingStarted → OrderCompleted
- 보상 플로우: PaymentFailed → InventoryRollback → OrderFailed
- Awaitility.await()로 비동기 이벤트 체인 완료 대기
- 최종 상태 검증 (Order 상태, 재고 수량 등)

### 3. DLQ 통합 테스트
- 실패 주입 패턴: Consumer에서 의도적 예외 발생
- @RetryableTopic 재시도 소진 → DLT(Dead Letter Topic) 자동 라우팅
- DLT Consumer로 실패 메시지 수신 검증
- DLT 메시지 헤더 검증 (원본 토픽, 예외 정보, 재시도 횟수)

### 4. 테스트 유틸리티 구축
- KafkaTestHelper: 메시지 전송(sendAndWait), 수신(consumeOne/consumeAll) 헬퍼
- AbstractKafkaTest: RedpandaContainer + @DynamicPropertySource 공통 추출
- 테스트 설정 클래스 (@TestConfiguration으로 테스트 전용 빈 분리)

### 5. Flaky 테스트 방지 패턴
- Thread.sleep() → Awaitility.await().atMost().until() 전환
- Container Reuse: .testcontainers.properties에 reuse.enable=true
- 토픽 격리: UUID 기반 고유 토픽명으로 테스트 간 간섭 제거
- @DirtiesContext 최소화 전략
