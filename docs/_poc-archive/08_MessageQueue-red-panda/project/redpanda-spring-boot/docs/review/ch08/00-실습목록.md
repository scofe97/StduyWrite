## 각 실습의 구현 범위

### 1. SAGA 이벤트 설계 (완료)
- `OrderSagaEvent` sealed interface
- 성공 이벤트 4종 + 실패 이벤트 3종 + 보상 이벤트 2종 (총 10파일)

### 2. 정상 플로우 구현
- Kafka JSON 설정 (ch03 전용 ProducerFactory, ConsumerFactory, KafkaTemplate)
- 토픽 설정 (9개 NewTopic 빈)
- 도메인 모델 (Order, Inventory, Payment 등 인메모리)
- 4개 서비스 (OrderService, InventoryService, PaymentService, ShippingService)
- 정상 시나리오에서 이벤트가 체인으로 흐르는 것까지

### 3. 보상 트랜잭션
- 기존 서비스에 보상 리스너 추가 (PaymentFailed→재고복구, ShippingFailed→환불)
- 보상 역순 검증

### 4. Kafka 트랜잭션
- Transactional Producer 설정 (transactional.id, idempotence)
- `executeInTransaction` 적용
- Consumer `isolation.level=read_committed` 설정

### 5. 통합 테스트
- 정상 플로우 테스트 (Order COMPLETED 확인)
- 실패+보상 플로우 테스트 (Order FAILED + 재고 복구 확인)

### 6. KafkaTemplate vs @SendTo 비교
- InventoryService를 @SendTo 방식으로 대안 구현
- 장단점 비교 문서

### 7. 멱등성 보장
- correlationId 기반 이벤트 중복 처리 필터
- 보상 멱등성 (같은 복구 요청 여러 번 처리해도 안전)

### 8. Choreography 한계 대응
- correlationId 기반 SAGA 상태 조회 API
- MDC에 correlationId 넣어서 서비스 간 로그 추적
