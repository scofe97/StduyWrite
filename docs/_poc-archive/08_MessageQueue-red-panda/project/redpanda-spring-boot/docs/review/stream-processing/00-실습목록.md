## 각 실습의 구현 범위

### 1. 기본 Topology 구성
- StreamsBuilder로 입력 토픽(orders)에서 KStream 생성
- filter()로 조건 필터링 (예: 1만원 이상 주문)
- mapValues()로 값 변환 (예: 할인 적용)
- to()로 출력 토픽(high-value-orders)에 결과 쓰기
- Topology의 DAG 구조 시각화 (Source → Processor → Sink)

### 2. KStream vs KTable
- KStream: 주문 이벤트 스트림 — 같은 키에 여러 레코드 보존
- KTable: builder.table()로 상품 정보 테이블 — 같은 키의 최신 값만 유지
- KStream → KTable 변환: groupByKey().count()로 사용자별 주문 건수 집계
- KTable → KStream 변환: toStream()으로 변경 이벤트 스트림 생성
- 두 추상화의 의미적 차이 확인 ("무슨 일이 일어났는가" vs "현재 상태가 무엇인가")

### 3. Stream-Table Join
- 주문 이벤트(KStream)와 상품 정보(KTable) 준비
- selectKey()로 조인 키 설정 (주문의 productId → 상품 테이블 키)
- join()으로 주문에 상품명/가격 보강 (EnrichedOrder 생성)
- 조인 결과를 출력 토픽에 쓰기
- Left Join vs Inner Join 차이 확인

### 4. State Store + Interactive Queries
- Materialized.as("store-name")로 명시적 State Store 생성
- RocksDB 로컬 저장소 디렉토리 구조 확인 (/tmp/kafka-streams/)
- Changelog 토픽 자동 생성 확인 (cleanup.policy=compact)
- ReadOnlyKeyValueStore로 State Store 외부 조회
- REST API 엔드포인트에서 실시간 상태 반환

### 5. Windowing — Tumbling & Hopping
- Tumbling: TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5))로 매 5분 집계
- Hopping: TimeWindows.ofSizeAndGrace().advanceBy()로 겹치는 윈도우 집계
- Windowed<String> 키에서 윈도우 시작/종료 시간 추출
- 윈도우 집계 결과를 출력 토픽에 전송
- Tumbling(겹치지 않음)과 Hopping(겹침)의 결과 차이 비교

### 6. Windowing — Session
- SessionWindows.ofInactivityGapWithNoGrace(Duration.ofMinutes(30))으로 세션 윈도우 설정
- 사용자 클릭 이벤트 기반 세션 분석
- 비활성 간격 초과 시 새 세션 자동 생성 확인
- 세션별 이벤트 수, 세션 지속 시간 집계
- Tumbling/Hopping과의 차이: 고정 크기 vs 이벤트 간격 기반 동적 크기

### 7. Exactly-Once Semantics (EOS)
- processing.guarantee=exactly_once_v2 설정
- At-most-once / At-least-once / Exactly-once 3가지 보장 수준 비교
- 트랜잭션 흐름 확인: 읽기 → 처리 → (출력 쓰기 + 오프셋 커밋) 원자적 실행
- 장애 시뮬레이션: 처리 중 애플리케이션 종료 → 재시작 후 중복/손실 없음 확인
- EOS와 AT_LEAST_ONCE 처리량 차이 확인

### 8. Spring Boot 통합 + Aggregation
- @EnableKafkaStreams + KafkaStreamsConfiguration 빈 설정
- StreamsBuilder 빈으로 Topology 정의
- WordCount 구현: flatMapValues → groupBy → count
- 지역별 주문 건수 집계: selectKey → groupByKey → count(Materialized.as())
- application.yml 설정: application-id, bootstrap-servers, processing.guarantee, state.dir
