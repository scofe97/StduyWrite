## 진행 순서 및 이유

| 순서 | 실습 | 이유 |
|------|------|------|
| 1 | Fire-and-Forget 비동기 발행 | KafkaTemplate.send() + whenComplete의 기본 패턴을 먼저 익혀야 다른 Producer 구현의 토대가 됨 |
| 2 | Command 패턴 Producer | Avro 스키마 기반의 타입 안전 발행 패턴으로, acks=all과 파티션 키 전략을 함께 학습 |
| 3 | 에러 핸들링 Producer | 기본 발행 패턴에 실패 처리를 추가하는 단계 — 실운영에서 필수적인 재시도·알람 로직 |
| 4 | Correlation 추적 | 비동기 발행 후 응답을 어떻게 연결하는지 이해해야 Request-Reply 패턴으로 자연스럽게 넘어갈 수 있음 |
| 5 | Request-Reply 패턴 | 앞선 4가지 개념(발행, 타입 안전, 에러 처리, Correlation)을 모두 통합하는 심화 패턴 |

## 각 실습의 구현 범위

### 1. Fire-and-Forget 비동기 발행
- `FireAndForgetProducer.java`에 `kafkaTemplate.send(topic, key, message)` 호출 구현
- `.whenComplete((result, ex) -> {...})` 콜백 작성: 성공 시 topic/partition/offset 로깅, 실패 시 에러 로깅
- AS-IS `CompletableFuture.runAsync()` + `catch → log.error()` 패턴과 비교하여 차이점 주석으로 명시
- `CommandRequest.avsc` 스키마 기반 Avro 직렬화 설정 확인

### 2. Command 패턴 Producer
- `CommandProducer.java`에 `execute(CommandRequest request)` 메서드 구현
- Producer 설정에 `acks=all`, `retries=3` 적용 (`Ch06TopicConfig.java` 또는 application.yml)
- `kafkaTemplate.send(topic, request.getCommandId(), request)` — commandId를 파티션 키로 사용하여 같은 커맨드 관련 이벤트가 같은 파티션으로 라우팅되도록 설정
- 발행 성공 시 `SendResult`에서 `RecordMetadata` 추출하여 topic/partition/offset/timestamp 로깅

### 3. 에러 핸들링 Producer
- `ErrorHandlingProducer.java`에 발행 실패 감지 및 분류 로직 구현 (일시적 오류 vs 영구 오류)
- 일시적 실패: retries 자동 재시도에 의존, 콜백에서 재시도 대기 로그 출력
- 영구 실패: `meterRegistry.counter("kafka.send.failure", "topic", topicName).increment()` 메트릭 수집
- 임계치 초과 시 알람 처리 패턴 구현 (로그 레벨을 ERROR로 올리거나 별도 알람 인터페이스 호출)

### 4. Correlation 추적
- `CorrelationTracker.java`에 요청 발행 시 `ConcurrentHashMap<String, CompletableFuture<CommandResponse>>`에 correlationId 등록
- 응답 수신 시 correlationId로 대기 중인 Future를 찾아 complete() 호출
- 타임아웃 처리: 일정 시간(예: 30초) 내 응답 없으면 `completeExceptionally(TimeoutException)`
- `CommandRequest.avsc`의 correlationId 필드와 `CommandResponse.avsc`의 correlationId 필드 연동 확인

### 5. Request-Reply 패턴
- `ReplyingKafkaConfig.java`에 `ReplyingKafkaTemplate` 빈 설정: reply 토픽, reply 파티션 설정
- `ReplyingTemplateService.java`에 `sendAndReceive(ProducerRecord<String, CommandRequest>)` 호출 구현
- `RequestReplyFuture<String, CommandRequest, CommandResponse>`로 비동기 응답 대기
- `Ch06FeignToProducerTest.java`에서 전체 흐름 검증: CommandRequest 발행 → Consumer가 CommandResponse 반환 → ReplyingTemplateService가 응답 수신

## 패키지 구조

```
src/main/java/com/study/redpanda/ch06/
├── config/
│   ├── Ch06TopicConfig.java          # 토픽 설정 (pipeline.execute.request/response)
│   └── ReplyingKafkaConfig.java      # ReplyingKafkaTemplate 빈 설정
└── producer/
    ├── CommandProducer.java           # acks=all Command 발행 Producer
    ├── CorrelationTracker.java        # 비동기 요청-응답 correlationId 추적
    ├── ErrorHandlingProducer.java     # 실패 감지·메트릭·알람 Producer
    ├── FireAndForgetProducer.java     # whenComplete 콜백 기본 발행 패턴
    └── ReplyingTemplateService.java   # Request-Reply 동기 패턴 서비스

src/main/avro/ch06/
├── CommandRequest.avsc               # 요청 Avro 스키마 (commandId, correlationId 포함)
└── CommandResponse.avsc              # 응답 Avro 스키마 (correlationId, status 포함)

src/test/java/com/study/redpanda/ch06/
└── Ch06FeignToProducerTest.java      # 전체 Producer 패턴 통합 테스트
```
