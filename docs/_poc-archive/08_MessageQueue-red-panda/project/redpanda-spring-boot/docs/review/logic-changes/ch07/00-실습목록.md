## 각 실습의 구현 범위

### 1. 기본 이벤트 Consumer
- `EventConsumer.java`: `@KafkaListener(topics, groupId)` 어노테이션 구현
- `ConsumerRecord<String, PipelineExecuteRequest>` 파라미터로 파티션/오프셋 로깅
- `Ch07TopicConfig.java`: `pipeline.execute.request` 토픽 NewTopic 빈 설정
- `ConsumerGroupConfig.java`: `jenkins-worker-group` Consumer Group 팩토리 설정

### 2. 배치 Consumer
- `BatchConsumer.java`: `@KafkaListener` + `batchListener=true` 설정
- `List<ConsumerRecord<String, PipelineExecuteRequest>>` 파라미터로 배치 수신
- `max.poll.records=10` 설정하여 배치 크기 제어
- 배치 내 개별 메시지 처리 후 일괄 오프셋 커밋

### 3. 수동 커밋 Consumer
- `ManualCommitConsumer.java`: `Acknowledgment ack` 파라미터 추가
- `AckMode.MANUAL` 팩토리 설정 (`ENABLE_AUTO_COMMIT_CONFIG=false` 병행)
- 처리 성공 시 `ack.acknowledge()` 호출, 예외 시 미호출로 재처리 유도
- `DefaultErrorHandler(FixedBackOff(1000, 3))` 설정으로 3회 재시도

### 4. 리밸런스 리스너
- `RebalanceListenerImpl.java`: `ConsumerRebalanceListener` 인터페이스 구현
- `onPartitionsAssigned()`: 파티션 할당 시 오프셋 초기화/복구 로직
- `onPartitionsRevoked()`: 파티션 해제 직전 보류 중인 오프셋 강제 커밋
- `@KafkaListener`의 `containerFactory`에 리스너 연결

### 5. 통합 테스트
- `Ch07RestToConsumerTest.java`: Spring Kafka Test 기반 통합 테스트
- 시나리오 1: Consumer 정상 수신 → 오프셋 커밋 확인
- 시나리오 2: Consumer 처리 실패 → ack 미호출 → 재전달 확인
- 시나리오 3: Consumer 재시작 후 미커밋 오프셋부터 재처리 확인
