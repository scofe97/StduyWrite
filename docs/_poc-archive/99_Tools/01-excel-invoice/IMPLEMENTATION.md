# 거래명세표 Excel 구현 상세

## 목적

거래명세표를 인쇄하면 동일한 내용의 3부가 나온다.
3부를 각각 공급자, 공급받는자, 보관용으로 나눠 가진다.
1부(공급자용)에만 데이터를 입력하면 2부, 3부에 자동으로 반영된다.

---

## 전체 구조

```
거래명세표.xlsx
├── Sheet 1: "거래명세표"
│   ├── Row 1~23:   Form 1 (공급자용) ← 여기에 입력
│   ├── Row 24~25:  빈 행 (간격)
│   ├── Row 26~48:  Form 2 (공급받는자용) ← 자동 복사
│   ├── Row 49~50:  빈 행 (간격)
│   └── Row 51~73:  Form 3 (보관용) ← 자동 복사
│
└── Sheet 2: "코드매핑"
    └── 221개 품목 조회 테이블 (코드번호 → 품명, 규격, 단가, 비고)
```

---

## 핵심 구현: 3부 자동 동기화

### 문제

하나의 시트에 동일한 양식 3개를 배치하되,
1번째 양식에 입력한 내용이 2번째, 3번째에 자동으로 나타나야 한다.

### 해결: 셀 참조(`=`) 수식

Form 2, 3의 모든 데이터 셀은 Form 1의 대응 셀을 `=` 수식으로 참조한다.

```
Form 1 (마스터)              Form 2 (미러)              Form 3 (미러)
┌──────────────┐            ┌──────────────┐            ┌──────────────┐
│ B9: (사용자 입력) │  ──→   │ B34: =B9     │  ──→      │ B59: =B9     │
│ E9: (수량 입력)   │  ──→   │ E34: =E9     │  ──→      │ E59: =E9     │
│ H9: =E9*G9       │  ──→   │ H34: =H9     │  ──→      │ H59: =H9     │
│ H18: =SUM(...)   │  ──→   │ H43: =H18    │  ──→      │ H68: =H18    │
└──────────────┘            └──────────────┘            └──────────────┘
```

### 왜 이 방식인가

| 대안 | 문제점 |
|------|--------|
| VBA 매크로로 복사 | 매크로 보안 설정 필요, 사용자 거부감 |
| 별도 시트 3개 | 인쇄 시 시트별로 따로 출력해야 함 |
| 수동 복사 | 입력 누락, 불일치 위험 |
| **셀 참조 수식** | **매크로 불필요, 자동 동기화, 한 시트에서 연속 인쇄** |

## 양식 레이아웃 (23행 × 9열)

각 양식은 23행으로 구성된다. 양식 간에 2행의 빈 간격이 있다.

```
행 오프셋  내용              열 범위
────────  ────────────────  ─────────
+0        제목              A:I (병합)
+1        날짜              A:I (병합, 입력 셀)
+2~+6     업체 정보          A:D (공급받는자), E:I (공급자)
+7        테이블 헤더        A~I (NO, 코드번호, 품명, ...)
+8~+16    거래 내역 9행      A~I
+17       공급가액 합계      A:G (병합), H (SUM)
+18       세액 (10%)        A:G (병합), H (계산)
+19       합계 (원정)        A:G (병합), H (합산)
+20       (빈 행)
+21       서명란            A:D, F:I
+22       인수일자          A:D
```

## 수식 체계

### Form 1 (마스터) — 원본 수식

| 셀 | 수식 | 설명 |
|----|------|------|
| C9 | `=IFERROR(VLOOKUP(B9,코드매핑!$A:$E,2,FALSE),"")` | 코드번호 → 품명 |
| D9 | `=IFERROR(VLOOKUP(B9,코드매핑!$A:$E,3,FALSE),"")` | 코드번호 → 규격 |
| F9 | `=IF(B9="","","EA")` | 코드 있으면 단위 표시 |
| G9 | `=IFERROR(VLOOKUP(B9,코드매핑!$A:$E,4,FALSE),"")` | 코드번호 → 단가 |
| H9 | `=IF(AND(E9<>"",G9<>""),E9*G9,"")` | 수량 × 단가 |
| I9 | `=IFERROR(VLOOKUP(B9,코드매핑!$A:$E,5,FALSE),"")` | 코드번호 → 비고 |
| H18 | `=SUM(H9:H17)` | 공급가액 합계 |
| H19 | `=INT(H18*0.1)` | 세액 (원 단위 절사) |
| H20 | `=H18+H19` | 총합계 |

### Form 2, 3 (미러) — 참조 수식

| Form 2 셀 | 수식 | 참조 대상 |
|-----------|------|----------|
| B34 | `=B9` | Form 1의 코드번호 |
| C34 | `=C9` | Form 1의 품명 (VLOOKUP 결과) |
| E34 | `=E9` | Form 1의 수량 |
| H34 | `=H9` | Form 1의 공급가액 |
| H43 | `=H18` | Form 1의 합계 |

Form 3도 동일하게 Form 1을 참조한다 (B59=B9, H68=H18 등).

├── build_master_form()          # Form 1 생성 (입력 + 수식)
├── build_mirror_form()          # Form 2, 3 생성 (셀 참조)

└── main()                       # 실행 진입점

### 핵심 함수: `build_mirror_form()`

```python
def ref(master_row, col):
    """마스터 셀 참조 수식 생성. ref(9, 2) → '=B9'"""
    return f"={get_column_letter(col)}{master_row}"

# 미러 양식의 각 셀:
set_cell(ws, R+8, 2, ref(master_R+8, 2))  # B34 = =B9
```

- `R` = 미러 양식의 시작행
- `master_R` = 마스터 양식의 시작행
- 모든 데이터 셀에 `=마스터셀주소` 수식을 넣는다
- 스타일(폰트, 테두리, 숫자 포맷)은 마스터와 동일하게 적용
- 제목만 다르게 설정 (공급자용 / 공급받는자용 / 보관용)

### 셀 병합 관리: `get_merge_ranges()`

셀 병합 범위를 **상대 좌표** (오프셋)로 정의하여 3개 양식에 동일하게 적용:

```python
def get_merge_ranges():
    return [
        (0, 0, 1, 9),    # 제목 행 A:I 병합
        (2, 6, 1, 1),    # 공급받는자 세로 병합
        (17, 17, 1, 7),  # 합계 행 A:G 병합
        ...
    ]
```

`apply_merges(ws, start_row, ranges)` 함수가 시작행을 더해서 실제 좌표로 변환한다.

## 인쇄 설정

| 항목 | 설정 |
|------|------|
| 용지 | A4 |
| 방향 | 가로 (landscape) |
| 너비 맞춤 | 1페이지 |
| 높이 맞춤 | 자동 (3페이지) |
| 여백 | 상하좌우 0.5인치 |
| 페이지 나누기 | Row 25, Row 50 (양식 간 빈 행에서 분리) |

인쇄 시 Form 1, 2, 3이 각각 한 페이지씩 출력된다.

## 사용 흐름

```
1. 거래명세표.xlsx 열기
2. Form 1 (Row 1~23)에서:
   - B9~B17: 코드번호 드롭다운 선택
   - E9~E17: 수량 입력
3. 자동으로:
   - 품명, 규격, 단가, 비고가 VLOOKUP으로 채워짐
   - 공급가액 = 수량 × 단가
   - 합계, 세액, 총합계 계산
   - Form 2, 3에 동일 내용 반영
4. Ctrl+P로 인쇄 → 3부 출력
5. 3부를 공급자, 공급받는자, 보관용으로 배분
```
