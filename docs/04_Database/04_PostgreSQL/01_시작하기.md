### 3. Docker로 Postgres 실행

```bash
# 기본 실행 (데이터 휘발성)
docker run --name postgres-server \
    -e POSTGRES_USER=postgres \
    -e POSTGRES_PASSWORD=password \
    -p 5432:5432 \
    -d postgres:17

# 프로덕션용 (볼륨 마운트로 데이터 영속성)
docker volume create postgres-volume
docker run --name postgres-server \
    -e POSTGRES_USER=postgres \
    -e POSTGRES_PASSWORD=password \
    -p 5432:5432 \
    -v postgres-volume:/var/lib/postgresql/data \
    -d postgres:17
```

### 5. generate_series 함수

```sql
-- 기본 사용법: 1부터 10까지 정수 생성
SELECT * FROM generate_series(1, 10);

-- 단계 지정: 짝수만 생성
SELECT * FROM generate_series(2, 10, 2);

-- 날짜 범위 생성
SELECT * FROM generate_series(
    '2025-01-01'::date,
    '2025-01-07'::date,
    '1 day'::interval
);

-- 목 데이터 생성 예시
INSERT INTO users (id, name, created_at)
SELECT
    g,
    'User_' || g,
    NOW() - (random() * interval '365 days')
FROM generate_series(1, 1000) AS g;
```

```
generate_series 활용 패턴
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  【 목 데이터 생성 】                                    │
│  INSERT ... SELECT FROM generate_series(1, 1000)        │
│  → 테스트용 대량 데이터 빠르게 생성                      │
│                                                         │

│  → 균등 간격 샘플링                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 비교표

### OLTP vs OLAP 비교

| 구분 | OLTP | OLAP |
|------|------|------|
| **목적** | 트랜잭션 처리 | 데이터 분석 |
| **쿼리 유형** | INSERT, UPDATE, DELETE | SELECT (집계) |
| **데이터 양** | 개별 레코드 | 대량 데이터 |
| **응답 시간** | 밀리초 단위 | 초~분 단위 |
| **동시성** | 많은 사용자 | 소수 분석가 |
| **예시** | 주문 처리 | 매출 리포트 |

### psql vs GUI 도구 비교

| 구분 | psql (CLI) | GUI (DBeaver 등) |
|------|------------|------------------|
| **접근성** | 터미널만 있으면 됨 | 별도 설치 필요 |
| **자동화** | 스크립트 연동 용이 | 제한적 |
| **학습곡선** | 명령어 암기 필요 | 직관적 UI |
| **기능** | 필수 기능 중심 | 시각화, ERD 등 |
| **서버 접속** | SSH + psql | SSH 터널 설정 |

## 면접 예상 질문 및 모범 답안

### Q1. PostgreSQL을 선택한 이유는 무엇인가요?

**모범 답안:**

PostgreSQL을 선택하는 주요 이유는 세 가지입니다.

첫째, **확장성(Extensibility)**입니다. PostgreSQL은 Extensions를 통해 기능을 확장할 수 있습니다. pgvector로 벡터 검색, PostGIS로 지리공간 데이터, TimescaleDB로 시계열 데이터를 처리할 수 있어, 하나의 데이터베이스로 다양한 use case를 커버할 수 있습니다.

둘째, **표준 준수와 고급 기능**입니다. SQL 표준을 가장 잘 준수하면서도 CTE, 윈도우 함수, JSONB 등 고급 기능을 지원합니다. 이로 인해 복잡한 비즈니스 로직을 SQL로 표현할 수 있습니다.

셋째, **오픈소스와 커뮤니티**입니다. MIT 유사 라이선스로 상업적 사용에 제약이 없고, 활발한 커뮤니티 덕분에 문제 해결과 학습 자료가 풍부합니다.

### Q2. Docker로 PostgreSQL을 운영할 때 데이터 영속성은 어떻게 보장하나요?

**모범 답안:**

Docker 컨테이너는 기본적으로 휘발성이므로, 컨테이너가 삭제되면 데이터도 함께 사라집니다. 데이터 영속성을 위해 **Docker Volume**을 사용합니다.

```bash
# 볼륨 생성
docker volume create postgres-volume

# 볼륨을 마운트하여 컨테이너 실행
docker run -v postgres-volume:/var/lib/postgresql/data ...
```

`/var/lib/postgresql/data`는 PostgreSQL이 데이터 파일을 저장하는 경로입니다. 이를 호스트의 볼륨에 마운트하면 컨테이너가 삭제되어도 데이터는 볼륨에 보존됩니다.

프로덕션 환경에서는 추가로 정기 백업(pg_dump), 레플리케이션, 그리고 볼륨의 정기적인 스냅샷을 고려해야 합니다.

### Q3. generate_series 함수는 어떤 상황에서 사용하나요?

**모범 답안:**

`generate_series`는 연속된 값의 집합을 생성하는 함수로, 주로 네 가지 상황에서 사용합니다.

**1. 목 데이터 생성:**
```sql
INSERT INTO orders (id, amount)
SELECT g, random() * 1000
FROM generate_series(1, 10000) AS g;
```

**2. 캘린더/시간 테이블:**
```sql
SELECT generate_series('2025-01-01'::date, '2025-12-31', '1 day');
```

**3. 시계열 갭 분석:**
날짜별 데이터가 없는 날도 0으로 표시하려면 generate_series로 전체 날짜를 생성하고 LEFT JOIN합니다.

**4. 숫자 범위 조건:**
WHERE 절에서 특정 패턴의 ID를 필터링할 때 사용합니다.

이 함수를 활용하면 복잡한 반복문 없이 선언적으로 데이터를 생성할 수 있습니다.

### Q4. psql에서 자주 사용하는 명령어를 설명해주세요.

**모범 답안:**

psql의 명령어는 크게 세 가지로 분류됩니다.

**메타데이터 조회:**
- `\l`: 데이터베이스 목록
- `\dt`: 현재 스키마의 테이블 목록
- `\d table_name`: 테이블 컬럼 구조, 인덱스, 제약조건 확인
- `\di`: 인덱스 목록

**세션 제어:**
- `\c dbname`: 다른 데이터베이스로 연결
- `\timing`: 쿼리 실행 시간 표시 토글
- `\x`: 세로 출력 모드 (컬럼이 많을 때 유용)

**작업 효율화:**
- `\e`: 외부 에디터로 쿼리 편집
- `\i file.sql`: SQL 파일 실행
- `\watch 5`: 마지막 쿼리를 5초마다 반복 실행

특히 `\d+`를 사용하면 테이블 크기와 상세 정보까지 확인할 수 있어 성능 분석에 유용합니다.

### Q5. PostgreSQL이 "Beyond SQL"을 지원한다는 것은 무엇을 의미하나요?

**모범 답안:**

"Beyond SQL"은 PostgreSQL이 전통적인 관계형 데이터베이스를 넘어 다양한 데이터 모델과 use case를 지원한다는 의미입니다.

**확장 기능 예시:**
- **JSONB**: 문서 데이터베이스처럼 JSON 저장/쿼리
- **pgvector**: 벡터 임베딩 저장, 유사도 검색 (AI/ML)
- **PostGIS**: 지리공간 데이터, 거리 계산, 영역 검색
- **TimescaleDB**: 시계열 데이터 자동 파티셔닝
- **pgmq**: 메시지 큐 기능

이러한 확장 덕분에 여러 전문 데이터베이스를 도입하는 대신 PostgreSQL 하나로 통합할 수 있습니다. 이를 통해 아키텍처 복잡도를 줄이고, 데이터 일관성을 유지하며, 운영 부담을 줄일 수 있습니다.

### Q6. PostgreSQL과 MySQL의 주요 차이점은 무엇인가요?

**모범 답안:**

PostgreSQL과 MySQL은 모두 인기 있는 오픈소스 RDBMS지만 설계 철학이 다릅니다.

**PostgreSQL:**
- SQL 표준 준수에 집중
- 고급 기능 풍부 (CTE, 윈도우 함수, JSONB, 확장)
- MVCC 구현이 정교 (UPDATE가 새 버전 생성)
- 확장성이 뛰어남 (Extensions)

**MySQL:**
- 단순성과 속도에 집중
- 다양한 스토리지 엔진 (InnoDB, MyISAM)
- 레플리케이션이 상대적으로 간단
- 웹 애플리케이션에 전통적으로 많이 사용

선택 기준으로는, 복잡한 쿼리나 확장 기능이 필요하면 PostgreSQL, 단순하고 빠른 읽기/쓰기가 주요하면 MySQL이 적합합니다. 다만 최근에는 두 DB 모두 격차가 줄어들고 있습니다.
