### 2. 확장 관리 명령어

```sql
-- 사용 가능한 확장 목록
SELECT * FROM pg_available_extensions;

-- 설치된 확장 목록
SELECT * FROM pg_extension;

-- 확장 설치
CREATE EXTENSION pgcrypto;

-- 특정 스키마에 설치
CREATE EXTENSION postgis WITH SCHEMA geo;

-- 확장 업그레이드
ALTER EXTENSION pgcrypto UPDATE TO '1.4';

-- 확장 제거
DROP EXTENSION pgcrypto;

-- 의존 객체와 함께 제거
DROP EXTENSION pgcrypto CASCADE;
```

### 4. pgcrypto 확장

```sql
-- 설치
CREATE EXTENSION pgcrypto;

-- 해시 함수
SELECT encode(digest('password', 'sha256'), 'hex');

-- 암호화 해시 (비밀번호 저장용)
SELECT crypt('password', gen_salt('bf', 8));
-- 결과: $2a$08$... (bcrypt 해시)

-- 비밀번호 검증
SELECT crypt('password', stored_hash) = stored_hash;

-- 대칭키 암호화 (AES)
-- 암호화
SELECT pgp_sym_encrypt('sensitive data', 'secret_key');

-- 복호화
SELECT pgp_sym_decrypt(
    pgp_sym_encrypt('sensitive data', 'secret_key'),
    'secret_key'
);

-- UUID 생성
SELECT gen_random_uuid();
```

### 5. Foreign Data Wrapper (FDW)

```
FDW 아키텍처
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Local PostgreSQL                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Foreign Table                                       │   │
│  │  (remote_users)                                      │   │
│  │       │                                              │   │
│  │       ↓                                              │   │
│  │  FDW Extension                                       │   │
│  │  (postgres_fdw)                                      │   │
│  │       │                                              │   │
│  │       ↓                                              │   │
│  │  Foreign Server                                      │   │
│  │  (remote_server)                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│              │                                              │
│              ↓ 네트워크                                     │
│  Remote PostgreSQL                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  users 테이블                                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```sql
-- FDW 설정 단계
-- 1. 확장 설치
CREATE EXTENSION postgres_fdw;

-- 2. 외부 서버 정의
CREATE SERVER remote_server
    FOREIGN DATA WRAPPER postgres_fdw
    OPTIONS (host 'db.example.com', port '5432', dbname 'remote_db');

-- 3. 사용자 매핑
CREATE USER MAPPING FOR local_user
    SERVER remote_server
    OPTIONS (user 'remote_user', password 'password');

-- 4. 외부 테이블 정의
CREATE FOREIGN TABLE remote_users (
    id INT,
    name TEXT,
    email TEXT
)
SERVER remote_server
OPTIONS (schema_name 'public', table_name 'users');

-- 또는 스키마 전체 가져오기
IMPORT FOREIGN SCHEMA public
FROM SERVER remote_server
INTO local_schema;

-- 5. 쿼리 실행 (로컬 테이블처럼 사용)
SELECT * FROM remote_users WHERE id = 1;
```

## 면접 예상 질문 및 모범 답안

### Q1. PostgreSQL Extension의 개념과 장점을 설명해주세요.

**모범 답안:**

Extension은 PostgreSQL의 기능을 확장하는 플러그인 시스템입니다.

**구성요소:**
- Control 파일: 메타데이터 (이름, 버전, 의존성)
- SQL 스크립트: 타입, 함수, 연산자 정의
- Shared Library: C로 구현된 고성능 함수 (선택)

**장점:**
1. **모듈화**: 필요한 기능만 선택적 로드
2. **버전 관리**: 확장별 독립적 업그레이드
3. **네임스페이스 격리**: 스키마 단위 설치
4. **표준화**: `CREATE EXTENSION` 통일된 설치

**예시:**
```sql
CREATE EXTENSION pgvector;  -- AI/ML 벡터 연산
CREATE EXTENSION postgis;   -- 지리공간
CREATE EXTENSION pgcrypto;  -- 암호화
```

이로 인해 PostgreSQL은 "하나의 DB로 여러 use case"를 커버할 수 있습니다.

### Q2. pgcrypto 확장의 주요 함수와 사용 사례를 설명해주세요.

**모범 답안:**

pgcrypto는 암호화 관련 함수를 제공합니다.

**1. 비밀번호 해싱 (가장 일반적):**
```sql
-- 저장
INSERT INTO users (password_hash)
VALUES (crypt('user_password', gen_salt('bf', 10)));

-- 검증
SELECT * FROM users
WHERE password_hash = crypt('input_password', password_hash);
```

**2. 데이터 암호화:**
```sql
-- 민감 정보 암호화 저장
UPDATE customers SET
    ssn = pgp_sym_encrypt(ssn_plain, encryption_key);

-- 복호화
SELECT pgp_sym_decrypt(ssn, encryption_key) FROM customers;
```

**3. UUID 생성:**
```sql
SELECT gen_random_uuid();  -- UUID v4
```

**권장사항:**
- 비밀번호: `crypt()` + `gen_salt('bf')` (bcrypt)
- 데이터 암호화: `pgp_sym_encrypt()` (AES)
- 키 관리는 DB 외부에서

### Q3. Foreign Data Wrapper(FDW)의 개념과 활용 사례를 설명해주세요.

**모범 답안:**

FDW는 외부 데이터 소스를 PostgreSQL 테이블처럼 쿼리할 수 있게 하는 기능입니다.

**아키텍처:**
```
Local DB → FDW Extension → Foreign Server → Remote Data
```

**설정 단계:**
1. 확장 설치: `CREATE EXTENSION postgres_fdw`
2. 서버 정의: `CREATE SERVER`
3. 사용자 매핑: `CREATE USER MAPPING`
4. 외부 테이블: `CREATE FOREIGN TABLE`

**활용 사례:**

1. **데이터 통합:**
   - 레거시 MySQL 데이터 + PostgreSQL 데이터 조인

2. **데이터 마이그레이션:**
   - 소스 DB를 FDW로 연결 후 점진적 마이그레이션

3. **분석:**
   - 여러 DB의 데이터를 한 쿼리로 집계

**주의점:**
- 네트워크 지연
- 푸시다운 최적화 확인 (WHERE 절이 원격에서 실행되는지)

### Q4. Postgres 호환 데이터베이스들의 차이점은?

**모범 답안:**

Postgres 생태계에는 여러 유형의 호환 솔루션이 있습니다.

**1. Postgres 기반 (Fork):**
- Aurora PostgreSQL, AlloyDB
- Postgres 코드 기반
- 스토리지/복제 계층 변경
- 확장 호환성 높음

**2. Postgres 확장:**
- Citus (수평 샤딩)
- TimescaleDB (시계열)
- 표준 Postgres에 확장으로 설치
- 기존 확장과 호환

**3. Postgres 프로토콜 호환:**
- YugabyteDB, CockroachDB, Neon
- SQL 문법과 프로토콜 호환
- 내부 구현은 완전히 다름
- 확장 호환성 제한적

**선택 기준:**
- 확장 호환성 필요 → Fork나 확장 기반
- 글로벌 분산 필요 → 분산 SQL (YugabyteDB, CockroachDB)
- 관리형 원함 → 클라우드 서비스 (Aurora, AlloyDB)

### Q5. 확장을 선택할 때 고려해야 할 사항은?

**모범 답안:**

**1. 클라우드 지원:**
```
RDS        → 허용 목록 확인
Cloud SQL  → 지원 확장 제한적
자체 호스팅 → 제한 없음
```

**2. 버전 호환성:**
- Postgres 버전에 맞는 확장 버전
- 업그레이드 경로 확인

**3. 라이선스:**
- PostgreSQL License: 자유로움
- AGPL: 소스 공개 요구 가능
- 상용 라이선스: 비용

**4. 성능 영향:**
- Hook 기반 확장은 모든 쿼리에 영향
- 메모리 사용량

**5. 커뮤니티/지원:**
- 활발한 개발 여부
- 문서화 수준
- 보안 업데이트 빈도

**체크리스트:**
```
□ 클라우드 환경에서 지원되는가?
□ 현재 Postgres 버전과 호환되는가?
□ 라이선스가 사용 조건에 맞는가?
□ 성능 오버헤드가 허용 범위인가?
□ 충분한 문서와 커뮤니티가 있는가?
```

### Q6. 표준 PostgreSQL 대신 Aurora나 Citus를 선택하는 기준은?

**모범 답안:**

**Aurora PostgreSQL 선택 시:**
- AWS 생태계 활용
- 자동 스케일링 필요
- 읽기 레플리카 쉬운 관리
- 고가용성 (Multi-AZ)
- 관리 부담 최소화

**Citus 선택 시:**
- 단일 노드 한계 도달 (TB급 데이터)
- 수평 샤딩 필요
- 멀티테넌트 아키텍처
- 기존 확장 호환성 유지
- 셀프 호스팅 또는 Azure 사용

**표준 PostgreSQL 유지:**
- 단일 노드로 충분한 워크로드
- 모든 확장 사용 필요
- 클라우드 종속 회피
- 최대 유연성 필요

**결정 흐름:**
```
단일 노드 충분? → 표준 Postgres
  ↓ No
읽기 확장 필요? → Aurora 또는 읽기 레플리카
  ↓ No (쓰기 확장)
수평 샤딩 필요? → Citus 또는 분산 SQL
```
