# Chapter 14: Doing the Right Thing

---

## 📌 핵심 요약
> 이 장에서는 **데이터 집약적 애플리케이션을 구축할 때의 윤리적 책임**을 다룬다. 데이터는 추상적 개념이 아니라 **사람에 대한 것**이며, 인간의 존엄성을 존중해야 한다. **예측 분석(Predictive Analytics)**은 편향과 차별을 증폭시킬 수 있고, **대규모 감시 인프라**는 개인의 프라이버시를 위협한다. 엔지니어로서 우리는 기술의 결과에 대해 책임을 져야 하며, **우리가 살고 싶은 세상**을 만들기 위해 노력해야 한다.

---

## 🎯 학습 목표
이 내용을 읽고 나면:
- [ ] 예측 분석이 개인의 삶에 미치는 영향을 설명할 수 있다
- [ ] 알고리즘의 편향과 차별 문제를 이해할 수 있다
- [ ] 자동화된 의사결정에서의 책임과 책무 문제를 인식할 수 있다
- [ ] 피드백 루프의 위험성을 설명할 수 있다
- [ ] 프라이버시와 감시의 관계를 이해할 수 있다
- [ ] 동의(Consent)의 한계를 인식할 수 있다
- [ ] 데이터를 자산이자 독성 물질로 보는 관점을 이해할 수 있다
- [ ] 엔지니어로서의 윤리적 책임을 인식할 수 있다

## 📖 본문 정리

### 1. 예측 분석 (Predictive Analytics)

#### 1.1 예측의 두 얼굴

```
┌─────────────────────────────────────────────────────────────────┐
│                    예측 분석의 적용 영역                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  비교적 무해한 예측:                                             │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  🌤️ 날씨 예측                                               ││
│  │  🦠 질병 확산 예측                                          ││
│  │  📦 재고 수요 예측                                          ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  개인의 삶에 직접적 영향:                                        │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  ⚖️ 재범 가능성 예측 → 보석/가석방 결정                      ││
│  │  💳 대출 상환 가능성 → 대출 승인/거절                        ││
│  │  🏥 보험 청구 가능성 → 보험료/보장 범위                       ││
│  │  💼 고용 적합성 → 채용 결정                                  ││
│  │  ✈️ 보안 위험 → 탑승 허용/거부                               ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 1.2 알고리즘 감옥 (Algorithmic Prison)

```
┌─────────────────────────────────────────────────────────────────┐
│                    "알고리즘 감옥" 현상                           │
│                                                                 │
│  조직의 관점:                                                    │
│  • 놓친 기회의 비용 < 잘못된 결정의 비용                          │
│  • 의심스러우면 거절하는 것이 안전                                │
│                                                                 │
│  개인에게 미치는 영향:                                           │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  "위험"으로 분류된 개인                                      ││
│  │         │                                                   ││
│  │         ▼                                                   ││
│  │  ┌───────────────────────────────────────────────────────┐  ││
│  │  │  ❌ 취업 거부                                          │  ││
│  │  │  ❌ 항공 탑승 거부                                      │  ││
│  │  │  ❌ 보험 가입 거부                                      │  ││
│  │  │  ❌ 주택 임대 거부                                      │  ││
│  │  │  ❌ 금융 서비스 거부                                    │  ││
│  │  │  ❌ 기타 사회 참여 기회 차단                            │  ││
│  │  └───────────────────────────────────────────────────────┘  ││
│  │                                                             ││
│  │  → 증거 없이, 항소 기회 없이 사회에서 체계적으로 배제         ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  형사 사법: "유죄가 입증될 때까지 무죄"                           │
│  알고리즘: "위험하다고 분류되면 증명 없이 배제"                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> 💬 **비유**: 알고리즘 감옥은 보이지 않는 감옥이다. 물리적 벽은 없지만, 사회의 모든 문이 당신에게 닫혀있다.

### 2. 편향과 차별 (Bias and Discrimination)

#### 2.1 알고리즘 편향의 메커니즘

```
┌─────────────────────────────────────────────────────────────────┐
│                  편향이 증폭되는 과정                             │
│                                                                 │
│  과거 데이터 (편향 내포)                                         │
│       │                                                         │
│       │  예: 과거 채용 데이터에서 특정 그룹이 덜 채용됨           │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                   ML 모델 학습                               ││
│  │                                                             ││
│  │  • 데이터에서 패턴 추출                                      ││
│  │  • 왜 그런 패턴인지는 모름 (상관관계 ≠ 인과관계)              ││
│  │  • 과거의 편향을 "객관적 사실"로 학습                         ││
│  └─────────────────────────────────────────────────────────────┘│
│       │                                                         │
│       ▼                                                         │
│  모델 출력 (편향 증폭)                                           │
│       │                                                         │
│       │  예: 특정 그룹에 낮은 점수 → 더 적게 채용                │
│       ▼                                                         │
│  새로운 데이터 생성 (더 편향된)                                   │
│       │                                                         │
│       └──────────────→ 피드백 루프 ───────────────┐             │
│                                                   │             │
│                                                   ▼             │
│                                            편향 영속화           │
│                                                                 │
│  "Machine Learning is like money laundering for bias"           │
│   (머신러닝은 편향을 세탁하는 것과 같다)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.2 보호 속성과 프록시 변수

```
┌─────────────────────────────────────────────────────────────────┐
│              보호 속성과 숨겨진 차별                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  법적으로 차별 금지된 속성 (Protected Traits):                    │
│  • 인종, 민족                                                   │
│  • 나이                                                         │
│  • 성별                                                         │
│  • 성적 지향                                                    │
│  • 장애                                                         │
│  • 종교, 신념                                                   │
│                                                                 │
│  하지만 다른 변수들이 이와 상관관계가 있다면?                     │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  우편번호  ─────────→  인종의 강력한 예측자                  ││
│  │  (인종 분리 지역)                                           ││
│  │                                                             ││
│  │  IP 주소  ──────────→  지역 → 인종                          ││
│  │                                                             ││
│  │  구매 패턴 ─────────→  소득 수준 → 사회 계층                 ││
│  │                                                             ││
│  │  이름     ──────────→  성별, 민족 예측                       ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  문제:                                                          │
│  알고리즘이 보호 속성을 직접 사용하지 않아도                      │
│  프록시 변수를 통해 간접적으로 차별할 수 있음                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> 💬 **핵심 통찰**: 예측 분석은 과거를 외삽할 뿐이다. 과거가 차별적이었다면, 그 차별을 코드화하고 증폭한다. 더 나은 미래를 원한다면, **도덕적 상상력(Moral Imagination)**이 필요하고, 이는 오직 인간만이 제공할 수 있다.

### 3. 책임과 책무 (Responsibility and Accountability)

#### 3.1 자동화된 의사결정의 책임 문제

```
┌─────────────────────────────────────────────────────────────────┐
│                    책임 소재의 모호성                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  인간의 결정:                                                    │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  결정자 → 실수 → 책임 추궁 가능 → 항소 가능                  ││
│  │                                                             ││
│  │  의사결정 과정을 설명할 수 있음                              ││
│  │  잘못된 결정에 대한 배상 가능                                ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  알고리즘의 결정:                                                │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  알고리즘 → 실수 → 누가 책임? → 항소 어떻게?                 ││
│  │                                                             ││
│  │  • 개발자? (의도하지 않은 결과)                              ││
│  │  • 회사? (모델 작동 방식 모를 수 있음)                       ││
│  │  • 데이터 제공자? (편향된 데이터)                            ││
│  │  • 사용자? (잘못된 적용)                                    ││
│  │                                                             ││
│  │  블랙박스: 결정 이유를 설명할 수 없는 경우 많음              ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  실제 사례들:                                                    │
│  • 자율주행차 사고 → 누구 책임?                                 │
│  • 신용 점수 차별 → 어떻게 이의 제기?                           │
│  • ML 시스템이 사법 심사를 받으면 → 판사에게 어떻게 설명?        │
│                                                                 │
│  "알고리즘 탓"으로 책임을 회피해서는 안 된다                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.2 신용 점수 vs 예측 분석

| 관점 | 전통적 신용 점수 | 예측 분석 기반 점수 |
|------|-----------------|-------------------|
| **기반** | 실제 차입 이력 | 유사한 사람들의 행동 |
| **질문** | "과거에 어떻게 행동했나?" | "당신 같은 사람들은 어떻게 행동했나?" |
| **투명성** | 상대적으로 투명 | 불투명 (블랙박스) |
| **오류 수정** | 기록 정정 가능 | 거의 불가능 |
| **함의** | 개인의 행동 기반 | 스테레오타이핑 (집단 일반화) |

> 💬 **핵심 문제**: 통계적으로 맞더라도 개별 사례는 틀릴 수 있다. 평균 수명이 80세라고 해서 당신이 80세 생일에 죽는 것은 아니다.

### 4. 피드백 루프 (Feedback Loops)

#### 4.1 자기 강화 피드백의 위험

```
┌─────────────────────────────────────────────────────────────────┐
│                   악순환 피드백 루프                              │
│                                                                 │
│  예시: 고용과 신용 점수                                          │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  좋은 직장, 좋은 신용 점수                                   ││
│  │           │                                                 ││
│  │           ▼                                                 ││
│  │  예상치 못한 불행 (병, 사고 등)                              ││
│  │           │                                                 ││
│  │           ▼                                                 ││
│  │  청구서 연체 → 신용 점수 하락                                ││
│  │           │                                                 ││
│  │           ▼                                                 ││
│  │  신용 기반 채용 평가 → 취업 어려움                           ││
│  │           │                                                 ││
│  │           ▼                                                 ││
│  │  실업 → 빈곤 → 신용 점수 더 하락                             ││
│  │           │                                                 ││
│  │           ▼                                                 ││
│  │  취업 더 어려움 → ...                                       ││
│  │                                                             ││
│  │  "수학적 엄밀함과 데이터의 위장 뒤에 숨은 독이 든 가정"       ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  예시: 알고리즘 가격 담합                                        │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  독일 주유소 가격 알고리즘                                   ││
│  │  → 알고리즘들이 서로 학습                                   ││
│  │  → 암묵적 담합 발생                                         ││
│  │  → 소비자 가격 상승                                         ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│  │        │          ┌──────────┐            │                 ││
│  │        └──────────│  사람    │←───────────┘                 ││

│  │                        │                                    ││

│  │                   │ (결과)   │                              ││

│  │                                                             ││

│                                                                 │
│  핵심 질문들:                                                    │

│  • 부자는 더 부자로, 가난한 자는 더 가난하게 만드는가?            │
│  • 의도치 않은 결과는 무엇인가?                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5. 프라이버시와 추적 (Privacy and Tracking)

#### 5.1 사용자 vs 제품

```
┌─────────────────────────────────────────────────────────────────┐
│                  비즈니스 모델에 따른 관계                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  사용자가 고객인 경우:                                           │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  사용자 ──$──→ 서비스                                       ││
│  │    ↑              │                                         ││
│  │    └──── 가치 ────┘                                         ││
│  │                                                             ││
│  │  사용자가 명시적으로 입력한 데이터 저장                       ││
│  │  사용자의 이익을 위해 서비스 제공                             ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  광고가 비즈니스 모델인 경우:                                    │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  광고주 ──$──→ 서비스                                       ││
│  │                   │                                         ││
│  │                   ▼                                         ││
│  │               사용자 데이터                                  ││
│  │                   │                                         ││
│  │                   ▼                                         ││
│  │  사용자 ←── "무료" 서비스                                    ││
│  │    │                                                        ││
│  │    └──→ 행동 데이터 ──→ 프로필 ──→ 광고주                   ││
│  │                                                             ││
│  │  광고주가 진짜 고객, 사용자 이익은 2순위                      ││
│  │  → 이것이 바로 "감시(Surveillance)"                          ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 5.2 감시 인프라

```
┌─────────────────────────────────────────────────────────────────┐
│           "Data"를 "Surveillance"로 바꿔보기                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  원래 표현:                                                      │
│  "In our data-driven organization we collect real-time         │
│   data streams and store them in our data warehouse.           │
│   Our data scientists use advanced analytics and data          │
│   processing in order to derive new insights."                 │
│                                                                 │
│  바꿔보면:                                                       │
│  "In our SURVEILLANCE-driven organization we collect real-time │
│   SURVEILLANCE streams and store them in our SURVEILLANCE      │
│   warehouse. Our SURVEILLANCE scientists use advanced          │
│   analytics and SURVEILLANCE processing in order to            │
│   derive new insights."                                        │
│                                                                 │
│  여전히 좋게 들리는가?                                           │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  현실:                                                          │
│  • 모든 거주 공간에 인터넷 연결 마이크 존재                       │
│    (스마트폰, 스마트TV, 음성 비서, 베이비 모니터, 장난감)         │
│                                                                 │
│  • 위치, 이동, 사회 관계, 통신, 구매, 결제, 건강 데이터            │
│    → 거의 피할 수 없는 감시                                      │
│                                                                 │
│  • 가장 억압적인 독재 정권도 꿈꾸지 못한 수준                     │
│    → 그러나 우리는 기술의 혜택 때문에 자발적으로 수용             │
│                                                                 │
│  차이점: 정부 기관이 아닌 기업이 데이터 수집                      │
│         → 그렇다고 덜 위험한 것은 아님                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> 💬 **이 책의 다른 제목**: "Designing Surveillance-Intensive Applications" (감시 집약적 애플리케이션 설계)

### 6. 동의와 선택의 자유 (Consent and Freedom of Choice)

#### 6.1 "동의"의 문제점

```
┌─────────────────────────────────────────────────────────────────┐
│                    동의의 허구성                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  주장: "사용자가 약관에 동의했으므로 문제없다"                    │
│                                                                 │
│  문제점 1: 추적이 정말 "필요"한가?                                │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  서비스 개선에 필요:                                         ││
│  │  • 검색 결과 클릭률 → 랭킹 개선                              ││
│  │  • 제품 구매 패턴 → 추천 개선                                ││
│  │                                                             ││
│  │  사용자 이익인지 불분명:                                     ││
│  │  • 사용자 프로필 구축 → 광고 타겟팅                          ││
│  │  • 장기 행동 데이터 보관 → 마케팅 목적                       ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  문제점 2: 의미 있는 동의가 아님                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • 데이터가 어떻게 수집/처리되는지 이해 불가                  ││
│  │  • 개인정보 처리방침은 숨기는 데 더 효과적                    ││
│  │  • 사용자가 아닌 다른 사람에 대한 정보도 추론                 ││
│  │  • 파생 데이터셋은 이해하기 불가능                           ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  문제점 3: 일방적 관계                                           │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • 데이터 추출은 일방적, 상호성 없음                         ││
│  │  • 공정한 가치 교환 아님                                     ││
│  │  • 협상 옵션 없음 (약관은 서비스가 정함)                      ││
│  │  • 매우 비대칭적 관계                                        ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 6.2 EU GDPR의 동의 요건

```
┌─────────────────────────────────────────────────────────────────┐
│                    GDPR 동의 요건                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  동의는 다음 조건을 충족해야 함:                                  │
│                                                                 │
│  ✅ "자유롭게 주어진" (Freely given)                             │
│     → 동의 없이도 불이익 없이 서비스 이용 가능해야 함            │
│                                                                 │
│  ✅ "구체적인" (Specific)                                        │
│     → 각 목적별로 별도 동의                                      │
│                                                                 │
│  ✅ "충분한 정보에 기반한" (Informed)                             │
│     → 명확하고 이해하기 쉬운 언어로 설명                         │
│                                                                 │
│  ✅ "명확한" (Unambiguous)                                       │
│     → 침묵, 사전 체크된 박스, 비활동은 동의가 아님               │
│                                                                 │
│  현실:                                                          │
│  • 시행이 약함                                                  │
│  • 기술 업계 문화와 관행은 크게 변하지 않음                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 6.3 선택의 자유 문제

```
┌─────────────────────────────────────────────────────────────────┐
│                  "사용하지 않으면 된다"의 허구                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  서비스가 "기본적 사회 참여에 필수적"이라면?                      │
│                                                                 │
│  현대 사회에서 사실상 필수:                                       │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  📱 스마트폰 소지                                            ││
│  │  👥 소셜 네트워크로 사회적 교류                              ││
│  │  🔍 Google로 정보 검색                                       ││
│  │  📧 이메일/메시징 앱                                         ││
│  │  💼 LinkedIn으로 직업적 기회                                 ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  네트워크 효과:                                                  │
│  • 서비스를 사용하지 않으면 사회적 비용 발생                      │
│  • 친구들이 다 쓰면 나도 써야 함                                 │
│                                                                 │
│  플랫폼 설계:                                                    │
│  • 게임 역학, 도박 전술로 사용자 유인                            │
│  • 중독성 있게 설계됨                                           │
│                                                                 │
│  특권의 문제:                                                    │
│  • 개인정보 정책 이해할 시간/지식이 있는 소수만 거부 가능         │
│  • 사회적/직업적 기회 포기할 여유가 있는 사람만 거부 가능         │
│  • 덜 특권적인 위치의 사람에게는 선택의 자유 없음                 │
│                                                                 │
│  → 감시는 피할 수 없게 됨                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7. 프라이버시의 본질

#### 7.1 프라이버시 ≠ 비밀

```
┌─────────────────────────────────────────────────────────────────┐
│                    프라이버시의 진정한 의미                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  잘못된 주장: "프라이버시는 죽었다"                               │
│  근거: 사람들이 소셜 미디어에 모든 것을 공유                      │
│                                                                 │
│  실제 의미:                                                      │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                                                             ││
│  │  프라이버시 = 모든 것을 비밀로 유지 ❌                        ││
│  │                                                             ││
│  │  프라이버시 = 선택할 자유 ✅                                 ││
│  │              • 무엇을 누구에게 공개할지                      ││
│  │              • 무엇을 공개하고 비밀로 할지                   ││
│  │              • 각 상황에서 투명성의 수준을 결정              ││
│  │                                                             ││
│  │  프라이버시 = 결정권 (Decision Right)                        ││
│  │  → 개인의 자유와 자율성의 중요한 측면                        ││
│  │                                                             ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  예시:                                                          │
│  • 희귀 질환 환자가 연구 목적으로 데이터 제공 → 자발적 선택       │
│  • 같은 데이터가 보험/고용에 영향 줄 수 있다면 → 공유 꺼릴 것     │
│  • 중요한 것: 누가 접근하고 어떤 목적인지 본인이 결정             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 7.2 프라이버시 권리의 이전

```
┌─────────────────────────────────────────────────────────────────┐
│                  프라이버시 권리의 기업 이전                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  감시 인프라를 통해 데이터가 추출되면:                            │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  프라이버시 권리가 파괴되는 것이 아니라                       ││
│  │  개인 → 데이터 수집 기업으로 "이전"됨                        ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  기업의 입장:                                                    │
│  • "우리를 믿고 데이터를 맡겨주세요"                             │
│  • 무엇을 공개하고 비밀로 할지 결정권이 기업에                    │
│                                                                 │
│  기업의 행동:                                                    │
│  • 감시 결과의 대부분을 비밀로 유지                              │
│    (공개하면 "섬뜩하게" 느껴지고 비즈니스 모델에 해로움)          │
│  • 친밀한 정보는 간접적으로만 공개 (광고 타겟팅 도구)             │
│                                                                 │
│  결과:                                                          │
│  • 사용자는 자신의 친밀한 정보 공개에 대한 결정권 상실            │
│  • 개인 선호가 아닌 기업 이익 극대화 기준으로 결정                │
│                                                                 │
│  "개인에서 기업으로의 대규모 프라이버시 권리 이전은                │
│   역사적으로 전례가 없다" - Shoshana Zuboff                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 8. 데이터: 자산인가, 독성 물질인가?

#### 8.1 데이터의 가치와 위험

```
┌─────────────────────────────────────────────────────────────────┐
│                    데이터의 두 얼굴                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  "데이터 배기가스" 관점:                                         │
│  • 행동 데이터 = 서비스 사용의 부산물                             │
│  • 버려질 것에서 가치를 추출하는 재활용                           │
│                                                                 │
│  다른 관점:                                                      │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • 타겟 광고가 서비스 비용을 지불한다면                       ││
│  │    → 사용자 활동 = 일종의 "노동"                             ││
│  │                                                             ││
│  │  • 앱 = 사용자를 감시 인프라에 유인하는 미끼                  ││
│  │                                                             ││
│  │  • 인간의 창의성과 사회적 관계가                             ││
│  │    데이터 추출 기계에 의해 착취됨                            ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  데이터의 가치:                                                  │
│  • 데이터 브로커 산업 존재 (비밀스럽게 운영)                      │
│  • 스타트업 가치 = 사용자 수 = 감시 역량                         │
│                                                                 │
│  누가 데이터를 원하는가:                                         │
│  • 기업 (수집 목적)                                              │
│  • 정부 (비밀 거래, 강압, 법적 강제, 도난)                        │
│  • 범죄자 (해킹)                                                 │
│  • 파산 시 데이터도 자산으로 매각됨                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 8.2 데이터 = 독성 자산?

```
┌─────────────────────────────────────────────────────────────────┐
│                    데이터의 위험성 비유                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  데이터는...                                                     │
│                                                                 │
│  🥇 새로운 금(Gold)?                                             │
│     → 가치 있고 추출해야 한다                                    │
│                                                                 │
│  🛢️ 새로운 석유(Oil)?                                            │
│     → 정제하면 가치 있지만 환경 영향 있음                         │
│                                                                 │
│  ☢️ 새로운 우라늄(Uranium)?                                      │
│     → 엄청나게 강력하지만 놀랍도록 위험함                         │
│                                                                 │
│  🧪 유해 물질(Hazardous Material)?                               │
│     → 필요성과 원하는 것의 균형 필요                              │
│                                                                 │
│  ☠️ 독성 자산(Toxic Asset)?                                      │
│     → 잘못된 손에 들어갈 위험 상시 존재                           │
│                                                                 │
│  데이터 수집 시 고려해야 할 위험:                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  • 범죄자/적대국 정보기관에 의한 시스템 침해                  ││
│  │  • 내부자에 의한 데이터 유출                                 ││
│  │  • 우리 가치를 공유하지 않는 경영진 교체                      ││
│  │  • 인권을 존중하지 않는 정권에 의한 국가 인수                 ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  "경찰 국가를 가능하게 할 수 있는 기술을 설치하는 것은             │
│   좋지 않은 시민적 위생이다" - Bruce Schneier                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 8.3 지식 = 권력

```
┌─────────────────────────────────────────────────────────────────┐
│                    데이터와 권력                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  "Knowledge is power" (지식은 권력이다)                          │
│                                                                 │
│  "타인을 감시하면서 자신은 감시당하지 않는 것은                   │
│   권력의 가장 중요한 형태 중 하나이다"                            │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  전체주의 정권이 감시를 원하는 이유:                          ││
│  │  → 인구를 통제할 권력을 부여하기 때문                        ││
│  │                                                             ││
│  │  오늘날 기술 기업:                                           ││
│  │  → 정치적 권력을 노골적으로 추구하지는 않지만                ││
│  │  → 축적된 데이터와 지식이 우리 삶에 많은 권력 부여           ││
│  │  → 대부분 은밀하게, 공적 감시 밖에서                         ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 9. 산업혁명의 교훈

#### 9.1 역사적 유사성

```
┌─────────────────────────────────────────────────────────────────┐
│                산업혁명 vs 정보화 시대                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  산업혁명:                                                       │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  긍정적 결과:                                                ││
│  │  • 지속적 경제 성장                                          ││
│  │  • 장기적 생활 수준 향상                                     ││
│  │                                                             ││
│  │  부정적 결과 (초기):                                         ││

│  │                                                             ││

│  │  • 환경 보호 규제                                            ││

│  │  • 아동 노동 금지                                            ││
│  │  • 식품 위생 검사                                            ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  정보화 시대:                                                    │

### 10. 법률과 자율 규제

#### 10.1 규제의 딜레마

```
┌─────────────────────────────────────────────────────────────────┐
│                    규제의 균형 문제                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  GDPR의 데이터 최소화 원칙:                                      │
│  "개인 데이터는 명시적이고 정당한 목적을 위해 수집되어야 하며,    │
│   그 목적과 양립할 수 없는 방식으로 추가 처리되어서는 안 된다"    │
│                                                                 │
│  vs                                                             │
│                                                                 │
│  빅데이터 철학:                                                  │
│  "데이터 수집 극대화, 다른 데이터와 결합, 새로운 통찰 탐색"       │
│  → 예상치 못한 목적으로 사용 = "명시적" 목적과 정반대             │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  과잉 규제의 위험:                                           ││
│  │  • 의료 데이터 공유의 프라이버시 위험 ↔ 잠재적 기회          ││
│  │  • 데이터 분석으로 더 나은 진단/치료 가능성                   ││
│  │  • 규제가 이런 돌파구를 막을 수 있음                         ││
│  │                                                             ││
│  │  과소 규제의 위험:                                           ││
│  │  • 차별, 감시, 착취                                          ││
│  │  • 데이터 유출, 오용                                         ││
│  │  • 권력 불균형                                               ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  GDPR 현실:                                                      │
│  • 온라인 광고 산업에 일부 영향                                  │
│  • 시행이 약함                                                  │
│  • 기술 업계 문화와 관행은 크게 변하지 않음                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 면접 대비 질문

**Q1: 머신러닝 시스템에서 편향이 발생하는 주요 원인은?**

```
A: 편향은 여러 단계에서 발생할 수 있습니다:

   1. 학습 데이터의 편향
      • 과거의 차별적 결정이 데이터에 반영됨
      • 특정 그룹의 과소/과대 표현

   2. 레이블링의 편향
      • 인간 레이블러의 무의식적 편견
      • 주관적 판단이 개입되는 태스크

   3. 피처 선택의 편향
      • 보호 속성과 상관관계가 높은 프록시 변수
      • 예: 우편번호 → 인종 예측

   4. 피드백 루프
      • 모델 결과가 새 데이터 생성에 영향
      • 편향이 자기 강화됨

   해결 접근법:
   • 데이터 감사 및 편향 측정
   • 공정성 제약 조건 적용
   • 다양한 테스트 세트로 평가
   • 인간 검토 프로세스
   • 결정에 대한 설명 가능성 확보
```

**Q2: 프라이버시와 개인화 서비스 사이의 균형은 어떻게 맞출 수 있나요?**

```
A:
   1. 데이터 최소화 원칙
      • 목적에 필요한 최소 데이터만 수집
      • 사용 후 즉시 삭제

   2. 온디바이스 처리
      • 민감한 처리는 사용자 기기에서
      • 집계된 통계만 서버로 전송

   3. 차등 프라이버시 (Differential Privacy)
      • 개인 식별 불가능하도록 노이즈 추가
      • 전체 통계적 유용성은 유지

   4. 연합 학습 (Federated Learning)
      • 데이터는 로컬에 유지
      • 모델 업데이트만 공유

   5. 투명한 선택권
      • 사용자가 데이터 공유 수준 선택
      • 명확한 가치 교환 설명

   6. 목적 제한
      • 수집 목적 외 사용 금지
      • 목적별 별도 동의
```

**Q3: "알고리즘 감옥"을 방지하기 위해 시스템 설계 시 고려해야 할 점은?**

```
A:
   1. 이의 제기 메커니즘
      • 자동화된 결정에 대한 인간 검토 경로
      • 명확한 항소 프로세스

   2. 설명 가능성
      • 결정 이유를 이해할 수 있는 형태로 제공
      • "왜 거부되었는가"에 대한 답변 가능

   3. 데이터 정정 권리
      • 잘못된 데이터 수정 가능
      • 정정이 전파되도록 보장

   4. 2차 기회
      • 단일 시스템의 거부가 모든 곳에 영향 미치지 않도록
      • 시간 경과에 따른 재평가

   5. 인간 개입 지점
      • 높은 영향을 미치는 결정에는 인간 검토
      • 완전 자동화 제한

   6. 감사 및 모니터링
      • 시스템 결과의 공정성 정기 감사
      • 그룹 간 영향 분석
```

**Q4: 기술 기업에서 일하는 엔지니어의 윤리적 책임은 무엇인가요?**

```
A:
   1. 제품 영향 고려
      • "이 기능이 누구에게 해를 끼칠 수 있는가?"
      • 의도치 않은 결과 예측

   2. 목소리 내기
      • 윤리적 우려 제기
      • 잘못된 방향에 대한 이의 제기

   3. 교육과 인식
      • 팀 내 윤리 논의 촉진
      • 최신 연구 및 사례 학습

   4. 투명성 추구
      • 사용자에게 정직하게 소통
      • 블랙박스 결정 설명

   5. 거부할 용기
      • 명백히 해로운 프로젝트 거부
      • 대안 제시

   6. 장기적 관점
      • 단기 이익보다 장기 영향 고려
      • "우리가 살고 싶은 세상" 만들기

   "기술의 사회적 영향을 고려하지 않는다면,
    우리는 우리 일을 제대로 하고 있지 않은 것이다"
```

## ✅ 체크리스트

### 핵심 개념 이해
- [ ] 예측 분석이 개인의 삶에 미치는 영향을 설명할 수 있다
- [ ] "알고리즘 감옥" 개념을 이해한다
- [ ] 알고리즘 편향이 증폭되는 메커니즘을 설명할 수 있다
- [ ] 보호 속성과 프록시 변수의 관계를 이해한다
- [ ] 자동화된 의사결정에서의 책임 문제를 인식한다

### 프라이버시와 감시
- [ ] 프라이버시의 진정한 의미(선택의 자유)를 이해한다
- [ ] 비즈니스 모델에 따른 사용자-서비스 관계를 구분할 수 있다
- [ ] 동의의 한계를 설명할 수 있다
- [ ] 네트워크 효과가 선택의 자유에 미치는 영향을 이해한다
- [ ] 프라이버시 권리의 기업 이전 개념을 이해한다

### 데이터의 양면성
- [ ] 데이터를 "독성 자산"으로 보는 관점을 이해한다
- [ ] 데이터 수집의 잠재적 위험을 나열할 수 있다
- [ ] 데이터와 권력의 관계를 설명할 수 있다
- [ ] 산업혁명과 정보화 시대의 유사성을 비교할 수 있다

### 윤리적 책임
- [ ] 엔지니어로서의 윤리적 책임을 인식한다
- [ ] 시스템 설계 시 윤리적 고려사항을 적용할 수 있다
- [ ] 피드백 루프의 위험성을 식별할 수 있다
- [ ] 시스템 사고(Systems Thinking) 접근법을 이해한다
- [ ] 규제와 자율 규제의 균형 필요성을 인식한다
