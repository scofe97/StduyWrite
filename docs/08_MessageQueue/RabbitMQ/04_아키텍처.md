# RabbitMQ 아키텍처

## 1. 전체 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                         RabbitMQ Broker                              │
│                                                                     │
│  ┌──────────┐    ┌──────────────┐    ┌─────────────┐              │
│  │ Producer │───▶│   Exchange   │───▶│    Queue    │───▶ Consumer │
│  └──────────┘    └──────────────┘    └─────────────┘              │
│                         │                                          │
│                    ┌────┴────┐                                     │
│                    │ Binding │  (라우팅 규칙)                       │
│                    └─────────┘                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 핵심 구성요소

| 구성요소 | 역할 |
|---------|------|
| **Producer** | 메시지 생성 및 Exchange로 발행 |
| **Exchange** | 라우팅 규칙에 따라 Queue로 메시지 분배 |
| **Binding** | Exchange와 Queue 연결 규칙 |
| **Queue** | 메시지 저장 버퍼 |
| **Consumer** | Queue에서 메시지 소비 |
| **Virtual Host** | 논리적 격리 단위 |

## 2. 메시지 흐름 상세

### 발행(Publish) 흐름

```
Producer
    │
    │ 1. Connection 생성 (TCP)
    ▼
┌─────────────────────────┐
│       Connection        │
│  ┌───────────────────┐  │
│  │     Channel       │  │  2. Channel 생성 (가상 연결)
│  │         │         │  │
│  │    ┌────┴────┐    │  │
│  │    │ Publish │    │  │  3. 메시지 발행
│  │    └────┬────┘    │  │
│  └─────────┼─────────┘  │
└────────────┼────────────┘
             │
             ▼
┌─────────────────────────┐
│       Exchange          │  4. Exchange가 메시지 수신
│                         │
│  Routing Key: "order.*" │  5. Routing Key로 대상 Queue 결정
│            │            │
│   ┌────────┼────────┐   │
│   ▼        ▼        ▼   │
│ Queue1  Queue2  Queue3  │  6. Binding된 Queue로 복사
└─────────────────────────┘
```

### 소비(Consume) 흐름

```
┌─────────────────────────┐
│         Queue           │
│  ┌───┬───┬───┬───┬───┐  │
│  │M1 │M2 │M3 │M4 │M5 │  │  메시지 대기 중
│  └───┴───┴───┴───┴───┘  │
└───────────┬─────────────┘
            │
            │ Push (기본) 또는 Pull (basic.get)
            ▼
┌─────────────────────────┐
│       Consumer          │
│                         │
│  1. 메시지 수신          │
│  2. 처리                 │
│  3. Ack 전송             │
│                         │
└─────────────────────────┘
            │
            │ Acknowledgement
            ▼
┌─────────────────────────┐
│         Queue           │
│                         │
│  메시지 삭제 (Ack 수신 시) │
└─────────────────────────┘
```

## 3. Exchange 타입

### 3.1 Direct Exchange

```
                    Routing Key = "error"
Producer ──────────────────────────────────▶ Exchange (Direct)
                                                  │
                         ┌────────────────────────┼────────────────────────┐
                         │                        │                        │
                    Binding:               Binding:                  Binding:
                    "error"                "warning"                 "info"
                         │                        │                        │
                         ▼                        ▼                        ▼
                    ┌─────────┐             ┌─────────┐             ┌─────────┐
                    │ Queue A │             │ Queue B │             │ Queue C │
                    │   ✓     │             │         │             │         │
                    └─────────┘             └─────────┘             └─────────┘

→ Routing Key가 정확히 일치하는 Queue로만 전달
```

### 3.2 Fanout Exchange

```
Producer ──────────────────────────────────▶ Exchange (Fanout)
                                                  │
                         ┌────────────────────────┼────────────────────────┐
                         │                        │                        │
                         │                        │                        │
                         ▼                        ▼                        ▼
                    ┌─────────┐             ┌─────────┐             ┌─────────┐
                    │ Queue A │             │ Queue B │             │ Queue C │
                    │   ✓     │             │   ✓     │             │   ✓     │
                    └─────────┘             └─────────┘             └─────────┘

→ Routing Key 무시, 바인딩된 모든 Queue로 브로드캐스트
```

### 3.3 Topic Exchange

```
                    Routing Key = "order.created.kr"
Producer ──────────────────────────────────▶ Exchange (Topic)
                                                  │
                         ┌────────────────────────┼────────────────────────┐
                         │                        │                        │
                    Binding:               Binding:                  Binding:
                    "order.#"              "order.created.*"         "*.*.us"
                         │                        │                        │
                         ▼                        ▼                        ▼
                    ┌─────────┐             ┌─────────┐             ┌─────────┐
                    │ Queue A │             │ Queue B │             │ Queue C │
                    │   ✓     │             │   ✓     │             │         │
                    └─────────┘             └─────────┘             └─────────┘

패턴:
  * = 정확히 1단어
  # = 0개 이상의 단어
```

### 3.4 Headers Exchange

```
Producer ──▶ Message Headers: {type: "report", format: "pdf"}
                                    │
                                    ▼
                           Exchange (Headers)
                                    │
         ┌──────────────────────────┼──────────────────────────┐
         │                          │                          │
    Binding:                   Binding:                   Binding:
    x-match: all               x-match: any               x-match: all
    type: report               type: report               type: log
    format: pdf                format: xlsx               format: pdf
         │                          │                          │
         ▼                          ▼                          ▼
    ┌─────────┐               ┌─────────┐               ┌─────────┐
    │ Queue A │               │ Queue B │               │ Queue C │
    │   ✓     │               │   ✓     │               │         │
    └─────────┘               └─────────┘               └─────────┘

x-match:
  all = 모든 헤더가 일치해야 함
  any = 하나라도 일치하면 됨
```

## 4. Connection과 Channel

### 구조

```
┌─────────────────────────────────────────────────────────┐
│                    Application                          │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │              Connection (TCP)                    │   │
│  │                                                  │   │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐        │   │
│  │  │Channel 1 │ │Channel 2 │ │Channel 3 │  ...   │   │
│  │  │          │ │          │ │          │        │   │
│  │  │ Producer │ │ Consumer │ │ Consumer │        │   │
│  │  └──────────┘ └──────────┘ └──────────┘        │   │
│  │                                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
                         │
                         │ TCP Connection
                         ▼
┌─────────────────────────────────────────────────────────┐
│                   RabbitMQ Broker                        │
└─────────────────────────────────────────────────────────┘
```

### Connection vs Channel

| 항목 | Connection | Channel |
|------|------------|---------|
| **물리적/논리적** | TCP 연결 (물리적) | 가상 연결 (논리적) |
| **비용** | 높음 (소켓 생성) | 낮음 (멀티플렉싱) |
| **스레드 안전** | Yes | **No** (스레드당 1개 권장) |
| **권장 사용** | 애플리케이션당 1개 | 스레드당 1개 |

## 5. 클러스터 아키텍처

### 5.1 기본 클러스터

```
┌─────────────────────────────────────────────────────────────┐
│                    RabbitMQ Cluster                          │
│                                                             │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐       │
│  │   Node 1    │   │   Node 2    │   │   Node 3    │       │
│  │             │   │             │   │             │       │
│  │ ┌─────────┐ │   │ ┌─────────┐ │   │ ┌─────────┐ │       │
│  │ │Metadata │◀┼───┼▶│Metadata │◀┼───┼▶│Metadata │ │       │
│  │ │ (복제)  │ │   │ │ (복제)  │ │   │ │ (복제)  │ │       │
│  │ └─────────┘ │   │ └─────────┘ │   │ └─────────┘ │       │
│  │             │   │             │   │             │       │
│  │ ┌─────────┐ │   │ ┌─────────┐ │   │ ┌─────────┐ │       │
│  │ │ Queue A │ │   │ │ Queue B │ │   │ │ Queue C │ │       │
│  │ │ (로컬)  │ │   │ │ (로컬)  │ │   │ │ (로컬)  │ │       │
│  │ └─────────┘ │   │ └─────────┘ │   │ └─────────┘ │       │
│  └─────────────┘   └─────────────┘   └─────────────┘       │
│                                                             │
│  Metadata = Exchange, Binding, Virtual Host 등 (모든 노드에 복제) │
│  Queue Data = 기본적으로 단일 노드에만 존재 (Classic Queue)    │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 Quorum Queue 클러스터

```
┌─────────────────────────────────────────────────────────────┐
│                Quorum Queue Replication                      │
│                                                             │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐       │
│  │   Node 1    │   │   Node 2    │   │   Node 3    │       │
│  │             │   │             │   │             │       │
│  │ ┌─────────┐ │   │ ┌─────────┐ │   │ ┌─────────┐ │       │
│  │ │ Queue A │ │   │ │ Queue A │ │   │ │ Queue A │ │       │
│  │ │ LEADER  │◀┼───┼▶│ FOLLOWER│◀┼───┼▶│ FOLLOWER│ │       │
│  │ └─────────┘ │   │ └─────────┘ │   │ └─────────┘ │       │
│  │      │      │   │      ▲      │   │      ▲      │       │
│  │      │      │   │      │      │   │      │      │       │
│  │      └──────┼───┼──────┴──────┼───┼──────┘      │       │
│  │             │   │   Raft 복제  │   │             │       │
│  └─────────────┘   └─────────────┘   └─────────────┘       │
│                                                             │
│  쓰기: Leader만 수신 → Follower에 복제 → Quorum 확인 → Commit  │
│  읽기: Leader에서 수행 (Strong Consistency)                   │
└─────────────────────────────────────────────────────────────┘
```

## 7. 메시지 확인 (Acknowledgement)

### Publisher Confirms

```
Producer                           Broker
   │                                  │
   │ ──── 1. Publish Message ───────▶ │
   │                                  │ 2. Queue에 저장/복제
   │ ◀─── 3. Confirm (ack/nack) ───── │
   │                                  │

모드:
- 개별 확인: 메시지마다 confirm
- 배치 확인: N개마다 confirm
- 비동기 확인: callback으로 처리
```

### Consumer Acks

```
Broker                            Consumer
   │                                  │
   │ ──── 1. Deliver Message ───────▶ │
   │                                  │ 2. 메시지 처리
   │ ◀─── 3. Ack ──────────────────── │
   │                                  │
   │ 4. Queue에서 메시지 삭제          │

Ack 모드:
- auto-ack: 전송 즉시 삭제 (위험)
- manual-ack: Consumer가 명시적으로 ack
- nack: 처리 실패 시 (requeue 옵션)
- reject: 단일 메시지 거부
```

## 8. 흐름 제어 (Flow Control)

### Prefetch (QoS)

```
┌─────────────────────────────────────────────────────────────┐
│                     Flow Control                            │
│                                                             │
│  Queue: [M1][M2][M3][M4][M5][M6][M7][M8][M9][M10]          │
│                                                             │
│  prefetch_count = 3                                         │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────────────────────────────────────┐           │
│  │              Consumer                        │           │
│  │                                              │           │
│  │   Unacked: [M1][M2][M3]  (최대 3개)         │           │
│  │                                              │           │
│  │   M1 처리 완료 → Ack                         │           │
│  │   → M4 전달                                  │           │
│  │                                              │           │
│  └─────────────────────────────────────────────┘           │
│                                                             │
└─────────────────────────────────────────────────────────────┘

prefetch_count로 Consumer가 한 번에 받을 수 있는 미확인 메시지 수 제한
→ Consumer 과부하 방지
```

### Memory/Disk Alarm

```
정상 상태                    경고 상태                   차단 상태
    │                           │                          │
    ▼                           ▼                          ▼
┌─────────┐              ┌─────────┐               ┌─────────┐
│ Memory  │              │ Memory  │               │ Memory  │
│  40%    │              │  75%    │               │  95%    │
│         │              │ WARNING │               │ BLOCKED │
└─────────┘              └─────────┘               └─────────┘
    │                           │                          │
 Publish                   Slow down                  Block
 정상 동작                 백프레셔                    Publish 차단

설정: vm_memory_high_watermark = 0.4 (기본 40%)
```

## 10. 포트 구성

| 포트 | 용도 |
|------|------|
| **5672** | AMQP 0-9-1 (기본) |
| **5671** | AMQP + TLS |

| **15671** | Management UI (HTTPS) |
| **25672** | 클러스터 내부 통신 |
| **4369** | epmd (Erlang Port Mapper) |
| **1883** | MQTT |
| **8883** | MQTT + TLS |
| **61613** | STOMP |
| **5552** | Streams (전용 프로토콜) |
