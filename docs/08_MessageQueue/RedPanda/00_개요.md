# RedPanda 개요

## 1. RedPanda란?

RedPanda는 **2020년**에 출시된 Kafka 호환 스트리밍 플랫폼입니다. C++로 작성되어 JVM 없이 동작하며, **Kafka API와 100% 호환**되어 기존 Kafka 클라이언트를 그대로 사용할 수 있습니다.

### 핵심 특징

| 항목 | 설명 |
|------|------|
| **출시** | 2020년 |
| **개발사** | Redpanda Data (구 Vectorized) |
| **언어** | C++ (Seastar 프레임워크) |
| **프로토콜** | Kafka Protocol 호환 |
| **라이선스** | BSL 1.1 (Business Source License) |
| **외부 의존성** | 없음 (ZooKeeper/KRaft 불필요) |

---

## 2. 왜 RedPanda인가?

### Kafka의 한계

```
Apache Kafka:
├── JVM 기반 → GC 지연, 메모리 오버헤드
├── ZooKeeper 의존성 (4.0 이전) → 운영 복잡성
├── 파티션 당 하나의 스레드 → CPU 활용 비효율
└── 복잡한 튜닝 필요
```

### RedPanda의 해결책

```
RedPanda:
├── C++ 네이티브 → GC 없음, 낮은 메모리 사용
├── 내장 Raft → 외부 의존성 제로
├── Thread-per-core → 최대 CPU 활용
└── 자동 튜닝 (autotuner)
```

## 3. 아키텍처

### Thread-per-core 모델

```
┌─────────────────────────────────────────────────┐
│                RedPanda Node                     │
│                                                 │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────┐ │
│  │ Core 0  │ │ Core 1  │ │ Core 2  │ │Core N │ │
│  │ Thread  │ │ Thread  │ │ Thread  │ │Thread │ │
│  │    │    │ │    │    │ │    │    │ │   │   │ │
│  │ ┌──┴──┐ │ │ ┌──┴──┐ │ │ ┌──┴──┐ │ │┌──┴──┐│ │
│  │ │Shard│ │ │ │Shard│ │ │ │Shard│ │ ││Shard││ │
│  │ └─────┘ │ │ └─────┘ │ │ └─────┘ │ │└─────┘│ │
│  └─────────┘ └─────────┘ └─────────┘ └───────┘ │
│                                                 │
│  각 코어가 독립적으로 파티션 처리                   │
│  코어 간 공유 없음 → 락 경합 최소화                │
└─────────────────────────────────────────────────┘
```

### Seastar 프레임워크

- ScyllaDB에서 사용하는 고성능 C++ 프레임워크
- 비동기 I/O
- Zero-copy 네트워킹
- DPDK 지원 (optional)

### Raft 합의

```
Partition (Topic-Partition)
├── Leader (Node A)
├── Follower (Node B)
└── Follower (Node C)

메타데이터와 데이터 모두 내장 Raft로 관리
→ ZooKeeper/KRaft 불필요
```

## 4. Kafka 호환성

### 호환되는 것

| 항목 | 호환성 |
|------|--------|
| Kafka 클라이언트 | ✅ 그대로 사용 |
| kafka-console-producer/consumer | ✅ |
| Kafka Connect | ✅ |
| Schema Registry | ✅ (내장 지원) |
| Kafka Streams | ✅ |
| ksqlDB | ✅ |
| MirrorMaker | ✅ |

### 연결 예시

```java
// 기존 Kafka 코드 그대로 사용
Properties props = new Properties();
props.put("bootstrap.servers", "redpanda:9092");  // 주소만 변경
props.put("key.serializer", StringSerializer.class);
props.put("value.serializer", StringSerializer.class);

KafkaProducer<String, String> producer = new KafkaProducer<>(props);
producer.send(new ProducerRecord<>("topic", "key", "value"));
```

## 5. 주요 기능

### 1. 내장 Schema Registry

```bash
# Kafka: 별도 Confluent Schema Registry 필요
# RedPanda: 내장!

curl -X POST http://redpanda:8081/subjects/my-topic-value/versions \
  -H "Content-Type: application/vnd.schemaregistry.v1+json" \
  -d '{"schema": "{\"type\": \"string\"}"}'
```

### 2. 내장 HTTP Proxy (Pandaproxy)

REST API로 메시지 생산/소비:

```bash
# 메시지 생산
curl -X POST http://redpanda:8082/topics/my-topic \
  -H "Content-Type: application/vnd.kafka.json.v2+json" \
  -d '{"records":[{"value":"hello"}]}'

# 메시지 소비
curl http://redpanda:8082/topics/my-topic/partitions/0/records
```

### 3. Tiered Storage

클라우드 스토리지로 오래된 데이터 자동 오프로드:

```yaml
# S3로 티어링
redpanda:
  cloud_storage_enabled: true
  cloud_storage_bucket: "my-bucket"
  cloud_storage_region: "us-east-1"
```

### 4. 자동 튜닝

```bash
# 시스템에 맞게 자동 최적화
rpk redpanda tune all
```

## 6. 설치 및 실행

### Docker

```bash
docker run -d --name=redpanda \
  -p 9092:9092 \
  -p 8081:8081 \
  -p 8082:8082 \
  docker.redpanda.com/redpandadata/redpanda:latest \
  redpanda start --smp 1 --memory 1G
```

### rpk CLI

```bash
# 토픽 생성
rpk topic create my-topic --partitions 3 --replicas 3

# 토픽 목록
rpk topic list

# 메시지 생산
echo "hello" | rpk topic produce my-topic

# 메시지 소비
rpk topic consume my-topic
```

### Kubernetes (Helm)

```bash
helm repo add redpanda https://charts.redpanda.com
helm install redpanda redpanda/redpanda
```

## 8. 사용 사례

### 적합한 경우
- **Kafka 대체**: 기존 Kafka 인프라를 더 적은 리소스로 운영
- **저지연 요구**: 금융 거래, 실시간 분석
- **운영 단순화**: ZooKeeper 없이 단일 바이너리
- **비용 절감**: 더 적은 노드로 동일 워크로드 처리
- **클라우드 네이티브**: Kubernetes 친화적

### 고려사항
- Kafka 에코시스템의 일부 고급 기능은 아직 개발 중
- BSL 라이선스 제약 확인 필요
- 커뮤니티 규모가 Kafka보다 작음
