# Redis 메시지 큐 개요 (2025-2026)

## 1. Redis란?

Redis(Remote Dictionary Server)는 **인메모리 데이터 구조 저장소**로, 데이터베이스, 캐시, 메시지 브로커로 활용된다.

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| **최신 버전** | Redis 8.4 (2025년 말 기준) |
| **개발** | Redis Ltd. (구 Redis Labs) |
| **언어** | C |
| **라이선스** | AGPLv3 (2025년 오픈소스 전환) |
| **특징** | 인메모리, 초저지연, 다양한 데이터 구조 |

### 1.2 Redis 8.x 주요 변경사항 (2025)

| 버전 | 주요 기능 |
|------|----------|
| **8.0** | I/O 스레딩, 새로운 복제 메커니즘, Query Engine |
| **8.2** | XACKDEL, XDELEX 등 Stream 명령어 개선 |
| **8.4** | 하이브리드 검색, 멀티스레드 I/O, 30% 처리량 향상 |

### 1.3 Redis 8.0 성능 개선

```
- 최대 87% 빠른 명령어 처리
- 최대 2x OPS 처리량
- 최대 18% 빠른 복제
- 최대 16x Query 처리 능력 (Query Engine)
- I/O 스레딩으로 최대 112% 처리량 향상 (8스레드 기준)
```

## 2. Redis의 메시지 큐 방식

Redis는 **4가지 방식**으로 메시지 큐를 구현할 수 있다:

| 방식 | 데이터 구조 | 특징 | 도입 시점 |
|------|-----------|------|----------|
| **Pub/Sub** | 채널 | 실시간 브로드캐스트, 메시지 비보존 | 초기부터 |
| **List Queue** | List | FIFO 큐, BRPOP 블로킹 | 초기부터 |
| **Sorted Set Queue** | Sorted Set | 우선순위/지연 큐 | 초기부터 |
| **Streams** | Stream | Kafka 스타일, Consumer Group, 메시지 보존 | Redis 5.0 (2018) |

### 2.1 방식별 비교

```
┌─────────────────────────────────────────────────────────────────┐
│                    Redis 메시지 큐 방식 비교                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Pub/Sub        : 실시간 브로드캐스트 (Fire-and-Forget)          │
│  List Queue     : 단순 작업 큐 (FIFO)                           │
│  Sorted Set     : 우선순위 큐, 지연 큐                           │
│  Streams        : 이벤트 스트리밍, Consumer Group                │
│                                                                 │
│  메시지 보존:    Pub/Sub < List < Sorted Set < Streams          │
│  복잡도:        Pub/Sub < List < Sorted Set < Streams          │
│  기능:          Pub/Sub < List < Sorted Set < Streams          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 선택 가이드

| 요구사항 | 추천 방식 |
|---------|----------|
| 실시간 알림, 메시지 유실 허용 | **Pub/Sub** |
| 단순 작업 큐, FIFO 처리 | **List Queue** |
| 우선순위 처리, 지연 실행 | **Sorted Set Queue** |
| 메시지 보존, Consumer Group, 재처리 | **Streams** |

## 3. Redis vs Kafka vs RabbitMQ

### 3.1 성능 비교

| 항목 | Redis | Kafka | RabbitMQ |
|------|-------|-------|----------|
| **처리량 (1KB 메시지)** | ~100,000 msg/s | ~500,000 msg/s | ~25,000 msg/s |
| **지연시간** | ~0.1-0.15ms | ~81ms | ~0.13ms |
| **저장 방식** | 인메모리 | 디스크 | 메모리/디스크 |

### 3.2 특성 비교

| 특성 | Redis | Kafka | RabbitMQ |
|------|-------|-------|----------|
| **메시지 보존** | Streams만 | 기본 보존 | 처리 후 삭제 |
| **Consumer Group** | Streams | 기본 지원 | 기본 지원 |
| **복잡한 라우팅** | 제한적 | 제한적 | Exchange 지원 |
| **수평 확장** | 제한적 | 우수 | 중간 |
| **학습 곡선** | 낮음 | 높음 | 중간 |

### 3.3 Redis가 유리한 경우

```
✅ 이미 Redis를 캐시로 사용 중
✅ 초저지연이 필수
✅ 단순한 큐 패턴
✅ 소규모~중규모 워크로드
✅ 빠른 개발/프로토타이핑
```

### 3.4 Redis가 불리한 경우

```
❌ 대용량 이벤트 스트리밍 (→ Kafka)
❌ 복잡한 라우팅 필요 (→ RabbitMQ)
❌ 장기 메시지 보존 필수 (→ Kafka)
❌ 수백만 msg/s 처리 (→ Kafka)
```

## 4. Redis 8.x 새로운 데이터 구조

Redis 8.0에서 8개의 새로운 데이터 구조가 추가되었다:

| 데이터 구조 | 설명 | 용도 |
|------------|------|------|
| **Vector Set (Beta)** | 벡터 저장/검색 | AI/ML, 유사도 검색 |
| **JSON** | 네이티브 JSON | 문서 저장 |
| **Time Series** | 시계열 데이터 | 모니터링, IoT |
| **Bloom Filter** | 확률적 존재 확인 | 중복 체크 |
| **Cuckoo Filter** | Bloom 대안 | 삭제 지원 중복 체크 |
| **Count-Min Sketch** | 빈도 추정 | 스트림 분석 |
| **Top-K** | 상위 K개 추적 | 인기 항목 |
| **T-Digest** | 분위수 추정 | 백분위 계산 |

## 5. Redis 라이선스 변경 (2025)

### 5.1 라이선스 히스토리

| 시점 | 라이선스 | 비고 |
|------|---------|------|
| ~2024 | BSD 3-Clause | 완전 오픈소스 |
| 2024 | SSPL/RSALv2 | 클라우드 제한 |
| 2025 | **AGPLv3** | 오픈소스 복귀 |

### 5.2 현재 상태 (2025)

- Redis는 **삼중 라이선스** (AGPLv3, SSPL, RSALv2)
- AGPLv3로 오픈소스 커뮤니티 복귀
- 조건 충족 시 자유롭게 사용/수정 가능
