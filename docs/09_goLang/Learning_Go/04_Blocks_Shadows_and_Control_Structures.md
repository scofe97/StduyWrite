## 1. 핵심 개념 상세 설명

### 1.1 Block과 스코프

Go에서 Block은 식별자(변수, 상수, 타입, 함수)의 유효 범위를 결정합니다. Block은 계층 구조를 가지며, 내부 Block에서 외부 Block의 모든 식별자에 접근할 수 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Block 계층 구조                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ Universe Block                                           │   │
│   │ • 내장 타입: int, string, bool, error, ...              │   │
│   │ • 내장 상수: true, false, iota                          │   │
│   │ • nil                                                    │   │
│   │ • 내장 함수: make, len, cap, append, copy, delete, ...  │   │
│   │                                                          │   │
│   │ ┌─────────────────────────────────────────────────────┐ │   │
│   │ │ Package Block                                        │ │   │
│   │ │ • 패키지 레벨 변수, 상수, 타입, 함수                  │ │   │
│   │ │                                                      │ │   │
│   │ │ ┌─────────────────────────────────────────────────┐│ │   │
│   │ │ │ File Block                                      ││ │   │
│   │ │ │ • import된 패키지명                             ││ │   │
│   │ │ │                                                 ││ │   │
│   │ │ │ ┌─────────────────────────────────────────────┐││ │   │
│   │ │ │ │ Function Block                              │││ │   │
│   │ │ │ │ • 파라미터, 반환값, 지역 변수               │││ │   │
│   │ │ │ │                                             │││ │   │
│   │ │ │ │ ┌─────────────────────────────────────────┐│││ │   │
│   │ │ │ │ │ Inner Block (if, for, switch, {})       ││││ │   │
│   │ │ │ │ │ • 블록 내 선언된 변수                   ││││ │   │
│   │ │ │ │ └─────────────────────────────────────────┘│││ │   │
│   │ │ │ └─────────────────────────────────────────────┘││ │   │
│   │ │ └─────────────────────────────────────────────────┘│ │   │
│   │ └─────────────────────────────────────────────────────┘ │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                   │
│   접근 규칙: 내부 → 외부 접근 가능, 외부 → 내부 접근 불가         │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

Universe Block은 Go 언어 자체에 내장된 식별자들을 포함합니다. 모든 Go 코드에서 명시적인 import 없이 사용할 수 있습니다. Package Block은 패키지 레벨에서 선언된 식별자들을 포함하며, 해당 패키지의 모든 파일에서 접근 가능합니다. File Block은 import 문에서 선언된 패키지명을 포함하며, 해당 파일에서만 유효합니다.

### 1.2 Shadowing

Shadowing은 내부 Block에서 외부 Block과 동일한 이름의 식별자를 선언할 때 발생합니다. 내부에서 그 이름을 사용하면 내부에서 선언된 식별자가 참조되고, 외부의 식별자는 "가려집니다".

```
┌─────────────────────────────────────────────────────────────────┐
│                    Shadowing 예시                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   func main() {                                                  │
│       x := 10        // 외부 x                                   │
│       fmt.Println(x) // 10                                       │
│                                                                   │
│       if true {                                                  │
│           x := 5     // 내부 x (새 변수!) - shadowing           │
│           fmt.Println(x) // 5                                    │
│       }                                                          │
│                                                                   │
│       fmt.Println(x) // 10 (외부 x는 변경되지 않음)              │
│   }                                                              │
│                                                                   │
│   문제가 되는 경우:                                               │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func process() error {                                    │ │
│   │     var err error                                         │ │
│   │                                                            │ │
│   │     if condition {                                        │ │
│   │         result, err := doSomething()  // 새 err! 외부와 다름│ │
│   │         if err != nil {               // 내부 err 체크     │ │
│   │             return err                // OK               │ │
│   │         }                                                 │ │
│   │         use(result)                                       │ │
│   │     }                                                     │ │
│   │                                                            │ │
│   │     return err  // 항상 nil! (외부 err는 할당되지 않음)     │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   해결 방법:                                                      │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func process() error {                                    │ │
│   │     var err error                                         │ │
│   │     var result int                                        │ │
│   │                                                            │ │
│   │     if condition {                                        │ │
│   │         result, err = doSomething()  // = 사용, 외부 err  │ │
│   │         // ...                                            │ │
│   │     }                                                     │ │
│   │                                                            │ │
│   │     return err  // 올바른 err 반환                        │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

특히 위험한 것은 Universe Block의 식별자를 shadowing하는 것입니다. `true`, `false`, `nil`, `int`, `make` 같은 이름을 변수로 사용하면 해당 스코프에서 원래 의미가 가려집니다. 이는 심각한 버그와 혼란을 초래합니다.

### 1.3 for 문의 4가지 형태

Go는 유일한 반복문 키워드로 `for`를 사용하며, 4가지 형태로 모든 반복 패턴을 처리합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    for 문의 4가지 형태                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   1. Complete for (C 스타일)                                     │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for i := 0; i < 10; i++ {                                 │ │
│   │     // 초기화; 조건; 증감                                  │ │
│   │ }                                                         │ │
│   │ 용도: 인덱스 기반 반복, 정해진 횟수 반복                   │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   2. Condition-only for (while 대체)                             │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for condition {                                           │ │
│   │     // 조건이 true인 동안 반복                            │ │
│   │ }                                                         │ │
│   │ 용도: 조건 기반 반복, 횟수가 정해지지 않은 반복            │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   3. Infinite for                                                │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for {                                                     │ │
│   │     // 무한 반복 (break로 탈출)                           │ │
│   │ }                                                         │ │
│   │ 용도: 이벤트 루프, 서버 메인 루프, do-while 패턴          │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   4. for-range                                                   │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for index, value := range collection {                    │ │
│   │     // 컬렉션 순회                                        │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ 지원 타입:                                                 │ │
│   │ • slice/array: index, value                               │ │
│   │ • map: key, value                                         │ │
│   │ • string: byte index, rune (문자)                         │ │
│   │ • channel: value (index 없음)                             │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

for-range에서 중요한 점은 value가 복사본이라는 것입니다. range에서 받은 value를 수정해도 원본 컬렉션은 변경되지 않습니다. 원본을 수정하려면 인덱스를 사용하여 직접 접근해야 합니다.

### 1.4 switch 문의 특징

Go의 switch는 다른 언어와 몇 가지 중요한 차이가 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Go switch 특징                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   C/Java                              Go                         │
│   ──────────────────────────────────────────────────────────────│
│   switch (x) {                        switch x {                 │
│     case 1:                             case 1, 2, 3:  ◄── 여러 값│
│       doA();                              doA()                  │
│       break;  ◄── 필수                  case 4:                  │
│     case 2:                               doB()                  │
│     case 3:  ◄── fall through           default:                │
│       doA();                              doC()                  │
│       break;                          }                          │
│     case 4:                                                      │
│       doB();                          • break 불필요 (자동)      │
│       break;                          • fall through 없음        │
│     default:                          • 빈 case는 아무것도 안 함 │
│       doC();                                                     │
│   }                                                              │
│                                                                   │
│   Blank Switch (조건 switch)                                     │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ switch {                          // 비교 대상 없음        │ │
│   │ case score >= 90:                                         │ │
│   │     grade = "A"                                           │ │
│   │ case score >= 80:                                         │ │
│   │     grade = "B"                                           │ │
│   │ case score >= 70:                                         │ │
│   │     grade = "C"                                           │ │
│   │ default:                                                  │ │
│   │     grade = "F"                                           │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ 용도: if/else if/else 체인을 더 깔끔하게 표현              │ │
│   │       관련된 조건들을 비교할 때 특히 유용                   │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   switch 내에서 for 탈출 (Label 필요)                            │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ loop:                                                     │ │
│   │     for i := 0; i < 10; i++ {                            │ │
│   │         switch i {                                        │ │
│   │         case 5:                                           │ │
│   │             break loop  // for 탈출 (label 없으면 switch 탈출)│ │
│   │         }                                                 │ │
│   │     }                                                     │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

blank switch는 조건 표현식을 생략하고 각 case에 조건을 직접 작성하는 형태입니다. 범위 비교나 여러 변수를 포함한 복잡한 조건을 처리할 때 if/else 체인보다 가독성이 좋습니다.

### 1.5 Label과 break, continue

Label은 중첩된 루프에서 특정 루프를 제어할 때 사용합니다. break는 해당 label이 붙은 루프를 완전히 탈출하고, continue는 해당 루프의 다음 반복으로 넘어갑니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Label 사용 패턴                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   outer:                                                         │
│       for _, file := range files {                               │
│           for _, line := range file.Lines() {                    │
│               if isInvalid(line) {                               │
│                   continue outer  // file 루프의 다음 반복       │
│               }                                                   │
│               if shouldStop(line) {                              │
│                   break outer     // 모든 루프 탈출              │
│               }                                                   │
│               process(line)                                      │
│           }                                                       │
│           finalize(file)  // 모든 line이 유효할 때만 실행        │
│       }                                                          │
│                                                                   │
│   Label 없이 break/continue:                                     │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for i := 0; i < 10; i++ {                                 │ │
│   │     for j := 0; j < 10; j++ {                            │ │
│   │         if condition {                                    │ │
│   │             break  // 내부 for만 탈출                     │ │
│   │         }                                                 │ │
│   │     }                                                     │ │
│   │     // 여기로 옴 (외부 for는 계속)                        │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

Label은 코드의 복잡성을 증가시킬 수 있으므로, 정말 필요한 경우(중첩 루프에서 외부 루프 제어)에만 사용해야 합니다.

### 1.6 Go 1.22의 루프 변수 변경

Go 1.22에서 for-range 루프 변수의 동작이 변경되었습니다. 이전에는 루프 전체에서 하나의 변수가 재사용되었지만, 이제는 매 반복마다 새 변수가 생성됩니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Go 1.22 루프 변수 변경                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   Go 1.21 이전 (문제 있음)                                       │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ values := []int{1, 2, 3}                                  │ │
│   │ var funcs []func()                                        │ │
│   │                                                            │ │
│   │ for _, v := range values {                                │ │
│   │     funcs = append(funcs, func() {                        │ │
│   │         fmt.Println(v)  // 모두 같은 v를 캡처!            │ │
│   │     })                                                    │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ for _, f := range funcs {                                 │ │
│   │     f()  // 출력: 3, 3, 3 (마지막 값만)                   │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   Go 1.22+ (수정됨)                                              │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ // 동일한 코드가 다르게 동작                               │ │
│   │ for _, f := range funcs {                                 │ │
│   │     f()  // 출력: 1, 2, 3 (각각의 값)                     │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ // 각 반복마다 새 변수 생성                               │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   Go 1.21 이전의 해결책 (여전히 유효)                            │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ for _, v := range values {                                │ │
│   │     v := v  // 루프 내에서 새 변수 생성 (copy)            │ │
│   │     funcs = append(funcs, func() {                        │ │
│   │         fmt.Println(v)                                    │ │
│   │     })                                                    │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 비교표

### 2.1 Block 종류 비교

| Block | 범위 | 포함하는 식별자 | 접근 가능한 곳 |
|-------|------|----------------|---------------|
| Universe | 모든 Go 코드 | int, string, true, false, nil, make, len, ... | 모든 곳 |
| Package | 패키지 전체 | 패키지 레벨 선언 | 패키지 내 모든 파일 |
| File | 해당 파일 | import된 패키지명 | 해당 파일만 |
| Function | 함수 본문 | 파라미터, 지역 변수 | 함수 내부 |
| Inner | {} 블록 | 블록 내 선언 | 해당 블록과 내부 블록 |

### 2.2 for 문 형태 비교

| 형태 | 문법 | 용도 | 다른 언어 대응 |
|------|------|------|---------------|
| Complete | `for i := 0; i < n; i++` | 인덱스 기반 반복 | C for |
| Condition-only | `for condition` | 조건 반복 | while |
| Infinite | `for { }` | 무한 루프 | while(true) |
| for-range | `for i, v := range x` | 컬렉션 순회 | foreach |

### 2.3 switch 비교 (C/Java vs Go)

| 특성 | C/Java | Go |
|------|--------|-----|
| break | 필수 (없으면 fall through) | 불필요 (자동 break) |
| fall through | 기본 동작 | 명시적 `fallthrough` 필요 |
| 여러 값 매칭 | 불가 (case 나열) | `case 1, 2, 3:` |
| 빈 case | fall through | 아무것도 안 함 |
| 조건 switch | 불가 | blank switch |

## 3. 면접 예상 질문 및 모범 답안

### Q1. Go에서 shadowing이 무엇이고, 왜 주의해야 하는지 설명해주세요.

**모범 답안:**

Shadowing은 내부 Block에서 외부 Block과 동일한 이름의 변수를 선언하여 외부 변수가 "가려지는" 현상입니다. 내부에서 그 이름을 사용하면 내부에서 선언된 변수가 참조되고, 외부 변수는 접근할 수 없게 됩니다.

주의해야 하는 이유는 크게 세 가지입니다. 첫째, `:=` 연산자가 의도치 않은 shadowing을 만들 수 있습니다. `result, err := doSomething()`에서 err가 외부에 이미 있어도 `:=`는 새 변수를 만듭니다. 이로 인해 외부 err가 업데이트되지 않는 버그가 발생합니다.

둘째, 코드의 동작을 예측하기 어렵게 만듭니다. 같은 이름이 다른 값을 가리키면 코드를 읽는 사람이 혼란스러워집니다.

셋째, Universe Block의 식별자를 shadowing하면 매우 위험합니다. `true := false`처럼 쓰면 해당 스코프에서 true의 의미가 바뀝니다. nil, int, make 같은 이름도 shadowing 가능하여 심각한 버그를 만들 수 있습니다.

해결 방법은 `:=` 대신 `=`를 사용하여 기존 변수에 할당하는 것입니다. 단, 이를 위해 변수를 미리 선언해야 합니다. 또한 go vet나 staticcheck 같은 도구로 shadowing을 탐지할 수 있습니다.

### Q2. Go에 while 문이 없는 이유와, while을 대체하는 방법을 설명해주세요.

**모범 답안:**

Go는 언어의 단순성을 추구하여 for 키워드 하나로 모든 반복 패턴을 처리합니다. while 문이 필요한 상황은 for로 완벽히 대체됩니다.

조건 기반 반복(while)은 `for condition { }` 형태로 작성합니다. 초기화와 증감 부분을 생략하고 조건만 남기면 while과 동일하게 동작합니다.

무한 루프(while true)는 `for { }` 형태로 작성합니다. 조건도 생략하면 무한히 반복하며, break로 탈출합니다.

do-while(최소 1회 실행)은 Go에 직접적인 대응이 없지만, 무한 루프와 조건부 break로 구현합니다.

```go
for {
    // 최소 1회 실행할 코드
    if !condition {
        break
    }
}
```

이 설계의 장점은 개발자가 하나의 키워드만 알면 되고, 코드를 읽을 때도 for만 보면 반복임을 알 수 있다는 것입니다. 다양한 키워드(for, while, do-while, foreach)를 구분하는 인지적 부담이 사라집니다.

### Q3. for-range에서 value가 복사본인 것이 왜 중요한지, 원본을 수정하려면 어떻게 해야 하는지 설명해주세요.

**모범 답안:**

for-range에서 `for i, v := range slice`를 사용하면 v는 slice[i]의 복사본입니다. v를 수정해도 원본 slice는 변경되지 않습니다. 이것을 모르면 원본을 수정한다고 생각하고 v를 수정하는 버그가 발생합니다.

예를 들어 `for _, v := range numbers { v *= 2 }`는 numbers를 변경하지 않습니다. 각 반복에서 v는 해당 요소의 복사본이고, 복사본에 2를 곱한 것이기 때문입니다.

원본을 수정하려면 세 가지 방법이 있습니다. 첫째, 인덱스를 사용하여 직접 접근합니다. `for i := range numbers { numbers[i] *= 2 }`처럼 인덱스로 원본 요소에 접근합니다.

둘째, 포인터 slice를 사용합니다. `[]*Type` 형태의 slice라면 for-range의 value가 포인터이므로 그 포인터를 통해 원본을 수정할 수 있습니다.

셋째, struct의 경우 포인터를 담은 slice를 사용하거나, 인덱스로 직접 접근합니다. `for i := range people { people[i].Age++ }`처럼 사용합니다.

복사본이 기본인 이유는 안전성입니다. 의도치 않은 원본 수정을 방지합니다. 원본 수정이 필요하면 명시적으로 인덱스를 사용하여 의도를 표현합니다.

### Q4. Go의 switch가 다른 언어와 어떻게 다른지, blank switch가 무엇인지 설명해주세요.

**모범 답안:**

Go의 switch는 세 가지 주요 차이가 있습니다.

첫째, 자동 break입니다. C나 Java에서는 각 case 끝에 break를 쓰지 않으면 다음 case로 fall through됩니다. Go에서는 매 case가 자동으로 종료되므로 break가 필요 없습니다. fall through가 필요한 드문 경우에는 `fallthrough` 키워드를 명시적으로 사용합니다.

둘째, 여러 값 매칭입니다. `case 1, 2, 3:`처럼 하나의 case에 여러 값을 나열할 수 있습니다. C에서는 case 1: case 2: case 3:처럼 여러 case를 나열해야 했습니다.

셋째, blank switch입니다. `switch {`로 시작하고 비교 대상 없이 각 case에 조건식을 직접 작성합니다. if/else if/else 체인을 더 깔끔하게 표현할 수 있습니다.

blank switch는 범위 비교나 여러 변수를 포함한 복잡한 조건에 유용합니다. 예를 들어 점수에 따른 학점 계산에서 `case score >= 90:`, `case score >= 80:` 형태로 작성하면 각 조건의 관계가 명확히 보입니다. if/else 체인보다 각 조건이 동등한 수준에서 비교됨을 시각적으로 표현합니다.

### Q5. Go 1.22에서 for-range 루프 변수의 동작이 어떻게 변경되었고, 왜 변경되었는지 설명해주세요.

**모범 답안:**

Go 1.22 이전에는 for-range 루프에서 인덱스와 값 변수가 루프 전체에서 하나만 생성되어 재사용되었습니다. 매 반복마다 같은 메모리 위치에 새 값이 할당되었습니다.

이 동작은 클로저와 함께 사용할 때 문제가 되었습니다. 루프 안에서 고루틴이나 함수 리터럴을 생성하면, 그 클로저는 루프 변수를 캡처합니다. 루프가 끝난 후 클로저를 실행하면 모든 클로저가 마지막 값을 참조하게 됩니다.

Go 1.22에서는 각 반복마다 새로운 변수가 생성됩니다. 클로저가 변수를 캡처하면 해당 반복의 값을 가진 별도의 변수를 캡처합니다. 이제 고루틴에서 루프 변수를 사용해도 예상대로 동작합니다.

변경 이유는 이 문제가 Go에서 가장 흔한 버그 중 하나였기 때문입니다. 초보자뿐만 아니라 경험 많은 개발자도 자주 실수했습니다. Go 팀은 하위 호환성보다 언어의 직관성과 안전성을 우선시하여 이 변경을 결정했습니다.

Go 1.21 이전 코드를 유지보수할 때는 여전히 `v := v` 패턴(루프 내에서 변수 복사)이 필요할 수 있으므로 알아두어야 합니다.

### Q6. if 문에서 조건 앞에 변수를 선언하는 패턴의 장점을 설명해주세요.

**모범 답안:**

Go에서는 `if n := rand.Intn(10); n == 0 { }` 형태로 조건 앞에 변수를 선언할 수 있습니다. 이 변수는 if/else if/else 블록 전체에서 유효하고, 블록이 끝나면 스코프를 벗어납니다.

첫 번째 장점은 변수의 스코프를 최소화한다는 것입니다. 변수가 필요한 범위에서만 존재하므로, 코드의 다른 부분에서 실수로 사용하거나 변경하는 것을 방지합니다.

두 번째 장점은 코드의 의도가 명확해진다는 것입니다. 변수 선언과 사용이 한 곳에 있으면, 이 변수가 이 조건 검사를 위해서만 존재함을 알 수 있습니다.

세 번째 장점은 함수 호출과 에러 처리를 간결하게 표현할 수 있다는 것입니다. `if result, err := doSomething(); err != nil { return err }` 형태로 함수 호출, 에러 확인, 조기 반환을 한 문장으로 표현합니다.

이 패턴은 Go의 관용적 코드에서 매우 자주 사용됩니다. 특히 에러 처리에서 `if err := validate(input); err != nil { return err }` 형태가 표준 패턴입니다.

## 4. 실무 체크리스트

### Shadowing 방지 체크리스트

- [ ] `:=` 사용 시 외부 스코프의 같은 이름 변수 확인
- [ ] 에러 변수(err) 처리 시 외부 err 재사용 필요한지 확인
- [ ] Universe Block 식별자(true, false, nil, make, len 등) 변수명으로 사용 금지
- [ ] go vet 또는 staticcheck로 shadowing 검사

### for 문 선택 체크리스트

- [ ] 컬렉션 순회 → for-range 사용
- [ ] 인덱스 기반 부분 순회 → complete for 사용
- [ ] 조건 기반 반복 → condition-only for 사용
- [ ] 무한 루프/이벤트 루프 → infinite for 사용
- [ ] for-range에서 원본 수정 필요 → 인덱스 사용

### switch 사용 체크리스트

- [ ] 한 값의 여러 경우 비교 → switch 사용
- [ ] 범위 비교 → blank switch 사용
- [ ] 관련 없는 조건들 → if/else 사용
- [ ] switch 내에서 for 탈출 필요 → label 사용
