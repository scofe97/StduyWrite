## 1. 핵심 개념 상세 설명

### 1.1 Go 함수의 특징

Go 함수는 일급 객체(First-class citizen)입니다. 이는 함수를 변수에 할당하고, 다른 함수의 파라미터로 전달하며, 함수에서 반환할 수 있다는 의미입니다. 다중 반환값을 지원하여 결과와 에러를 함께 반환하는 패턴이 표준입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Go 함수 특징                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   일급 객체 (First-class Citizen)                                │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ • 변수에 할당 가능                                         │ │
│   │   var fn func(int) int = myFunc                           │ │
│   │                                                            │ │
│   │ • 파라미터로 전달 가능                                      │ │
│   │   sort.Slice(data, func(i, j int) bool { ... })           │ │
│   │                                                            │ │
│   │ • 반환값으로 사용 가능                                      │ │
│   │   func makeMult(base int) func(int) int { ... }           │ │
│   │                                                            │ │
│   │ • map에 저장 가능                                          │ │
│   │   opMap := map[string]func(int,int)int{ "+": add }        │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   다중 반환값 (Multiple Return Values)                           │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func divide(a, b int) (int, int, error) {                 │ │
│   │     if b == 0 {                                           │ │
│   │         return 0, 0, errors.New("division by zero")       │ │
│   │     }                                                     │ │
│   │     return a / b, a % b, nil                              │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ // 관례: 마지막 반환값은 error                             │ │
│   │ result, remainder, err := divide(10, 3)                   │ │
│   │ if err != nil {                                           │ │
│   │     return err                                            │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 Variadic 파라미터

Variadic 파라미터(`...`)는 가변 개수의 인자를 받을 수 있게 합니다. 함수 내부에서는 slice로 처리됩니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Variadic 파라미터                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   함수 정의                                                      │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func sum(nums ...int) int {                               │ │
│   │     total := 0                                            │ │
│   │     for _, n := range nums {  // nums는 []int            │ │
│   │         total += n                                        │ │
│   │     }                                                     │ │
│   │     return total                                          │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   호출 방법                                                      │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ // 개별 값으로 호출                                        │ │
│   │ sum()              // 0                                   │ │
│   │ sum(1)             // 1                                   │ │
│   │ sum(1, 2, 3, 4, 5) // 15                                  │ │
│   │                                                            │ │
│   │ // slice로 호출 (... 연산자 필수!)                         │ │
│   │ numbers := []int{1, 2, 3}                                 │ │
│   │ sum(numbers...)    // 6                                   │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   규칙                                                           │
│   • Variadic 파라미터는 반드시 마지막에 위치                     │
│   • 함수 내부에서 slice로 사용됨                                 │
│   • 0개 이상의 인자 전달 가능                                    │
│   • slice 전달 시 ... 연산자 필수                                │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 Closure

Closure는 함수가 자신이 정의된 스코프의 변수를 "캡처"하여 접근하고 수정할 수 있는 기능입니다. 캡처된 변수는 함수가 반환된 후에도 살아있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Closure 동작 원리                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   기본 예제                                                      │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func makeCounter() func() int {                           │ │
│   │     count := 0           // 외부 변수                      │ │
│   │     return func() int {                                   │ │
│   │         count++          // 캡처 및 수정                   │ │
│   │         return count                                      │ │
│   │     }                                                     │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ counter := makeCounter()                                  │ │
│   │ fmt.Println(counter())  // 1                              │ │
│   │ fmt.Println(counter())  // 2                              │ │
│   │ fmt.Println(counter())  // 3                              │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   메모리 동작                                                    │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ makeCounter() 호출                                        │ │
│   │     │                                                     │ │
│   │     ▼                                                     │ │
│   │ ┌─────────────┐     ┌─────────────┐                      │ │
│   │ │   count=0   │ ←── │  closure    │                      │ │
│   │ │   (힙에 존재)│     │  함수 반환  │                      │ │
│   │ └─────────────┘     └─────────────┘                      │ │
│   │     │                      │                              │ │
│   │     │                      ▼                              │ │
│   │     │              counter() 호출마다                     │ │
│   │     └─────────────── count 증가                           │ │
│   │                                                            │ │
│   │ • count는 힙에 할당되어 closure가 살아있는 동안 유지       │ │
│   │ • 각 makeCounter() 호출은 독립적인 count를 가짐           │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   실무 활용: sort.Slice                                          │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ people := []Person{{Name: "Alice", Age: 30}, ...}        │ │
│   │                                                            │ │
│   │ // people을 캡처하여 정렬 기준 함수 전달                   │ │
│   │ sort.Slice(people, func(i, j int) bool {                  │ │
│   │     return people[i].Age < people[j].Age                  │ │
│   │ })                                                        │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.4 defer

defer는 함수 종료 시 실행될 코드를 등록합니다. LIFO(Last In First Out) 순서로 실행되며, 리소스 정리에 필수적입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    defer 동작 원리                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   실행 순서 (LIFO)                                               │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func example() {                                          │ │
│   │     defer fmt.Println("first")   // 1번째 등록            │ │
│   │     defer fmt.Println("second")  // 2번째 등록            │ │
│   │     defer fmt.Println("third")   // 3번째 등록            │ │
│   │     fmt.Println("done")                                   │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ 출력:                                                     │ │
│   │ done                                                      │ │
│   │ third    ← 마지막 등록이 먼저 실행                       │ │
│   │ second                                                    │ │
│   │ first                                                     │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   파라미터 평가 시점 (등록 시점!)                                │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func example() {                                          │ │
│   │     x := 10                                               │ │
│   │     defer fmt.Println(x)  // x=10이 캡처됨                │ │
│   │     x = 20                                                │ │
│   │ }                                                         │ │
│   │ // 출력: 10 (등록 시점의 값)                              │ │
│   │                                                            │ │
│   │ // closure로 최신 값 사용                                  │ │
│   │ func example2() {                                         │ │
│   │     x := 10                                               │ │
│   │     defer func() {                                        │ │
│   │         fmt.Println(x)  // 실행 시점의 x 참조             │ │
│   │     }()                                                   │ │
│   │     x = 20                                                │ │
│   │ }                                                         │ │
│   │ // 출력: 20 (실행 시점의 값)                              │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   리소스 정리 패턴                                               │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func readFile(path string) ([]byte, error) {              │ │
│   │     f, err := os.Open(path)                               │ │
│   │     if err != nil {                                       │ │
│   │         return nil, err                                   │ │
│   │     }                                                     │ │
│   │     defer f.Close()  // 함수 종료 시 반드시 닫힘          │ │
│   │                                                            │ │
│   │     // 파일 읽기...                                        │ │
│   │     return data, nil                                      │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.5 Call by Value

Go는 Call by Value 언어입니다. 모든 값이 함수에 전달될 때 복사됩니다. 하지만 map과 slice는 내부적으로 포인터를 포함하므로 함수 내 수정이 원본에 영향을 미칩니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Call by Value 동작                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   기본 타입 (완전한 복사)                                        │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func modify(x int, s string) {                            │ │
│   │     x = 100           // 복사본 수정                       │ │
│   │     s = "changed"     // 복사본 수정                       │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ func main() {                                             │ │
│   │     a := 10                                               │ │
│   │     b := "hello"                                          │ │
│   │     modify(a, b)                                          │ │
│   │     fmt.Println(a, b)  // 10 hello (원본 변경 없음)       │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   Struct (완전한 복사)                                           │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ type Person struct { Name string; Age int }               │ │
│   │                                                            │ │
│   │ func modifyPerson(p Person) {                             │ │
│   │     p.Age = 100       // 복사본 수정                       │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ p := Person{Name: "Alice", Age: 30}                       │ │
│   │ modifyPerson(p)                                           │ │
│   │ fmt.Println(p.Age)    // 30 (원본 변경 없음)              │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   Map (포인터 복사 → 원본 수정됨)                                │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │                  호출 전            호출 후               │ │
│   │                                                            │ │
│   │ 원본 m ──┐                    원본 m ──┐                 │ │
│   │          │                             │                  │ │
│   │          ▼                             ▼                  │ │
│   │     ┌────────┐                   ┌────────┐              │ │
│   │     │ map    │                   │ map    │              │ │
│   │     │데이터  │                   │데이터  │◄── 수정됨   │ │
│   │     └────────┘                   └────────┘              │ │
│   │          ▲                             ▲                  │ │
│   │          │                             │                  │ │
│   │ 파라미터 ─┘                    파라미터 ─┘                 │ │
│   │ (포인터 복사)                  (같은 데이터 가리킴)        │ │
│   │                                                            │ │
│   │ m := map[string]int{"a": 1}                               │ │
│   │ func modify(m map[string]int) { m["a"] = 100 }            │ │
│   │ modify(m)                                                 │ │
│   │ fmt.Println(m["a"])  // 100 (원본 변경됨!)                │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   Slice (헤더 복사 → 요소 수정됨, 길이 변경 안 됨)               │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func modifySlice(s []int) {                               │ │
│   │     s[0] = 100        // 요소 수정 → 원본 반영 ✓          │ │
│   │     s = append(s, 4)  // 길이 변경 → 원본 미반영 ✗        │ │
│   │ }                                                         │ │
│   │                                                            │ │
│   │ slice := []int{1, 2, 3}                                   │ │
│   │ modifySlice(slice)                                        │ │
│   │ fmt.Println(slice)    // [100 2 3] (요소 변경됨)          │ │
│   │                       // 길이는 여전히 3                   │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.6 Named Return Values

Named return values는 반환값에 이름을 붙이는 기능입니다. zero value로 자동 초기화되며, 문서화 효과가 있습니다. 하지만 blank return과 함께 사용하면 가독성이 떨어집니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Named Return Values                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   기본 문법                                                      │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func divide(a, b int) (result int, err error) {           │ │
│   │     // result = 0, err = nil로 자동 초기화               │ │
│   │     if b == 0 {                                           │ │
│   │         err = errors.New("division by zero")              │ │
│   │         return result, err  // 명시적 반환 권장           │ │
│   │     }                                                     │ │
│   │     result = a / b                                        │ │
│   │     return result, err                                    │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   ⚠️ Blank Return (사용 금지!)                                   │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ // 나쁜 예: 무엇이 반환되는지 불명확                       │ │
│   │ func divide(a, b int) (result int, err error) {           │ │
│   │     if b == 0 {                                           │ │
│   │         err = errors.New("division by zero")              │ │
│   │         return  // ← 무엇이 반환되는지?                   │ │
│   │     }                                                     │ │
│   │     result = a / b                                        │ │
│   │     return  // ← 무엇이 반환되는지?                       │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
│   유효한 사용: defer에서 반환값 수정                             │
│   ┌───────────────────────────────────────────────────────────┐ │
│   │ func doWork() (err error) {                               │ │
│   │     defer func() {                                        │ │
│   │         if err != nil {                                   │ │
│   │             err = fmt.Errorf("doWork: %w", err)           │ │
│   │         }                                                 │ │
│   │     }()                                                   │ │
│   │                                                            │ │
│   │     // 작업...                                             │ │
│   │     return someOperation()                                │ │
│   │ }                                                         │ │
│   └───────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 비교표

### 2.1 함수 파라미터 전달 방식

| 타입 | 전달 방식 | 함수 내 수정 | 원본 영향 |
|------|----------|-------------|----------|
| int, float, bool | 값 복사 | 가능 | 없음 |
| string | 값 복사 | 가능 | 없음 |
| struct | 값 복사 | 가능 | 없음 |
| array | 값 복사 | 가능 | 없음 |
| map | 포인터 복사 | 가능 | **있음** |
| slice 요소 | 포인터 복사 | 가능 | **있음** |
| slice 길이 | 헤더 복사 | 가능 | 없음 |
| pointer | 포인터 복사 | 가능 | **있음** |

### 2.2 defer 파라미터 평가 시점

| 방식 | 평가 시점 | 사용 사례 |
|------|----------|----------|
| `defer fn(x)` | 등록 시점 | 등록 시점의 값 저장 필요 시 |
| `defer func() { fn(x) }()` | 실행 시점 | 최신 값 사용 필요 시 |

### 2.3 Named Return Values 사용 지침

| 상황 | 사용 여부 | 이유 |
|------|----------|------|
| 문서화 목적 | O | 반환값 의미 명확화 |
| defer에서 반환값 수정 | O | 유일한 방법 |
| Blank return | X | 가독성 저하 |
| 일반적인 경우 | X | 불필요한 복잡성 |

## 3. 면접 예상 질문 및 모범 답안

### Q1. Go에서 함수가 일급 객체(First-class citizen)라는 것이 무엇을 의미하고, 어떻게 활용할 수 있는지 설명해주세요.

**모범 답안:**

Go에서 함수가 일급 객체라는 것은 함수를 다른 값(정수, 문자열 등)처럼 다룰 수 있다는 의미입니다. 구체적으로 네 가지가 가능합니다.

첫째, 변수에 함수를 할당할 수 있습니다. `var fn func(int) int = myFunc`처럼 함수 타입 변수를 선언하고 함수를 할당합니다. 함수 타입은 시그니처(파라미터 타입과 반환 타입)로 정의됩니다.

둘째, 함수를 다른 함수의 파라미터로 전달할 수 있습니다. 대표적인 예가 `sort.Slice`입니다. 정렬 기준을 함수로 전달하여 커스텀 정렬을 구현합니다.

셋째, 함수에서 함수를 반환할 수 있습니다. 팩토리 패턴이나 미들웨어 패턴에서 사용됩니다. `makeMult(2)`가 "2를 곱하는 함수"를 반환하는 식입니다.

넷째, map이나 slice에 함수를 저장할 수 있습니다. 연산자에 따른 함수 선택 로직을 `map[string]func(int,int)int` 형태로 구현하면 switch 문 없이 깔끔한 코드가 됩니다.

실무 활용 예로는 HTTP 미들웨어가 있습니다. `func(Handler) Handler` 타입의 함수로 로깅, 인증, 압축 등의 기능을 체인으로 연결합니다. 또한 콜백 패턴, 이벤트 핸들러, 전략 패턴 등에서 함수를 값으로 전달하여 유연한 설계가 가능합니다.

### Q2. Closure가 무엇이고, Go에서 어떻게 동작하는지 설명해주세요.

**모범 답안:**

Closure는 함수가 자신이 정의된 스코프의 변수를 "캡처"하여 접근하고 수정할 수 있는 기능입니다. 캡처된 변수는 원래 스코프가 끝나도 closure가 살아있는 한 유지됩니다.

Go에서 closure가 동작하는 방식은 이렇습니다. 함수 내에서 익명 함수를 정의하면, 익명 함수는 외부 변수에 대한 참조를 가집니다. Go 런타임은 이 변수를 힙에 할당하여 함수가 반환된 후에도 접근 가능하게 합니다.

대표적인 예가 카운터 함수입니다. `makeCounter` 함수가 count 변수를 선언하고, 그 변수를 증가시키는 함수를 반환합니다. 반환된 함수를 호출할 때마다 같은 count 변수가 증가합니다. 각 `makeCounter` 호출은 독립적인 count를 가진 closure를 생성합니다.

실무에서 closure는 `sort.Slice`에서 정렬 대상 slice를 캡처하거나, HTTP 핸들러에서 설정값을 캡처하거나, defer에서 반환값을 수정할 때 사용됩니다.

주의할 점은 closure 내에서 `:=`를 사용하면 새 변수가 생성되어 외부 변수를 수정하지 못한다는 것입니다. 외부 변수를 수정하려면 `=`로 할당해야 합니다.

### Q3. defer의 실행 순서와 파라미터 평가 시점을 설명해주세요.

**모범 답안:**

defer는 두 가지 중요한 특성이 있습니다. 실행 순서와 파라미터 평가 시점입니다.

첫째, defer는 LIFO(Last In First Out) 순서로 실행됩니다. 먼저 등록된 defer가 나중에 실행됩니다. 이는 리소스 정리에 적합합니다. 예를 들어 파일을 열고 나서 버퍼를 할당했다면, 버퍼를 먼저 해제하고 파일을 닫아야 합니다. 할당 순서대로 defer를 등록하면 역순으로 정리됩니다.

둘째, defer의 파라미터는 등록 시점에 평가됩니다. `defer fmt.Println(x)`에서 x의 값은 defer가 등록될 때 캡처됩니다. 이후 x가 변경되어도 등록 시점의 값이 출력됩니다.

실행 시점의 값을 사용하려면 closure를 활용합니다. `defer func() { fmt.Println(x) }()`처럼 작성하면 closure가 x에 대한 참조를 캡처하고, 실행 시점에 현재 값을 읽습니다.

이 특성은 실수를 유발할 수 있습니다. 루프에서 `defer f.Close()`를 반복 호출하면 모든 defer가 마지막 f를 닫습니다. 이를 피하려면 각 반복에서 f를 closure로 캡처하거나, 별도 함수로 분리해야 합니다.

### Q4. Go가 Call by Value라면, 왜 map은 함수에서 수정하면 원본이 변경되나요?

**모범 답안:**

Go는 확실히 Call by Value입니다. 모든 인자는 함수에 전달될 때 복사됩니다. map이 수정 가능한 것처럼 보이는 이유는 map 변수가 실제로 포인터를 담고 있기 때문입니다.

map 변수는 내부적으로 해시 테이블 데이터 구조를 가리키는 포인터입니다. 함수에 map을 전달하면 이 포인터가 복사됩니다. 복사된 포인터는 여전히 같은 해시 테이블을 가리키므로, 함수 내에서 map을 수정하면 원본 map도 변경됩니다.

slice도 비슷하지만 약간 다릅니다. slice는 포인터, 길이, 용량을 담은 구조체입니다. 함수에 전달하면 이 구조체가 복사됩니다. 복사된 포인터는 같은 backing array를 가리키므로 요소 수정은 원본에 반영됩니다. 하지만 길이와 용량은 복사본의 필드이므로, `append`로 길이를 변경해도 원본에는 반영되지 않습니다.

이것은 Go가 Call by Value라는 원칙과 일치합니다. map 변수 자체(포인터 값)가 복사되는 것이지, 예외적으로 Call by Reference가 되는 것이 아닙니다. 완전한 값 복사가 필요하면 명시적으로 `maps.Clone`이나 직접 복사하는 코드를 작성해야 합니다.

### Q5. Blank return이 왜 나쁜 습관인지 설명해주세요.

**모범 답안:**

Blank return은 named return values와 함께 사용할 때 값을 명시하지 않고 `return`만 쓰는 것입니다. 이것이 나쁜 이유는 코드 가독성을 심각하게 해치기 때문입니다.

첫째, 무엇이 반환되는지 return 문만 보고 알 수 없습니다. 함수 전체를 훑어보며 named return 변수에 어떤 값이 할당되었는지 추적해야 합니다. 긴 함수에서는 이것이 매우 어렵습니다.

둘째, 코드 수정 시 실수하기 쉽습니다. named return 변수 할당을 놓치거나, 다른 경로에서 다른 값을 기대하는 버그가 발생할 수 있습니다.

셋째, 코드 리뷰어가 의도를 파악하기 어렵습니다. 의도적으로 zero value를 반환하는 것인지, 실수로 값을 할당하지 않은 것인지 구분이 안 됩니다.

named return values를 사용하더라도 `return result, err`처럼 명시적으로 반환값을 작성해야 합니다. named return values의 유효한 용도는 문서화 목적(반환값 이름으로 의미 전달)과 defer에서 반환값 수정(named return만 가능)입니다.

### Q6. Variadic 파라미터를 사용할 때 주의할 점을 설명해주세요.

**모범 답안:**

Variadic 파라미터를 사용할 때 세 가지 주의점이 있습니다.

첫째, variadic 파라미터는 반드시 마지막 파라미터여야 합니다. `func f(a ...int, b string)`은 컴파일 에러입니다. 컴파일러가 어디서 variadic 인자가 끝나고 다음 파라미터가 시작하는지 알 수 없기 때문입니다.

둘째, slice를 variadic 함수에 전달할 때 `...` 연산자가 필요합니다. `sum(numbers)`는 컴파일 에러이고, `sum(numbers...)`로 작성해야 합니다. 이 연산자는 slice를 개별 인자로 "펼칩니다".

셋째, variadic 파라미터는 함수 내부에서 slice로 처리됩니다. 아무 인자도 전달하지 않으면 빈 slice(길이 0)가 됩니다. nil이 아닌 빈 slice이므로 range로 순회해도 panic이 발생하지 않습니다.

추가로, variadic 파라미터와 일반 slice 파라미터의 차이를 이해해야 합니다. `func f(args ...int)`는 0개 이상의 인자를 개별적으로 받을 수 있고, `func f(args []int)`는 반드시 slice 하나를 받아야 합니다. 전자가 호출 시 더 유연하지만, 명시적으로 slice를 전달할 때는 후자가 더 명확합니다.
